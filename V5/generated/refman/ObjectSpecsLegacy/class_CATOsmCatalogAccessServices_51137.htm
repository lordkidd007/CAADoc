public class CATOsmCatalogAccessServices

Class grouping Feature Modeler services related to Catalog access.
Role: This class allows to manage links between container and StartUp Catalogs.
Note: It can be useful to remove from a document useless persistent links to StartUp Catalogs due to the use of AccessCatalog method, which is now deprecated.
When opening a document holding a link to a StartUp Catalog, a warning message will be raised if this Catalog is not found. This warning will appear even if there is no Feature instances in the document.

  • All methods taking startup late types as argument, such as CATICatalog or , actually use the links between the container and StartUp Catalogs to find the startup in Catalogs.
    Note that the more catalogs are linked to a container, the less efficient the search is.
  • These links are useless for StartUp Catalogs saved after release CXR18. In later releases, there is indeed a new mecanism allowing to find startups in a more efficient way.
    This mecanism is implemented by class and it is recommended to use it, giving the catalog name as input argument to the constructor for maximum efficiency.
  • The patterns AccessCatalog / CATICatalog / , and AccessCatalog / must be replaced by and methods.

Checks if a StartUp Catalog is linked to a container.
Role: This method checks if there is a link between the StartUp Catalog and the container.
Parameters:
iCatalogName
[in] The name of the StartUp Catalog, with its extension.
iClientId
[in] The client identifier of the StartUp Catalog.
ipiContainer
[in] An input container.
oIsLinked
[out] A (caller provided) boolean value in which the result will be stored.
Legal values:
  • TRUE: if the StartUp Catalog is linked with the container.
  • FALSE: if the StartUp Catalog is not linked with the container.

If the method fails oIsLinked is not modified, it is otherwise overwritten with the retrieved value.
Returns:
An HRESULT value.
Legal values:
S_OK
The catalog is linked to the container.
S_FALSE
The catalog is not linked to the container.
E_INVALIDARG
The catalog is not a CAA catalog.
E_POINTER
The input container is null.
E_ACCESSDENIED
The client identifier is not valid.
E_UNEXPECTED
There is no current session.
E_FAIL
Another failure has occured.

Links a StartUp Catalog to a container.
Role: This method creates a link between the StartUp Catalog and the container, provided the right client identifier is supplied.
Note: A container can be linked to several StartUp Catalogs.
After its creation, the link will persist in the document until the method is called. It means that the link will still exists even if there is no Feature instances in the document.
The only way to remove the link is to call the method .
Parameters:
iCatalogName
[in] The name of the StartUp Catalog, with its extension.
iClientId
[in] The client identifier of the StartUp Catalog.
ipiContainer
[in] A container to which the StartUp Catalog will be linked.
iIID
[in] The interface identifier for which a pointer is requested.
oPtr
[out, CATBaseUnknown#Release] The address where the returned pointer to the interface is located.
Returns:
An HRESULT value.
Legal values:
S_OK
The catalog has been successfully linked to the container.
E_INVALIDARG
The catalog is not a CAA catalog.
E_POINTER
The input container or the output pointer is null, or both.
E_ACCESSDENIED
The client identifier is not valid.
E_UNEXPECTED
There is no current session.
E_FAIL
Another failure has occured.
