o getAttributeNames
public virtual getAttributeNames( oAttributeNames,
oTypes)
Gets the list of available attribute names on the current object.
Parameters:
oAttributeNames
: List of attribute names for which the value is available in the result.
oTypes
: Type of the attribute.
o getAttributeValue
public virtual getAttributeValue( const iAttributeName,
oValue,
oIsSet)
Gets the value of one attribute on the current object.
Parameters:
iAttributeName
: Name of the attribute. This name must be in the list given by getAttributeNames.
oValue
: Values of the attribute.
oIsSet
: Flag to know if the attribute is set or not. It can have 3 values : 1 if the attribute is set. 0 if the attribute is not set. In that case, the value oValue is empty. -1 if the object itself doesn't exist. See the CAA use case for an example.

o getDocuments
public virtual getDocuments( oDocuments,
iDocCriteria= LEX_NoDocuments )
Gets the documents attached to the current object.
Parameters:
oDocuments
: The list of documents attached to the current object. Do not delete the list, do not create it either. Do not delete the pointers inside this list, deleting the current object will delete them, and the list. If there are no documents, the pointer oDocuments will be null.
iDocCriteria
: The type of document to retrieve.

oLevel

o getLockInfo
public virtual getLockInfo( oLockOwner,
iIsRef= FALSE )
Gets the lock information of the current object.
Parameters:
oLockOwner
: The lock owner of the object. If the object is not locked, the string is empty.
iIsRef
: For a Part Instance and a Product Instance, the lock is retrieved on the instance if iIsRef is false and on the reference if iIsRef is true.
o getName
public virtual getName( oName)

Parameters:
oName
: The name of the object to be displayed.
o getParent
public virtual getParent( oParentElem)
Gets the parent index of the current object, in the list of results.
Parameters:
oParentElem
: The index of the parent of the current object in the list of results. If oParentElem is 0, it means that the parent is the root object that got expanded - the object is a first level object. If oParentElem is -1, it means that the parent index has not been computed (expand LEX_MODE_DMU). If the object is a document or a TechPack, it gives the part on which this object is attached.
