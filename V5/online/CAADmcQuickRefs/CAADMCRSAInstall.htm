How to install RSx

Abstract

Before You Start

Launch  the "CAA V5 Development Environment Setup" window (CATVBTSetup executable program), select "Eclipse/IBM Rational Application" as target application, then select components to Install with at least "ENOVIA Designer Studio". To find out more, see customizing the RADE Interactive Tools.

Installing ENOVIA Designer Studio plugin in RSx

 To perform installation of Dassault Systemes extensions for RSx 9.0, proceed as follows:

  1. Start RSx 9.0
  2. From the Help menu, select "Install New Software...". The Install window is displayed
  3. Click on Add button to declare a new repository for plugins
  4. In the Add Repository panel, enter a label for the new repository (ex: DS Plugins for DMC V5)
  5. Click on "Local" button
  6. In the file selection box, select the CAAExtensionSiteRSx directory from your RADE V5 installation (ex: C:/Program Files (x86)/Dassault Systemes/B25/intel_a/resources/CATVBTServer/CAAExtensionSiteRSx) 
  7. Click OK in panels.
  8. Now a list of plugins is displayed. If no list is displayed, be sure the "Group items by category" option is unchecked
  9. Select all plugins, or at least "Feature DMC"
  10. Click Next
  11. Validate Licence agreement
  12. Click Yes when asked to restart RSx

32 bits Server support on Windows 2008 64 bits platform

It is possible to run a 32 bits ENOVIA Server on Windows 2008 64bits platform. By default, on 64 bits platforms, CAA tools (mkmk, mkGetPreq, etc...) are setup to manage 64 bits runtime and buildtime: paths are configured to reference win_b64 directories.

Therefore, if a 32 bits ENOVIA Server is installed on a 64 bits platform, it is necessary to set the _MkmkOS_BitMode variable to 32 before launching RSx and using the DMC product.

For more information, refer to dedicated article: CAA V5 64-bits Operating System Support
