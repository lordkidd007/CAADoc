Preparing for CAA RADE V5 Products Installation
You may have a previous RADE installation already running with a different Visual Studio than the one required by the current release.

You must always use the highest release levels of Visual Studio and RADE to work with the different V5 CAA installations you are dealing with. For example, if you deal with applications using the V5R20, V5R21, and V5-6R2012 (V5R22) release levels, you need to use Visual Studio 2008 required by V5-6R2012 and the Visual Studio add-ins installed with CAA RADE V5-6R2012. Nevertheless, you should not uninstall previous Visual Studio and CAA RADE levels, since they are needed to access the build tools they require, such as the compilers.

As a rule, you should have all the Visual Studio and V5 CAA RADE installed to match all the V5 release levels on top of which you develop and maintain your applications, but only the latest Visual Studio customized with the latest available customization.

This task shows you how to prepare the CAA RADE Version 5 Products installation for V5-6R2012 on a single computer running a supported Windows operating system.

Note: if you develop applications running on top of both V5 and V6, you should use the highest level of Visual Studio and V6 ENVOVIA Studio required. Refer to the V6 documentation to carry out this preparation task.
Clean up VS2005 or VS 2008: from start to finish, this task should take approximately 5 minutes.
1. Log on as an administrator.
You must belong to the Administrators group, or have the privileges assigned to the Administrators group. Otherwise, you will not be able to remove the add-ins from Visual Studio.
2. Start Visual Studio 2005 or 2008. With Windows 7, use the Run as Administrator command.
3. Remove the commands from the menus. Since you can have up to three add-ins installed, up to three remove commands may be necessary.

On the Tools menu, successively click:

  • Remove Code-Generator Add-in commands from menus
  • Remove Code-Designer Add-in commands from menus
  • Remove Mkmk Add-in commands from menus.

The commands belonging to these add-ins are removed from the menus.
4. Still in Visual Studio 2005 or 2008, disable the add-ins from being loaded.

On the Tools menu, click Add-in Manager.
The Add-in Manager dialog box opens.

Uncheck each CAA RADE add-in, in both the Available Add-ins column and in the Startup column.

Click OK.
5. Exit Visual Studio 2005 or 2008.
6. Unregister the add-ins.

Open an explorer, locate, and double click the following file:

C:\Program Files (x86)\Dassault Systemes\B22\intel_a\resources\CATVBTServer\UninstallAddinVS2005.reg

The add-ins are unregistered.

You can now install the new CAA RADE. See Installing CAA RADE V5 Products
