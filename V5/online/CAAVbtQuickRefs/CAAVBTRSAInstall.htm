Launching the Setup Tool

After installation of the RADE CD on a workstation, all users are provided with a default working environment enabling them to work with the C++ Interactive Dashboard within Visual Studio or with the Java Interactive Dashboard within IRAD.

This default environment however may not be adapted to the user's individual needs, and many options are open to him. These options are conditioned by the authorizations he has on his workstation, and an administrator has more possibilities than a simple user.

In order to launch the interactive Setup Tool, simply, with administrator privileges, start the program called CATVBTSetup located in the RADE installation directory. Right click CATVBTSetup .exe in an explorer, and click Run as administrator, then press Yes in the User Account Control dialog box.

"c:\Program Files (x86)\Dassault Systemes\B25\intel_a\code\bin\CATVBTSetup.exe"

If no ssh implementation exists on your computer, the following prompt opens:

Click OK.

The Setup Tool looks like this, but appearance may vary from one configuration to the other:

Click Eclipse/IBM Rational applications.

Click the components to install. The installation directory box is normally filled. This refers to RADE installation directory.

Note on the left-hand side the various target applications you wish to install the products on. Currently, only Eclipse/IBM Rational applications is available.

The list of components to install is displayed in the box named Components to Install. Check the components to install depending on your RADE products:

  •  Java Dashboard (JID), check CAA V5 - MKMK and CAA V5 - CodeGenerator.
  •  Data Model Customizer (DMC), check ENOVIA Designer Studio.
  •  Web Application Composer (WAC), check CAA - Web Application Composer.

Setting your Options

The following options do not require administrative privileges, and can be modified by an individual user.

Setting Default Variables

The Settings tab page enables you to customize variables used by the various interactive tools.

 

There are currently three categories of variables:

  • ENOVIA Publish variables. These variables enable you to pre-valuate the standard values needed to publish an ENOVIA customization to the database in order to avoid the individual user to know these values.
    • CATSettingPath: The setting path of the ENOVIA server installation
    • ENOVDataBaseAliasName: ENOVIA database alias name
    • ENOVDataBaseType: the type of the database. Possible values: db2 or oracle
    • ENOVDB2_HOME: Installation path of DB2 if db2 database installed
    • ENOVDBAID: Id of the ENOVIA database administrator
    • ENOVDBAPSSD: Password of the ENOVIA database administrator (this value is not kept)
    • ENOVORACLE_HOME: Installation path of ORACLE if ORACLE database installed
    • ENOVOutputDirectory: UNIX directory where generated libraries, SQL orders and metadata files are copied to facilitate deployment.
    • ENOVSimulatePublish: Default value for publish (YES for simulation, NO for NoSimulation, i.e. database modification)
    • ENOVTableSpace: Name of the ENOVIA tablespace
    • TNS_ADMIN: For Oracle only. The default value is the installation path of oracle with /network/admin suffixed.
  • CodeGenerator variables. These variables enable you to customize values for the Code Generation wizards.
    • CATCompanyName: The generated code has a standard header. You can modify the name of the company developing the software. Default value is Dassault Systemes, but you can insert your own company name in its place when developing your code.
  • WebApplicationComposer
    • JAVA_HOME: Installation path for java. For example, C:\Program Files\Java\jdk1.7.0_11

To edit a variable, double click it. A dialog box opens:

 

Type the appropriate value, and click OK.

Changing the tck Server

The Tool Access tab page allows you to modify several tool initialization options.

  • tck_init Profile Path When installing the RADE CD, a tck initialization file is provided. This file initializes the tools that let you point to various software levels (called profiles). Refer to the tck documentation for full explanations. It is possible to point to tck initialization files installed with other levels of the RADE CD, but in practice this should not be necessary as an initialization file provided on a given level gives you access to profiles set up on all other levels below.

Building your Software on UNIX

The Tool Access tab page also enables you to build your software on a UNIX platform. CAA-RADE V5 Products CD-ROM should have been previously installed on Unix before running this optional step.

In addition, to be able to send commands to the UNIX computer, you must have an implementation of either rsh or ssh installed on your Windows computer and available from the PATH environment variable.

When connecting to a UNIX machine from the C++ Interactive Dashboard, the connection is established under a given user only if that user has authorized the connection by adding a line in his .rhosts file in his/her home directory specifying the Windows machine name and the Windows user name.

  • Activate Remote Access: This check-box determines whether the interactive tools will allow you to build your software on a UNIX platform.
  • using rsh or using ssh: Check the option you want to use.
  • Location of RADE tools on UNIX: In this field, enter the directory of the UNIX path where you have installed the RADE tools. Note that you must enter the platform in the path. For example, if you installed the tools on AIX under /usr/DassaultSystemes/RADE, enter /usr/DassaultSystemes/RADE/aix_a64.
  • Unix tck_init Profile Path: In this field, enter the location of the UNIX tck initialization file. In most installation case, this path should start the same as the previous one. In our example, this would be: /usr/DassaultSystemes/RADE/aix_a64/code/command/tck_init.

Running the Setup Tool

Once you have set all the options you required, click Install to modify the options in your directory.

Click OK.

In the Log tab page, notice that the various steps and warnings are listed to guide you in case a step should fail. If the user does not have administrator privileges, he is warned first, but setup will continue for those operations not needing it.

Note that in some cases, a target tab page could appear. This page allows you to copy the installation of the RADE products to another location. This option requires administrative privileges because it modifies the registry and is not supported.

Installing DS plugins for Eclipse, RAD or RSA

No distinction is made between RSA 9.0 (Rational Software Architect), RAD 9.0 (Rational Application Developer) or Eclipse, meaning that there is only one installation process. To perform this installation, you need to create an Eclipse extension. To do so, proceed as follows:

  1. Start Eclipse (or RAD or RSA)
  2. From the Help menu, select "Install New Software...". The Install window is displayed
  3. Click on Add button to declare a new repository for plugins
  4. In the Add Repository panel, enter a label for the new repository (ex: DS Plugins for DMC V5)
  5. Click on "Local" button
  6. In the file selection box, select the CAAExtensionSiteRAD directory from your RADE V5 installation (ex: C:/Program Files (x86)/Dassault Systemes/B25/intel_a/resources/CATVBTServer/CAAExtensionSiteRAD) 
  7. Click OK in panels.
  8. Now a list of plugins is displayed. If no list is displayed, be sure the "Group items by category" option is unchecked
  9. Select one or all features ("Feature JID" or "Feature WAC or "Feature DMC")
  10. Click Next
  11. Validate Licence agreement
  12. Click Yes when asked to restart Eclipse (or RAD or RSA)
