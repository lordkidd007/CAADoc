Exploring Black Box Document Using UUID of Root Part Reference

Black box navigation in impacted by direction using Root PV
Use Case:CAALightExploreChildDocsFromRootPV

Abstract

This article discusses the CAALightExploreChildDocsFromRootPV use case. This use case explains how to use CATIVpmLightNavigationServices::GetConnectedDocuments and CATIVpmLightNavigationServices::ExploreDocumentLinks APIs to explore the BlackBox in Impacted By direction from root PV.

What You Will Learn With This Use Case

This use case is intended to use the service "GetConnectedDocuments" and "ExploreDocumentLinks" APIs exposed in CATIVpmLightNavigationServices Public Interface in succession.

[Top]

The CAALightExploreChildDocsFromRootPV Use Case

CAALightExploreChildDocsFromRootPV is a use case of the CAAVPMPersistency.edu framework that illustrates VPMInterfaces framework capabilities.

[Top]

What Does CAALightExploreChildDocsFromRootPV Do

The goal of CAALightExploreChildDocsFromRootPV use case is to show how to use CATIVpmLightNavigationServices::GetConnectedDocuments and CATIVpmLightNavigationServices::ExploreDocumentLinks Service in succession to explore the black box children document in impacted by direction using root PV.

[Top]

How to Launch CAALightExploreChildDocsFromRootPV

To launch CAALightExploreChildDocsFromRootPV, you will need to set up the build time environment, then compile CAALightExploreChildDocsFromRootPV along with its prerequisites, set up the run time environment, and then execute the use case [1].

Launch the use case as follows:

CAALightExploreChildDocsFromRootPV utility requires following argument in same order:

  • <user> : The user ID of a Admin user to run the utility.
  • <Password> : Password for the Admin User.
  • <Role> : This must be a ADMIN Role. i.e. VPMADMIN.ADMIN.DEFAULT
  • <V_ID OF Part> : V_ID Of Part where BlackBox document is attached.
  • <version String > : version string of the PV.
  • With Windows 
    e:>CAALightExploreChildDocsFromRootPV <user> <Password> <Role> <V_ID of Part> <Version String>
  • With UNIX 
    $ CAALightExploreChildDocsFromRootPV <user> <Password> <Role> <V_ID of Doc> <Revision String>

Step-by-Step

Following are the main steps in CAALightExploreChildDocsFromRootPV Code:

  1. Prerequisite
  2. Open Enovia session
  3. Get the UUID of root Part Version.
  4. Call the CATIVpmLightNavigationServices::GetConnectedDocuments to get the attached Document Revision from representation link on this Part
  5. Call the CATIVpmLightNavigationServices::ExploreDocumentLinks to get the children Documents

We will now comment each of these sections in detail.

[Top]

Prerequisite

To run and test this utility we must have created Black Box document with several children in enovia data base.

[Top]

Open Enovia session

...
//--Get Session
  VPMSession* session = VPMSession::OpenSession();
	 
//--Get FactoryManager
  CATIVpmFactoryManager_var FactoManager = NULL_var;
  session->GetVPMObjectFactory(FactoManager);
...

The Factory manager is retrieved from the session to further get the pointer of CATIVpmLightQueryManager interface to run the light query to get the UUID of input PV and also to get the pointer of CATIVpmLightNavigationServices interface to call GetConnectedDocuments and ExploreDocumentLinks service.

[Top]

Get the UUID of root Part Version.


//- Get CATIVpmLightQueryManager pointer.
	CATIVpmLightQueryManager_var LightQueryManager( spTheFactory );

//- Execute query and get the Light Query result.
	LightQueryManager->RunLightQuery( Query, SelectClause, SortAttr, TypeOfQuery, oQueryResult );

//- Query the CATIVPMLightNavigation Interface from pointer.
	spTheFactory->QueryInterface(IID_CATIVpmLightNavigationServices, (void**)& navPTR)) && navPTR);

//- Get the UUID of root Document Revision
	navPTR->GetUUIDFromObjectIdentity(

For more information on how to use light query please refer the documentation on CATIVpmLightQueryManager and ENOVLightQueryResult.

[Top]

Call the CATIVpmLightNavigationServices::GetConnectedDocuments to get the attached Document Revision from representation link on this Part

The input and output parameters for GetConnectedDocuments are as below:

  • iListofPVUUID : Input list of PV UUIDs in _SEQUENCE_octet.
  • iSelectClause : Select clause to get the attributes of Document Revision.
  • iDocCriteria : Input document criteria in DocumentCriteria enum.[LEX_Documentation/LEX_Representation]
  • oLightObjects : Output in format of light expand Objects.
...
	
	navPTR->GetConnectedDocuments(iListofPVUUID,iSelectClause,iDocCriteria,oLightObjects);
...
	// Now get the UUID of representation doc from oLightObjects
	(*oLightObjects)[1]->getUUID(objUUID); 

...

For more information on Select Clause Please refer the Documentation Light query.

For more information on LightExpandObject Please refer documentation on CATIVpmLightExpandObject interface.

[Top]

Call the CATIVpmLightNavigationServices::ExploreDocumentLinks to get the children Documents

The input and output parameters for ExploreDocumentLinks are as below:

  • iRootDocUuid : UUID of root of black box document.
  • iSelectClauseOnDoc : Select clause to get the attributes of Child Document Revision.
  • iSelectClauseOnRef : Select clause to get the attributes of Part Version attached with child document revision.
  • oLightObjects : Output in format of light expand Objects.
  • iRecusively : iRecusively = 0 to specify one level only. iRecusively = 1 to explore all level.
...
	
	navPTR->ExploreDocumentLinks(iRootDocUuid,iSelectClauseOnDoc,iSelectClauseOnRef,oLightObjects,iRecusively);

...

For more information on Select Clause Please refer the Documentation Light query.

For more information on LightExpandObject Please refer documentation on CATIVpmLightExpandObject interface.
