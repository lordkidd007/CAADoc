What You Will Learn With This Use Case

This use case is intended to use the service "ExploreDocumentLinks" exposed in CATIVpmLightNavigationServices Public Interface.

[Top]

The CAALightExploreDocLinks Use Case

CAALightExploreDocLinks is a use case of the CAAVPMPersistency.edu framework that illustrates VPMInterfaces framework capabilities.

[Top]

What Does CAALightExploreDocLinks Do

The goal of CAALightExploreDocLinks case is to show how to use CATIVpmLightNavigationServices::ExploreDocumentLinks Service to explore the black box children document in impacted by direction.

[Top]

How to Launch CAALightExploreDocLinks

To launch CAALightExploreDocLinks, you will need to set up the build time environment, then compile CAALightExploreDocLinks along with its prerequisites, set up the run time environment, and then execute the use case [1].

Launch the use case as follows:

CAALightExploreDocLinks utility requires following argument in same order:

  • <user> : The user ID of a Admin user to run the utility.
  • <Password> : Password for the Admin User.
  • <Role> : This must be a ADMIN Role. i.e. VPMADMIN.ADMIN.DEFAULT
  • <V_ID OF Document > : V_ID Of Root Document Revision to explore.
  • <Revision String > : Revision string of the document.
  • <level> : 0 for one level explore and 1 for all level explore.
  • With Windows 
    e:>CAALightExploreDocLinks <user> <Password> <Role> <V_ID of Doc> <Revision String> <level[0/1]>
  • With UNIX 
    $ CAALightExploreDocLinks <user> <Password> <Role> <V_ID of Doc> <Revision String> <level[0/1]>

Step-by-Step

Following are the main steps in CAALightExploreDocLinks Code:

  1. Prerequisite
  2. Open Enovia session
  3. Get the UUID of root Document Revision
  4. Call the CATIVpmLightNavigationServices::ExploreDocumentLinks to get the children Documents)

We will now comment each of these sections in detail.

[Top]

Prerequisite

To run and test this utility we must have created Black Box document with several children in enovia data base.

[Top]

Open Enovia session

...
//--Get Session
  VPMSession* session = VPMSession::OpenSession();
	 
//--Get FactoryManager
  CATIVpmFactoryManager_var FactoManager = NULL_var;
  session->GetVPMObjectFactory(FactoManager);
...

The Factory manager is retrieved from the session to further get the pointer of CATIVpmLightQueryManager interface to run the light query to get the UUID of input document revision and also to get the pointer of CATIVpmLightNavigationServices interface to call ExploreDocumentLinks service.

[Top]

Get the UUID of root Document Revision


//- Get CATIVpmLightQueryManager pointer.
	CATIVpmLightQueryManager_var LightQueryManager( spTheFactory );

//- Execute query and get the Light Query result.
	LightQueryManager->RunLightQuery( Query, SelectClause, SortAttr, TypeOfQuery, oQueryResult );

//- Query the CATIVPMLightNavigation Interface from pointer.
	spTheFactory->QueryInterface(IID_CATIVpmLightNavigationServices, (void**)& navPTR)) && navPTR);

//- Get the UUID of root Document Revision
	navPTR->GetUUIDFromObjectIdentity(

For more information on how to use light query please refer the documentation on CATIVpmLightQueryManager and ENOVLightQueryResult.

[Top]

Call the CATIVpmLightNavigationServices::ExploreDocumentLinks service to explore the black box.

The input and output parameters for ExploreDocumentLinks are as below:

  • iRootDocUuid : UUID of root of black box document.
  • iSelectClauseOnDoc : Select clause to get the attributes of Child Document Revision.
  • iSelectClauseOnRef : Select clause to get the attributes of Part Version attached with child document revision.
  • oLightObjects : Output in format of light expand Objects.
  • iRecusively : iRecusively = 0 to specify one level only. iRecusively = 1 to explore all level.
...
	
	navPTR->ExploreDocumentLinks(iRootDocUuid,iSelectClauseOnDoc,iSelectClauseOnRef,oLightObjects,iRecusively);

...

For more information on Select Clause Please refer the Documentation Light query.

For more information on LightExpandObject Please refer documentation on CATIVpmLightExpandObject interface.
