Abstract

The main goal of this use case is to describe how to implement the CATIInputDescription interface in order to correctly insert a V5V6ExtCombinedCurve within an ordered set.
This use case is the same as the original CombinedCurve use case, "Integrating Combined Curves inside Ordered Sets" [5] rewritten for the new V5V6ExtCombinedCurve Feature.

What You Will Learn With This Use Case

The main goal of this use case is to describe how to implement the CATIInputDescription interface on the V5V6ExtCombinedCurve StartUp. This interface is taken into account when the feature is aggregated into an ordered set [1]. It enables you to:

  • Define the type of your geometrical feature (Creation or Modification)
  • Define the absorbed features in case of feature of modification.

You can you refer to the "Creation versus Modification Feature" section of the referenced technical article [2] to find more details behind each concept and each method of this interface. Moreover, the "Integrating a New Mechanical Feature in V5" article [3] explains for which type of StartUp this interface should be implemented. 

Additionally, you will learn how to implement this interface by the BOA mechanism [4].
