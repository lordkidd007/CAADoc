Accessing Settings

    The Options dialog box opens.

  2. Possibly check Show all settings at the bottom left of the dialog box to display all setting categories.

Modifying the Tools Initialization Command Path

This is the path of the tck_init command which initializes the build tools environment when, for example, you open a workspace. It is set, when you have installed RADE in the default folder, to:

C:\Program Files (x86)\Dassault Systemes\B25\intel_a\code\command\tck_init.bat.

To change this command path:

  1. In Tools Initialization Command Path, type the new path of the tck_init.bat command. You can use a Windows (c:\xxx) or a UNC path (\\psrdsy\c$\xxx).

  2. Click OK.

Important: You must restart Visual Studio to take this new path into account.

Modifying the Start Program

The start program is the executable launched when running/testing the application. It is set by default to 3DEXPERIENCE.exe.

When managing you prerequisites, the start program you enter, or the default 3DEXPERIENCE.exe, is copied in you workspace. See Managing Prerequisites.

Important: Do not leave this field empty.

To change this start program:

  1. In Start Program, type the new start program.

  2. Click OK.

The start program is used as output file for each project of the solution. To see it:

  1. Right click any project, that is, any framework or module in the Solution Explorer.

  2. On the contextual menu, click Properties.

    The Property Pages dialog box of the selected project opens.

  3. In the right pane, click Configuration Properties to expand it.

    The relative path of the start program is displayed in Output File.

If you expand Configuration Properties and click Linker, it is also displayed in Output File

Important: Once you have changed the start program, refresh the project:

  1. In the Workspace Explorer, selecting the frameworks and modules projected in the Solution Explorer.

  2. Right click them.

  3. In the contextual menu, click Refresh Solution Project

Updating the Run Time View after Build

The run time view is not automatically updated when you rebuild a module, a framework, or the whole workspace. In this case, you should use the Refresh Run Time View command of the Workspace contextual menu to synchronize the run time view with your last changes. To make sure your run time view is always up-to-date, you can request it to be refreshed after each build.

To update the run time view whenever you have rebuilt a part of, or your whole workspace:

  1. Check Update run time view after compilation.

  2. Click OK.

Resetting Open Workspace Dialog Box History

In General, click Reset Open List to reset the history of the Open Workspace dialog box lists:

  • The Workspace Folder list.
  • The Tools Level list.
  • The Associated Visual Studio Solution list.

See Opening a Workspace.
