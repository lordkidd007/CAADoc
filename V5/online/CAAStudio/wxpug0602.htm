Running Tests

Running tests implies that you have a license for C++ Unit Test Manager.
Before you begin:
  • Make sure that you have a license for C++ Unit Test Manager.
  • Open a workspace with at least a framework and a module containing C++ sources, a test framework and a test object to run.

  1. In the 3DS Workspace Explorer, right click the workspace name, point to Test, and click Run Tests.

    The 3DS Wizard dialog box opens.

  2. Select the test objects to replay:

    • Select all test objects at once.

      1. Click Standard to replay only the standard test objects, click Switch to replay only switch test objects, or click both to replay all test objects.

    • Select only a subset of the test objects.

      1. In the tree, click the + signs to open a test framework, and the TestCases folder of this framework.

      2. Check one or several test cases.

  3. Click Next.

  4. Check Display traces if you want to display the test object traces in the Output window.

  5. Check Keep TMP directory to prevent mkOdt from deleting the TMP directory when it ends the test object run.

  6. Check No Maximum Time to prevent mkOdt from stopping test objects if they reach the time out limit.

  7. In Simultaneous Jobs, type the number of test object you want to run simultaneously. The default is 2.

  8. In Specific ENOVIA environment, type the ENOVIA environment required to run the selected test objects, if any.

  9. In Options, type the possible options, if any.

  10. Click Finish.

    The selected test objects are run.
