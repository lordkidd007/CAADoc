Getting Started

This task shows you how to get started with C++ Interactive Dashboard, using a basic common scenario and application.

Before you begin: Note that:
  • You should have first installed the C++ Interactive Dashboard.
  • You should also have installed a Dassault Systèmes brand, such as CATIA or DELMIA, as well as the associated API CD ROM. The scenario uses some of the installed frameworks to populate the workspace and run the tools.

Opening C++ Interactive Dashboard

  1. Click Microsoft Visual Studio 2012 shortcut on your desktop, or on the Start menu, point to All Programs, then point Microsoft Visual Studio 2012, and click click Microsoft Visual Studio 2012.

    Microsoft Visual Studio 2012 opens and the Start Page is displayed.

  2. On the View menu, point to 3DS Views, and click 3DS Workspace Explorer.

    The 3DS Workspace Explorer opens.

  3. Drag the Workspace Explorer and drop it onto the left hand arrow.

    The Workspace Explorer is docked on the left of the Microsoft Visual Studio 2012 window.

  4. On the View menu, click Solution Explorer.

  5. Drag the Solution Explorer and drop it onto the right hand arrow.

    The Solution Explorer is docked on the right of the Microsoft Visual Studio 2012 window.

    C++ Interactive Dashboard is ready for creating a workspace.

Creating a Workspace

  1. On the File menu, point to New, and click 3DS Workspace.

    The 3DS - New Workspace dialog box opens.

  2. In Location, type a folder path to create a new folder, such as E:\MyWorkspace.
    You can also click Browse to select an existing folder and fill the Location box, and then complete the selected path by typing the folder name into which you want to store the workspace frameworks.

    Warning: The folder you set for the workspace must not exist. Otherwise, a prompt is issued to warn you that a workspace cannot be created in an existing, even empty, folder. 

  3. In Tools Level, select the tools level you want to use for this workspace, such as V5_6R2015_B25.

  4. Click OK.

    • The folder E:\MyWorkspace is created if it did not exist.
    • The empty workspace MyWorkspace is displayed in the Workspace Explorer.
    • An empty project corresponding to this workspace is created and displayed in the Solution Explorer.

Populating the Workspace

  1. In the Workspace Explorer, right click MyWorkspace, and click Copy Framework.

    The 3DS - Copy Framework dialog box opens.

  2. In Framework to Copy, click Browse.

    The Browse for Folder dialog box opens.

  3. Locate the API CD ROM installation folder (by default C:\Program Files\Dassault Systemes\B25), then in this folder, locate the CAADoc folder, click the CAADialog.edu framework, and click OK.

  4. Click Finish.

    The framework, its modules and data are displayed in the Workspace Explorer.

  5. Right click the framework, and click Add All to Solution from the contextual menu to display the framework and the modules in the Solution Explorer.

    The projects corresponding to the framework and to the modules are displayed in the Solution Explorer.

Declaring the Workspace Prerequisites

  1. In the Workspace Explorer, right click the workspace root node MyWorkspace, and click Prerequisites.

    The 3DS - Manage Prerequisites dialog box opens.

  2. Click .

    A field appears prompting you to select or type a folder.

  3. Click .

    The Browse for Folder dialog box opens.

  4. Locate the prerequisite workspace root folder, that is, the folder where you installed the brand CD ROM (by default C:\Program Files\Dassault Systemes\B25), click it, and click OK.

    The selected folder is displayed in the 3DS - Manage Prerequisites dialog box.

  5. Click OK.

    The workspace prerequisites are updated.

Building a Module

  1. In the Workspace Explorer, expand the CAADialog.edu framework.

  2. Right click the module CAADlgOnIdle.m, and click Build with mkmk.

    The 3DS - Build Options dialog box opens.

  3. Click Force mkdata update (-u).

  4. Click Build.

    mkmk starts to build the module. The Output window displays the mkmk build log. When completed, the Error List window opens.

Creating the Run Time View

  1. In the Workspace Explorer, right click the workspace root node MyWorkspace, and click Refresh Run Time View.

    The Output window displays the mkRtv command build log which displays an error for each non built module. When completed, the Refresh the Run Time View prompt opens.

  2. Click Close in this prompt.

Creating a Test Framework

  1. In the Workspace Explorer, right click the workspace root node MyWorkspace, and click New Framework.

    The 3DS New Framework dialog box opens.

  2. In the Framework Name box, type CAADialog as the framework name. The wizard will add the tst extension when creating the framework.

  3. Check Test Framework.

  4. Click Finish.

    The framework CAADialog.tst is created in the workspace. Its IdentityCard.xml file is opened in the Code Editor window.

  5. In the Workspace Explorer, right click CAADialog.tst, and click Refresh Solution Project.

    The Solution Project is refreshed with CAADialog.tst.

Creating a Test Case

  1. In the Solution Explorer, right click CAADialog.tst

  2. In the contextual menu, point to Add, and click New Item.

    The Add New Item dialog box opens.

  3. In Installed Templates, click the triangle in front of 3DS Templates to expand its contents, if not yet expanded.

  4. Click Tests.

    The test case templates are displayed in the central pane.

  5. In the central pane, click XML test case.

  6. In the Name box, type CAADlgOnIdle. This is the name assigned to the test case.

  7. Click Add.

    The CAADlgOnIdle.xml opens in the Code Editor window.

  8. Edit this file:

    1. In the Target tag, replace the value attribute with This test validates the CAADlgOnIdle module.

    2. In the Command tag, replace the value attribute with CAADlgOnIdle.

    <?xml version="1.0"?>
<!-- COPYRIGHT Dassault Systemes 2014 -->

<ODT Version="1.0">
  <Definition>
    <Target value="This test validates the CAADlgOnIdle module"/>
  </Definition>
  <Execution>
    <Command value="CAADlgOnIdle"/>
  </Execution>
</ODT>

  9. On the File, menu, click Save CAADlgOnIdle.xml.

    The file is saved in the CAADialog.tst\FunctionTests\TestCases.

Running the Test Case

  1. In the Workspace Explorer, right click the workspace root node MyWorkspace, point to Tests, and click Run Tests.

    The ODT Replay Selection dialog box opens

  2. Check the box in front of TestCases.

  3. Click Next.

  4. Check Display Traces.

  5. Click Finish.

    mkodt starts replaying the test case. The Output window displays the mkodt log. A command prompt window opens, and is closed when the test cases completes. Then, a opens with the test case result page.
