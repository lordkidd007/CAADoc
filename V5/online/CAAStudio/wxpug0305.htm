Generating Shell Test Cases

This task shows you how to generate a shell test case.
Before you begin:
  • Open a workspace with at least a framework and a module.
  • Display the Solution Explorer.

  1. In the Solution Explorer, right click a test framework.

  2. In the contextual menu, point to Add, and click New Item.

    The Add New Item dialog box opens.

  3. In Installed Templates, click the triangle in front of 3DS Templates to expand its contents, if not yet expanded.

  4. Click Tests.

    The Tests installed templates are displayed in the central pane.

  5. In the central pane, click Test case.

  6. In the Name box, type the name you want to assign to the Test Case.

    Warning: This name is used to create the test file name. Bear in mind that:

    • All the characters are allowed except those listed below:

      * ? \ / < > " ' | :

    • With Windows, full path names must not exceed 260 characters.

  7. Click Add.

    The 3DS - New Test Case dialog box opens.

  8. In Type, select Batch or Record. If you select Record, the name in the Executable will automatically be initialized to the value of the Start Program box value in the Tools/Options/3DS dialog box, but will be editable.

  9. In Executable, specify the name of the executable to launch, along with its arguments.

  10. In Provider, select NONE or ENOVIA.

  11. In Replay on, select the configuration where the test case will be replayed:

    • CLIENT to replay on a client.
    • SERVER to replay on a server.
    • COMMON to replay on a both a client and a server.

    A tree view appears below with the various operating systems that are commonly supported for the chosen configuration.

  12. In the tree view, unselect the operating systems on which you specifically do not want the test case to be replayed.

  13. Click Finish.

    The .sh file, such as CAAxxxTest.sh is created and opens in the Code Editor window. See the C++ Unit Test Manager documentation to get information about filling a test sh file.

  14. On the File menu, click Save CAAxxxTest.sh.
