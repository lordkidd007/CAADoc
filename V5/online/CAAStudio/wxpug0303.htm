Generating Interfaces

This task shows you how to generate the source files of a C++ or IDL interface.
Before you begin:
  • Open a workspace with at least a framework and a module for a C++ interface, or an Automation framework for an IDL interface. See Creating a Framework.
  • Display the Solution Explorer.

  1. In the Solution Explorer, right click a module.

  2. In the contextual menu, point to Add, and click New Item.

    The Add New Item dialog box opens.

  3. In Installed Templates, click the triangle in front of 3DS Templates to expand its contents, if not yet expanded.

  4. Click C++.

    The C++ installed templates are displayed in the central pane.

  5. In the central pane, click Interface.

  6. In the Name box, type the name you want to assign to the interface. Refer to C++ Naming Rules.

    Warning: This name is also used to create the header and source files. Bear in mind that:
    • All the characters are allowed except those listed below: 
      * ? \ / < > " ' | :
    • With Windows, full path names must not exceed 260 characters.

  7. Click Add.

    The 3DS - New Interface dialog box opens.

  8. In Parent class, type the parent interface name, usually CATBaseUnknown.

  9. In Header exposition, select how you want the interface header to be seen from.

    Exposition Description Folder
    Local Visibility from this module only. LocalInterfaces
    Private Visibility from this framework only. PrivateInterfaces
    Protected Visibility from the modeler only ProtectedInterfaces
    Public External visibility. PublicInterfaces

  10. If you want to create an Automation interface, check IDL interface.

  11. If you want to create an C++ interface accessible using a smart pointer, check Create interface handler.

  12. Click Finish.

    The following is created or updated:

    • For a C++ interface:
      • The header file is created where you have requested it to be, the cpp and tsrc files are created in the module src folder. These files are opened in the Code Editor window.
      • The dico and the iid files, if they did not exist in the framework, in the CNext\code\dictionary folder. The iid file is updated with the interface IID.
    • For an Automation interface:
      • The IDL header file is created where you have requested it to be, and is opened in the Code Editor window.
