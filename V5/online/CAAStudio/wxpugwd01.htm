Interface Description

C++ Interactive Dashboard adds commands to the Visual Studio menus, and adds new toolbars.

  • File Menu
  • Edit Menu
    • 3DS Edit
    • 3DS Navigate
  • View Menu
  • Build Menu
    • Build Workspace
    • Build Module
  • Tools Menu
  • Windows Menu
  • Help Menu
  • Mkmk Toolbar
  • Workspace Contextual Menu
  • Framework Contextual Menu
  • Module Contextual Menu
  • Data Contextual Menu

File Menu


 The File menu includes two new commands:
  1. 3DS Workspace in the New submenu. See Creating a Workspace.
  2. 3DS Workspace in the Open submenu. See Opening a Workspace.

Edit Menu

The Edit menu includes two new submenus: 3DS Edit and 3DS Navigate.

3DS Edit

The 3DS Edit submenu includes four new commands:
  1. Add Include. See Adding an Include.
  2. Add forward Declaration. See Adding a Forward Declaration.
  3. Add Comments. See Adding a Comment.
  4. Check Source. Runs mkscc against the opened file. See Checking Sources.

3DS Navigate

The 3DS Navigate submenu includes three new commands:
  1. Go To Header. Go To Header.
  2. Swap Source Header. See Swap Source Header.
  3. Copy relative path. See Copying the Path Name of a File.

View Menu

The View menu includes the new 3DS Views submenu with the following commands:
  1. 3DS Workspace Explorer. See 3DS Workspace Explorer.
  2. 3DS Task List. See 3DS - Tasks List.
  3. 3DS Workspace Finder. See 3DS - Workspace Finder.
  4. Refresh 3DS Workspace Explorer. Refreshes the 3DS Workspace Explorer.

Build Menu

The Build menu includes two submenus: Build 3DS Workspace and Build 3DS Module.

Build 3DS Workspace

The Build 3DS Workspace submenu includes five new commands:
  1. Build Workspace. Builds the workspace with the previously set options.
  2. Build Workspace. Sets build options and builds the workspace.
  3. Link Workspace. Link-edits all the workspace modules.
  4. Build Id Cards. Builds the Identity Card files only.
  5. Clean Build. Removes the frameworks derived objects.

See Compiling.

Build 3DS Module

The Build 3DS Module submenu includes three new commands:
  1. Build Module. Builds the module with the previously set options.
  2. Build Module. Sets build options and builds the module.
  3. Link All Module. See Link-edits all the framework modules.

See Compiling.

Tools Menu

The Tools menu includes the new 3DS Environment Views commands. See Accessing and Customizing Environment.

Windows Menu

The Windows menu includes the new 3DS Windows submenu with the following commands:
  1. Explore 3DS Workspace. See Explore 3DS Workspace.
  2. Run Time Prompt. See Run Time Prompt.
  3. Build Time Prompt. See Build Time Prompt.

Help Menu

The Help menu includes two new commands:
  1. 3DS Help Viewer. Opens the 3DS help viewer. See 3DS Help Viewer.
  2. 3DS Object Browser. Runs the 3DS Object Browser. See 3DS Object Browser.

Mkmk Toolbar

The Mkmk toolbar includes the following commands.
Build Workspace. Builds the workspace with the previously set options.
Build Workspace. Sets build options and builds the workspace.
Link All Workspace. Link-edits all the workspace modules.
Build IdCard. Builds the Identity Card files only.
Clean Build. Removes the frameworks derived objects.
See Compiling.

Workspace Contextual Menu

Right clicking the workspace root opens a contextual menu with the following commands:
  1. Open Solution. Opens the Visual Studio solution corresponding to the workspace, and display it in the Solution Explorer.
  2. Search file. Open the 3DS - Workspace Finder to search for a file within the workspace. See 3DS - Workspace Finder.
  3. Build submenu:
    1. Build with mkmk. Builds the workspace with the previously set options.
    2. Build All with mkmk. Sets build options and builds the workspace.
    3. Link All. Link-edits all the workspace modules.
    4. Build Id Cards. Builds the Identity Card files only.
    5. Clean Build. Removes the derived objects of all the frameworks in the workspace.

    See Compiling.

  4. Prerequisites. See Managing Prerequisites.
  5. Refresh Run Time View. Refreshes the workspace run time view. See Refreshing the Run Time View.
  6. Refresh Workspace Explorer. Refreshes the 3DS Workspace Explorer.
  7. New Framework. See Creating a Framework.
  8. Copy Framework. See Copying a Framework.
  9. Delete. See Deleting a Framework.
  10. Test submenu:
    1. Run Tests. See Running Tests.
  11. Check Sources. See Checking Sources.
  12. Generate Intellisense. See Intellisense.
  13. Explore. Opens a Windows explorer sets to the workspace folder. See Explore from Object.

Framework Contextual Menu

Right clicking a framework opens a contextual menu with the following commands:
  1. Edit Id Card Opens the framework identity card in code editor.
  2. Build with mkmk. Sets build options and builds the framework.
  3. Build All with mkmk. Builds the framework with the previously set options.
  4. Link All. Link-edits all the framework modules.
  5. Build ID Cards. Builds the framework Identity Card file only..
  6. Clean Build. Removes the derived objects of the framework.
  7. New Module. See Creating a Module.
  8. New Data. See Creating Data.
  9. Delete. See Deleting a Framework.
  10. Add All to Solution. Recursively creates a Visual Studio project for the selected node and its child nodes, and add these projects to the current Visual Studio Solution.
  11. Add to Solution. Creates a Visual Studio project for this node and adds the project as current to Visual Studio Solution.
  12. Refresh Solution Project. Refreshes the Visual Studio project and the Solution Explorer.
  13. Check Sources. See Checking Sources.
  14. Explore. Opens a Windows explorer sets to the framework folder. See Explore from Object.
  15. Build Time Prompt. Opens a command prompt with the framework folder as the current one, and with the build time environment set. See Build Time Prompt.

Module Contextual Menu

Right clicking a module opens a contextual menu with the following commands:
  1. Build with mkmk. Sets build options and builds the module.
  2. Link Module. Link-edits the module.
  3. Delete. Delete the module.
  4. Add to Solution. Adds the module as a project to the current solution.
  5. Refresh Solution Project. Refreshes the Solution Explorer.
  6. Check Sources. See Checking Sources.
  7. Explore. Opens a Windows explorer sets to the module folder. See Explore from Object.
  8. Build Time Prompt. Opens a command prompt with the module folder as the current one, and with the build time environment set. See Build Time Prompt.

Data Contextual Menu

Right clicking a data folder opens a contextual menu with the following commands:
  1. Delete. Delete the data folder.
  2. Explore. Opens a Windows explorer sets to the data folder. See Explore from Object.
  3. Build Time Prompt. Opens a command prompt with the data folder as the current one, and with the build time environment set. See Build Time Prompt.
