3DS Object Browser

The 3DS Object Browser scans the dictionary files of the frameworks located in the run time views of your workspace and of its prerequisites. It displays the features or object that implement CAA exposed interfaces, along with the libraries containing the implementation run time code, and the frameworks containing the dictionary files. It provides navigation capabilities and a tree or table display for conveniently arranging the data related to a given feature, interface, library, or framework.

Before you begin:
  • Open Visual Studio.
  • Open a workspace with its prerequisites set.

Opening the 3DS Object Browser

  1. Open Visual Studio.

  2. Open a workspace as described in Opening a Workspace.

  3. Make sure that the workspace prerequisites are set, or set them as described in Managing Prerequisites.

  4. On the Help menu, click 3DS Object Browser.

    The 3DS Object Browser opens.

    The 3DS Object Browser includes a toolbar and two window panes.

Toolbar

The toolbar includes a text box and commands:

Icon Command Description
  Search Search for the entered character string.
  Refresh Empty the text box and display all the features.
  Configure Opens a prompt to set the path of the index file to use.
  Tree View Show the selected object and its related objects in a tree.
  Table View Show the selected object and its related objects in a table.

Left Pane

This pane shows, when opening the 3DS Object Browser, or when clicking , all the lines describing the implementation of an interface found in your workspace framework dictionary files. It also display the lines of the dictionary files of the frameworks belonging to your workspace prerequisites, provided these lines describe the implementation of a CAA exposed interface. These interface implementations are describe in a four-column table:

  1. Feature: the object that implement the interface. Note that all these objects are nor always features of the Feature Modeler. Some are Object Modeler implementation declared classes.
  2. Interface: the implemented interface.
  3. Library: the DLL or exe which contains the interface implementation executable code.
  4. Framework: the framework containing the dictionary file.

When opening the 3DS Object Browser, the rows are displayed in the dictionary file reading order. You can then sort the table according to the alphabetical order of the data in a given column by clicking the header of this column. Clicking again this column header sorts the table in the reverse alphabetical order.

Double clicking a cell in the table selects the entity in the cell and displays the data for this entity in the right pane.

Right Pane

This pane shows data of a selected entity in a cell of the table displayed in the left pane. This data can be shown in a tree or in a table.

Tree View

The tree depends on the type of the selected entity.

  • Feature

    The feature is the root node, and the interfaces it implements are its child nodes. The library and the framework declaring the interface implementation is a child node of the interface.

    Selecting the Pad feature displays all the CAA exposed interfaces this Pad feature implements, and for each of these interfaces, the DLL containing the implementation run time code, and the framework in which this interface implementation is declared.

    For example, the Pad feature implements the CATICkeObject interface, the implementation code of which is located in the CATPdgExposedView DLL. The interface implementation declaration is located in the PartDesignFeature framework.

  • Interface

    The interface is the root node, and the features implementing the interface are its child nodes. The library and the framework declaring the interface implementation is a child node of the feature.

     

    Selecting the CATICamera interface displays all the features implementing this interface, and for each of these features, the DLL containing the implementation run time code, and the framework in which this interface implementation is declared.

    For example, the CATFrm2DCamera object (feature) implements the CATICamera interface, the implementation code of which is located in the CATAfrFoundation DLL. The interface implementation declaration is located in the AfrFoundation framework.

  • Library

    The library, along with the framework where the interface implementations are declared, is the root node. The features implementing interfaces are its child nodes, and the feature implemented interfaces in this library are the feature child nodes.

     

    Selecting the libCATPdgExposedView library (DLL) displays all the features implementing interfaces in this DLL, and for each of these features, the interfaces the implementation run time code of which is contained in this DLL.

    For example, the libCATPdgExposedView library (DLL)  contains the RectPattern feature implementation code of the CATIAttributesDescription, CATICkeFeature, and CATICkeObject iinterfaces.

  • Framework

    The framework containing dictionary entries is the root node. The DLL declared in this framework dictionary as containing interface implementation run time code are its child nodes, and the features implementing interfaces are the DLL child nodes. Finally, the interfaces implemented by these features in this DLL are the feature child nodes.

     

    Selecting the CATPartUseImpl framework display all the libraries (usually only one) declared in this framework dictionary as containing the implementation run time code of features for the interfaces they implement in that library.

    For example, the CATPartUseImpl framework contains the libCATPartUseImpl library declared in this framework dictionary as containing the implementation run time code of the CircPattern feature for the interfaces CATIPdgUseCircPatt, CATIPdgUsePrtPattern, and CATIPdgUdeShape.

You can double click a feature, an interface, or a library in the tree to replace the current tree with the one displaying this feature, interface, or library, as the root node, along with its related data.

Table View

The table view has the same layout whatever the selected entity.

 

The feature is displayed with a separator, such as Chamfer of CircPattern above. Each interface this feature implements, as well as the framework containing the dictionary entry, and the library containing the implementation run time code, are displayed on a row.

You can click an interface to display all the features implementing this interface, whatever the rows displayed and the cell selected in the left pane.
