PLM Data Model

Description

Fig.1 PLMProductDS Model

The PLMProductDS customizes all the PRODUCT Modeler classes by adding PLM attributes:

  • VPMReference is customized as PLMProductDS.
  • VPMInstance is specialized as PLMProductInstanceDS.
  • VPMRepReference is specialized as PLMRepresentationDS.
  • VPMRepInstance is specialized as PLMRepresentationInstanceDS.
  • VPMPort is specialized as PLMPortDS.
  • VPMConnection is specialized as PLMConnectionDS.

Refer to the PLM Data Model documentation to see the PLM attributes [2].

Specificities

Mandatory Attributes: All PLMProductDS classes have two or three mandatory PLM attributes, but all are read-only, it means that they are instantiated either by the PRODUCT modeler itself, or the server, but never it can be never done by the end-user.

Identifier Set Attributes: Only PLM Reference (PLMProductDS) and the PLM Representation Reference (PLMRepresentationDS) have identifier PLM Attributes . For both, they are PLM_ExternalID and V_version. It means that to open PLMProductDS or a PLMRepresentationDS the two PLM attributes are mandatory. But to create such PLM Objects, only PLM_ExternalID is necessary. V_version is automatically valuated server side. PLM_ExternalID is valuated by the "init" Business logic when calling the API of creation.

CAA Use Cases

Some CAA Use Case are classified as "Use case requiring a customization matching specific characteristics". When the use case specifies the PLMProductDS customization it means that the use case will only work if referenced PLM objects, issued or to be created, from a PRODUCT modeler customization whose characteristics are as follows:

  • For all PLM customization classes, no writable public PLM Attributes.
  • For PLM Reference and PLM Representation Reference only PLM_ExternalID and V_version within the Identifier Set.

In Short

This article has described the PLMProductDS, one provided example of a PRODUCT Modeler customization.
