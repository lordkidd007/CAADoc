Working with Fix Instances
Use Case

Abstract

This Use case is working with CATIPrdInstanceFixityManagement interface to understand when an instance is movable or not.

This use case supposes that you have read the referenced technical articles [1] [2].

What You Will Learn With This Use Case

The use case mainly teaches us to use CATIPrdInstanceFixityManagement.

The CAAPstInstanceFixity Use Case

CAAPstInstanceFixity is a use case of the CAAProductStructure.edu framework that illustrates the ProductStructureUseItf framework capabilities.

What Does CAAPstInstanceFixity Do

Once the instance/reference data model is created:

  • We check that the instance LeftWheel is movable ( default behavior).
  • We move LeftWheel , using CATIMovable, it is OK.
  • We set RightWheel as fixed.
  • We try to move RightWheel, using CATIMovable, it is KO.

Then the occurrence data model is created:

  • We move the FrontLeftWeek, using CATIMovable, it is OK.
  • We try to move FrontRightWheel, using CATIMovable, it is KO - since moving the occurrence means modify its instance.

Then we set as flexible the Axle reference (CATIPrdRepositionableDefinition), and repositionable (CATIPrdRepositionable) its two instances (*):

  • We move the FrontLeftWeek, using CATIMovable, it is still OK.
  • We try to move RightFrontRightWheel Wheel, using CATIMovable, and now it is OK - since moving the occurrence means modify the occurrence matrix.

(*) Since the instance/reference data model is modified AFTER the occurrence creation, the occurrence data model must be synchronized [4] before a safe use.

How to Launch CAAPstInstanceFixity

To launch CAAPstInstanceFixity, you will need to set up the build time environment, then compile CAAPstInstanceFixity along with its prerequisites, set up the run time environment, and then execute the use case [3].

To launch the use case execute the command:

mkrun -c "CAAPstInstanceFixity Repository Server User Password SecurityCtx

where:

Repository
Server The server as host::name:port_number/RootURI
User The user name
Password The user password to authenticate the user
SecurityCtx a string representing the security context (Role.Organization.Project). Choose Role as Designer (as the use case modifies/creates the PLM Data).

Where to Find the CAAPstInstanceFixity Code

The CAAPstInstanceFixity use case is located in the CAAPstInstanceFixity.m module of the CAAProductStructure.edu framework:

InstallRootFolder\CAAProductStructure.edu\CAAPstInstanceFixity.m\

where InstallRootFolder [3] is the folder where the API CD-ROM is installed.

Step-by-Step

Following are the main steps in CAAPstInstanceFixity:

  1. Checking Fixity
  2. Setting Fixity / Setting Movable

Checking Fixity

... 
CATIPrdInstanceFixityManagement_var spFixOnLeftWheelInstance = pLeftWheelInstance ;
...
CATBoolean isFix = spFixOnLeftWheelInstance->IsFixed();
...

Setting Fixity / Setting Movable

... 
CATIPrdInstanceFixityManagement_var spFixOnRightWheelInstance= pRightWheelInstance ;
if ( NULL_var == spFixOnRightWheelInstance )   return 1 ;

hr = spFixOnRightWheelInstance->SetAsFix();
...
hr = spFixOnRightWheelInstance->SetAsMovable();
...
