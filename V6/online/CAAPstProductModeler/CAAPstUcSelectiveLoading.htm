What You Will Learn With This Use Case

This use case is intended to show:

  • How to select a PLM Representation Reference in the specification tree.
  • How to use the CATIPsiRepresentationLoadingMode interface to display and change the loading mode of the selected PLM Representation Reference. You will see the impact of each kind selected loading mode.

The CAAPstSelectiveLoading Use Case

CAAPstSelectiveLoading is a use case of the CAAProductStructure.edu framework that mainly illustrates CAT3DPhysicalRepInterfaces frameworks capabilities.

What Does CAAPstSelectiveLoading Do

The CAAPstSelectiveLoading use case is an interactive command.

Launch CATIA, when the application is ready:

  • Connect to the database.
  • Select your environment.
  • Open a Product Model (See Provided 3D XML File section).
  • On the View menu click Toolbar and check, if necessay, the CAA PLM Product Model Commands (See Add-in Access).
  • Push the Representation Loading Mode command button

    The following dialog box appears:

    Fig.1: The Representation Loading Mode Dialog Box
  • Select a Representation instance

    On this model, the CAAPrdBigWheel (the PLM component with the following icon ) has been selected. The "Rep Ref Loading Mode Selection" dialog box is updated such as:

    • Rep ref = The identification set of the PLM Representation Reference which is the reference of the selected PLM Representation Instance (here the PLM_ExternalID + V_version attributes). It is not detailed in the step by step section, because it is not the direct topic of the article, but the identification set is displayed thanks the CATIPLMIdentifierSet interface applied on the PLM Representation Reference.
    • Loading Mode = The current loading mode of the PLM Representation Reference.
  • Select a new loading mode

    In the Sel New Loading Mode combo, select one of the possible loading mode, and then push the Change Loading Mode button.

    Here is the result if you select the PLM Mode

    Note that:

    • In the dialog box, the Loading Mode editor has changed (it contains the new current loading mode of the CAAPrdBigWheel Rep Ref).
    • In the 3D viewer: on the front and back axles, all the instances of the CAAPrdBigWheel are not displayed.
  • Close the dialog box to finish the command.

The use case will only detail the PLM concepts, refer to the UI encyclopedia for dialog [1] & state command [2], Addin [3] concepts used in this use case.

Step-by-Step

There are four logical steps in CAAPstSelectiveLoading use case:

  1. Selecting a Representation Instance
  2. Retrieving the Representation Reference
  3. Displaying the Loading Mode
  4. Changing the Loading Mode

Selecting a Representation Instance

Here is an extract of the state command, the CAAPstLoadingModeDemoCmd class. It is a description of the dialog agent [2] enabling to select a PLM Representation Instance (In Product Context).

...
_daPathEltRepInstance = new CATPathElementAgent("SelRepInstance");
_daPathEltRepInstance->AddElementType(IID_CATIPrdRepInstance);
...

Here the dialog agent, _daPathEltRepInstance, is a CATPathElementAgent class [5] instance enabling to select object in the 3D Viewer or in the specification tree. The AddElementType method enables you to define the type of object to select. Only the object implementing the CATIPrdRepInstance interface could be selected, otherwise the mouse's cursor indicates that the selection is forbidden. The CATIPrdRepInstance is only implemented by PLM Representation Instance (In Product Context).

Retrieving the Representation Reference

Once a PLM Representation Instance is selected, you must retrieve it, and retrieve its reference.

...
CATPathElement * pathmodel = _daPathEltRepInstance->GetValue();
CATBaseUnknown * pobj = NULL ;
if ( ( NULL != pathmodel) && (pathmodel->GetSize()> 0) )
{
   pobj = (*pathmodel)[pathmodel->GetSize()-1];
...

Here it is an extract of the callback method when an object implementing CATIPrdRepInstance is selected. You retrieve the dialog agent, _daPathEltRepInstance, instantiated in the the previous step. The GetValue method retrieves the selected object, in fact the path from the root to the selected object. So the PLM Representation Instance is the leaf of this path. pobj is the PLM Representation Instance

Now, retrieving its reference

...
CATIPLMNavRepInstance *pInstRef = NULL ;
HRESULT rc = pobj->QueryInterface(IID_CATIPLMNavRepInstance, (void**) &pInstRef);
if ( SUCCEEDED(rc) )
{
   CATIPLMNavRepReference * pRepRef = NULL ;
   rc = pInstRef->GetRepReferenceInstanceOf(pRepRef);
...

You use the GetRepReferenceInstanceOf method of the CATIPLMNavRepInstance ( CATPLMComponentInterfaces fw) to retrieve it. pRepRef is the PLM Representation Reference.

The PLM Representation Reference, retrieved in the previous step, has been given to the CAAPstLoadingModeDemoDlg class thanks a call to the NewRepRef method, and kept as data member with _pCurrentRepRef . This data member is a CATIPsiRepresentationReference interface pointer. The translation from CATIPLMNavRepReference to CATIPsiRepresentationReference has been done in the state command. See the code for detail.

Now, the code to retrieve the loading mode of the main stream descriptor _pCurrentRepRef.

...
CATIPsiRepresentationLoadMode * pIRepLoadModeOnCurrentRepRef = NULL ;
rc = _pCurrentRepRef->QueryInterface(IID_CATIPsiRepresentationLoadMode , (void**) &pIRepLoadModeOnCurrentRepRef );
if ( SUCCEEDED(rc) )
{
   CATIPsiRepresentationLoadMode::LoadingMode ModeValue ;
   pIRepLoadModeOnCurrentRepRef->ComputeLoadingMode(ModeValue);

The CATIPsiRepresentationLoadMode interface implemented on the The PLM Representation Reference is the interface to retrieve the current mode. ModeValue is the value displayed in the Loading mode editor .[Fig.1]

Changing the Loading Mode

When the end user pushes the Change Loading Mode button [Fig.1] the callback below is executed. Here is an extract.

...
int SelLine = _pRepRefLoadingModeCombo->GetSelect();
...
CATIPsiRepresentationLoadMode::LoadingMode ModeValue ;
switch ( SelLine )
{
   case 0: ModeValue = CATIPsiRepresentationLoadMode::PLMMode ; break ;
   case 1: ModeValue = CATIPsiRepresentationLoadMode::VisuMode ; break ;
   case 2: ModeValue = CATIPsiRepresentationLoadMode::LinkableMode ; break ;
   case 3: ModeValue = CATIPsiRepresentationLoadMode::EditMode ; break ;
}
rc = pIRepLoadModeOnCurrentRepRef->ChangeLoadingMode(ModeValue);
...

_pRepRefLoadingModeCombo is a CATDlgCombo class instance created in the Build method of the CAAPstLoadingModeDemoDlg dialog class. Refer the code for detail. The GetSelect method of the combo box class retrieves the choice of the end user. According to this value, and a translation in the LoadingMode enum of the CATIPsiRepresentationLoadMode interface, the loading mode is changed using the ChangeLoadingMode method of the CATIPsiRepresentationLoadMode interface. pIRepLoadModeOnCurrentRepRef is a CATIPsiRepresentationLoadMode interface pointer on _pCurrentRepRef (see the previous step).
