Customize the Manufacturing System Definition Workbench Toolbars
Use Case

Abstract

This article discusses the CAAManufacturingSystemDefinitionToolbar use case. It explains how to implement the CATIManufacturingSystemDefinitionWorkbenchAddin to create and customize toolbars associated to Manufacturing System Definition Workbench.

What You Will Learn With This Use Case

This use case is intended to help you to create your own toolbars in Manufacturing System Definition workbench with command headers lauching different System functionalities.

  • Describing the different steps dedicated to the knowhow of the company through the customization.
  • Implementing  the methods of CATIManufacturingSystemDefinitionWorkbenchAddin to create toolbars in the Manufacturing System Definition Workbench.

The CAAManufacturingSystemDefinitionToolbar Use Case

CAAManufacturingSystemDefinitionToolbar is a use case of the CAADELPPRLiveSystemItf.edu framework that illustrates Manufacturing System Definition Workbench customization capabilities.

What Does CAAManufacturingSystemDefinitionToolbar Do

This use case illustrate adds a new toolbar in the Manufacturing System Definition Workbench

  • Add a toolbar in the Manufacturing System Definition Workbench
  • Create commands headers that will launch Manufacturing System functionality.

How to Launch CAAManufacturingSystemDefinitionToolbar

To use CAAManufacturingSystemDefinitionToolbar, you will need to:

  • Set up the build time environment, then compile CAAManufacturingSystemDefinitionToolbar.m module along with its prerequisites [1]
  • Edit the interface dictionary, that is, the CAADELPPRLiveSystemItf.edu.dico file located in:
    Windows InstallRootDirectory\CAADoc\CAADELPPRLiveSystemItf.edu\CNext\code\dictionary\
    Unix InstallRootDirectory/CAADoc/CAADELPPRLiveSystemItf.edu/CNext/code/dictionary/

    where InstallRootDirectory is the directory where the CAA CD-ROM is installed, and decomment the following line by removing the '#CAA#' character:

    CAAManufacturingSystemDefinitionToolbarExt CATIManufacturingSystemDefinitionWorkbenchAddin libCAAManufacturingSystemDefinitionToolbar
  • Set up the run time environment
  • Launch a CATIA V6 session
  • Go to the Manufacturing System Definition Workbench
  • Look for the "CAA Manufacturing System Definition Workbench Addin Toolbar" Toolbar (if this toolbar is not visible, make it visible from the list that is displayed when right clicking any toolbar)
  • Select the first icon in the toolbar, that will launch a command on the system.

        Step-by-Step

    

        The class CAAManufacturingSystemDefinitionToolbar implements the Interface CATIManufacturingSystemDefinitionWorkbenchAddin,
        this implementation is located on the file CAAManufacturingSystemDefinitionWorkbenchAddin.m\src\CAAManufacturingSystemDefinitionToolbar.cpp,
        Its purpose is to create the toolbar and to layout the commands on it. Only two
        methods need be defined: CreateToolbars
        and CreateCommands.

    

        Here is the declaration of this class:

    

        class CAAManufacturingSystemDefinitionToolbar: public CATBaseUnknown {
        

            CATDeclareClass;
        

            public:
        

                // Standard constructors and destructors
        for an implementation class
        

                // -----------------------------------------------------------------
        

                CAAManufacturingSystemDefinitionToolbar();
        

                virtual ~CAAManufacturingSystemDefinitionToolbar();
        

        

                void CreateCommands();
        

               
        

                CATCmdContainer *CreateToolbars();
        

        

            private:
        

                // The copy constructor and the equal
        operator must not be implemented
        

                // -------------------------------------------------------------------
        

                CAAManufacturingSystemDefinitionToolbar (CAAManufacturingSystemDefinitionToolbar
        &);
        

                CAAManufacturingSystemDefinitionToolbar&
        operator=(CAAManufacturingSystemDefinitionToolbar &);
        

        

                static CATCmdContainer* _pCreatedToolbar;
        

        };

    

        There are two logical steps in CAAManufacturingSystemDefinitionToolbar:

    
    
        
  1. Creating the commands headers to launch Manufacturing System functionalities
        
    2. 
        
  2. Adding a Toolbar inside Manufacturing System Definition Workbench

        We now comment each of those sections by looking at the code.

    

        Creating a command header to launch Manufacturing System functionalities

    

        The command that is going to be called when the toolbar button is clicked is is
        created on the method: CAAManufacturingSystemDefinitionToolbar::CreateCommands.
    

    

        void CAAManufacturingSystemDefinitionToolbar::CreateCommands()
            

            {
            

                CAAManufacturingSystemDefinitionToolbarTRACE(">>
                CAAManufacturingSystemDefinitionToolbar::CreateCommands()");
            

                // Instantiates the command
                header for the command 
            

                CATAfrCommandHeader::CATCreateCommandHeader("CAAManufacturingSystemDefinitionToolbarCmd1",
                

                                                               
                "CAAManufacturingSystemDefinitionToolbar",
                

                                                               
                "CAAManufacturingSystemDefinitionCmd",
                

                                                               
                (void *)NULL,
                

                                                               
                "CAAManufacturingSystemDefinitionToolbarHeader",
                

                                                               
                CATFrmAvailable); 
            

                CAAManufacturingSystemDefinitionToolbarTRACE("<<
                CAAManufacturingSystemDefinitionToolbar::CreateCommands()");
            

        }

    

        Here a command named  CAAManufacturingSystemDefinitionToolbarCmd1 is created,
        this comand is going to be tied with the tool bar button, as explained on the next
        section. It is located in the CAAManufacturingSystemDefinitionToolbar shared library
        and its associated class is CAAManufacturingSystemDefinitionToolbarCmd. The base
        name for resource files for this command will be CAAManufacturingSystemDefinitionToolbarHeader
        as explained later.

        Adding a Live Process Editor toolbar

    

        The visual part of the toolbar (The Toolbar itself and the button to access the
        command CAAManufacturingSystemDefinitionToolbarCmd1 created previously) is created
        on the implementation of the method: CAAManufacturingSystemDefinitionToolbar::CreateToolbars
    

    

        CATCmdContainer *CAAManufacturingSystemDefinitionToolbar::CreateToolbars()

            {

                 CAAManufacturingSystemDefinitionToolbarTRACE(">> CAAManufacturingSystemDefinitionToolbar::CreateToolbars()");

                 
                // verify if tool bar has not been created already

                     if(_pCreatedToolbar==NULL)

                         {

          // create 
        the tool bar called: CAAManufacturingSystemDefinitionToolbarToolbar

          
        
                    NewAccess (CATCmdContainer,pToolBarPPRLiveProcess,CAAManufacturingSystemDefinitionToolbarToolbar);

        

                  
        // create the command starter called CAAManufacturingSystemDefinitionToolbarStarter

          NewAccess 
        (CATCmdStarter,pCommandStarter,CAAManufacturingSystemDefinitionToolbarStarter);

        

                  
        // tell to the system that the starter we have just created start the command 
        called CAAManufacturingSystemDefinitionToolbarCmd1

          SetAccessCommand 
        (pCommandStarter,"CAAManufacturingSystemDefinitionToolbarCmd1");

                   
            

                      // adding the command starter
            as a child of the new created toolbar

                          SetAccessChild (pToolBarPPRLiveProcess,pCommandStarter);

                       
                    

                             
                        // add the tool bar,

                                     
                                // the tool bar is by default Visible and on the Top side

                                              int
                    visibleFlag = 1;

                                  AddToolbarView
                    (pToolBarPPRLiveProcess,visibleFlag,Top);

                                  CAAManufacturingSystemDefinitionToolbarTRACE("<< CAAManufacturingSystemDefinitionToolbar::CreateToolbars()");

    

              
                _pCreatedToolbar = pToolBarPPRLiveProcess;

                          return pToolBarPPRLiveProcess;

    

           
        }else

                {

                              
                        // in case it was already created (this function was already called) return NULL,
                        

                                  // that avoids the creation
                        of the toolbar two times

                                     
         return NULL;

            }

    

        }

        User Interface

    

        There are two visible objects: the toolbar container and its command.

    

        The ToolBar Container

    

        The title of the toolbar named CAAManufacturingSystemDefinitionToolbarToolbar is defined
        on the file DELPPRLiveProcessWorkbenchAddinAdapter.CATNls which can
        be found in install_root/CAADELPPRLiveSystemItf.edu/CNext/resources/msgcatalog
        directory. The name of this file is predertermined ans it should be used as it is.
        All the custom tool bars that you will define should define its names on this file,
        following the same format:

    

    
 NAME_OF_THE_TOOLBAR.Title = "Localized Title Text";

    
On this example the title is defined as follows: 

CAAManufacturingSystemDefinitionToolbarToolbar.Title = "CAA Manufacturing System Definition Workbench Addin Toolbar";

    
Note: The NAME_OF_THE_TOOLBAR part is on this
        case replaced by: "CAAManufacturingSystemDefinitionToolbarToolbar" This is the
        name given to the toolbar when creating its container using the NewAccess macro,
        on the CreateToolbars function.

    

        The Commands

    

        CAAManufacturingSystemDefinitionToolbarCmd1 resources are defined by three files:

    
    
        
  1. CAAManufacturingSystemDefinitionToolbarHeader.CATNls (located in the

            directory):
            
    CAAManufacturingSystemDefinitionToolbarHeader.CAAFldUIFolderWorkbenchAddinCmd1.Title = "Example LPE Command";
CAAManufacturingSystemDefinitionToolbarHeader.CAAFldUIFolderWorkbenchAddinCmd1.Help = "Example LPE Command detailed help";
CAAManufacturingSystemDefinitionToolbarHeader.CAAFldUIFolderWorkbenchAddinCmd1.ShortHelp = "Example LPE Command short help";
    
            
    
                These texts correspond to the screen tip, the documentation line and the command
                prompt input.
    
            
    
                Note: The name "CAAManufacturingSystemDefinitionToolbarHeader" is the name given
                when calling CATAfrCommandHeader::CATCreateCommandHeader function. Please refers
                to the Code.
