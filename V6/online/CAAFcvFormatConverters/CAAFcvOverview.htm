Abstract

Format conversion services offer a set of Apis and components enabling the Implementation of Converters between Native Apps formats and ENOVIA V6

They  provide:

 - Common conversion Engine for all Converters based on high level semantic C++ APIs

 - Convert from/to Native Apps format to/from V6

 - Extendable Engine for all kind of semantics.

 

The value of these services is to provide thru an open architecture: 

      A reusable V6 writer in the purpose of the implementation of a Converter from Native App format to V6 (import scenarios)

      A reusable V6 reader in the purpose of the implementation of a Converter from V6 to a Native App format (export scenarios)

      A comprehensive Integration infrastructure enabling to plug dynamically a converter in one of the Following main scenarios:

      - Import of Native xAPP Format From a xPDM, or File

      - Export of Native xAPP Format to xPDM or File
