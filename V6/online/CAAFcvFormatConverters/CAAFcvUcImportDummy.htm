What You Will Learn With This Use Case

This use case is intended to show you how to implement a native APP format Reader to V6.
This use case implements several Format Conversion Services APIs dedicated to provide the information from the input file.
This use case exposes the main domains of Mechanical CAD: product structure, boundary and tessellated representations.

The CAAXCADInputDocument Use Case

CAAXCADInputDocument is an use case of the CAAXCADInputDocument.edu framework that illustrates Format Conversion Services capabilities for importing external CAD data in V6.

What Does CAAXCADInputDocument Do

CAAXCADInputDocument simulates an implementation of a ProE reader, to be used in V6 import scenarios.

This example illustrates the previous sections about the conversion of App format to V6, and the plug of this converter in V6.

In particular, the following APIs are implemented :
- CATIExchangeInputDocument,
- CATIExchangeProductRepresentation, CATIExchangeProduct, CATIExchangeProductInstance
- CATIExchangeBoundaryRepresentation, CATIExchangeBody, CATIExchangeShell, CATIExchangeFace, CATIExchangeEdge, CATIExchangeLine, CATIExchangeLoop, CATIExchangePCurve, CATIExchangePlane
- CATIExchangeTessellatedRepresentation

How to Launch CAAXCADInputDocument

To launch CAAXCADInputDocument, you will need to set up the build time environment, then compile CAAXCADInputDocument along with its prerequisites, set up the run time environment, and then execute the use case [1].

As this use case is made up of a DLL, you have to launch it from CATIA (3DEXPERIENCE exe).
Then, select PLM Access Import from File command and choose "PROE_Dummy" usage.
Kind of supported samples (create an empty file with following name, depending of your use case):
- ProEAssembly.asm : to simulate the import of an Assembly.
- ProEBRepPart.prt : to simulate the import of a Part with exact geometry.
- ProETessPart.prt : to simulate the import of a Part with tessellated geometry.

Where to Find the CAAXCADInputDocument Code

The CAAXCADInputDocument use case is made of several classes/a single file located in the CAAfromPROE.m module of the CAAXCADInputDocument.edu framework: InstallRootFolder\CAADoc\CAAXCADInputDocument.edu\CAAfromPROE.m\

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.
