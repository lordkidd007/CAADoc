Abstract

This article describes an example of a converter implementation, using Format Conversion services APIs.

The export scenarios addressed in this example are:

- PLM Access Export from File in V6

What You Will Learn With This Use Case

This use case is intended to show you how to implement a native APP format Writer from V6.
This use case implements Format Conversion Services APIs dedicated to retrieve the information from V6 and generate the output file.
This use case exposes the main domains of Mechanical CAD: product structure, boundary and tessellated representations.

The CAAXCADReader Use Case

CAAXCADReader is an use case of the CAAXCADReader.edu framework that illustrates Format Conversion Services capabilities for exporting external CAD data from V6.

What Does CAAXCADReader Do

CAAXCADReader simulates an implementation of a writer, to be used in V6 export scenarios.

This example illustrates the previous sections about the conversion of App format from V6, and the plug of this converter in V6.

In particular, the following API is implemented :
- CATIExchangeOutputDocument.

How to Launch CAAXCADReader

To launch CAAXCADReader, you will need to set up the build time environment, then compile CAAXCADReader along with its prerequisites, set up the run time environment, and then execute the use case [1].

As this use case is made up of a DLL, you have to launch it from CATIA (3DEXPERIENCE exe).
Then, open the data you want to export, and select PLM Access Export File command.
The usage to choose is "SimulAlien".
The generated file .xcaddump will expose all the information retrieved by this sample.

Where to Find the CAAXCADReader Code

The CAAXCADReader use case is made of several classes/a single file located in the CAAXCADDumpCommon.m module of the CAAXCADReader.edu framework: InstallRootFolder\CAADoc\CAAXCADReader.edu\CAAXCADDumpCommon.m\

where This article describes the App format writer implementation.
