Basic Objective

Basic objective of this functionality is to provide monitoring capability to user. This can be used as alternative way for monitoring gateway operation by using gateway administration console.

This functionality enables automation of gateway monitoring. User can write his own application to automate monitoring capability.

Overview of gateway monitoring is given in techincal article [1].

Basic Scenario

This is a simple Java batch usecase illustrating methods mentioned in IXPGMonitoringService0.wsdl interface. To use this web services we have to follow below mentioned steps :

  • Instantiating Proxy
  • Perform different monitoring options

Instantiating proxy involves connecting to gateway URL through web services and providing handle for interface IXPGMonitoringService. We can perform different operations mentioned in interface by using this handle.

IXPGMonitoringService interface provides following facilities for gateway monitoring :

  • Retrieving gateway status
  • Restore gateway
  • Get list of queues
  • Retrieve count of events (User need to mention the search criteria for this operation)
  • Retrieve event (User need to mention the search criteria for this operation)
  • Retrieve event information
  • Retrieve event state
  • Remove event
  • Retry event

Searching Events

In Gateway Monitoring usecase we have illustrated retrieving event with only one criteria of event name and event state.

Gateway monitoring provides capability of retrieving events by using different criteria as mentioned below :

  • QueueType
  • QueueState
  • QueueName
  • EventState
  • Event Name
  • Creation Date
  • Event Procesing Result
  • Processing Return Code
  • Event Size
  • Event Duration

This usecase is illustrating retrieving events with these mentioned criteria. Refer to usecase article [3] for details.
