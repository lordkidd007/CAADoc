Preparing for Native Apps Development Toolset Installation
 

This article helps you determine if a clean up of any previous Visual Studio customization with CAA V5 RADE Products, V6 ENOVIA Studio, and/or 3DEXPERIENCE Native Apps Development Toolset is necessary, and shows you how to run this clean up prior to install a new  3DEXPERIENCE Native Apps Development Toolset release level.

You may have a previous V6 ENOVIA Studio/3DEXPERIENCE Native Apps Development Toolset customization already running with a different Visual Studio version than the one required by the current release. You must always use the highest release levels of Visual Studio and Native Apps Development Toolset customization to work with the different CAA installations you are dealing with.

In addition, if you develop applications running on top of V5 and V6/3DEXPERIENCE, you must use the highest levels of Visual Studio and 3DEXPERIENCE Native Apps Development Toolset customization for developing both V5 and V6/3DEXPERIENCE applications, without taking care of CAA V5 RADE requirements.

Nevertheless, you should not uninstall previous Visual Studio, CAA RADE, and V6 ENOVIA Studio/3DEXPERIENCE Native Apps Development Toolset release levels, since all are needed to access the build tools the Visual Studio customization require for these levels, such as the compilers.

As a rule, you should have all the Visual Studio, V5 CAA RADE, V6 ENOVIA Studio, and 3DEXPERIENCE Native Apps Development Toolset installed to match all the V5, V6, and 3DEXPERIENCE release levels on top of which you develop and maintain your applications, but only the latest Visual Studio customized with the latest available customization.

If in addition you want to stop supporting a previous release level of your applications, then you can uninstall the run time, CAA APIs, and either RADE or ENOVIA Studio/Native Apps Development Toolset for this release level after having run a clean up, if this clean up is necessary. See Uninstalling Native Apps Development Toolset.

Before you begin:
  • Before starting the installation, refer to the Program Directory where additional useful information could be given
  • To prevent the installation from hanging due to concurrently running programs such as screen savers or virus scanning programs, we recommend that you first shut down any such programs.
  • Once this or these cleaning steps are completed, install Native Apps Development Toolset. See Installing Native Apps Development Toolset.

Clean Up VS 2005

This clean up is required if you have a customization of Visual Studio 2005 for a V5 release, up to V5R20, and/or a V6 release up to V6R2010x.

  1. Log on as an administrator.
    You must belong to the Administrators group, or have the privileges assigned to the Administrators group. Otherwise, you will not be able to remove the add-ins from Visual Studio.

  2. Start Visual Studio 2005. With Windows 7, use the Run as Administrator command.

  3. Remove the commands from the menus. Since you can have up to three add-ins installed, up to three remove commands may be necessary.

    On the Tools menu, successively click:

    • Remove Code-Generator Add-in commands from menus
    • Remove Code-Designer Add-in commands from menus
    • Remove Mkmk Add-in commands from menus.

    The commands belonging to these add-ins are removed from the menus.

  4. Still in Visual Studio 2005, disable the add-ins from being loaded.

    On the Tools menu, click Add-in Manager.
    The Add-in Manager dialog box opens.

  5. Uncheck each CAA RADE add-in, in both the Available Add-ins column and in the Startup column.

  6. Click OK.

  7. Exit Visual Studio 2005.

  8. Unregister the add-ins.

    Open an explorer, locate, and double click the following file:

    InstallRootFolder\intel_a\resources\CATVBTServer\UninstallAddinVS2005.reg

    Where InstallRootFolder is the folder where you have installed Development Tools (default is C:\Program Files\Dassault Systemes\B214 for V6R2013x).

    The add-ins are unregistered.

Clean Up VS 2008

A clean up is always required:

  • If you have a previous customization of Visual Studio 2008 with add-ins for a V5 release, starting with V5R21, and/or a V6 release starting with V6R2011, run the procedure below.
  • If you have a previous customization of Visual Studio 2008 for V6 with the package installed from the MSI (starting with V6R2011x, and up to V6R2013x), simply uninstall this package. See in Installing Native Apps Development Environment the section dealing with uninstalling Native Apps Development Environment.

  1. Log on as an administrator.
    You must belong to the Administrators group, or have the privileges assigned to the Administrators group. Otherwise, you will not be able to remove the add-ins from Visual Studio.

  2. Start Visual Studio 2008. With Windows 7, use the Run as Administrator command.

  3. Remove the commands from the menus. Since you can have up to three add-ins installed, up to three remove commands may be necessary.

    On the Tools menu, successively click:

    • Remove Code-Generator Add-in commands from menus
    • Remove Code-Designer Add-in commands from menus
    • Remove Mkmk Add-in commands from menus.

    The commands belonging to these add-ins are removed from the menus.

  4. Still in Visual Studio 2008, disable the add-ins from being loaded.

    On the Tools menu, click Add-in Manager.
    The Add-in Manager dialog box opens.

  5. Uncheck each CAA RADE add-in, in both the Available Add-ins column and in the Startup column.

  6. Click OK.

  7. Exit Visual Studio 2008.

  8. Unregister the add-ins.

    Open an explorer, locate, and double click the following file:

    InstallRootFolder\intel_a\resources\CATVBTServer\UninstallAddinVS2008.reg

    Where InstallRootFolder is the folder where you have installed Development Tools (default is C:\Program Files\Dassault Systemes\B214 for V6R2013x).
