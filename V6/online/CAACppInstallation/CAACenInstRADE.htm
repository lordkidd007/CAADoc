Installing Native Apps Development Toolset
 

This task explains how to unload the Native Apps Development Toolset code from scratch on a single computer running a supported Windows operating system.

Installation and uninstallation rely on Windows-compliant tools enabling anyone familiar with Windows procedures and concepts to install the software without assistance.

Before starting the installation, refer to the Program Directory to check you have all the software prerequisites.

You must also have your license enrollment certificate (in electronic format) provided by your vendor. If you have the certificate, you will be able to register your license during the installation procedure.

Note also that, if a license server is running on the computer on which you are installing the software, you must stop the server before starting the installation.

Before you begin:
  • Before starting the installation, refer to the Program Directory where additional useful information could be given.
  • You must also have carried out the procedure described in Preparing for Native Apps Development Toolset Installation. This is very important to have Visual Studio correctly cleaned up from previous add-ins or packages.
  • To prevent the installation from hanging due to concurrently running programs such as screen savers or virus scanning programs, we recommend that you first shut down any such programs.
  • Additional steps are required to enroll your licenses and activate the customization in Visual Studio.

Installing the Files

  1. Log on as an administrator.

    You must belong to the Administrators group, or have the privileges assigned to the Administrators group. Otherwise, you will not be able to start the installation.

  2. Insert the CD-ROM into the drive.
    If needed double-click on setup.exe
    The setup program starts. A prompt asks you to run this command as an administrator. Click Yes in this prompt. The Welcome dialog box opens.

  3. Click Next.
    The setup program starts and the Choose Destination Location dialog box opens, displaying the default destination folder:

    C:\Program Files (x86)\Dassault Systemes\B417

  4. If the default destination folder is suitable, click Next, or type a new folder name, or click Browse, navigate to select another existing folder, and click Select Folder. Then click Next.

    The Select Processes dialog box opens. SDV - Application Development Suite is selected.

  5. Click Next.
    The Setup Type for Tool Configuration Management dialog box opens.
    Leave No Server Setup checked, which is the default. Refer to the Using TCKs for Accessing Tools for more information and possible future customization.

  6. Click Next.

  7. The Start Installation dialog box opens. It summarizes what will be installed.

  8. Click Install.
    The installation begins. A progress indicator shows you the installation progress, and the files are shown as they are installed. When the installation is done, the Setup Complete dialog box opens.

  9. Click Close.
    The installation is complete.

Additional Steps

Once the installation is complete:

  • Set up the licenses you wish to reserve for each Native Apps Development tool, for each user, and for each operating system. See Setting up Native Apps Development Toolset Licenses.
  • Activate the Visual Studio customization:

    1. Install the VSIX containing the Visual Studio customization. See Installing Native Apps Development Environment.

    2. Install the documentation. See Installing Native Apps Development Environment Documentation.

  • Check that the tools initialization command path (tck_init) referred to by Visual Studio is the correct one, and change it if it is wrong:

    1. On the Tools menu, click Options.
      The Options dialog box opens.

    2. In the left pane, click 3DEXPERIENCE Open Development Toolset.

    3. In the Tools Initialization Command Path box, check that the path displayed matches the one of your current installation, or change it to match it otherwise.

      The correct path is InstallRootFolder\intel_a\code\command\tck_init.bat, where InstallRootFolder is the folder where you have installed the Native Apps Development Toolset (default is C:\Program Files (x86)\Dassault Systemes\B417).
