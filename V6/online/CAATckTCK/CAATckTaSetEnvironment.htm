Getting Access to the Native Apps Development Toolset
Quick Reference

Abstract

The purpose of this article is to explain how to set up your environment to gain access to the Native Apps Development Toolset.

Unloading a Development Toolkit CD-ROM

Before unloading the Native Apps Development Toolset CD-ROM, the installation program asks for the folder where data will be copied. In this article, this folder is referenced as InstallRootFolder.

For each different CD-ROM, a label is generated to identify the level of tools to be installed. This label is known as a Tool Configuration Key or TCK.
Once the installation is completed, this TCK can be used to specified the level of tools to be used.

The naming of the TCK is something like V6Ryyyy_Bnnn where where yyyy is the release, such as 2011 or 2011x, and Bnnn is the default installation folder, such as V6R2011x_B210.

Purpose and Location of the tck_init Profile

The tck_init profile is a profile that is used to setup the working environment to have access to TCK commands. Once this profile has been executed, TCK commands can be launched without knowing their exact location. With Windows 32-bit or 64-bit, the path is InstallRootFolder\intel_a\code\command\tck_init.bat.

List and Select TCKs

Among TCK commands, two of them are interesting for setting up an environment:

  • tck_list lists the available TCKs, that is the different set of tools that can be used on a machine.
  • tck_profile sets up the current environment for a given TCK, then all tools associated with this TCK will be available.

To be able to execute these commands, first run tck_init from a command prompt.

The way to set up the current environment is the same whatever the current platform is. Given a TCK the command to execute is:

tck_profile TCK

Accessing Tools from the Native Apps Interactive Development Environment

When working with the Native Apps Interactive Development Environment, the available TCKs can be selected when opening a workspace. In this case, there is no need neither to know where the tck_init file is located nor to execute the tck_profile command. Please refer to the appropriate documentation to learn more about using Native Apps Development Toolset tools from within the Native Apps Interactive Development Environment.

In Short

The first command to run is tck_init, then run tck_profile by specifying the TCK referencing the tools to be used.
