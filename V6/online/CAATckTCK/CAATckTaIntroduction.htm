Introducing TCK
Technical Article

Abstract

TCK stands for Tool Configuration Key and is a label for identifying the installation of one tool or a set of tools.
The purpose of TCK commands is to manage various tools' installations and to help setting up a working environment by learning the less about tools' installations.

A TCK Sample

A TCK is a string used for registering a new level of tools and for referencing these tools afterwards.

The TCK corresponding to the installation of a brand API is something like V6Ryyyy_Bnnn where yyyy is the release, such as 2014 or 2014x, and Bnnn is the default installation folder, such as V6R2014x_B216.

Do I Need to Learn TCK Commands before Working?

If you intend to work either in a shell window (on UNIX platform) or in a command window (on Windows platform), you must execute the tck_init profile and then execute the tck_profile command specifying the TCK corresponding to the tool level you want to use. In this case, please read Getting Access to the Native Apps Development Toolset.

If you intend to work with the Native Apps Development Environment, then the tck_init and tck_profile commands are automatically called when opening a workspace.

What Are the Benefits of Using TCK Commands?

Referencing Several Installations

As explained above, each brand API installation is identified by a label known as a TCK.
If several installations are performed on the same machine, the installation program looks for existing TCKs and register them in the new installation. By the way, you can have several versions of tools installed on a machine and choose each of them by just specifying the associated TCK.

Adding Personal Tools

Another interesting point is that the registration of the Native Apps Development Toolset is done using TCK commands and that these commands are available for any users. By the way it is possible to add personal tools' profiles in existing TCKs and then to have only one command to run for setting up a working environment. Please consult Using TCKs for Accessing Tools to learn more about these functionalities.

It is also possible to create new TCKs from scratch.

Referencing Centralized Installations

The default proposed when installing an Native Apps Development Toolset CD-ROM is to use TCKs without any TCK server. In this case all information are found on the local machine.

It is possible to install a TCK server on a given machine whose purpose will be to gather all the TCKs corresponding to various tools on the local network. When using a TCK server, all the other machines own a TCK client installation allowing them to remotely access information managed by the TCK server. TCK can be created, modified, listed or removed from any machine on the local network.

In Short

Any Native Apps Development Toolset installation is identified by a label known as a TCK; tck_init and tck_profile are the two commands to know for accessing the Native Apps Development Toolset.

If you are working with the Native Apps Development Environment, these commands are automatically called for setting up your environment and you don't need to learn them.
