Assembly Design Removed Functions
Technical Article

Abstract

This article lists the removed functions in Assembly Design interactive product, having an impact on available APIs, and explains why these changes were made.

  • What Has Changed?

What Has Changed?

PLM transformations have required full rework of the Assembly Design application.

Some functions or concepts have been removed, and the associated CAA APIs were removed accordingly. So the CAA application code that uses these APIs must also be removed. These functions are:

  • Constraints sets: there is no need, and thus no possibility to organize constraints in sets any longer. PLM loading allows you to load only some instances, and only the loaded instances' constraints are available to the end user, removing the strong need for sets in previous versions.
  • Drafting properties: filtering of parts in drawings is done through other concepts, that can use the part's PLM attributes. Drafting properties can become one part attribute in customer's configuration, allowing more generic handling.
