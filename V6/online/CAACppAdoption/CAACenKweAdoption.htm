Knowledge Advisor

Set Management

CATICkeFunctionFactory::AddToCurrentParameterSet and CATICkeFunctionFactory::AddToCurrentRelationSet no longer exist in V5 and are replaced.

V5

CATIParmPublisher_var spRoot = ...
CATICkeFunctionFactory_var spFuncFact = CATCkeGlobalFunctions::GetFunctionFactory();

CATICkeParm_var spParm = ...
spFuncFact->AddtoCurrentParameterSet(spParm, spRoot);

CATICkeRelation_var spRelation = ...
spFuncFact->AddtoCurrentRelationSet(spRelation, spRoot);

V6

CATIParmPublisher_var spRoot = ...
CATIKweModelServices_var spModelSvcs = CATCkeGlobalFunctions::GetModelServices();

CATICkeParm_var spParm = ...
CATIParmPublisher_var spParameters = spModelSvcs->GetCurrentSet(CATIKweModelServices::Parameter,
	spRoot);
spParameters->Append(spParm);

CATICkeRelation_var spRelation = ...
CATIParmPublisher_var spRelations = spModelSvcs->GetCurrentSet(CATIKweModelServices::Relation,
	spRoot);
spRel	spRoot);
spRelations->Append(spRelation);

Knowledge Expert

Model Changes

In CATIA V5, there could only be one rulebase in a document. Moreover, that rulebase was directly under the Relations node.

In CATIA V6, there are two types of rulebases:

  1. The ones saved inside a representation reference : there can be several rulebases of this kind in a single representation reference. Those rulebases are under a common RuleBases node, that node being directly under the Relations node.
  2. The ones directly saved in the database, as PLMRuleSet objects.

Rulebase Import from a File

The file containing the rulebase has to be imported in the database, so a representation reference is created, containing that rulebase.

V5

// Creation of the rulebase
CATIRuleBase_var spRuleBase;
...

// Import the content of another rulebase
spRuleBase->Import("/path/to/a/document/containing/a/rulebase");
...

V6

// Creation of the rulebase
CATIRuleBase_var spRuleBase;
...

// Find the rulebase in that representation reference
CATIRuleBase_var spRuleBasetoBeImported;
...

spRuleBase->ImportRuleBase(spRuleBasetoBeImported);
...

Linked Rulebases

Rulebases linked to rulebases in another representation reference, are no longer supported. Now, they have to be linked to PLMRuleSet objects.

V5

// Ex. 1 : Creation of the linked rulebase
//----------------------------------------
CATIRuleBase_var spRuleBaseFactory = ...;
CATIRuleBase_var spOtherRuleBase = ...;
CATIRuleBase_var spRuleBase = spRuleBaseFactory->CreateLinkedRuleBase("MyRB", spOtherRuleBase);
...

// Ex. 2 : Import with link
//----------------------------------------
CATIRuleBase_var spRuleBaseToBeLinked = ...;
CATBaseUnknown_var spPartWhereToCreateALinkedRuleBase = ...;
spRuleBaseToBeLinked->ImportWithLink(spPartWhereToCreateALinkedRuleBase);

V6

// Ex. 1 : Creation of the linked rulebase
//----------------------------------------
CATIRuleBase_var spRuleBaseFactory = ...;
CATIRuleBase_var spRuleBase = spRuleBaseFactory->CreateLinkedBase("MyRB");

// Fetch the PLMRuleSet
CATIRuleSet_var spPLMRuleSet = ...;
spRuleBase->ImportRuleSet(spPLMRuleSet);
...

// Ex. 2 : Import with link
//----------------------------------------
CATIRuleSet_var spPLMRuleSetToBeLinked = ...;
CATBaseUnknown_var spRepRefWhereToCreateALinkedRuleBase = ...;

CATIKweModelServices_var spModelSvcs = CATCkeGlobalFunctions::GetModelServices();
CATIParmPublisher_var spRulebases = spModelSvcs->GetCurrentSet(CATIKweModelServices::RuleBase,
	spRepRefWhereToCreateALinkedRuleBase);
CATIRuleBaseFactory_var spRuleBaseFactory = spRulebases->GetContainer();
CATIRuleBase_var spLinkedRuleBase = spRuleBaseFactory->CreateLinkedBase("MyRB2");
spRulebases->Append(spLinkedRuleBase);

spLinkedRuleBase->ImportRuleSet(spPLMRuleSetToBeLinked);
