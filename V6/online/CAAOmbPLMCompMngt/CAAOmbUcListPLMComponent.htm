Retrieving PLM Component
Use Case

Abstract

The use case is an interactive command which retrieves and displays all the PLM References and PLM Representation References, as PLM Component, currently loaded in the session.

What You Will Learn With This Use Case

The use case educates the user to understand what are exactly the PLM Components representing a PLM References or a PLM Representation within an interactive session.

The CAAOmbListPLMComponen Use Case

CAAOmbListPLMComponent is a use case of the CAAObjectModelerBase.edu framework that mainly illustrates CATPLMComponentInterfaces frameworks capabilities.

What Does CAAOmbListPLMComponent Do

The CAAOmbListPLMComponent use case is fundamentally an interactive command for displaying the list of PLM Components (PLM Ref and PLM Rep Ref) the current session.

This command is added to the "CAA PLM Persistency" toolbar a part of the PLM Client Adapter Add-In to the General Workshop within CATIA. The Tools --> Customize --> Toolbar tab lists that toolbar, as seen in Fig. 1 below.

How to Launch CAAOmbListPLMComponent

To launch CAAOmbListPLMComponent , you will need to set up the build time environment, then compile CAAOmbListPLMComponent along with its prerequisites, set up the run time environment, and then execute the use case [3].

Launch a session, when the application is ready:

  • Connect to the database.
  • Select your environment.
  • Open any PLM object.
  • On the Tools menu click Customize and check that the CAA PLM Persistency (see Add-in Access) is existing ,as shown in Fig. 1 above. User can set the visibility of the created toolbar in the View->Toolbars option.

The What Does CAAOmbListPLMComponent Do section provides further details on the steps ahead.

Add-in Access

The PLM Client Adapter Add-In to the General workshop within CATIA, is available thanks to a data extension of the CAAOmbGeneralWksAddin component. This necessitates the following entries in the .dico file.

...
#CAAOmbGeneralWksAddin CATIWorkbenchAddin libCAAOmbGeneralWksAddin
#CAAOmbGeneralWksAddin CATIAfrGeneralWksAddin libCAAOmbGeneralWksAddin
...

Before launching CATIA, uncomment the above lines in the CAAObjectModelerBase.edu.dico. These primarily indicate that the add-in is fundamentally a CATIWorkbenchAddin and a CATIAfrGeneralWksAddin implementation residing in the CAAOmbGeneralWksAddin shared lib.

Step-by-Step

The only most important step in the CAAOmbListPLMComponent use case is:

  1. Listing the PLM Components Loaded in the Current Session

Listing the PLM Components Loaded in the Current Session

The core implementation of this use case which lists the PLM Components loaded in the current session is fundamentally a callback invoked in response to an Update button click event of the dialog-box based command.

...
CATLISTV(CATIPLMComponent_var) oList;
HRESULT hr = CATPLMComponentInterfacesServices::GetPLMComponentsInSession(oList);
if (SUCCEEDED(hr))
{
  for (int i=1; i<=oList.Size(); i++)
  {
    CATIPLMIdentifierSet* piIdentifier = NULL;

    CATIPLMComponent_var spPlmComp = oList[i];
    hr = spPlmComp->QueryInterface(IID_CATIPLMIdentifierSet,(void**)&piIdentifier);
    if (SUCCEEDED(hr))
    {
      CATUnicodeString strIdentifier;
      hr = piIdentifier->GetIdentifierSet(strIdentifier);
      if (SUCCEEDED(hr))
      {
        _pMultiListResult->SetColumnItem(0,strIdentifier);
...

The dialog-box based command CAAOmbListPLMComponentCmd has an Update button, whose click event is associated with a callback namely ListPLMComponentsInSession API, a member of the CAAOmbListPLMComponentCmd class.

The code above is in fact an implementation of the CAAOmbListPLMComponentCmd::ListPLMComponentsInSession method.

Its here that we retrieve the list of the PLM components (PLM Reference and PLM Representation Reference only) loaded in the current session as oList.

The Identifying attribute of each of element of this list is then sought and displayed through the multi-list element within the dialog box namely _pMultiListResult, a CATDlgMultiList instance.

In Short

This use case shows you that the GetPLMComponentsInSession method of the CATPLMComponentInterfacesServices class retrieve all the PLM reference and PLM Representation reference loaded in session as PLM Component.
