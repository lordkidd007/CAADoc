Basic Objective

Basic objective of this functionality is that user will be able to exchange data between any PDM and 3DEXPERIENCE Platform. Overview of this integration is explained in [1] technical article.Here,we have illustrated this functionality by using CAA APIs.

To import or export data between two PDMs PLMBatch can be used. SendToXPDM API is useful only in export scenario.

Basic Scenario

This section details all necessaary steps of exchange scenarios. We can exchange data between and PDM and 3DEXPERIENCE Platformby using following two ways :

Exchange Using PLM Batch

PLMBatch is a batch built on architecture.This batch is dedicated exclusively for querying and saving PLM components in PLM database. This batch is able to exchange the data between two PDMs without event bus connectivity. This batch can be used for both export and import of data.

Here we are using methods of CATPLMBatch API to exchange data. Basic steps involved in these exchange are listed below :

  • Generate input parameter XML file
  • Instantiate CATPLMBatch class with PLMBatch name to be used
  • Query PLM Component from database to be exchanged and set it in PLMBatch
  • set applicative parameters to PLMBatch
  • Generating parameter output XML file
  • Launch and Monitor Batch

PLMBatch does processing of events before sending the event to event bus so it bypasses dispatch and monitoring capability delivered by event bus.

Exchange Using SendToXPDM  API

SendToXPDM is a C++ API used at 3DEXPERIENCE Platform side. This API can be used from 3DEXPERIENCE Platform client machine.

This API takes full advantage of Event bus dispatch and monitoring capability. Users can write custom code to use this C++ API for initiating 3DEXPERIENCE Platform to XPDM exchange.

As name suggests this API is useful only in Export scenario. Basic steps involved in these exchange are as follows :

  • Instantiate SendToXPDM
  • Set Target Site ID
  • Query PLM components from the database and Set PLM Component to be transferred
  • Set Applicative Parameters
  • Invoke Send Method

Basic steps involved during export of data between X-PDM and 3DEXPERIENCE Platform are explained in Export Scenario section of technical article [2].  

In Short

This article describes about exchange of data between any PDM and 3DEXPERIENCE Platform by using CAA APIs. User can write his own custom code by using these APIs to exchange the data.

This article explain basic steps involved in exchange scenario.
