X-PDM Error Code
Use Case

Abstract

This article describes about error code retrieved in X-PDM exchanges. Purpose of this article is to help users in analyzing such errors. User can able to decode error faced in processing of X-PDM. 

Error Codes

The new SendToXPDM C++ API delivered in XPDM product enables to automate 3DEXPRIENCE Platform to XPDM exchange while taking full advantage of the Event Bus dispatch and monitoring capability.

This is achived with SendToXPDM::Send method. Failure of this method can be retrieved with CATError handle.Errors and warnings retrieved during this invocation are as follows :

Error Messages with ID

Error Id Error Message
1 Input object not supported
2 No target site provided
3 Object to export has not been set
6 No object corresponding to provided PLMIdentificator / PLMIdentificators
441 Gateway Server returned error response
442 3DEXPERIENCE server to Gateway Server communication failed
445 XPGAdapterConfig.properties file is blank in 3DEXPERIENCE server.Write URL of Gateway Server for X-PDM into the file
443-444 3DEXPERIENCE Server Internal Error
446-450 Internal Error, please contact support

Warning Messages with ID

Id Warning Message
1011 Some of the objects corresponding to provided PLM Identificators not found
1012 Export of some of the objects failed
