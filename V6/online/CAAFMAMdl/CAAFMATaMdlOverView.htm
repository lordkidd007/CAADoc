Introduction

This article explains the basic level of understanding and usage of the generic fastener modeler. In order to set the context, the next two aspects create the background on V6 fasteners and its model.

Origin

Fig 1: V6 Fastener Modeler Positioning

The first V6 Drop of PLM Fastener model was targeting the market of fastening applications requiring the design of series of fasteners made of a single flexible 3D Shape:

  • Canonical Spot Welds, Adhesives, Sealants, for Automotive BiW fasteners.
  • Complex V-Butts, Fillet Bead Shapes for Welding F&A applications.

The decision to extend PLM Fastener model with new PLM Spot Assembly Fastener Model is the result of the DS Analysis of the Key Requirements of our Aeronautics customers:

  • Design of Fasteners (Rivet, Bolts,) that are made of one or several rigid 3D Shape components (Rivet, Screw, Nut,..) instantiated from a Catalog.
  • Unitary PLM Management and Configuration of Fasteners.

Thus, V6 Fastener Model Generation enables the support of any Fastener Application targeting Automotive, Aeronautics, and Fabrication & Assembly markets. Additionally, the CAA modeling capacity of the new fastener modeler allows partners and customers to develop their own applications of "fastener design".

Spot Assembly Fastener

The Spot Assembly Fastener is an excellent example of CAA Aero Fastener application built on top of the generic fastener modeler. This article will take help of spot assembly fastener to explain the CAA modeling capability of the generic modeler.

External View

Fig 2: Spot Assembly Fastener - External View

It is defined as a specialized PLM Product assembly instantiated and designed in the Assembly context of the Parts to join. Fig.2 illustrates the PLM / CATIA Specifications Tree Integration of PLM Spot Assembly Fastener Model. In this example, the root product contains:

  • Instances of a reference PLM Part A and a reference PLM Part B that need to be joined.
  • One instance of a reference PLM Spot Assembly "Bolt like"; Fastener "Fastener-0001" that is made of one instance of a "Screw" reference and one instance of a "Nut" reference.
  • One engineering connection that defines the physical "fasten connection" of the parts realized by the bolt.

Internal View

Fig 3: Spot Assembly Fastener - Internal View

It is a PLM Product reference specialized by the addition of a PLM extension named "PLM_Fst_SpotAssyFastener" that owns the PLM Fastener exposed attributes:

  • A Fastener Type descriptor that identifies a Standard fastener Reference Type, e.g. a "Bolt" or a "Rivet.
  • The Fastener diameter (in mm).

The PLM Spot Assembly fastener 3D & technological specifications are stored in an instanciable only once 3D shape Representation attached to the PLM Fastener Reference that includes:

  • Coupling links of Fastener to its 3D Shape components instances:
    • Each fastener coupling link to a Fastener component is implemented by a simple PLM semantic relation pointing through an instance path the target product instance.
  • Fastener technological classification attributes, for instance : Criticality, Torque.
  • Fastener reference 3D location and orientation.

The PLM Fastener connection is stored in an Engineering Connection that includes:

  • Coupling links of Fastening to the Parts.
  • Coupling link to its corresponding fastener instance that implements the connection.

A Reference Fastener Type is a customizable Design set-up table (e.g. "Fst_SpotAssy_Bolt! CL0.xls") used to assist the user to valuate the Fastener Assembly specification values. Each Reference fastener Type table can be customized to capture:

  • Fastener Classifications attributes names & values.
  • Fastener components shape characteristics & positioning rules.

The Fastener Standard is a customizable set-up table that enables the specification of:

  • The list of Company Standard Reference Types.
  • The Company Standard Fastener Types Display & FTA Formatting.
  • The Company Standard fastener design naming and check rules.

Persistent Model

Fig 4: Spot Assembly Fastener Model

The picture illustrates the Spot Assembly Fastener data model. It consists of:

  • PLMCATIAFst_SpotAssyFastener - the PLM Fastener Extension on the ASMPRODUCT. It stores PLM attributes information like the Fastener diameter and Fastener type.
  • Fst_SpotAssy - the Fastener Representation feature aggregated in the Mechanical part. It is defined by DS in the catalog file.
  • CAA_Fst_SpotAssy - the CAA Fastener Representation feature defined by the user in the catalog file customized to the user's specifications.
  • Fst_SpotAssy_Base - the parent of Fst_SpotAssy. The CAA applicative feature start-up CAA_Fst_SpotAssy should be explicitly derived from this feature in the catalog file.
  • Fst_Fastener - the root parent of Fst_SpotAssy which stores the component information.
  • Fst_Component - the fastener component feature which has the component links information.
  • Fst_Coupling - the fastener coupling feature instantiated inside the engineering connection. This feature holds the fastener realization link information.
  • Fst_Zone - the fastener zone stored inside the Fst_Coupling feature as an attribute. This feature holds the fasten link information.

Capability

This generic fastener modeler delivers the CAA capability of software openness dedicated to:

  • Authoring fasteners.
  • Browsing fasteners.
  • Allowing specific fastener behavior activation / deactivation (choice of 3D Visualization for example).

Authoring APIs

The following pictures display the implementation of different authoring APIs on fastener data

Fig 5: Authoring APIs

These APIs allow creation and edition of fastener data. They are used to:

  • Set fastener product level information.
  • Set fastener 3D & technological specifications.
  • Set fastener standard & reference fastener type information.

Browsing APIs

The following picture display the implementation of different browsing APIs on fastener data.

Fig 6: Browsing APIs

These APIs allow reading of fastener data. They are used to:

  • Get fastener product level information.
  • Get fastener 3D & technological specifications.
  • Get fastener standard & reference fastener type information.
  • Get the fastener connection, navigation and positioning.

Specific Fastener Behavior APIs

Fig 7: Additional APIs

These APIs allow activation / deactivation of specific fastener behavior. They are used to change:

  • Fastener 3D Visualization
