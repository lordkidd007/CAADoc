AppResourceSet

Element

Name AppResourceSet
Type Locally-defined complex type
Nillable no
Abstract no
Documentation Define the Resource Set of for an application. A Resource Set is defined by a list of Resource Items.

Attribute

Name Id
Type xsd:Name
Use Required
Documentation The identifier of the application resource set. The name must be unique in the runtime view. We recommend to use a name starting by the brand trigam. It will be easier to ensure the uniqueness. The xml file name must be id.xml.

XML Example

<rsc:AppResourceSet Id="XXXMyappliname" 
      xsi:schemaLocation="urn:com:dassault_systemes:AppResourceSet AppResourceSetDescription.xsd" 
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
      xmlns:rsc="urn:com:dassault_systemes:AppResourceSet">
      ...
      <rsc:SetupData .....> </rsc:SetupData>  [0..N]
     ...
</rsc:AppResourceSet>

SetupData

Element

Name SetupData
Type Locally-defined complex type
Nillable no
Abstract no
Documentation Define a resource item.

Attributes

Name Id
Type xsd:Name
Use Required
Documentation The identifier of the resource item. The name must be unique in the xml file, therefore the naming convention is less important.
Name Type
Type Locally-defined simple type
Use Required
Documentation The CAA available types are:
  • Catalog
  • Chapter
  • PLMReference
  • PLMRepReference
  • TextDocument
  • XMLDocument
  • PowerCopy
  • UserDefinedFeature
  • Sheet
Name Mandatory
Type xsd:boolean
Use optional, default = true
Documentation The Mandatory attribute allows the validation of the data connections filled by the administrator. All mandatory resource item need to have a correct assigned data.
Name Multiple
Type xsd:boolean
Use optional, default = false
Documentation The Multiple attribute indicates if the application expects to retrieve only one data, or a list to provide a choice to the end-user.

XML Example

<rsc:SetupData Id="Resource1" Type="theType" Mandatory="true|false" Multiple="true|false">
    <rsc:Parameter .....> </rsc:Parameter> [0..N]
</rsc:SetupData >

Parameter

Element

Name Parameter
Type Locally-defined complex type
Nillable no
Abstract no
Documentation A Parameter is a way to categorize several sub items for a given Resource item. The Parameters will be used by administrator to define conditions on the data connections.

Attributes

Name Name 
Type xsd:Name
Use Required
Documentation The parameter name. This name must be unique for the current resource item.
Name Type
Type Locally-defined simple type
Use Required
Documentation It is a knowledge type, the CAA available values are:
  • Boolean
  • Integer
  • Real
  • String
  • LENGHT
  • ANGLE
  • ObjectType
Name min
Type xsd:double
Use optional
Documentation Minimal possible value. It is only valid for numerical parameter. 
Name max
Type xsd:double
Use optional
Documentation Maximal possible value. It is only valid for numerical parameter. 

XML Example

<rsc:Parameter Name="Resource1" Type="typeparameter"  min="..." max="...">
    <rsc:PossibleValue>...</rsc:PossibleValue> [0..N]
</rsc:Parameter >

PossibleValue

Element

Name PossibleValue
Type xsd:string
Nillable no
Abstract no
Documentation A possible discrete value for the parameter.

XML Example

<rsc:PossibleValue>TheValue</rsc:PossibleValue>
