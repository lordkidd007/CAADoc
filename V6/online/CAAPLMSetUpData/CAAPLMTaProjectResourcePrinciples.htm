Abstract

This article presents the V6 Application resource principles, then it presents the openness for the CAA applications.

The path of this documentation is as follows: "Client Collaborative Platform/Virtual Product Modeling/Infrastructure/Project Resource Management".

Principles

V6 applications rely on standard resources that modify their behaviors and that must be identified and localized when the application is running and exploiting them. Usually these data are required to set up the application. Some concrete examples:

  • Piping and tubing applications need to access some catalogs of piping & tubing components
  • Sheet Metal Design relies on a sheet that describes the possible values and limits for bend radius
  • still, Sheet Metal Design relies on a XML file that precise the visualization conventions for the features (line types, colors, ...)

V6 applications describe the list of resources that can be customized. This "description" is build-time: it consists in to define a Resource Set where each resource (called a Resource Item) is defined by a name and a type. For DS application the resource item types can be:

  • a PLM object ( a PLM Reference, a catalog, ...)
  • A knowledge object ( User feature, power copy,... )
  • A Microsoft file ( xls, cvs,..)
  • a txt or xml file
  • ...

Once the Resource set is defined and accessible in session, an administrator can associate the end user resources with the applications using the Project Resource Management workbench.

On the right, you have the list of Resource Set, example "Knowledge Resources". To connect a set of end user resources you have to create a Resource Table. Select a Resource Set Id, and click right, the contextual menu contains the "Create a new Resource Table" command".

A Resource Table is a PLM object, so the next dialog box is as follows:

Then a dialog box is opened showing all the resource item of the Resource Set. The resource item name are inside the column "Logical Name", and their type in the column besides it. This editor enables an administrator to connect a data to each resource.

Once the data are given, you close the dialog box.

The last step consists in to associate the Resource Table to a P&0 project.

You drag and drop a Resource Table from the right side to a project (left side). Here the Resource Set CAAResourceSet41 has been linked to the Standard project. For the users connected to the database with this project, the 'Knowledge Resource" application will use the data of CAAResourceSet41 .

Openness for CAA Application

For your CAA application, DS offers (almost) the same capabilities. It means:

  • Build-time : Define your Resource Set and its item [1]
  • Run-time : Retrieve the data contained by the Resource Table linked to the current P&O project [2]

In Short

You have the capability to create you own application resource set.
