Create Extensions
Technical Article

Abstract

This article shows how to create extensions.

Note: Extensions belonging to the DS parent packages which cannot be specialized are not displayed.

  • Create a Specialization Extension
  • Create a Customer Extension
  • In Short

Create a Specialization Extension

You can create a specialization extension by selecting a DS extension from the current package. Such extensions are used by applications to add behaviors and attributes to existing objects.

Note: Specialization Extensions names shall contain alphanumeric characters and the "_" character only.
  1. Click the Extensions tab, select the extension to which you want to add a subextension, and click New.

    The Create New Customization Extension window opens.

  2. Define the extension.

    1. Enter the extension name in the Name box.
    2. Optional: Enter a comment if need be.
    3. Optional: Add an icon to your extension by entering the icon name.

      Note: Icons must be located in the following folder: <OSDir>\ resources\graphic\icons\normal. Only the .bmp and the .png formats are supported.

    4. Click Done when you are finished with the extension creation.

      The extension appears in the List of Extensions window.

      • Name corresponds to the extension name.
      • Nature shows an icon indicating if the extension is a DS extension, a customer extension, or a specialization extension.
      • Scopes corresponds to the list of types the extension can be applied to.
      • Deployed shows the deployment information (yes, no, or partial).
      • Attributes shows the number of attributes.

Create a Customer Extension

You can create a root Customer Extension or a sub-extension if a customer extension is already selected.

Note:
  • When you create a customer extension, you cannot modify its scope if a scope was already defined on the root extension.
  • Customer Extensions names shall contain alphanumeric characters and the "_" character only.
  1. Click the Extensions tab, click New.

    The Create New Customization Extension window opens.

    Note: If you create a subextension, the Parent Extension box displays the name of the root extension.

  2. Define the extension.

    1. Enter the extension name in the Name box.
    2. In the Scopes box, click ... to display the Types List. This list shows the types the extension can be applied to.

      The Types List window opens.

    3. Click one of the types and click Submit.
    4. Optional: Enter a comment if need be.
    5. Click Done when you are finished with the extension creation.

      The extension appears in the List of Extensions window.

      • Name corresponds to the extension name.
      • Nature shows an icon indicating if the extension is a DS extension, a customer extension, or a specialization extension .
      • Scopes corresponds to the list of types the extension can be applied to.
      • Deployed shows the deployment information (yes, no, or partial).
      • Attributes shows the number of attributes.
    Note:
    • You can add attributes to extensions.
    • Uniqueness keys may be delivered along with DS packages. If the root package contains unique keys, they are displayed in the Attributes tab. For more information about unique keys, see Model Customization: Unique Key.
    • Once the extensions are defined, Server must be restarted.

In Short
