Site Map
 
This site map helps you understand what you have installed from the CD-ROMs and find your way in the Encyclopedia.

Several CD-ROMs can be installed:

  1. CAA API
  2. 3DEXPERIENCE Open Development Toolset for Native Apps, usually shortened in the CAA Encyclopedia as the Native Apps Development Toolset.

The API CD-ROM contain the APIs of the V6 brands, such as CATIA, DELMIA, SIMULIA, and ENOVIA brands. It also contains use cases and the CAA Encyclopedia along with the Native Apps Development Toolset documentation. The Native Apps Development Toolset CD-ROM contains only the Native Apps Development Toolset run time part.

The brands' are made up of apps gathered in business themes/domains which are organized according to the IFWE Compass User Experience quadrants. An app may be equivalent to a modeler, or may contain several modelers. Sometimes, a modeler may span over two apps. The common services  are also made up of modelers. An app or a modeler is in turn made up of one or several frameworks that contain the API that can be provided as C++ classes and interfaces, Automation objects, and/or web services.

The Encyclopedia navigation tree is classified according to these programming languages as:

  • Native Apps C++
  • Native Apps Automation.

For Native Apps C++, the Common Services section is divided as follows:

  • C++ Programming dealing with the Object Modeler, the collections, and the C++ services.
  • Abstractions that makes up a middleware layer
  • VPM Multidiscipline Collaborations that offers the PLM Core modeler, the PLM Dictionary, the PLM server access
  • User Experience that offers the user interface modeling components for Wintop applications
  • Representation Modelers that provides the main representation modelers, such as the Geometric modeler, the Feature modeler, the Knowledge Optimization and Knowledge Expert modelers, the Mechanical modeler, the Data Upward Compatibility modeler, and the Behavior modeler. The other representation modelers are located in the brands.
  • PLM Foundations that isolates the APIs and the applications from, but altogether allows for, operating systems and other system required software.
  • PLM Services that manage the query, open, expand, save, and life cycle services.
  • PLM Modelers that provides the core PLM data modelers.
  • Interfaces & MultiCAD to exchange data between either other CAD systems or standard formats.

The Foundations section is followed by the solutions and modelers you have installed.

A specific V6 Adoption section gives you information about the new concepts your existing applications should adopt to take advantage of Dassault Systèmes' technologies, shows you how to change your code, and give you tips to use the new APIs enabling these technologies.

Then, you will find:

  • Native Apps Development Toolset that supplies the tools to design, implement, build, check, and test your C++ applications.
  • Rules and Guides that gives you tips about the API resources supplied, naming conventions, stability, and C++ coding and naming rules, features. You will also find information for Automation and Web Services.
  • References that gathers reference information for the Data Models, the C++, Automation, and Web Services API, and the settings.
  • Installations that details the installation and de-installation procedures, as well as the installation preparation for the coexistence of several Visual Studio levels.

The documentation is organized as articles classified into the following categories using dedicated icons:

Icons Article Type Content
Overview Introduction to a given modeler or topic, overall information to figure out where the topic scope is and is not, relationship with the modeler neighborhood.
Getting Started Information helping to start, without having a previous knowledge of the modeler or topic. Usually a use case.
Understanding Explanation of a concept, a mechanism, or a protocol to understand how it works, on what it relies, and how it is interrelated with others.
How To Programming macro- and micro-tasks organized as a programming guide.
Use Case Detailed description of a coded example of a modeler's typical programming scenario that gives you the way to use the APIs supplied with the modeler. The code is delivered in a .edu framework located in the CAADoc directory below the one into which you have installed the API CD-ROM. Only the source files are supplied. You can copy these frameworks in a workspace, build them, and run their use cases to learn about the modeler.
Reference Dictionary-organized information. The documentation of the classes, interfaces, methods, and global functions for each framework fall into this category.

The types Overview, Getting Started, Understanding, and How To are referred to as Technical articles.
