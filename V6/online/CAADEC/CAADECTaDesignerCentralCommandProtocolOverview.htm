X-CAD Command Protocol - An Overview
Technical Article 

Abstract

3DEXPERIENCE Platform X-CAD is a framework which enables you to closely integrate your applications (CAD, non-CAD) with the 3DEXPERIENCE Platform products. This article provides an overview of the Design central protocol overview, its various components and the steps involved to realize this integration.

Introduction

This document briefly discusses the protocol used between CAD Integration applet and the Client Specific Extension in the Client Tool.

The Client Specific Extension will be referred as CSE henceforth in this document.

Classification

Depending upon the number of interactions between the applet and CSE, one can classify the commands as

  • Single interaction command:
    • CSE will send a command and gets a reply from applet
  • Multiple interaction command:
    • CSE will send a command. Applet will request for further information. CSE will send more information. This loop will continue till applet stops the response by saying command is complete/terminated.

Thumb rules in command interactions

Some of the thumb rules for command interactions are as mentioned below

  • CSE initiates commands. Applet will send the last response and terminates/completes the command.
  • CSE will NOT send other command unless it gets the response from the previous command
  • Most of the tag and attributes are of lower case.
  • CSE can directly send these commands through the socket

Or

Optionally it can write these commands into a file on local machine and specify its detail as follows in the xml request packet sent through socket

<?xml version="1.0" encoding="UTF-8" ?> 
<request dirname="C:\Program Files\MxSwInteg" filename="Request.xml" />

In such a case the Applet will write the response into this file and specify its detail as follows in the xml response packet sent through socket 

<?xml version="1.0" encoding="UTF-8" ?> 
<response filename="Response.xml" dirname="C:\Program Files\MxSwInteg"></response>

The CSE will need to read this response for the sent commands from the specified file on the local machine.

Types of commands

Depending upon the nature/execution of the command, we can classify commands into 2 categories.

  • Command
    • They are the statements (commands) like “Login”, “CheckIn”. These commands will start an interaction. In many case, they may end up in single interaction
  • Response
    • These are the commands following the initial command. When applet requires additional information, it requests the CSE for information. CSE will send the commands by name “response”. Response commands will not start any new interaction. They will continue the communication. Response commands will have an attribute which mentions for which command, it sends information
