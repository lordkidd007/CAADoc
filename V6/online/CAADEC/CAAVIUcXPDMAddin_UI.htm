MyX-PDM menu in MyX-CAD Application
Use Case

Abstract

This article describes how to store the file data in the MyX-PDM through the MyXCAD Application and integrate the data with the ENOVIA V6 through the MyX-PDM

  • Basic Objective
  • In Short
  • References

Basic Objective

The basic objective of this functionality is that, the user will be able to manage the files of MyXCAD Application in the MyX-PDM. and to integrate this data from MyX-PDM to the ENOVIA V6 server.

Launch MyX-PDM

When we click on the MyX-PDM menu in menu bar the following window will appear which include the MyX-PDM menu [Fig.1].

  • Connect to My-XPDM
  • Chekin to My-XPDM
  • Chekin as new version to My-XPDM
  • Checkout from My-XPDM
  • Disconnect of My-XPDM
  • Send to Enovia V6
Fig.1 MyX-PDM Menu

Next Step when we select the Connect to My-XPDM the following window will appear for login to MyX-PDM [Fig.2].

Fig.2 Login Window

Enter the login details to connect with the My-XPDM. But before login to My-XPDM Check the following details.

  • Start the My-XPDM application.
  • Save the file locally.

After login we can aceess the other command in the menu bar which are explained below.

Chekin to My-XPDM

Checkin as new version to My-XPDM

Checkout from My-XPDM

Disconnect of My-XPDM

Before the exit of application it is required to disconnect the My-XPDM from the My-XCAD Application for smooth operation [Fig.3].

Fig.3 Disconnect of MyX-PDM

Send to ENOVIA V6

This functionality is used when we want to save our file in My-XPDM (Local Server) and ENOVIA V6 (Remote Server). When we select this tab we get the window showing the Destination directory name. this means the folder name in the remote and local server where the file with its associated file get store [Fig.4].

In same window we have to enter the value for the XPDM Gateway URL and XPDM Shadow directory which are mandatory field.

For sending this file information to ENOVIA V6 we need to create the XPDMXML file which will map the file in it and relationship between the file. So the ENOVIA server able to understand the file type and there relation. This XPDMXML will only generate for sending data to ENOVIA V6.

Before executing this command check the following details

  • Start ENOVIA V6 server.
  • Start the V6 Gateway at the server site.
  • Start the XPDM gateway at the client site.
  • Check both the Gateway port working smoothly on the web.

Fig.4 Connection Window with EV6

Execute the command and if all things are in there place the 'Send To ENOVIA V6' message will appear.

In Short

We will have to follow a proper sequence of information flow (through xml), in order to manage our External Application data files through ENOVIA V6. And we can manage Standalone as well as Assembly structure through ENOVIA server
