WHY Integrating Client Application with 3DEXPERIENCE Platform ?

Business processes requires a various range of capabilities. Tight integrations between applications and especially with the IP repository referential is mandatory to manage efficiently global process.

Why do I need to integrate my client application to 3DEXPERIENCE Platform?

  • My application is file based and I want to manage versioning, life cycle, change management and all capabilities enabling full design collaboration.
  • I need to be navigate on my application data within 3DEXPERIENCE Platform .
  • I want to get my application integrated with other standard 3DEXPERIENCE Platform  applications though powerful processes.
  • My application users can be in different locations and I want a centralized data repository with distributed files vaults.

HOW does X-CAD Enablers tight integration of External Client Application Within 3DEXPERIENCE Platform  ?

  • X-CAD is a comprehensive workgroup design application that allows product designers and their extended supply chains to manage the design process and collaborate from one central application, independent of the authoring applications they use. X-CAD eliminates the need to support multiple CAD workgroup data managers.
  • X-CAD can be deployed and supported from a central location, while the file locations can be distributed and synchronized around the world. From their design applications (electrical , mechanical or others) users can easily access product design information stored in 3DEXPERIENCE Platform  Live Collaboration database, and use local workspace management on their own desktop to easily compare work-in-progress to the master designs in the 3DEXPERIENCE Platform  Live Collaboration.
  • 2 users interfaces are provided :
  • Designer or application user can save , retrieve and manage their data in 3DEXPERIENCE Platform  through specific embedded commands from the Client Application.
  • Manager, project leader (or any user) can navigate on Application data and perform life cycle or various operations from standard 3DEXPERIENCE Platform web applications.

WHAT are the Components of a X-CAD based Integration ?

3DEXPERIENCE Platform  X-CAD based integration consists of components depicted below :

3DEXPERIENCE Platform  X-CAD Server Side :

  • Consists of Integration Exchange Framework (IEF) schema (for all collaborative products), JPOs, servlets, beans, property files and JSPs apart from other contents such as images and resource bundles (valid for all centrals)
  • Includes the commands and menus used to customize X-CAD (valid for all centrals)
  • Performs business operations such as creating, revisioning, modifying, connecting business objects, checking files in or out from business objects and so on (valid for all 3DEXPERIENCE Platform  centrals; i.e. Enterprise web applications)
  • Handles File Collaboration Server (FCS) infrastructure to handle multiple collaborative products on the same server (valid for all centrals)
  • Configures the product using the Global Configuration Object and Local Configuration Product (specific to X-CAD)

Integration Exchange Framework (IEF) Client Workspace Service:

  • Has no independent UI and supports all applet mode commands which do not have UI and additional commands with their UI control delegated to the client applications
  • You cannot log in to this service directly. Logging into Collaborative products connects the service and allows the collaborative products to send other commands through this service

Client side Plug In (CSE) :

  • Client side Plug In handles the User Interface for the Integration and also the Socket communication with the Client Workspace Manager (IEF Client). The IEF Client in turn communicates with the X-CAD server and with the FCS (File Collaboration Server) for file upload/download using the https protocol.
  • The Business Logic is under the control of the IEF Client while the UI is delegated to the client tool native User Interface.

HOW and WHERE to implement Application Integration ?

The steps involved in implementing an embedded integration inside a client tool with X-CAD are listed below:

  • Defining and Installing the Collaborative Product specific schema
  • Defining, Configuring and Setting Integration specific preferences
  • Creating a CSE (Client Side Extension), also referred to as the Client Plug In

Defining and Installing the Collaborative Product-specific Schema

Each collaborative product defines and installs its own schema. Typically, this integration-specific schema is derived from the base types installed by X-CAD. Technical article "X-CAD Schema for Client Integrations" [] which delves in depth, on this subject.

Defining, Configuring and Setting Integration-specific preferences

An Integration is configured and preferences specific to it are set, using the Configuration Objects. The GCO (Global Configuration Object) is at the heart of the Integration on the server side implying that it is this Business Object which primarily governs the behavior of collaborative product.

The GCO governs the data structure (schema) for each integration. It maps the types and attributes on the client side to those at the server end. Additionally a LCO (Local Configuration Object) configures an individual user's preference for a collaborative product.  

Creating the Client Side Plug In

The client side plug In implements the native User Interface which allow us to launch (AddIns) and work with the Integration commands.

It further communicates with the IEF Client using sockets. It sends a command for execution to the IEF Client. The client processes the request command and forwards it to the 3DEXPERIENCE Platform  server for execution. The response from 3DEXPERIENCE Platform  is then routed through the IEF Client back to the client. Since this communication happens in a silent mode, with the UI deployed natively by the client, these commands are also termed as "Silent" commands.

This is how Integration clients communicate with the 3DEXPERIENCE Platform  server and thus realize the client embedded integration with 3DEXPERIENCE Platform .

WHAT capabilities are provided (for free) in addition to Client side plug in?

Providing the access to all 3DEXPERIENCE Platform  Entreprise Apps value

  • Incremental update of EBOM to reflect Application product structure changes

To Conclude

In short, the article dealt with the basics of Client applications (CAD, non-CAD) Integration with 3DEXPERIENCE Platform  X-CAD. Primarily, a client PlugIn (CSE) communicates with the IEF Client (X-CAD client installation) using the socket protocol. This IEF client processes the command and forwards it to the 3DEXPERIENCE Platform  server for execution. On command execution, its response is sent back to the Client, routed through the IEF Client. This communication between the Client plugIn and the IEF Client is fundamentally XML based, whose specifications are well defined by the Silent Commands Protocol.

The X-CAD Openness for client Integration is afforded since this client plugIn-IEF Client communication is fundamentally XML based. Use Cases limit their scope to delve into the relevance of the contents of this XML, and relies on a dummy Java-based client to execute commands on the server. The real integrations will involve implementing the Client side Plug-ins, the native UI, BL to generate and parse the XML etc.
