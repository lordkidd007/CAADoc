Configuring and Setting Preferences - Introduction to GCO

Technical Article 

Abstract

3DEXPERIENCE Platform X-CAD configures the preferences for each collaborative product, in other words the X-CAD functionality based on the company's business requirements. This is realized by the Integration Administrator primarily using the Global Configuration Object (GCO) and further configure it for individual users using the Local Configuration Object (LCO).

The Global Configuration Object (GCO)

The GCO primarily:

  • The Global Config Object also known as the GCO which is the heart of the integration in the server.
  • The behavior of the IEF is governed by the GCO. The GCO is specific to each integration. The GCO governs the Data Structure for each integration, it contains the information about what object is created for each file type, how the attributes are mapped from CSE and vice versa, etc.
  • There can be many GCO for each integration and only one default GCO for each integration, which is installed by the CSE.
  •  captures the rules that map the compound document structure in the authoring tool to object structure in 3DEXPERIENCE Platform which includes the CAD type, the relationship type and the attribute mapping
  • sets global preferences
  • stores the program name that implements the key functionalities
  • In case of CAA GCO will be by name “CAANewArch”, and with Revision 1

The MCADInteg-GlobalConfig type in the 3DEXPERIENCE Platform database holds all the global settings which enable configuring each installed collaborations. For eg. an instance of this type exists for each CAD Integration namely. SWNewArch for the 3DEXPERIENCE Platform-Solidworks collaboration, CATIAV5NewArch for the 3DEXPERIENCE Platform-CATIAV5 collaboration and so on, as depicted in the Fig. that follows. The settings are stored as attribute-value pairs.

The GCO settings can be done either using the X-CAD thin client interface or one can work directly on the GCO object using the Matrix Navigator, specifically if IEF alone is installed at client site.

About Local Configuration Objects

When an Integration Administrator assigns a collaborative product to a user, 3DEXPERIENCE Platform X-CAD creates an object of type MCADInteg-LocalConfig and copies the preferences (settings) from the GCO object for that collaborative product.

The LCO also contains the user's preferences and working set, the file that tracks the objects and files that the user checks out to a local machine. The LOC settings can be done either using the X-CAD thin client interface or one can work directly on the LCO object using the Matrix Navigator, specifically if IEF alone is installed at client site.

GCO Settings

The GCO object for an integration contains the values required for a typical business process.

The key attributes defined in a GCO object are detailed below:

Attribute: MCADInteg-CADToMxAttribMapping
Function: Defines the CAD-to-Matrix attribute mapping
Syntax: CAD-Type,CAD-Attribute|Matrix-Type,Matrix-Attribute
CAD Type values: component, componentInstance, componentFamily, assembly, assemblyInstance, assemblyFamily, drawing, ALL
Examples:
component,Test01|SW Component,Test01
component,$$Author$$|SW Component,Author
component,$$Comments$$|SW Component, $$Description$$

 

Attribute: IEF-Pref-IEF-BackgroundCheckinDirectory
Function: Define the temporary directory where design files are kept during background checkin, and where success and failure logs are created

If left blank, 3DEXPERIENCE Platform X-CAD uses the User Profile Directory.
Example: For Windows XP, the background checkin directory location is, C:\Documents and Settings\<user>\IEFBackgroundCheckin\BackgroundCheckin<date_time>
For Windows Vista and Windows 7, the background checkin directory location is,
C:\Users\<user>\IEFBackgroundCheckin\BackgroundCheckin<date_time>
Default Value: Blank

 

Attribute: IEF-CreateVersionObjects
Function: Define whether or not new version objects can be created for a Revision
Possible Values:
True--more than one minor object can be created in any revision stream
False--only one minor object can be created in one revision stream and Version column is not displayed in all UI pages
Default Value: True

 

Attribute:IEF-Webforms
Function: List of Webforms forms available to the user
Examples:DECObjectProperties

 

Attribute: IEF-Pref-MCADInteg-CheckOutDirectory
Function: Define the default location for downloading files during checkout
Default Value: C:\temp

 

Attribute: IEF-Pref-MCADInteg-LockObjectOnCheckin
Function: Enable or disable the ability to retain lock after checkin
Possible Values:
True--Users can choose to retain the lock on objects when checking in
False--Users can not retain the lock.
Default Value: False

 

Attribute: IEF-Pref-MCADInteg-WarnForFileOverwrite
Function: Enable or disable warning if files will be overwritten during checkout
Possible Values:
True-- warning for overwriting the file appears during checkout
False--no warning displays
Default Value: True

 

Attribute: IEF-Pref-MCADInteg-CreateVersionOnCheckin
Function: Set the default state of the Create Version check box
Possible Values:
True--every checkin creates the next version
False--the first object creates a version, and subsequently all the files are checked into the same version overwriting the earlier design
Default Value: True

 

Attribute: IEF-Pref-MCADInteg-CheckOutDirectory
Function: Set the default Checkout directory. Users can change their default Checkout directory from the Checkout or Preferences page
Default Value: c:\temp

 

Attribute: MCADInteg-CreateDerivedOutputObj
Function: Defines if derived outputs are associated to the parent object using a relationship to another object, or as a file checked into the parent object
Possible Values:
True--3DEXPERIENCE Platform X-CAD creates an object for each derived output formats and connects it to its respective parent CAD Model using the Derived Output relationship
False--the Derived Output file is checked in into its parent CAD Model in a different format

 

Attribute: IEF-Pref-MCADInteg-LockObjectOnCheckout
Function: Enable or disable the default setting for lock object on checkout
Possible Values:
True--Object is locked by default (but can be changed)
False--Object is not locked by default
Default Value: False

In Short

The article delves into the details of the Configuration objects (GCO, LCO) to configure client integrations with X-CAD. It discusses various attributes which are set on the GCO and LCO, their relevance and their impact on the behavior of integrations.
