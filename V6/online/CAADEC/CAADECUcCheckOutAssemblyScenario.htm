Checkout From 3DEXPERIENCE Platform Assembly Scenario

Abstract

The Use Case illustrates the steps involved for an Integration to check-out files into 3DEXPERIENCE Platform. An overview of the integrations of client applications with 3DEXPERIENCE Platform using the X-CAD is detailed in the technical article [1].

The CheckOut Command

This command is invoked to checkout an object and all its children from the 3DEXPERIENCE Platform database. You check out an object when you need to download a file that has been updated by another user, or if you have not previously downloaded that file.

Typically, the checkout command is sent by CSE to the IEF client, on launching 3DEXPERIENCE Platform New, Open and Insert UI commands, same as basic scenario[2].

Note : Please note that in response to this command IEF client will send Response xml. For answers user have to read this xml and process (Parse) for further use.

CheckoutEx Request XML

On CheckoutEx request XML is sent to the IEF client. The table that follows lists these nodes (attributes) and details their impact on the behavior of the checkout command.

Node     Attribute Name Description
command    cse Name of the CSE
ver X-CAD version that the current CSE was built with and is compatible with
  cname should be checkoutEx (Mandatory)
  csever CSE (Client Side Extension) version
  expandlevel All level we can mention or number of level
  expandrel  relation of expand objects 
  selectallchildern  true/false - selection of all children below input object  
  download this is an optional boolean attribute on the cadobject node. default value is false. If it is false then the file will not be downloaded
  workingdirectory Directory names where the files to be checked out. Default value taken from LCO
cadobject name Name of the business object to checkout. All children of this object will be checked out (Mandatory)
  busid Business Object ID of the document to checkout. Not mandatory

*The Request XML can specify a JPO to retrieve the revision. The JPO should be available in 3DEXPERIENCE Platform. It should define a method "getLatestRevisionString" which will take Context and string array as arguments (Type and Object name).

Path for xml generated by MyX-CAD

XML generated by MyX-CAD application could be found at following location. (where xxx is user name and its local folder on client windows machine)

C:\Users\XXX\AppData\Local\Temp\Request_Command\Checkin_Request.xml
