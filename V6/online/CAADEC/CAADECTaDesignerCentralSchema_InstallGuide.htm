CAA Schema (CSE) Installation Guide
Technical Article 

Abstract

This article describes about installation of the  the CAA Specific schema for X-CAD Integration. This schema is added by ENOVIA X-CAD for senario, or more specifically the Integration Exchange Framework (IEF) which is installed with ENOVIA products. 

This Types are specific for CAA to run CAA Usecases.

CAA X-CAD Integration Schema

Here we illustrating CAA schema installation in detail. Basically we could consider this as sample schema based on this user could create its own schema and use for own application. This Schema creation we divided into two parts. These two steps triggered from one single TCL file (installCAASchemaInstallerV6R2014x.tcl). Before running schema modify paths inside if required.

  • CAA Type and Format creation

In CAA Type creation we are creating 4 types and 2 formats for each type/format we have created one XML . User could create its own type instead of CAA also.  Please note that only bold marked types and formats reqired for basic scenario we will discuss rest in assembly scenario.

type_CAAAssembly.xml
type_CAAComponent.xml
type_CAAVersionedAssembly.xml
type_CAAVersionedComponent.xml
format_CAAasm.xml
format_CAAcmp.xml

Then we referring all these types/formats inside SchemaManifest_CAADesignerCentralIntegration.xml. For syntax you could refer XML files. next step is run emxSchemaManifest program with input of SchemaManifest_CAADesignerCentralIntegration.xml   

exec program emxSchemaManifest -construct "C:\enoviaV6R2014x\server\Apps\BusinessProcessServices\V6R2014x\Modules\ENOSchemaInstaller\SchemaManifest\SchemaManifest_CAADesignerCentralIntegration.xml" -construct "C:\enoviaV6R2014x\server\Apps\BusinessProcessServices\V6R2014x\Modules\ENOSchemaInstaller\SchemaDefinition" -construct "C:\enoviaV6R2014x\server\Apps\BusinessProcessServices\V6R2014x\Modules\ENOSchemaInstaller\AppInstall\AppInfo\AppInfo.rul" -method installSchema "C:\enoviaV6R2014x\server\Apps\BusinessProcessServices\CustomSchema\SchemaManifest" "C:\enoviaV6R2014x\server\Apps\BusinessProcessServices\CustomSchema\SchemaDefinition";

By referring this user could modify and create its own data and type. After creation of this user could check its own type in Business.

  • Admin/config Object creation and configuration

After creation of type create Admin type and configure them. Here mainly we are Creating GCO for CAA . the we are defining Policies , Roles, relationships, formats, types and mapping between them.

For better understanding of this we could refer InstallCAASchemaObjects.tcl. Please note that if user creating its own types user have to change InstallCAASchemaObjects.tcl this file and make appropriate changes for its own type. (Refer all CAA keywords and make changes accordingly)

After creation of CAANewArchi GCO user could check inside matrix.

Steps to run Schema installation

Before running Schema installation process some basic prerequisite

  • Check installation of Enovia
  • Copy files/Folders at desired location [#]
  • Modify paths if required inside installCAASchemaInstallerV6R2014x.tcl

Run Following commands on MQL

  • Go to MQL command prompt
    •  Start->  run -> mql command (For windows) or open MQL command prompt
  • execute TCL (Check your relative path of tcl and modify if required)

"Run C:\enoviaV6R2014x\server\Apps\BusinessProcessServices\V6R2014x\Modules\ENOSchemaInstaller\installCAASchemaInstallerV6R2014x.tcl;"

Note – If error occurred to run install schema command tcl. Copy “managed” folder to C:\enoviaV6R2014x\studio\ from C:\enoviaV6R2014x\server

or modify enovia.ini file replace "studio" -> "server"

In Short

This article has detailed the schema installed by IEF in an ENOVIA stack which is CAA Specific . Integrations inherit from this schema to run CAA Usecases to define their own integration specific schema created.
