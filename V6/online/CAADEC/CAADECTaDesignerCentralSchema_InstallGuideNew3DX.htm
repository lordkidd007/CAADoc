CAA Schema (CSE) Installation Guide
Technical Article 

Abstract

This article describes about installation of the  the CAA Specific schema for X-CAD Integration. This schema is added by 3DEXPERIENCE Platform  X-CAD for senario, or more specifically the Integration Exchange Framework (IEF) which is installed with 3DEXPERIENCE Platform  products. 

This Types are specific for CAA to run CAA Usecases.

CAA X-CAD Integration Schema

Here we illustrating CAA schema installation in detail. Basically we could consider this as sample schema based on this user could create its own schema and use for own application. This Schema creation we divided into two parts. These two steps triggered from one single TCL file (installCAASchemaInstaller.tcl). Before running schema modify paths inside if required.

  • CAA Type and Format creation

In CAA Type creation we are creating 2 types and 2 formats. User could create its own type instead of CAA also.  Please note that only bold marked types and formats reqired for basic scenario we will discuss rest in assembly scenario.

Then we referring all these types/formats inside yml files. For syntax you could refer yml files.

By referring this user could modify and create its own data and type. After creation of this user could check its own type in Business.

  • Admin/config Object creation and configuration

After creation of type create Admin type and configure them. Here mainly we are Creating GCO for CAA . the we are defining Policies , Roles, relationships, formats, types and mapping between them, you will find for this tcl files inside CAA_utils directory.

For better understanding of this we could refer CollaborativeDesignForCAATeam.tcl. Please note that if user creating its own types user have to change InstallCAASchemaObjects.tcl this file and make appropriate changes for its own type. (Refer all CAA keywords and make changes accordingly)

After creation of CAANewArchi GCO user could check inside matrix.

Steps to run Schema manual installation

Before running Schema installation process some basic prerequisite

  • Check installation of 3DEXPERIENCE Platform 
  • Copy files/Folders at desired location [#]
  • Modify paths if required inside installCAASchemaInstaller.tcl

Run Following commands on MQL

  • Go to MQL command prompt
    •  Start->  run -> mql command (For windows) or open MQL command prompt
  • execute TCL (Check your relative path of tcl and modify if required)

"Run C:\enoviaV6R2015x\server\Apps\BusinessProcessServices\V6R2015x\Modules\ENOSchemaInstaller\installCAASchemaInstaller.tcl;"

Note : This TCL is provided as a sample for CAA, It creates a Types which are available for CAA. User should  create their own "TCL" to create desired types , for Details about the TCL Programming and installation Please refer to Product Doc. 

In Short

This article has detailed the schema installed by IEF in an 3DEXPERIENCE Platform  stack which is CAA Specific . Integrations inherit from this schema to run CAA Usecases to define their own integration specific schema created.
