Logout From 3DEXPERIENCE Platform
Use Case

Abstract

To begin with, we recommend users to get an overview of client integrations with X-CAD, by studying the article [1].

This Use Case details the protocol defined by X-CAD to realize Logout in from integrations (Client Specific Extensions) inside the client tools. 

Dis-Connecting to 3DEXPERIENCE Platform

For client integrations to work with 3DEXPERIENCE Platform, the last step is to logout from 3DEXPERIENCE Platform.

logout Command

The logout command closes the user session on the server.

logout Request XML

A sample logout request XML is depicted below. A sample xml also exists at CAAVIXCADIntegration.edu\InputData\Request\LogoutRequest.xml.

<command cname="logout" cse="CAA" ver="R417" csever="R417"/>

The table that follows details the relevance of each attribute contained in that xml.

Attribute Name Description
cname should be logout
cse Name of the CSE
ver X-CAD version that the current CSE was built with and is compatible with
csever CSE (Client Side Extension) version

logout Response XML on logout SUCCESS

A sample logout response XML on success exists at CAAVIXCADIntegration.edu\InputData\Response\LogoutResponse.xml.

The logout response XML has primarily the following nodes

  • response node, whose result attribute is valuated to success or failure. In case of failure, there exists a reason node, which details the reason of failure
