Set Context to user

Security Context is necessary to assign the user for exchange the data with the database. Following fig shows the How to assign the Security context to the Existing User.

Before Assigning the context make sure that:

  • User is created.
  • Particular role has been assigned to the User.
  • Login to ENOVIA V6 with Administrative Credential as rights are assigned to Administrator only to change the User Details.

Select the Power View Drop Down menu and select the "VPLM Administration" command under People and Administration.

Now search the User Details.

MQL Command

We can also list user, Licence and assign, remove the licences of User by just command. The steps to Run the MQL command are given below:

Select the "Run MQL" Menu inside utilities under Power View.

Fig.18.

Run the command in the MQL Command line.

Fig.19.

Some commands for the MQL windows are:

Command Description
list user; List The all User which are present in the ENOVIA V6.
list product; List all product license which is possessed by ENOVIA V6
modify product "Trigram of Product" add person "User Credential"; To assign the product license to particular User. For ex.

modify product DEC add person MyUser;
modify product "Trigram of Product" remove person "User Credential"; To remove product license from User.
list person "User" select product; To check no. of licence allocated to the particular user..
