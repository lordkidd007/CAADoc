X-CAD Integration - Architecture
Technical Article 

Abstract

3DEXPERIENCE Platform  X-CAD Integration enables you to integrate your applications (CAD, non-CAD) with the 3DEXPERIENCE Platform  products. This article provides an overview of this integration architecture, its various components and the steps involved to realize this integration.

About X-CAD Architecture

The Architecture Diagram shown in Fig.1 illustrates the different components of xApp-X-CAD Integration architecture along with the sequence of operations being executed. The legends have been provided to easily understand the diagram

The sequence of the operations performed or being executed have been illustrated in the architecture diagram.

The architecture is mainly divided in two parts: Client and Server

On Client side the main components are:

  • External client Application (xApp) which can create and load files of their respective format (e.g. SaveAs and Open)
  • XML files folder where the xml files are stored
  • IEF Client - It mainly interacts with the X-CAD Foundation (server side) and does the basic function of sending and receiving XML file
  • X-CAD Web App: Gets the customer data from 3DEXPERIENCE Platform Server, modify and propagate the data

On Server side we have:

  • 3DEXPERIENCE Platform MCS (Metadata Collaboration Server): It performs the instantiation of Customer types within PLM Dictionary depending upon the metadata sent through TCL file
  • X-CAD Foundation: It interacts with the 3DEXPERIENCE Platform MCS depending upon the xml command request from IEF client
  • 3DEXPERIENCE Platform FCS (File Collaboration Server): Store or Return the customer files within the Stream
