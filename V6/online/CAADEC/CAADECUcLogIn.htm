Login To 3DEXPERIENCE Platform
Use Case

Abstract

To begin with, we recommend users to get an overview of client integrations with X-CAD, by studying the article [1].

This Use Case details the protocol defined by X-CAD to realize LogIn in from integrations (Client Specific Extensions) inside the client tools. 

Connecting to 3DEXPERIENCE Platform

For client integrations to work with 3DEXPERIENCE Platform, the first step is to login to 3DEXPERIENCE Platform.

loginEx Command

User must execute the loginEx command before sending any other command requests to the system. On successful completion of the command the IEF system will return a response XML to the calling function. This response XML has GCO & LCO embedded in.  The article [2] delves into the concepts of the GCO and LCO.

Please note that after sending this command to IEF client server will lounch unified login panel and context panel which will enable user to login as Unified login.

loginEx Request XML

A sample loginEx request XML is depicted below. A sample xml also exists at CAAVIXCADIntegration.edu\InputData\LoginExRequest.xml. For unified login we will give user name and Password  in Unified login panel which will invoke after sending this command to IEF client. 

<command cname="loginEx" cse="CAA" ver="R417" csever="R417" password="" username="" unifiedlogin = "true"  ssologin = "true" />

The table that follows details the relevance of each attribute contained in that xml.

Attribute Name Description
cname should be loginEx
cse Name of the CSE
username blank ""
password blank ""
ver X-CAD (MCAD Harness) version that the current CSE was built with and is compatible with
csever CSE (Client Side Extension) version
ssologin true
unifiedlogin true

loginEx Response XML on LogIn SUCCESS

XML generated by MyX-CAD application could be found at following location.

C:\Users\XXX\AppData\Local\Temp\Request_Command\LoginEx_Request.xml

A sample loginEx response XML on success exists at CAAVIXCADIntegration.edu\InputData\Request\FORCAA_Standalone_LoginEx_Request.xml.

The loginEx response XML has primarily the following nodes

  • response node, whose result attribute is valuated to success or failure. In case of failure, there exists a reason node, which details the reason of failure
  • gsettings node, which has the attribute-value sets of the GCO
  • lsettings node, which has the attribute-value sets of the LCO

CSE reads these settings from the response XML and thus governs the behavior of the CSE. The fundamentals of GCO and LCO have been delved in depth in the technical articles [2] [3].

loginEx Response XML on LogIn Failure

<response result="failure" loglevel="5" cname="loginEx" cse="CAA" ver="R417" csever="R417"/>
<info><reason> Invalid User Credential supplied </reason></info>

The result attribute is valuated to failure. It is supported with the reason node, which details the reason of failure. A sample login Ex response on failure exists at  CAAVIXCADIntegration.edu\InputData\Responce\LoginExFailureResponse.xml.
