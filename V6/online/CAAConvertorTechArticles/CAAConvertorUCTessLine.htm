What you will learn with this Use Case

This use case is intended to show you how to implement a native APP format Reader. The scenario addresses Importing a xCAD file, converting its geometry data (a Line) and generating a cgr.

It exposes main domains of Mechanical CAD: Tessellated Representations. CATIExchangeTessellatedRepresentation interface being the most important one

The CATIExchangeTessellatedRepresentation provides a view on the document as a Tessellated representation and retrieves all the Tessellated information.

It is intended to support the conversions of scene graphs, essentially to retrieve the graphic representation of parts, as an alternate representation of the Boundary Representation. The major application will be the Digital Mock-up, in graphic mode.

The CAATessellatedLineConvertor Use Case

CAATessellatedLineConvertor is an use case of the CAAVIConvertors.edu framework that illustrates Format Conversion Services capabilities for importing external CAD data in V6.

What Does CAATessellatedLineConvertor Do

CAATessellatedLineConvertor simulates an implementation of a MyXCAD Reader, to be used in V6 import scenarios. This Reader is a set of classes implementing the Convertor Services Interfaces. Using these implementation, we can import geometry data from a .CAAcmp extension file.

How to Launch CAATessellatedLineConvertor Use-case

To launch CAATessellatedLineConvertor, you will need to set up the build time environment, then compile CAATessellatedLineConvertor along with its prerequisites, set up the run time environment, and then launch the V6.

This Use-case is made up of a DLL. You have to launch it using CATIA (3DEXPERIENCE.exe)

Then please refer to the article How to launch Convertor Use-case [1]

The file to be selected while launching the Use-case is a .CAAcmp file which contains co-ordinates of points. The name of the file xApp_Line.CAAcmp

At last when you click on Open, the Convertors code is called and our DLL gets loaded. The flow of the execution code is explained below in this article.

Where to Find the CAATessellatedLineConvertor Code

The CAATessellatedLineConvertor use case is defined at the location:

InstallRootDirectory\CAADoc\CAAVIConvertors.edu\CAATessellatedLineConvertor.m\

where InstallRootDirectory [1] is the directory where the CAA CD-ROM is installed.

The above implemented Interfaces have the following Functions or IP exposed

INTERFACE Function/IP Exposed
CATICreateInstance Instantiating component/ Factory creation
CATIExchangeInputDocument File Access, Open, Document links
CATIExchangeTessellatedRepresentation Access to Tessellated Geometry

Call to CATCreateAndAddOMFactoryFunctionWithSpecificKeyName depending upon the LATE TYPE

As the user selects the xCAD file to be Imported and clicks on OPEN button, V6 Writer (provided by DS) searches for the component with specific Late-Type made from the combination of the APPFormat and the ConvertorID. In this case, the file extension that we wish to open is .CAAcmp. This extension is associated with the APPFormat 'MyXCAD' and the ConvertorID is 'CAAVILine'.

The corresponding Late-Type is XCAD_MyXCAD_CAAVILine_InputDocument. This should be the Late-Type input for the macro. And the class to be instantiated should be the second argument of the macro

Initializing the Input Document and retrieving the Type of Representation

Once the Input document object is created by the Factory, a series of callback starts from the V6 Writer. These callbacks have to be answered by V6 Reader implemented by CAA developer. Depending upon these answers, V6 Writer converts the geometry data.

Basically the callback APIs are for 2 purposes

  1. To get the information on Assembly Structure or Reference and Representation nodes
  2. To get the absolute geometry data incase the node is Representation node

For the first type of callbacks, the following two APIs are very important

  • Initialize(): To get the path of the first file selected interactively. If the first file is of the type .CAAasm i.e. Assembly file, then this API will also help to store and retrieve the subsequent file paths from this Assembly file.
  • GetRepresentation():  Depending upon the file type, it will call the further implementation classes

Implementation of CATIExchangeTessellatedRepresentation interface

If the Representation type is Tessellated, then the CATIExchangeTessellatedRepresentation implementation class object is created. The CATIExchangeTessellatedRepresentation provides a view on the document as a tessellated representation and retrieves all the Tessellated information. Hence the second Callback APIs are called. All these APIs get the information from the document which supports the conversion of 'Scene Graphs', essentially to retrieve the graphic representation of parts. Refer to article [2] for info on Scene graph

  • GetRootNodesCount(), GetRootNode(), GetChildCount(), GetChild(), GetNodeType(), GetGPType() - All these APIs navigate through the Product Structure to get the exact number of geometrical elements present in a .CAAcmp file. They create all these geometrical elements and display in 3D Viewer.

First we pass the RootNodes count to the Writer which is 1, then we pass that Root node ID through GetRootNode(). Then we count the number of child nodes for the root node. This includes the number of  element tags from XML file .CAAcmp i.e. Point element tags <PointRef> present in the file. Then we pass the NodeIDs for all these Child nodes through GetChild() method.

After this, we have to specify/send the NodeType for every child node. The types of node present are NOD_ContainerNode, NOD_SurfacicNode, NOD_SolidNode, NOD_CustomNode, NOD_CoordinateSystem. We send the NOD_ContainerNode

Then we have to specify the GP Type (Graphical Presentation) if the node is of the type 'NOD_GP'. The GP Types are GP_Point, GP_Line, GP_Plane, GP_Cylinder, GP_Cone, GP_Sphere, GP_Face, GP_Edge. For this use-case we send the GP Type as GP_Point and GP_Line

Then we have to implement the corresponding API which takes the geometrical co-ordinates for the corresponding GP type. e.g. GetPointData() in our use-case

At last a cgr file is saved into the local directory. The path of it is C:/Users/<user>/AppData/Local/DassaultSystemes/CATTemp/Temp01_TT.cgr

In Short

The Use Case demonstrates the capability of the MultiCAD Convertor Service to convert a .CAAcmp file to tessellated geometry and generate a cgr.
