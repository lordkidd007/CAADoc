Dynamic Behavior Modeler Overview

The Dynamic Behavior Modeler is a new generation product that lets you design multi-physics systems. These behaviors are described in a formal language called Modelica[1].

There are two main kinds of Dynamic Behavior Representation:

  • Behavior Representation Library : it can be assimilated to a global library containing Modelica classes that can be accessed from every where and instantiated in every kind of Dynamic Behavior Representation.
  • Logical Behavior Representation : it can describe Modelica classes which can be exclusively accessed locally inside a logical representation[2].

For now CAA APIS allowing to manipulate Dynamic Behaviors can only be used on Behavior under Logical representations.

Dynamic Behavior Prerequisites

Several frameworks are necessary to create a CAA application based on Dynamic Systems Behavior:

  • CATFLEditorItf: This framework is dedicated to manipulate logical components and contains the CATIFLBehaviorList API which is mandatory to retrieve Dynamic Behaviors
  • CATSysBehaviorUseItf: This framework contains all Dynamic System Behavior's API and allow access to the Modelica class.

Dynamic Behavior Structuring Objects

Behavior designs are mainly described within a PLM RepReference, with a feature data structure.

The use of a behavior in a logical model is achieved through the instantiation of a Rep Reference under a Logical Reference and connections of Logical ports with behavior features in the rep ref.

Dynamic Behavior Modeler Navigation

For now navigation through the Dynamic Behavior Modeler is only available through the Logical Modeler. Behaviors aggregated under a Logical Reference are retrived using CATIFlBehaviorList::GetBehaviors(). Dynamic Behaviors adheres to the CATIModelicaClass interface, it allows to retrieve the Modelica text of the behavior using CATIModeliacClass::GetFullText() or its children classes using CATIModeliacClass::GetLocalClasses().
