Creating PLM Technological Representation
Use Case

Abstract

This Use case is basically a description of a global routine to create and instantiate a PLM Technological Representation Reference (Techno Rep Ref) [1].

This article is one among a set of articles which together discuss the "Integrating Applicative Data in Product Context" use case [2].

What You Will Learn With This Use Case

This use case teaches the steps involved to create and instantiate a PLM Technological Representation.

The CAAPstINFTechnoRepAndContainerCreation Use Case

CAAPstINFTechnoRepAndContainerCreation is a use case of the CAAProductStructureExtend.edu framework that mainly illustrates ProductStructureExtItf and FeatureModelerExt frameworks capabilities.

What Does CAAPstINFTechnoRepAndContainerCreation Do

This use case describes the global routine CAAPstINFTechnoRepAndContainerCreation. Its signature is as follows:

HRESULT CAAPstINFTechnoRepAndContainerCreation( CATIPLMProducts* pIOnProduct, CATIAdpEnvironment * pIAdpEnvironment,
					  const IID&  iIIDRepRef, void**  opRepRef, 
					  const IID&  iIIDCont, void**  opContainer )

A PLM Technological Representation (opRepRef ) will be created and aggregated by pIOnProduct. The PLM Technological Representation is created inside the container, whose name and the type is hard-coded by the method.

How to Launch CAAPstINFTechnoRepAndContainerCreation

To launch CAAPstINFTechnoRepAndContainerCreation global routine, we have to launch other UC, which has a call to this global routine.

  • "Creating Data in PLM Technological Representation" UC [3].
  • "Establishing External Links" UC [4].

Where to Find the CAAPstINFTechnoRepAndContainerCreation Code

The CAAPstINFTechnoRepAndContainerCreation global routine (use case) located in the CAAProductStructureExtend.edu framework, at specified path below:

InstallRootFolder\CAADoc\CAAProductStructureExtend.edu\CAAPstINFModeler.m

where InstallRootFolder [5] is the folder where the API CD-ROM is installed.

Step-by-Step

Following are the steps involved with the CAAPstINFTechnoRepAndContainerCreation global routine implementation.

  1. Retrieving the Credential
  2. Creating and Instantiating the PLM Technological Representation
  3. Retrieving the Newly Created Container

Retrieving the Credential

First we retrieve the credential enabling us to create the PLM techno rep and its first container.

...
  CATFmCredentials * MyCredential = NULL ;
  rc = ::CAAPstINFRetrieveCredentialsModeler(MyCredential);
...

The local method, CAAPstINFRetrieveCredentialsModeler is described in another use case article [6].

Creating and Instantiating the PLM Technological Representation

We use the CATIPrdExtAggregatedRepresentations interface implemented by the PLM product Reference.

...
  CATIPrdExtAggregatedRepresentations * pIAggOnProduct = NULL ;
  rc = pIOnProduct->QueryInterface(IID_CATIPrdExtAggregatedRepresentations, (void**)& pIAggOnProduct);
...

Then we create and instantiate the PLM Technological Representation with its first container.

...
   CATIPsiRepresentationReference * pTechnoRepCreated= NULL ;
   CATListValCATICkeParm_var ListAttributeEmpty ;
   rc = pIAggOnProduct->AddTechnologicalRepresentation(pIAdpEnvironment,ListAttributeEmpty,*MyCredential,
	                                              "CAAPstINFCont","CAAPstINFContainer", pTechnoRepCreated);
...

The PLM Technological Representation is created, and instantiated to be aggregated by the input PLM Product Reference - pIAggOnProduct . AddTechnologicalRepresentation creates a first container of type "CAAPstINFCont", and which is identified by "CAAPstINFContainer".

Retrieving the Newly Created Container

...
CATIPLMNavRepReference_var spPLMNavRepRefOnNewTechno =  pTechnoRepCreated  ;
rc =  spPLMNavRepRefOnNewTechno->RetrieveApplicativeContainer("CAAPstINFContainer",iIIDCont, (void**) opContainer); 
...

In Short

This use case uses describes in detail to create and instantiate a Technological Representation Reference under a PRODUCT ref.
