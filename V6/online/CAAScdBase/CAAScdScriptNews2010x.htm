What's New with V6R2010x Automation
Technical Article

Abstract

This article presents Automation objects, methods, and properties that are added since V6R2010 GA to enhance V6 Automation capabilities, support functional evolutions, and correct defects. Some of those modifications may already exist in V6R2010 maintenance fixes. A few properties and methods might have been modified or removed. VB and VBScripts code using those methods may have to be modified. VB projects using the corresponding objects must be recompiled.

Consult About Automation Languages, Debug, and Compatibility for details on manual adjustments that may be required for existing VB/VBA and VSTA projects to be usable on a new release.

Modified Methods and Properties
Removed Objects, Collections, Methods, and Properties
PrintersSettingAtt (Object) Has been replaced by the new generic settings API. See the SettingController Object and the CATPrintersSettings Setting Repository.
HybridBody::Bodies (Property Read) In V6, Body objects cannot be aggregated anymore in an HybridBody object. This method was so useless and has been removed.
DrawingView::GenerativeBehavior (Property Read) Generative views management is now offered through the DrawingDefineGenView and DrawingGenView objects. Example and details are available in the Drafting Changes chapter of the V6 Adoption section.
DrawingView::GenerativeLinks (Property Read) Generative views links management is now offered through the DrawingGenView object. Example and details are available in the Drafting Changes chapter of the V6 Adoption section.
SystemService::ExecuteScript V5 Method providing access to file based macro libraries. Doesn't work anymore in V6. Removed.
