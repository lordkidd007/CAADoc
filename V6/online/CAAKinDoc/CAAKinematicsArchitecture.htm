Kinematics Overview

Kinematics deals with the motion of a set of related components defined by the relative or absolute motion of some of these components.

Product

A Product is a rigid body or mechanism links.
A door and its frame are examples of products.

Mechanism

A mechanism inside a product allows its components to move relative to each other according to certain predefined constraints. The set of predefined constraints are a selection of engineering connections from the product.

A mechanism is defined as a representation of a product that can be used to perform kinematics simulations.

Engineering Connection

A engineering connection identifies a kinematics joint defined with a set of constraints commonly between two products which limit their relative motion.

A engineering connection is characterized with a kinematics type, such as prismatic or revolute.A hinge is an example of revolute type : that means the frame or the door can have a rotational motion against each other.

The engineering connection unifies V5 kinematics joints and V5 assembly constraints into a single V6 object. Please refer to assembly section for more information.

Command

The kinematics command identifies a degree of freedom of the mechanism on which the user can set a numerical value for a kinematics simulation.

For instance a kinematics command can be used to define the degree of door openness.
A door can be closed or opened at 90 degrees.

The kinematics command is associated to a controlled parameter of a engineering connection inside a scope of delimited values( lower and upper values).

APIs

A set of APIs to browse mechanisms contents and to drive a mechanism thanks to the kinematics solver.

Capabilities can be summarized as follows:

  • Scan mechanism content
    • Retrieve the list of mechanisms under a product.
    • Retrieve the list of products impacted (moved) by the mechanism.
    • Retrieve the list of joints ( aka engineering connections) from a mechanism.
    • Retrieve the list of commands from a mechanism.
    • Retrieve from a command: pointed MCX, command type, current value upper and lower limits.
  • Run kinematics solver
    • Instantiate a kinematic solver with a given mechanism.
    • Run kinematics solver for a given set of command values in nominal mode.
    • Retrieve impacted product transformation matrices (relative to previous position).
    • Clean Kinematic solver.
  • Edit a kinematics mechanism
    • Initialize the mechanism with all the joints and all the possible commands.
    • Get for a given joint the supported type of kinematics command.
    • Attach a joint.
    • Create a kinematics command.
    • Detach a joint.
    • Remove a kinemtics command.
