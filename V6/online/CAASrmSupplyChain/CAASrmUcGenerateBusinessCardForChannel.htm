Generating a Business Card
Use Case

Abstract

This use case illustrates how to generate a business card in the context of a PLM Channel. This is the first step of the PLM Channel creation .

A PLM Channel is an exchange context created between two specific repositories in order to allow data exchange and ownership transfers of PLM entities. Refer the Technical article [1] for the details of PLM Channel creation.

What You Will Learn With This Use Case

This use case is intended to show you how to generate a business card file. A business card is a file describing the unique identification of a repository .

The business card enables us to exchange repository information. A site generates a business card to identify itself. The business card once generated can be sent to any remote site, which would like to exchange data.

Note: The files can be sent through any medium like email, file transfer etc. The files generated during Channel formation cannot be interchangeably used for channel creation with other sites. Only the business card can be used to initialize channel creation with any other site.

The CAASrmGenerateBusinessCardForChannel Use Case

CAASrmGenerateBusinessCardForChannel is a use case of the CAASRMExposedServices.edu framework that illustrates PLMSRMExposedServices framework capabilities.

What Does CAASrmGenerateBusinessCardForChannel Do

This use case generates a business card file of a repository. It retrieves a pointer to CATPLMExchangeChannelServices class and calls the GenerateBusinessCard API on it.

How to Launch CAASrmGenerateBusinessCardForChannel

To launch CAASrmGenerateBusinessCardForChannel, you will need to set up the build time environment, then compile CAASrmGenerateBusinessCardForChannel along with its prerequisites, set up the run time environment, and then execute the use case [5].

To launch the use case, execute the command:

mkrun -c "CAASrmGenerateBusinessCardForChannel Repository Server User Password SecurityCtx BusinessCardPath"

where:

Repository The name of the repository
Server The name and port of the server as name:port_number
User The user name
Password The user password to authenticate the user
SecurityCtx A string representing the security context (Role.Organization.Project). Choose Role as Designer
BusinessCardPath The complete path of the business card file, including the file name, to be generated by the use case.
Example: "C:\Temp\BusinessCard.xml". This use case fails if the file already exists.

Where to Find the CAASrmGenerateBusinessCardForChannel Code

The CAASrmGenerateBusinessCardForChannel use case is made of a single file located in the CAASrmGenerateBusinessCardForChannel module of the CAASRMExposedServices.edu framework:

InstallRootFolder\CAADoc\CAASRMExposedServices.edu\CAASrmGenerateBusinessCardForChannel.m\

where InstallRootFolder [6] is the folder where the API CD-ROM is installed.

Generate a Business Card

The first step of PLM Channel creation involves creation of a Business Card by one of the two sites involved in this transaction. A Business Card is a XML file describing the unique identification of a VPLM Repository. The details of Creating PLM Channel is illustrated in the Technical article [1].

...
   CATPLMExchangeChannelServices* pExchangeChannelServices = 
   			        new CATPLMExchangeChannelServices();
   if (NULL!= pExchangeChannelServices)
   {
    HRESULT Rc = pExchangeChannelServices->GenerateBusinessCard(businessCard);
   }
...

The CATPLMExchangeChannelServices::GenerateBusinessCard API takes as input the path of the output directory to which the business card is to be written.

A Business card file contains the site identification of the VPLM database to which the process is connected . Site sends this business card ( through email , file transfer etc ) to the other site who will then process the business card to initialize the PLM Channel [2].

Epilog

The CAASrmGenerateBusinessCardForChannel use case ends by simply closing the PLM session. Since all use cases necessarily conclude by closing the PLM session, we have a dedicated article on this topic namely "Creating a Basic PLM Batch" [6].

In Short

The use case illustrates the generation of the business card file. A site identifies itself with this business card . Site sends this file( through email, file transfer etc) to the other repository site who will use it for initializing the PLM Channel [2].
