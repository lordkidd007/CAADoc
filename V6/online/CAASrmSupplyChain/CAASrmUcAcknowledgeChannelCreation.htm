Acknowledging the PLM Channel Creation
Use Case

Abstract

This use case illustrates how to acknowledge the PLM Channel creation or refusal.

A PLM Channel is an exchange context created between two specific repositories in order to allow data exchange and ownership transfers. Refer the Technical article [1] for the details of PLM Channel creation.

What You Will Learn With This Use Case

This use case is intended to show you how to acknowledge the PLM Channel creation or refusal using the creation confirmation file.

When a site receives a creation confirmation file ,it activates or deprecates the channel accordingly. The site generates a report with the channel creation or refusal information from the other site. Once the PLM channel is activated , it can be used to transfer the data between the two sites.

The CAASrmAcknowledgeChannelCreation Use Case

CAASrmAcknowledgeChannelCreation is a use case of the CAASRMExposedServices.edu framework that illustrates PLMSRMExposedServices framework capabilities.

What Does CAASrmAcknowledgeChannelCreation Do

Finalizing PLM Channel [4] use case has generated a creation confirmation file stored into a PLM Briefcase file. This file has been sent (through email, file transfer etc) to the current site. Using this confirmation file, the current site process the final step of PLMChannel creation by generating a report ( with channel creation or refusal information from the other site).

How to Launch CAASrmAcknowledgeChannelCreation

To launch CAASrmAcknowledgeChannelCreation, you will need to set up the build time environment, then compile CAASrmAcknowledgeChannelCreation along with its prerequisites, set up the run time environment, and then execute the use case [6]

To launch the use case execute the command:

mkrun -c "CAASrmAcknowledgeChannelCreation Repository Server User Password SecurityCtx CreationConfirmationPath"

where:

Repository
Server The server as Host::port_number/RootURI
User The user name
Password The user password to authenticate the user
SecurityCtx A string representing the security context (Role.Organization.Project). Choose Role as Designer
CreationConfirmationPath The complete file path of the creation confirmation file required to activate the PLM Channel.
The file must exist already at remote site. Its extension must be ".PLMBriefcase".
Example : "C:\Temp\CreationConfirmationFile.PLMBriefcase"

Where to Find the CAASrmAcknowledgeChannelCreation Code

The CAASrmAcknowledgeChannelCreation use case is made of a single file located in the CAASrmAcknowledgeChannelCreation module of the CAASRMExposedServices.edu framework:

InstallRootFolder\CAADoc\CAASRMExposedServices.edu\CAASrmAcknowledgeChannelCreation.m\

where InstallRootFolder [7] is the folder where the API CD-ROM is installed.

Prolog

The CAASrmAcknowledgeChannelCreation use case begins by creating a session and connecting to the Input Repository with an appropriate role. The use case "Creating a Basic PLM Batch" [7] deals with Opening and Closing a PLM Session.

Acknowledge the PLM Channel creation

When the creation confirmation file ( generated during finalizing the PLM channel at the other site [4] ) is received at the current site, it will then acknowledge the PLM Channel creation( the final step of PLM Channel creation ). Using this confirmation file , the current site activates or deprecates the channel accordingly.

...
   CATPLMExchangeChannelServices* pExchangeChannelServices = 
   			     new CATPLMExchangeChannelServices();
   if (NULL!=pExchangeChannelServices)
   {
      ...
      HRESULT Rc = pExchangeChannelServices ->AcknowledgeChannelCreation
      			(icreationConfirmation, ochannelId, oreport);
      ...
   }
...

The CATPLMExchangeChannelServices::AcknowledgeChannelCreation takes three arguments:

  • icreationConfirmation, CATUnicodeString type, the path of the creation confirmation file (received from the other site)
  • ochannelId , CATString type, channel identification
  • oreport, CATUnicodeString type, the report generated with the creation or refusal information from the other site

The CATPLMExchangeChannelServices::AcknowledgeChannelCreation API generates a report with the channel creation or refusal information from the other site. Once the PLM channel is activated , it can be used to transfer the data between the two sites.

Epilog

The CAASrmAcknowledgeChannelCreation use case ends by simply closing the PLM session. Since all use cases necessarily conclude by closing the PLM session, we have a dedicated article on this topic namely "Creating a Basic PLM Batch" [7].

In Short

The use case illustrates how to acknowledge the PLM Channel creation and activate/deactivate the PLM Channel creation. Once the PLM Channel is activated, it can be used to transfer the data between the two sites [5].
