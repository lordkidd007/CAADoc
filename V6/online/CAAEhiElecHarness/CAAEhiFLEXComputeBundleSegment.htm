Testing Your Own Algorithm Implementation
Use Case

Abstract

This article discusses on how to test your own algorithm to compute the Equivalent Young Modulus, Equivalent RatioToBend and Equivalent RatioToTwist of a segment. We call "segment" a portion of a bundle segment where the physical properties are constant.

Launching the Document Containing a Bundle and Wires

The attached 3dxml is called "CAAElecHarnessItf.3dxml" and is contained in the following path:

InstallRootFolder\CAAElecHarnessItf.edu\InputData\CAAElecHarnessItf.3dxml

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

Import the provided 3dxml. It contains a root product with PLM ID "SampleProduct".

Computing the Shape with FLEX Algorithm using the Implementation Sample

The model opened was computed with the Standard algorithm. So first you have to select the FLEX algorithm in the settings of Harness Management.

Then you have to force the re-computation in the Wire Harness Assembly workbench.

The shape is computed by the FLEX algorithm using your own implementation.
