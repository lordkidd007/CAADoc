What You Will Learn With This Use Case

This use case is intended to help you to declare your own workbench as part of the Authoring Foundation workshop. Then all toolbars or commands defined in the Authoring Foundation workshop will be available in your own workbench.

  • Describing the different steps dedicated to the knowhow of the company through the customization.
  • Implementing the interface CATIDELPPRAuthFoundationWksConfiguration to declare your own workbench as part of the Authoring Foundation workshop.

The CAAPPRWorkbenchForAuthFoundationWks Use Case

CAAPPRWorkbenchForAuthFoundationWks is a use case of the CAADELPPRAuthFoundItf.edu framework that illustrates how to make your own workbench as part of the Authoring Foundation workshop.

What Does CAAPPRWorkbenchForAuthFoundationWks Do

This use case illustrate how to make your own workbench as part of the Authoring Foundation workshop

How to Launch CAAPPRWorkbenchForAuthFoundationWks

To use CAAPPRWorkbenchForAuthFoundationWks, you will need to:

  • Set up the build time environment, then compile CAAPPRWorkbenchForAuthFoundationWks.m module along with its prerequisites [1]
  • Edit the interface dictionary, that is, the CAADELPPRAuthFoundItf.edu.dico file located in:

    InstallRootFolder\CAADoc\CAADELPPRAuthFoundItf.edu\CNext\code\dictionary\

    where InstallRootFolder is the folder where the API CD-ROM is installed, and decomment the following line by removing the '#CAA#' character:

    CAAPPRWorkbenchForAuthFoundationWks CATIWorkbench libCAAPPRWorkbenchForAuthFoundationWks
    CAAPPRWorkbenchForAuthFoundationWks CATIDELPPRAuthFoundationWksConfiguration libCAAPPRWorkbenchForAuthFoundationWks
  • Set up the run time environment
  • Launch a CATIA V6 session
  • Go to the Manufacturing Planning category
  • Look for the "Example CAA New Workbench" workbench
  • Launch it.

Where to Find the CAAPPRWorkbenchForAuthFoundationWks Code

The CAAPPRWorkbenchForAuthFoundationWks use case code is located in the CAAPPRWorkbenchForAuthFoundationWks.m module of the CAADELPPRAuthFoundItf.edu framework:

InstallRootFolder\CAADoc\CAADELPPRAuthFoundItf.edu\CAAPPRWorkbenchForAuthFoundationWks.m

where InstallRootFolder is the folder where the API CD-ROM is installed.

Step-by-Step

The class CAAPPRWorkbenchForAuthFoundationWks, implements the Interface CATIDELPPRAuthFoundationWksConfiguration, this implementation is located on the file CAAPPRWorkbenchForAuthFoundationWks.m\src\CAAPPRWorkbenchForAuthFoundationWks.cpp. Its purpose is to declare the workbench as part of the Authoring Foundation workshop. There is no need to implement new methods. The only methods to implement are the ones corresponding to the CAA interface CATIWorkbench.

Here is the declaration of this class:

class CAAPPRWorkbenchForAuthFoundationWks: public CATIDELPPRAuthFoundationWksConfiguration
{
  CATDeclareClass;
  public:
    // Standard constructors and destructors for an implementation class
    // -----------------------------------------------------------------
    CAAPPRWorkbenchForAuthFoundationWks();
    virtual ~CAAPPRWorkbenchForAuthFoundationWks();
    void CreateCommands();

    CATCmdWorkbench *CreateWorkbench();
    CATClassId GetAddinInterface();

    void GetCustomInterfaces(CATListPV * iDefaultIIDList, CATListPV * iCustomIIDList);
  private:
    // The copy constructor and the equal operator must not be implemented
    // -------------------------------------------------------------------
    CAAPPRWorkbenchForAuthFoundationWks (CAAPPRWorkbenchForAuthFoundationWks &);
    CAAPPRWorkbenchForAuthFoundationWks&
    operator=(CAAPPRWorkbenchForAuthFoundationWks &);
};

All methods defined in this class are for implementing the CAA Interface CATIWorkbench. Please check the CAA documentation of this interface to know how to implement these methods.
