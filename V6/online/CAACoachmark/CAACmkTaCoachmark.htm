What is the Coachmark?

The Coachmark helps users to get basics of the user interface through 6 slides. Browsing over the slides will give all clues to use main components of the 3DExperience platform. The Coachmark is user friendly on Touchscreen devices. Users can swipe on slides to browse, or use the arrows on each side of the Coachmark.
The Coachmark is launched on start up of the platform. Users can choose to display or not the Coachmark on next start up. The Coachmark can also be launched via the TopBar by clicking on the menu item "Get Started...".

Include 3DExperience Platform Coachmark on the page

The Coachmark requires:

  • Coachmark module: In V6R2015x, the path in runtime view is: \webapps\Coachmark

Check with your environment if you have to add UWA library on your own:

<!--UWA library & CSS -->
<link rel="stylesheet" type="text/css" href="../api/ext/init/resources/lib/UWA/assets/css/inline.css" media="all"/>
<script type="text/javascript" src="../api/ext/init/resources/lib/UWA/js/UWA_Swym_Alone_full.js"></ script>

Then to embed Coachmark in a web page, prerequisite is to add these lines:

<!--Coachmark CSS-->
<link rel="stylesheet" type="text/css" href="<pathOfCoachmark>/css/coachmmark-min.css" media="all" />
< Coachmark JS -->
<script type="text/javascript" src="<pathOfCoachmark>/js/Coachmark-min.js"></script>

Init the Coachmark

This section will teach you how to initialize the Coachmark. The Coachmark need to be initialized with the "myAppsBaseURL", which is a server where the “Do not show again” option is stored. This option is to know if there is a need to show the Coachmark or not.

Important:
  • Use options.forceDisplay=true to display it whatever the value stored in user preferences.
  • Use options.lang to give the Coachmark the language it should use

Init the Coachmark with its options:

Parameters
Object options: key/value pair

Key Value type Description Rule Default
myAppsBaseURL String MyApps server URL where the “Do not show again” may be retrieved and stored. Optional ""
parentEltSelector String Selector to identify DOM element in which the Coachmark will be added. If not provided, will be added in the body (not recommended) Optional "body"
forceDisplay Boolean True: Display the Coachmark even if “Do not show again” option has been checked before. To be used for example when user wants to launch Coachmark from help menu Optional false
lang String Could be : 'en', 'fr', 'ko', 'es', 'ru', 'zh', 'de', 'ja' or 'it' Optional en

Returns
Void

Example

require('Coachmark', function(Coachmark){
    Coachmark.init({
        myAppsBaseURL: 'http://3ds.com',
        parentEltSelector: 'body',
        forceDisplay: false,
        lang: ‘en’
    });
});

Include 3DExperience Platform Coachmark for Rich Client

The Coachmark requires UWA library and 1 module, the Coachmark is instantiated in a HTML page, Cochmark.html. (webroot/plugins/coachmark/coachmark.html)
This would mean it will be installed inside ENOVIA server, so it could be sourced from there.
Ex: http://ve4al418dsy:8090/ematrix/plugins/coachmark/coachmark.html

Launch on start up

  1. Retrieve the ‘Do not show Again’ status, from calling the myApps web service or from a setting.
  2. If the dnsa status is false : Open the Coachmark.html inside a window (Internet Explorer Active X or iOS system in Safari) followed by the dnsa status, the link should be an URL + a query string. In this case the query string you provide should be : ?dnsa=false.
  3. Give the Coachmark the language it should use via query string.

Exemple

http://ve4al418dsy:8090/ematrix/plugins/Coachmark/Coachmark.html?dnsa=false&lang=en

Launch on click

  1. Retrieve the ‘Do not show Again’ status, from calling the myApps web service or from a setting.
  2. Open the Coachmark.html inside a window (Internet Explorer Active X or iOS system in Safari) followed by the dnsa status, the link should be an URL + a query string. The query string you provide has to got the key is dnsa, paired with its Boolean value retrieve in step 1 and the language.

Exemple

 http://ve4al418dsy:8090/ematrix/plugins/Coachmark/Coachmark.html?dnsa=true&lang=en

Close scenario with the Coachmark close button

  1. The function sendNotification({string}dnsa, {Boolean}) is called, so you can store the new dnsa.
  2. Close the tab.

Close scenario with the tab close button

  1. When the user closes the Coachmark with the window close button, execute those lines, to get the dnsa status 
    CATUnicodeString script = " window.getDNAStatus();";
MyCAfrUrlViewer-> RunScript ( script );
    This will perform a sendNotification({string}dnsa, {Boolean}).
  2. Store the new dnsa status.
