Logical Modeler Behaviors
Quick Reference

This article describes for each PLM Class of the Logical modeler how you can invoke the Qeops, Crud, and lifecycle operations [1] either thanks by a generic API [2], or with a modeler specific APIs.

Logical Reference

On PLM Object in Database

Behaviors Supported How Is It Supported
Query Yes CATAdpPLMQueryServices
Expand Yes CATAdpOpener
Open Yes CATAdpOpener
Save Yes CATAdpSaver
Lock/Unlock Yes CATAdpLockServices
Versioning Yes CATAdpVersionningServices
Change Maturity Yes CATAdpMaturityServices

On PLM Object in Session - Modeler APIs

Behaviors Supported How Is It Supported Comments
Edition Yes CATIFL2DEntity  
Creation Yes CATILogReferenceFactory or CATIFLEditorReferenceFactory For logical Reference with 2D Layout (created with CATIFLEditorReferenceFactory)
Deletion Yes CATLogDeleteServices  
Navigation Yes CATILogInstances, CATILogPorts, CATILogRepresentationInstances, CATILogConnections Prefer using generic API

On PLM Object in Session - Generic APIs

Behaviors Supported How Is It Supported Comments
Navigation No CATIPLMNavInstance  
No CATIPLMNavOccurrence
Yes CATIPLMNavReference Use it instead the Modeler APIs
No CATIPLMNavRepInstance  
No CATIPLMNavRepReference  
Public PLM Attributes  Edition (RW) Yes CATICkeObject  
Public PLM Attributes Browse (R) Yes CATIPLMNavEntity  
PLM class type and PLM object Identifier Browse Yes CATIPLMComponent
Identifier Set Browse Yes CATIPLMIdentifierSet
