Document Modeler Behaviors
Quick Reference

This article describes for each PLM Class of the Document modeler how you can invoke the Qeops, Crud, and lifecycle operations [1] either thanks by a generic API [2], or with a modeler specific APIs.

Note: Document Modeler is not RACE Compliant.

Document Object

Behaviors Supported How Is It Supported
Query Yes CATAdpQueryServices
Expand No
Open Yes PLMDocumentServices::GetDocument
Save No Operation on document are managed in short transaction meaning there is no need to explicitly save
Lock/Unlock No  
Versioning No

On PLM Object in Session - Modeler APIs

Behaviors Supported How Is It Supported Comments
Edition Yes PLMIDocument to modify a document  
Creation Yes PLMDocumentServices::CreateDocument
Deletion Yes PLMDocumentServices::DeleteDocument  
Navigation Yes PLMDocumentServices::ListDocuments to list document attached to an object, GetParents on PLMIDocument to list parent of the current document
