JavaScript Naming Guides
Technical Article

Abstract

This article is about naming conventions when writing in Javascript. Most of them are ordinary advice and conventions leading to a code more readable and maintanable.

Naming Conventions

In JavaScript the common naming convention is the CamelCase, but Pascal case can be used too.

CamelCase

CamelCase looks like as follows : "theCamelCaseSymbol", where

  • A CamelCase name always starts with a lowercase letter,
  • The words in the name, except the first word (to respect the previous rule), must have their initial letter in uppercase,
  • No special character separate the words.

Pascal case

Pascal case looks like as follows: "ThePascalCaseSymbol", where all words start with a uppercase letter.

Variable

Variable names are usually CamelCase and begin with a noun, pronoun or adjective.

var theCommunityName;
var myString;
var listOfCars;

Try to have meaningful values for variable so that one does not have to read the full source to get the role of the variable.

Constant

We recommend using the C convention which is to use all UPPERCASE letters with underscores separating words.

var MY_CONSTANT_VALUE = 112 ;

Function

Function names uses CamelCase and start with a verb.

function isRed() {
    ...                
}
function setColor() {
   ...
}

Constructor

For a constructor use the Pascal case convention. They can begin with a noun or pronoun which helps differentiate them from variables and functions. 

function Car() {
    ...                
}
function Color() {
   ...
}

AMD Module

AMD modules [1] are located in JavaScript files included in directories dedicated for the web, the mweb modules [2]. A JavaScript file must contain only one AMD module which is defined through the define function:

 define('AMDName',[..,..], 
    function (..,..) {
         ....
    }
)();

where the first argument of the define function is the AMD module name.

For such file : 

myFw
    webappname.mweb
        src
            dir1
			    ...
                   dir2
                       jsnamefile.js

The naming rule is as follows:

partnername/webappname/dir1/.../dir2/jsnamefile

Where:

Name Meaning
partnername A short string representing for example your brand, your company in order to differentiate your code from others. The DS AMD modules all start with 'DS'.
Strongly recommendated.
webappname The name of the webApp. By default it is the name of the mweb module, but you can overload it by using the PROGRAM_NAME variable into the mweb module imakefile.
dir1/../dir2 the path between the src directory of the mweb module, and the JavaScript file
jsnamefile he name of the JavaScript file without its js extension

In the imakefile, you can change the name of the web app:

PROGRAM_NAME=webappnamebis

In this case, the AMD module name is: partnername/webappnamebis/dir1/.../dir2/jsnamefile

Global Identifier

The use of global identifiers is not recommended, but if absolutely required and really no possibility to avoid it, the only global should be namespaces, and their use should generally not been needed thanks to AMD encapsulation.

The only namespaces should be brand.

BRANDNAME.NAMECOMPONENT.myVariable

Then with the brand name space, the component , process ,framework or webapp name should be used to encapsulate the other variable
