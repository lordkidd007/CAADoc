Objectives

Architecture

Fig. 1: Conceptual Model for MSR

The Categories in more details:

Model

Model refers to the PLM level data model that describes the aggregated structure of physical and/or logical abstractions of a product model (RFLP) or production system model (PPR). Model also describes other aspects of the abstractions such as properties, connections, ports, interfaces, sensors, actuators, etc. Typically, this Model will be subjected to a Scenario and executed in order to produce a Result enabling the user to "Experience" the product or production system model in life and nature.
Scenario

Scenario refers to the PLM level data model description of the control flow (structure and sequence) and contents of activities/operations including magnitudes and time or frequency dependent amplitudes of excitations to which a Model is subjected/executed to produce Results in an Experience.
RESULT

Result refers to the PLM data model level description of the capture and "Play" of an Experience - the solved/executed outcome of a Model subjected to a Scenario.
