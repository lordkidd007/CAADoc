What You Will Learn With This Use Case

This use case is intended to show you how to navigate into a simulation.

You will learn:

  • How to use the CATSimPLMAppModelerItf framework, and specially the APIs of navigation in a simulation.
  • How to retrieve the simulated component.
  • How to retrieve the objects stored under each category.
  • How to retrieve the root manager of a representation.
  • How to check in or check out a representation.

The representation is created in the SIMObjSimulationRepReferenceGeneric Modeler.

The CAASimPLMSimulationNAV Use Case

CAASimPLMSimulationNAV is a use case of the CAASimPLMInterfaces.edu framework that mainly illustrates CATSimPLMAppModelerItf framework capabilities.

What Does CAASimPLMSimulationNAV Do

The CAASimPLMSimulationNAV use case shows how to navigate on a Simulation structure using the CATSimPLMAppModelerItf API's.

How to Launch CAASimPLMSimulationNAV

First, you need to import the GHASimulation.3dxml into your data base. You can find it under : InstallRootFolder\CAADoc\CAASimPLMInterfaces.edu\CNext\resources\graphic

To launch CAASimPLMSimulationNAV , you will need to set up the build time environment, then compile CAASimPLMSimulationNAV along with its prerequisites, set up the run time environment, and then execute the use case [1].

mkrun -c "CAASimPLMSimulationNAV repository server user password securityCtx  AttributeName1"

where the description of the arguments is the following:

Position Description Example Category
1 PLM repository PLM1 Connection
2 PLM server e4au2dsy:1571
3 Username username
4 Password password
5 Security Context VPMDESIGNER.VPM.DEFAULT
6 AttributeName1 GHASimulation Component Attribute
  • The first five are related to the connection [2].
  • The last argument respectively define the attribute names and their values for the attribute.

Where to Find the CAASimPLMSimulationNAV Code

The CAASimPLMSimulationNAV use case is made of a single file located in the CAASimPLMSimulationNAV.m module of the CAASimPLMInterfaces.edu framework:

InstallRootFolder\CAADoc\CAASimPLMInterfaces.edu\CAASimPLMSimulationNAV.m

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

Step-by-Step

There are eleven logical steps in CAASimPLMSimulationNAV use case. The prolog and epilog parts, which correspond to Creating/Closing a PLM session are already described in another document [2]. We will only focus on the five steps concerning the navigation on simulation:

  1. Retrieving the Simulated Component under the Model Category
  2. Retrieving the Representation Reference under a Given Category
  3. Retrieving the Root Manager from the Aggregated Representation
  4. Aggregating a Document to the Simulation
  5. Checking in and Checking out the Aggregated Representation

Retrieving the Simulated Component under the Model Category

This step consists in to retrieve a pointer on the simulated component pointed by our simulation under the model category.

...
CATIPLMComponent* opSimulatedComponent = NULL;
oHR =iSimulation ->RetrieveSimulatedComponent( opSimulatedComponent);
...

Retrieving the Representation Reference under a Given Category

This step consists in to retrieve a pointer on the representation reference under a given category.

...
CATListValCATBaseUnknown_var opiSimulationPLMObjects;
oHR =iSimulation ->ListRepRefUnderCategory( TypeCategory, opiSimulationPLMObjects);
...

Retrieving the Root Manager from the Aggregated Representation

This step consists into retrieving a pointer on the root manager feature of the aggregated representation under a given category.

CATBaseUnknown* oRoot = NULL;
oHR =iSimulation ->GetRootManagerFromSO( 
			TypeCategory, oRoot);
...

Aggregating a Document to the Simulation

Here are extracted the code lines valuating some data from inputs. iSimulation is the input simulation object imported before launching this use case.

...
CATUnicodeString Path = "path"
CATUnicodeString extension = "txt"
oHR = iSimulation -> AggregateSimulationDocument(TypeCategory, iType, Path, extension, pRepresentationAggregated);
...
  • TypeCategory is the type of the category (Scenario or Result) aggregating the representation.
  • Path is the path of the aggregated representation.
  • extension is the extension type of the aggregated representation (txt, jpeg etc.).
  • pRepresentationAggregated is the aggregated representation object.

Checking in and Checking out the Aggregated Representation

This step consists in checking in or checking out the aggregated representation (opPLMSimulationDocument).

...
CATUnicodeString endPath ;
endPath += "resources" ; 
endPath += DIR_DELIMITER ; 
endPath += "graphic" ; 
endPath += DIR_DELIMITER ;         
endPath +="MySpecs.txt"  ; 

const char * localFullPath = CATFindPath(endPath.CastToCharPtr(),  installPath ) ;
//CheckIn 
oHR = opPLMSimulationDocument -> CheckInContent(localFullPath);
...
//CheckOut
oHR = opPLMSimulationDocument -> CheckOutContent(localFullPath);
...
