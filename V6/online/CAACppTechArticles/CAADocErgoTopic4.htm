4.1 Design of Visual Elements

4.1.1 Visual States for Controls

Rule 4.1.1.1: If a control or an item is not available (because the command or the data associated is not available), it must be displayed with an unavailable appearance (grayed).

4.2 User Interface Text

4.2.1 Terminology

Rule 4.2.1.1: For each object, use only one term, to maintain the interface consistency

Example:

Correct Incorrect
Save Commit
Configurable View CO or CV
Search Query
Rule 4.2.1.2: Use a common vocabulary, generic and easy to understand. Do not use any computer vocabulary.
Rule 4.2.1.3: Translate all the object names and commands into professional vocabulary known by the end-user.

Example:

Correct Incorrect
Planes
Products
Plane Version		 Product Root Class
Product PRC
Product Container CATProd Cont
Rule 4.2.1.4: Texts in labels respects the rules of vocabulary and grammar of title of books.
  • In English, the first letter of each word is a capital letter except articles, conjunctions and preposition not positioned at the begin of the end of the sentence. (ex: New Folder, Go To, Select All, Table of Contents).
  • In French, only the first letter of the first word is a capital letter
    ex: Nouveau dossier, Aller à, Sélectionner tout, Collage spécial
Rule 4.2.1.5: There is no abbreviation in labels.

Example:

Correct Incorrect
Action IDX_Action
Item VPM_Item
Rule 4.2.1.6: The only authorized characters are letters and space and "..." [1].

Note:

[1] In a menu, "..." indicates that the command will not terminate immediately. The users expect a continuation in the dialog sequence (for example a dialog box)

Rule 4.2.1.7: Avoid redundancies in dialog boxes titles.

Example:

Correct Incorrect
The title describes the content of the dialog box.
This dialog box can appears both during the creation or the edition.		 Once selected the "Area" command for the creation, the word creation is redundant.
Rule 4.2.1.8: The dialog box title is the name of the related command (or slight modification).

Example:

Correct Incorrect


Name of the command of creation of a product and its dialog box are consistent.

Title of the dialog box has no relation with the command.

4.3 Design of Graphic Images

4.3.1 Icon Design

Rule 4.3.1.1: The sizes of the icons are:
Graphical Object Useful Drawing Size [1]
Command icon in normal size (P1 style [2]) 22x22 (minus a frame of 1 pixel around)
Command icon in small size (P1 style[2]) 16x16 - 1 pixel at the top left corner
Mask 10x10 - 1 pixel at the top left corner
Tree node 16x16 - 1 pixel at the top left corner

Notes:

  • [1] The useful size is the dimension total of the graphical object minus the zone used to define the transparency color and the margins.

    In all icons cases, the transparency is defined by the upper left pixel. All pixel that has the same color will be considered as transparent.

  • [2] P2 command icons in the three states Unpressed, Focused, and Pressed are dynamically generated at run time from the P1 icons (normal or small) and so respectively follow the standard dimensions 24x24 and 18x18.
    Icon State Normal Size Small Size
    Unpressed 24x24 18x18
    Focused 24x24 18x18
    Pressed 24x24 18x18
Rule 4.3.1.2: An icon has a single meaning. (If two similar command icons are identical, they perform the same action)

Example:

Correct Different icons for "Standard Print" and "Captured Image Print" (for Save and Copy too)

or

Use the same command (for Save and Copy too).
Incorrect
Same Save and Print icons for Document Save and Capture Save
Same Print icon for Document Print and Capture Print
Same Copy icon for Selection Copy and Capture Copy
