Introduction

The resources define the layout of the factory and execute the operations. The operations are balanced between physical resources.The physical resources are programmed with tasks to execute what is specified by the operation.

The physical resources are authored as a physical product. The tasks definition are based on LCM technology.

Manufacturing Resource Topology

Physical Resource Modeler

It offers extended functionalities of the Physical Product Modeler to model the PLM Resources.

A Resource is a Reference Product used in a manufacturing context, as such resources are decomposed through Instance-Reference aggregation mechanism. The resource related PLM Extensions can be assigned to extend the product reference with attributes supporting the usage of the product in the manufacturing context. The PLM extensions are used to extend the product ports with attributes defining Logical & Physical interfaces of the resource.

The specializing attributes are, for example:

  • Type (Robot, Worker, NC Machine, etc.).
  • Cost.
  • Cycle time.

Behavior Representation Modeler

It offers functionalities for Resource Detailing as:

  • Task modeling.
  • Sequencing of tasks.
  • Trajectory modeling.

Scopes

A "scope" link between reference of resource and reference of system is mandatory to create implement links. It describes in which organizational part of the factory the children system's operations will be realized.

Implement Links

The "Who" implement link relation is the real link to the implementing task. The "who" is a "control" port exposing a task. The other implement link relations are optional:

  • The "Where" relation specifies the physical area where the operation is performed.
  • The "With" relation specifies the others resources involved in the implementation of the operation.

Manufacturing resource types

There are three categories of resources:

  1. Organizational (WHERE)
    Containing two types:
    • Area
    • Organizational.
  2. Working (WHO)
    Containing nine types:
    • Robot
    • Worker
    • Transport
    • Conveyor
    • NC Machine
    • Industrial Machine
    • Inspect
    • Control Device
    • Logic Controller.
  3. Not Working (WITH)
    Containing four types:
    • Tool Device
    • Storage
    • Sensor
    • User Defined Resource.

For a given operation:

  • 0...1 Working resource can realize the operation (Who link, Executing resource for Operation).
  • 0...1 Organizational resource can realize the operation (Where link, Localization resource for Operation).
  • 0...n Non Working resource can realize the operation (With link, Used resource for Operation).
