What You Will Learn With This Use Case

This use case is intended to show you how to retrieve data inside a PLM Representation reference whose the main Stream descriptor contains data from .model (V4 file format).

The CAAV4iEduV4DataAccess Use Case

CAAV4iEduV4DataAccess is a use case of the CAAV4V5Interoperability.edu framework that illustrates CATIAV4Interfaces framework capabilities.

What Does CAAV4iEduV4DataAccess Do

CAAV4iEduV4DataAccess begins by opening the PLM representation reference whose the Stream Descriptor contains V4 elements coming from a .model (V4 file format).

Then, it scans the stream descriptor using functions of CATIAV4Interfaces to get the element to be analyzed [1]. These elements are objects of the class CATV4iV4Element.In this particular case, it retrieves the first element of the first set of the workspace MASTER.

Finally, data of the element are retrieved : the identifier, the primary and secondary types, the number of the layer of the element and some graphical information about the element.The elements are removed.

How to Launch CAAV4iEduV4DataAccess

To launch CAAV4iEduV4DataAccess , you will need to set up the build time environment, then compile CAAV4iEduV4DataAccess along with its prerequisites, set up the run time environment, and then execute the use case [2].

mkrun -c "CAAV4iEduV4DataAccess Repository Server UserId Password SecurityCtx PLMRepresentationRefPLMType PLMExternalID V_version OutputFile"

where:

Repository
Server The name and port of the server as name:port_number
User The user name
Password The user password to authenticate the user
SecurityCtx A string representing the security context (Role.Organization.Project). Choose Role as Designer
PLMRepresentationRefPLMType A Representation Reference PLMType containing
PLM_ExternalID The PLM_ExternalID of a PLM Reference or PLM Representation Reference
V_version The V_version of a PLM Reference or PLM Representation Reference
OutputFile The name of the output text file

You can directly execute this use case if you know how to locate your own representation reference. If not, you can import a 3D XML file CAAV4iDataAccessUseCase.3dxml supplied in the CAAV4V5Interoperability.edu framework , at the path specified below:

InstallRootFolder\CAADoc\CAAV4V5Interoperability.edu\InputData\CAAV4iDataAccessUseCase.3dxml

where InstallRootFolder [2] is the folder where the API CD-ROM is installed.

This 3D XML file contains the Input Model with the following attributes:, which needs to be imported to the PROVIDER, you are working with.

  • PLM_ExternalID : CAAV4iDataAccessUseCases (with a Prefix).
  • Version: ---

Attention: Its mandatory that you provide a suitable prefix to the PLM_ExternalID of the Input Model, while importing a 3D XML file, to ensure that data stored in the PROVIDER, is unique.

Please refer to the document [2] for how to import and re-import such a 3D XML file in your PLM repository.

Where to Find the CAAV4iEduV4DataAccess Code

where InstallRootFolder [2] is the folder where the API CD-ROM is installed.

Step-by-Step

There are three logical steps in CAAV4iEduV4DataAccess:

  1. Opening the PLM Representation Reference
  2. Scanning the Model
  3. Retrieving the Data

Opening the PLM Representation Reference

...
    CATIAdpPLMIdentificator* opiIDComp = NULL;
    const char *iStrPLMType = argv[6];
    
    rc = ::CAARetrieveIDofUniquePLMObject (iListAttributeName,iListAttributeValue,														
				       iStrPLMType,opiIDComp);
...	
    CATOmbLifeCycleRootsBag Bag;
    CATAdpOpener opener(Bag);
    CATIPsiRepresentationLoadMode *piLoadModeOnRepRef  =  NULL;
    rc= opener.CompleteAndOpen(opiIDComp,IID_CATIPsiRepresentationLoadMode, (void**) &piLoadModeOnRepRef);
...
	rc= piLoadModeOnRepRef->ChangeLoadingMode(CATIPsiRepresentationLoadMode::EditMode);
...
	rc = piLoadModeOnRepRef->QueryInterface(IID_CATIPsiRepresentationReference ,(void**)&piPsiRepRefOnModel );
...

The CAARetrieveIDofUniquePLMObject method retrieves the identificator of Reference by using the Attribute Set and PLM Type. Details of retrieving identificator are given in "Querying, Expanding and Opening PLM Components" [3].

The CompleteAndOpen [3] method of CATAdpOpener class opens the PLM representation reference using the input arguments with .model as a stream descriptor.

Scanning the Model

...
  CATV4iV4Element* masterElem=NULL;

...

CATV4iGetMaster is a function that retrieves the first workspace of the model: the MASTER. If piPsiRepRefOnModel is not a PLM representation reference, the function fails.

CATV4iGisset retrieves the first set of the workspace masterElem.

CATV4iGisels retrieves the first element of the set setElem.

Retrieving the Data

...
        CATString identificator=element->GetId();
...
        int itp, its;
        element->GetType(itp, its);
...
        int layer =-1;
        result = CATV4iGirlay(element, layer, ier);
...
        int oShow=-1, oPick=-1, oCol=-1, oBlink=-1, oThick=-1, oLine=-1;
        result = CATV4iGirvis( element, oShow, oPick, oCol, oBlink, oThick, oLine, ier);
...

If element has been correctly created by CATV4iGisels, the data can be retrieved.

element->Id() retrieves the identifier of the element,

element->GetType(...) retrieves the primary and the secondary types of element

...CATV4iGirlay(... retrieves the number of the layer.

...CATV4iGirvis(... retrieves graphical data about the element: show, pickable, color, blink, steady, thickness, line type.

In Short

This use case provides a way to retrieves information of elements of a model.
