Functional Modeler Behaviors
Quick Reference

This article describes for each PLM Class of the Functional modeler how you can invoke the Qeops, Crud, and lifecycle operations [1] either thanks by a generic API [2], or with a modeler specific APIs.

Functional Reference

On PLM Object in Database

Behaviors Supported How Is It Supported
Query Yes CATAdpPLMQueryServices
Expand Yes CATAdpOpener
Open Yes CATAdpOpener
Save Yes CATAdpSaver
Lock/Unlock Yes CATAdpLockServices
Versioning Yes CATAdpVersioningServices
Change Maturity Yes CATAdpMaturityServices

On PLM Object in Session - Modeler APIs

Behaviors Supported How Is It Supported Comments
Edition Yes CATIFct2DPictures
Creation Yes CATIFctReferenceFactory
Deletion Yes CATFctDeleteServices
Navigation Yes CATIFctInstances , CATIFctPorts, CATIFctRepresentationInstances, CATIFctConnections, CATIFctConnectable, CATIFctFlowExpositionInstances Prefer using generic Api's
