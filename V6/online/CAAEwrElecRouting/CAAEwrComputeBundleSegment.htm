Testing Your Own Algorithm Implementation
Use Case

Abstract

This article discusses on how to test your own algorithm to compute the diameter and the minimum bend radius of a bundle segment according to the wires contained in it.

  • Editing the Diameters of a Bundle Segment and of a Wire Contained in It

Launching the Document Containing Bundles and Wires

The attached 3dxml is called "CAAElecRoutingItf.3dxml" and is contained in the following path:

InstallRootFolder\CAAElecRoutingItf.edu\InputData\CAAElecRoutingItf.3dxml

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

Import the provided 3dxml. It contains a root product with PLM ID "Session".

Editing the Diameters of a Segment and of a Wire Contained in It

To visualize the diameter of the segment called "Segment.1", you have to select it and make appear the Properties dialog box with the contextual menu. The diameter is equal to 5 mm.

To see the wires passing through the selected segment, you can use the Link Review command in the Wire Harness Assembly workbench as follows:

The following panel is displayed: Wire02 is contained in Segment.1 of Multi_Branchable3.1.

The diameter of the wire can be displayed in the Properties dialog box. It is equal to 5 mm.

Modifying the Diameter of a Wire of the Bundle Segment

To modify the diameter of the wire, you can modify the diameter value in the Properties panel, as shown in the following picture:

In our example, new diameter of the wire is set to 8 mm.

By successively pressing the 'Apply' and the 'OK' buttons, the icon 'Update All' is set to active. By pressing the 'Update All' button, the diameter of the bundle segment is changed.
