Getting Started
Technical Article

This task shows you how to get started with Native Apps Unit Tester, using a basic test scenario relying on a module of an education framework installed with the API CD ROM [1].
Before you begin: Note that:
  • You should have first installed the Native Apps Development Toolset  and the Native Apps Interactive Development Environment.
  • You should also have installed a Dassault Systèmes brand, such as CATIA or DELMIA, as well as the associated API CD ROM. The scenario uses some of the installed frameworks to populate the workspace and run the tools.
  • You should have already created a workspace, such as the one used as an example in Getting Started with Native Apps Developement Environment. Otherwise, create a simple folder that ill be used as the workspace root folder.

Populating the Workspace

Warning: The workspace is supposed to be located in E:\MyWorkspace in this example.

  1. Locate the API CD ROM installation folder (by default C:\Program Files\Dassault Systemes\B216).

  2. Open the CAADoc folder in this installation folder.

  3. Copy the CAASystem.edu framework into your workspace root folder.

The use cases of this framework will be used as the code to test.

Creating the Test Framework

  1. In you workspace, create a folder named CAASystem.tst

  2. In CAASystem.tst, create a folder named IdentityCard.

  3. In the IdentityCard folder, create an XML file named IdentityCard.xml with the following contents:

    <?xml version='1.0' ?>
<testFramework xmlns="http://www.3ds.ic"
               xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
               xsi:schemaLocation="http://www.3ds.ic ICModel.xsd">
  <prerequisite name = "System" access = "Public"/>
  <prerequisite name = "SystemTS" access = "Public" />
  <prerequisite name = "CAASystem.edu" access = "Public"/>
  <prerequisite name = "SpecialAPI" access = "Public"/>
</testFramework>

  4. In CAASystem.tst, create a folder named FunctionTests.

  5. In the FunctionTests folder, create a folder named TestCases.

Creating a Test Object

  1. In the TestCases folder, create an XML file named CAASysCallBack.xml with the following contents:

    <?xml version="1.0"?>
<ODT Version="1.0">
  ...
</ODT>

  2. Add the test object definition section.

    <?xml version="1.0"?>
<ODT Version="1.0">
  <Definition>
    <Replayed value="false" osConstraint="!win_b64" />
  </Definition>
  ...
</ODT>

    The definition section is required and is enclosed within the tag <Definition>. In this example, it prevents the test object replay if the operating system is not one of the Windows 64 bit operating systems.

  3. Add the test object execution section.

    <?xml version="1.0"?>
<ODT Version="1.0">
  <Definition>
    <Replayed value="false" osConstraint="!win_b64" />
  </Definition>
  <Execution>
    <Command value="CAASysCallback" osConstraint=”win_b64” />
  </Execution>
</ODT>

    The execution section is required and is enclosed within the tag <Execution>. In this example, it requests CAASysCallback.exe to be executed, with the consition that the operating system is one of the Windows 64 bit operating systems.

  4. Save the file CAASysCallBack.xml.

Building the Workspace

Warning: The Dassault Systemes 3DEXPERIENCE run time and API are supposed to be installed in C:\Program Files\Dassault Systemes\B216 and the Native Apps Development Toolset in C:\Program Files (x86)\Dassault Systemes\B216. Replace these paths with your own Dassault Systemes 3DEXPERIENCE installation folder and the Native Apps Development Toolset installation folder respectively if you installed them in other places.

  1. Open a command prompt.

  2. Set the current folder at the root folder of your workspace.

  3. Type:

    C:\Program Files (x86)\Dassault Systemes\B216\intel_a\code\command\tck_init.bat

    This command initializes the tools to be used to build the workspace.

  4. Type:

    tck_list

    This command lists the available TCKs. Choose a value similar to V6B2014x_1330_B216.

  5. Type:

    tck_profile V6B2014x_1330_B216

    The build environment is now set up.

  6. Type:

    mkGetPreq -p "C:\Program Files\Dassault Systemes\B216"

    The prerequisites of the frameworks in your workspace are now set up.

  7. Type:

    mkmk -au

    The frameworks are now built.

  8. Type:

    mkCreateRunTimeView

    The run time view of your workspace is created.

Running the Test Object

  1. Still in the command prompt set at the root folder of your workspace, type:

    mkodt -s CAASysCallBack

    Note that the test object name if the name of the XML file without its extension.

The test object output is:

## ------------------------------------------------------------
## Workspace Directory : E:\MyWorkspace
## Concatenation : E:\MyWorkspace
## Concatenation : C:\Program Files\Dassault Systemes\B216
## OS : win_b64
## Machine : LW5PSRDSY
## Display :
## 1 ODT to replay
## ------------------------------------------------------------
## RUNNING ODT [CAASystem.tst CAASysCallback] (1 / 1)
elapsed_time: 00 min 01 sec
## ++ CAASysCallback: Test execution successful code 0
## ------------------------------------------------------------
##
## 1 ODT replayed, 0 KO.

The test object traces are located in E:\MyWorkspace\CAASystem.tst\FunctionTests\Output\win_b64. The trace file name is the ODT name, CAASysCallback, with extension traces, that is, CAASysCallback.traces. It contains the traces issued by the tested code, as well as possible error messages.
