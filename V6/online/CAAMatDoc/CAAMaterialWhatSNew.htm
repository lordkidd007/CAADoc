V6R2011x

Usage of CATOmbObjectInContext for links creation

In order to manage correctly links into the Product Structure, CATOmbObjectInContext has been created. CATMatNewMaterialSupport interface has been impacted for two methods:

SetMaterialCore:

HRESULT CATMatNewMaterialSupport::SetMaterialCore( CATIMaterialReference* iCoreMaterial, CATIMaterialReference* ipiMaterial, 
                            int& oErrorCode, const int iReplace, CATIMatNewMaterialInstance** opiMatCnx )

is replaced with:

HRESULT CATMatAppliedAccess::SetMaterialCore( const CATOmbObjectInContext* const ipObjInCont, CATIMaterialReference* ipiMaterial, 
                            int& oErrorCode, const int iReplace, CATIMatNewMaterialInstance** opiMatCnx )

and SetMaterialCovering:

HRESULT CATMatNewMaterialSupport::SetMaterialCovering(CATIMaterialReference* iCoveringMaterial, CATIMaterialReference* ipiMaterial, 
                            int& oErrorCode, const int iReplace, CATIMatNewMaterialInstance** opiMatCnx )

is replaced with:

HRESULT CATMatAppliedAccess::SetMaterialCovering( const CATOmbObjectInContext* const ipObjInCont, CATIMaterialReference* ipiMaterial, 
                            int& oErrorCode, const int iReplace, CATIMatNewMaterialInstance** opiMatCnx )

Please, note that the new services have been provided on CATMatAppliedAccess class.

V6R2010x

Removal of CATISpecObject from Public View

Consequently, Applicative Features stored inside material domains are now manipulated through CATIMaterialDomainContent interface that they have to implement. This impacts two classes:

In CATIMaterialDomain:

HRESULT GetApplicativeSpec( CATUnicodeString& iType,
                            CATISpecObject *& opiSpec )

is replaced with:

HRESULT GetApplicativeSpec( CATUnicodeString& iType,
                            CATIMaterialDomainContent *& opiCont )

and in CATMaterialTools:

static HRESULT GetDomain( CATISpecObject * ipiFeat,
	                  CATIMaterialDomain *& opiDom ) ;

is replaced with:

static HRESULT GetDomain( CATIMaterialDomainContent * ipiCont,
                          CATIMaterialDomain *& opiDom ) ;
