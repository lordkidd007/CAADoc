Mechanical Modeler Objective

The Mechanical Modeler is a DS Representation Modeler dedicated to Mechanical and Shape applications, such as Part Design, Generative Shape Design and Sketcher. It provides a complete set of tools to manage the content of PLM Representations for these applications.

Mechanical Modeler Concepts

3D Shape

The PLM Representation's main data stream for Mechanical Modeler is called the 3D Shape. It contains the full design of a Mechanical Part. The 3D Shape in V6 is structured as the V5 Part Document, namely by a set of containers that store different types of Features.

The 3D Shape Data Stream

The article "From PLM Representation to the 3D Shape"[4] explains the link between the PLM Representation and the 3D Shape.

Mechanical Features

The design of Mechanical Parts relies on different types of Mechanical Features based on the Feature Modeler infrastructure. Each type of Feature is associated with interfaces that drive their specific behaviors. Some interfaces are common to all types of Mechanical Features, such as CATIMmiMechanicalFeature.

There are three main categories of Mechanical Features:

  • The Part Feature: the top level feature in the 3D Shape.
  • The Geometrical Feature sets: features that aggregate either other geometrical features sets or geometrical features.
  • The Geometrical Features: features that generate a geometry.

The design of a mechanical part is represented in the Specification Tree of CATIA. The structure of the graph and the icons associated to each type of feature enable users to see at a glance the type of the features and how they were built, it reflects the internal model of the part.

The Specification Tree

Representation1 is the Part Feature.

PartBody is the main Geometrical Feature Set.

xy plane, yz plane, zx plane, Pad.1 are Geometrical Features.

Sketch.1 and Sketch.2 are features that are managed by the Sketcher application, they derive from Mechanical Features.

The three categories of features are detailed in the following technical articles: "The Part Feature"[5], "Geometrical Feature Sets[6], "Geometrical Features"[7].

Spec/Result Model

Mechanical Modeler relies on the Feature Modeler Spec/Result model. The Specifications are modelized by the Mechanical Features from which the system is able to build the geometrical Result, visualized in the geometry area of CATIA.

The Spec/Result Model

Here is a reminder on the Feature Modeler's qualifications for attributes that define features:

  • Input attributes are the specification attributes.
  • Output attribute contains a generated result and needs to be updated each time an input attribute is modified.
  • Neutral attributes contain additional information.

The input attributes of a feature can reference other features. Thus, the design of complex mechanical parts can lead to enormous networks of inter-dependent features. The Feature Modeler's Build/Update mechanism drives the computation of the result and ensures the consistency of the networks of features, while each feature deals with the building of its own result.

Update Mechanism for Mechanical Features

The article "Specification/Result Mechanism applied to Mechanical Features" [8] explains how Mechanical Results are built.

Generic Naming

During the design, it can be useful to select a topological sub-element such as an edge or a face as input of a new Feature.

Selection of an Edge to Create a Fillet

Topological objects are unstable because they can be deleted and rebuilt during the update process. Therefore Mechanical Features should not reference them directly. The solution implemented in Mechanical Modeler to overcome this problem is to use a name instead of a direct reference to the topology. Such a name is called a Generic Name, it provides a stable way to reference a topological cell.

The Generic Name is built through a temporary object representing the geometry selection, called BRep Access. It can be made persistent by generating a BRep Feature. The BRep Feature can be referenced by a Mechanical Feature.

The article "Generic Naming Overview "[9] explains the Generic Naming mechanism.

In Short

The Mechanical Modeler is a feature based, specification-driven and generative modeler that enables users to concentrate on the design specifications while the system computes the resulting geometry. It provides a complete set of tools to create, modify and browse 3D Shapes.
