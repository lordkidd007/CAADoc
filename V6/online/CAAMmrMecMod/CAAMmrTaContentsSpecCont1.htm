What Are Geometrical Feature Sets?

Geometrical Feature Sets are features that represent either the resulting shape of a mechanical part, either subparts of it. They aggregate Geometrical Features or other Geometrical Feature Sets. They are also called Tools.

As seen in the article "The Part Feature"[2], a 3D Shape necessarily contains a PartBody and eventually Bodies, Geometrical Sets and Ordered Geometrical Sets. These features are all Geometrical Feature Sets, they are each represented in the Specification Tree with a specific icon.

Geometrical Feature Sets

Three categories of Geometrical Feature Sets

Geometrical Features are distinguished by their type: Solid features for 3D objects excepted volumes, Surfacic features for all other objects (points, wires, surfaces and volumes). Refer to the article "Geometrical Features"[3] for further information.

There are three categories of Geometrical Feature Sets, depending on the type of Geometrical Features that they can contain:

  • Solid and Surfacic Feature Sets: they can contain both solid and surfacic features. PartBody and Body belong to this category when environment is set to hybrid design mode (hybrid design is the default mode from V5R14 onwards).These features are also called Hybrid Bodies.
  • Surfacic Feature Sets: they can only contain surfacic features. Geometrical Sets and Ordered Geometrical Sets belong to this category. These features are also called GSM Tools.
  • Solid Feature Sets: they can only contain solid features. PartBody and Body belong to this category when created with hybrid design mode deactivated (before V5R14, hybrid mode did not exist, so Bodies were necessarily Solid Feature Sets). These features are also called Solid Bodies or Mechanical Tools.

Ordered Sets

Ordered Geometrical Sets and Hybrid Bodies are both ordered sets. This section explains two concepts that define an ordered set: the Order Concept and the Absorption Concept. Refer to the "Order and Absorption Concepts" article [4] for further technical information.

  • Order Concept

    An ordered set is a set in which features appear in the specification tree by order of dependency, from the basic features at the top of the graph to the most complex features at the bottom. The purpose of ordering is to help user with the design in the case of very complex geometry, where features can have a huge historical graph.

    The main characteristics of an ordered set are:

    • Each feature inside an ordered set can only have input specifications that are above it in the specification tree.
    • A new feature is inserted just below the current feature and automatically becomes the current feature.
    • Only features above the current feature are visualized in the Geometry Area.
  • Absorption Concept

    Some features in a set may not be relevant. The absorption concept is the fact of not taking into account the non relevant features, these are "absorbed".

    Let us consider the following example:

    Extrude and Split Operation

    We can see the 3D view of the extruded surface in the first picture. In the second picture, the 3D view represents the result of the split operation on the extruded surface. Extrude.1 has been "absorbed" by Split.1.

    Features such as Extrude.1 are named absorbed features, and those as Split.1 are named absorbent features.

    The main particularities of an absorbed feature are:

    • It's not visualized as long as the absorbent feature is visualized, there is no superimposition.
    • It's not selectable as long as the absorbent feature is visualized.
  • Interfaces specific to ordered sets

    CATIGSMTool interface is implemented on Surfacic Sets or Hybrid Sets. It enables you to manage characteristics of the sets, in particular the fact that it is ordered or not.

    CATIInputDescription interface gives information on the features in an Ordered Set.

Solid and Surfacic Feature Sets - Solid Feature Sets

For ease of use, and because it's the nomenclature in the specification tree (PatBody and Body), Solid and Surfacic Feature Sets and Solid Feature Sets are usually called Body features. In order to distinguish hybrid design from non hybrid design, we will use the following wording:

  • Hybrid Body for Solid and Surfacic Feature Set.
  • Solid Body for Solid Feature Set.

Body features enable to model objects in 3D. Each Body has one resulting 3D shape. PartBody is the main body, it contains the final resulting 3D Shape of the Mechanical Part. Secondary Bodies enable to design separately subparts of a mechanical Part, that can then be assembled with boolean operations.

The icon represents the PartBody in the Specification Tree.

The icon represents all other Bodies in the Specification Tree.

Note: there is a color code on these icons, to differentiate Hybrid Bodies from Solid Bodies, depending on the current hybrid design mode. They can appear either green, grey or yellow. The hybrid design and this color code are fully detailed in the online documentation of CATIA, chapter Mechanical/Part Design/Hybrid Design.

Body Feature

Creating Body Features

A Body feature can be aggregated by a Part Feature. If it's a Hybrid Body, it can also be aggregated by an Ordered Geometrical Set.

The method to create a Body feature is the CreatePRTTool method of the CATIMmiUseSetFactory interface implemented on the Specification container.

...
  CATIMmiUseSetFactory_var spSetFactory = piPrtCont;
  CATIMmiMechanicalFeature_var spPart = ...;
...
  CATIMmiMechanicalFeature_var spNewBody;
  HRESULT rc= spSetFactory->CreatePRTTool("MyNewBody", spPart, spNewBody); 
...

piPrtCont is a pointer to the specification container.

spPart is a smart pointer to the part feature that will aggregate the new Body.

Refer to the article "the Part Feature"[2] for details on how to retrieve both the container and the Part.

Retrieving the Main Body Feature

The main Body feature is the PartBody. It is automatically created at the 3D Shape creation and cannot be deleted.

The method to retrieve the PartBody feature is GetMainBody method of the CATIPartRequest interface, implemented on the part feature.

...
  CATIPartRequest_var spPartRequest = spPart;
  CATBaseUnknow_var PartBody;
  CATUnicodeString ViewContext ="MfDefault3DView" ;
  HRESULT rc = spPartRequest->GetMainBody(ViewContext,PartBody);
...

spPart is a smart pointer to the part feature under which the Bodies are aggregated.

About the view context: There is a default view context which is represented by the MfDefault3DView string. For some applications, such as Sheet Metal, another view context can be used to represent the same object in a different way, MfUnfoldedView for an unfolded view. If you give an empty string, the default value is used.

Retrieving Body Features

The method to retrieve the Body features is GetSolidBodies method of the CATIPartRequest interface, implemented on the part feature.

...
  CATIPartRequest_var spPartRequest = spPart;
  CATLISTV(CATBaseUnknow_var) ListBodies;
  CATUnicodeString ViewContext ="MfDefault3DView" ;
  HRESULT rc = spPartRequest->GetSolidBodies(ViewContext,ListBodies);
...

spPart is a smart pointer to the part feature under which the Bodies are aggregated.

ListBodies contains the list of Body features aggregated by the part.

Differences between an Hybrid Body and a Solid Body

Hybrid bodies were created as from V5R14 to facilitate the design.

  • The main difference of course is that the Solid Body can only aggregate solid features, whereas the Hybrid Body can aggregate both solid and surfacic features.
  • Solid Bodies are non ordered sets, whereas Hybrid Bodies are necessarily ordered.
  • Aggregation rules are different.

Surfacic Feature Sets

The Surfacic Feature Set, also called GSMTool, is mostly made of elements that do not have a thickness, such as planes, surfaces, wireframe geometry, but it can also contain volumes.

There are two types of Surfacic Feature Sets : Geometrical Set and Ordered Geometrical Set.

The icon represents the Geometrical Set in the Specification Tree.

The icon represents an Ordered Geometrical Set in the Specification Tree.

For ease of use, Surfacic Feature Sets are usually designated by their abbreviations:

  • GS for Geometrical Set.
  • OGS for Ordered Geometrical Set.

MyGeometricalSet is a GSand MyOrderedGeometricalSet is an OGS.

Creating a Surfacic Feature Set

A GS can be aggregated by a Part Feature, another GS or a Solid Body.

An OGS can be aggregated by a Part feature, another OGS or a Hybrid Body.

The method to create a GS is the CreateGeometricalSet method of the CATIMmiUseSetFactory interface implemented on the Specification container.

The method to create an OGS is the CreateOrderedGeometricalSet method of the CATIMmiUseSetFactory interface implemented on the Specification container.

...
  CATIMmiUseSetFactory_var spSetFactory = piPrtCont;
  CATIMmiMechanicalFeature_var spPart = ...;
...
  CATIMmiMechanicalFeature_var spNewGeomSet;
  HRESULT rc= spSetFactory->CreateGeometricalSet("MyNewGeomSet", spPart, spNewGeomSet); 
...

piPrtCont is a pointer to the specification container.

spPart is a smart pointer to the part feature that will aggregate the new GS.

The CreateGeometricalSet and CreateOrderedGeometricalSet methods check the validity of the aggregating feature based on the previous rules.

Retrieving Surfacic Features Sets

The method to retrieve Surfacic Feature Sets is GetSurfBodies method of the CATIPartRequest interface.

  ...
  CATIPartRequest_var spPartRequest = spPartFeature;
  CATListValCATBaseUnknown_var ListSurfBodies ;
  CATUnicodeString ViewContext ="MfDefault3DView";
  HResult rc = spPartRequest->GetSurfBodies(ViewContext,ListSurfBodies);
  ...

spPartFeature is a smart pointer to the part feature under which the Surfacic Feature Sets are aggregated.

ListSurfBodies contains the list of OGS aggregated by the part and the list of all the GS of the 3D Shape.

Private Surfacic Feature Sets

A private Surfacic Feature Set is protected from interactive modifications. It means that end users cannot modify the structure of such a set:

  • No deletion of its features.
  • No insertion of features.
  • No reordering of its features.

CATIGSMTool interface implemented on Surfacic Feature Sets enables to set or retrieve the private mode.

In Short

Geometrical Feature Sets are mechanical features that represent the resulting shape of a mechanical part or subparts of it. They aggregate Geometrical Features or other Geometrical Feature Sets.

Their type depends of the type of Geometrical features that they can aggregate:

  • The Solid and Surfacic Feature Set : aggregates solid and surfacic features, except volumes. It's also called Hybrid Body.
  • The Solid Feature Set: aggregates solid features. It's also called Solid Body.
  • The Surfacic Feature Set: aggregates surfacic features and volumes. There are two types, ordered and not ordered, called Ordered Geometrical Set (OGS) and Geometrical Set (GS).
Mechanical Feature Aggregation Rules

Note: a Body (hybrid or solid body) can only aggregate another body through a boolean operation.
