Introduction

The manufacturing process model describes the genealogy of the Manufactured product. One manufacturing process represents one transformation of the product.

Each manufacturing process may contains estimated duration, resource requirements, documentation. A manufacturing process holds all information to generate the MBOM.

Manufacturing Process Topology

Aggregation

The manufacturing process references specifies a manufacturing function. It can have:

  • Data output/inputs : 0 to N
  • Time output/inputs : 0 to N.

The manufacturing process can be de-composed into sub-processes. This decomposition is described through an instance-reference hierarchy: sub-processes are sub-functions of the parent function.

The sum of the sub-functions realizes the father function. The hierarchy allows to support PLM life cycle: configuration, reuse.

  • Data requirement connections (DR): explicit DATA driven specifications between instances.

Scope

Connects a product reference with a process reference that is by definition a description of the processes necessary and sufficient to realize the transformation. There can be {0..n} process references pointing to one product reference. These references represent different possible solutions to realize the transformation.

Implement Link

Connects implemented features to a N Process Instance (and not occurrence).
