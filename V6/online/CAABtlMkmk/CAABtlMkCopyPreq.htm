mkCopyPreq
Quick Reference

Abstract

The mkCopyPreq command copies the necessary build and run time part of the prerequisite frameworks.

  • Usage
  • Options
  • Examples
  • References

Synopsis

mkCopyPreq [-W WSPath] -p Path1;|:Path2;|:... [-t TargetPath] [-o OS1 [-o OS2 ...]] [-v] [-debug] [-copysrc | copyall] [-simul] [-d] [-f] [-jobs #] [-h | -help | -?]

Usage

mkCopyPreq copies the prerequisite frameworks into a workspace. Contrary to mkGetPreq [1], physical copies are made. This isolates a workspace from the changes of its prerequisites to stabilize the environment.

Please note that mkCopyPreq replaces the existing frameworks only if a previous mkCopyPreq run had already copied these frameworks.

For disk space economy and performance reasons, mkCopyPreq does not copy the entire data of a prerequisite framework. For instance, the source code is not copied locally, which won't allow a local work on these frameworks.

mkCopyPreq reprocesses the identity card of the local framework.

If one of the frameworks is already existing in the target workspace, it is not copied, an error message is displayed but the copy continues.

Options

mkCopyPreq accepts the following options:

Option Description
-copysrc Copy only the module source code.
-copyall Copy all files.
-d Copy the CNext data folder.
-debug Debug mode.
-f Copy FunctionTests folders.
-h|-help|-? Display the command help.
-jobs # Multiprocessing mode. # is the maximum number of parallel tasks mkCopyPreq forks. Useful to limit the number of tasks when the machine is heavily used. Default is 5.
-nopdb Do not copy .pdb files and remove existing .pdb for prerequisite frameworks in target directory.
-o Set an operating system to consider. Default is all available operating systems.
-p Path1[;|:Path2...] Concatenate workspaces where to search for the prerequisite frameworks. Path1 and Path2 represent workspace root folder paths, and must be separated using a ":" (colon) with Unix and a ";" (semi-colon) with Windows. Windows workspace paths can be mapped drive paths, such a e:\WorkspaceToAdd, or UNC paths, such as \\dsone\rnd\R211rel\BSF.
-simul Simulate the command execution.
-t TargetPath Path of the folder where the prerequisite frameworks are copied. Default is -W option value.
-v Verbose mode.
-W WSPath Path of the workspace to process. Default is the current folder.

Examples

Copy what is necessary for build and run time for the prerequisites of the current workspace from r:\R211rel to the current workspace.

mkCopyPreq -p r:\R211rel\BSF

Copy what is necessary for build and run time for the prerequisites of the current workspace from r:\R211rel\BSF into e:\users\dave\Preq.

mkCopyPreq -p r:\R211rel\BSF -t e:\users\dave\Preq
