mkRmFw
Quick Reference

Abstract

The mkRmFw command deletes frameworks from a workspace.

  • Usage
  • Options
  • Examples

Synopsis

mkRmFw [-W WSPath] [FW1 [FW2 ...] | -lFW FWlist] [-preq | -nopreq] [-fw | -nofw] [-simul] [-jobs #] [-v] [-debug] [-h | -help | -?]

Usage

mkRmFw deletes frameworks into a workspace.

Note that this command addresses frameworks which have been installed by the commands mkCopyPreq [1] or mkCopyFw [2].

Options

mkCopyFw accepts the following options:

Option Description
-debug Debug mode.
-fw Delete frameworks that were copied using mkCopyFw only.
-h|-help|-? Display the command help.
-jobs # Multiprocessing mode. # is the maximum number of parallel tasks mkRmFw forks. Useful to limit the number of tasks when the machine is heavily used. Default is 5.
-lFW FWlist Delete the frameworks the names of which are listed in the file located at the path FWlist.
-nofw Delete all the frameworks except those that were copied using mkCopyFw.
-nopreq Delete all the frameworks except those that were copied using mkCopyPreq.
-preq Delete frameworks that were copied using mkCopyPreq only.
-simul Simulate the command execution.
-v Verbose mode.
-W WSPath Path of the workspace to process. Default is the current folder.
FW1 [FW2]... Delete FW1, FW2, etc.

Examples

Delete any frameworks in the current workspace which were copied using the mkCopyPreq command:

mkRmFw -preq

Delete any frameworks in the current workspace which were copied using the mkCopyPreq and mkCopyFw commands:

mkRmFw -preq -fw
