mkCreateRuntimeView
Quick Reference

Abstract

The mkCreateRuntimeView creates or refreshes the run time view of a workspace.

Synopsis

mkCreateRuntimeView [-W WSPath] [-FW Fw1 [Fw2] &| -lFW FWList] [-k Type [-k Type]] [-doc] [-c | -f | -l] [-d] [-x] [-t TargetPath [-s] [-external]] [-xml XmlFile] [-v,] [-debug] [-h | -help | -?]

Usage

They are two kinds of file trees: one for the build time and one for the run time. Some objects are used during the run time like icons or resource files and are stored in the frameworks build time file tree. In order to be used during the run time they must be copied in a run time file tree named RunTimeView.

Run time software knows only the RunTimeView.

The mkCreateRuntimeView is a command to copy (copy or symbolic link) automatically files from the build time file tree into the run time file tree.

CNext.xxx means specific operating systems.

In this kind of directories are resources which are available only on some operating systems. These directories are named CNext.specifics_$OS. The following table shows the different values of $OS.

$OS Operating Systems
aix_a AIX UNIX
aix_a64 AIX 64 bit
hpux_a HP-UX 10
hpux_b HP-UX 11
linux_a64 Linux 64 bit
linux_b64 Linux 64 bit
solaris_a SunOS
solaris_a64 SunOS 64 bit
solaris_b64 SunOS 64 bit
irix_a IRIX
macos_a64 Mac 64 bit  
intel_a Windows  
win_b64 Windows 64 bit  

The values are hierarchical. It means if a file is both in CNext, CNext.specifics_AIX and CNext_specifics_UNIX, the run time view will contain the file coming from:

  • CNext.specifics_aix-a for AIX 32 bit.
  • CNext_specifics_UNIX on the other UNIX platforms.
  • CNext on Windows.

Options

mkCreateRuntimeView [-W WSPath] [-FW Fw1 [Fw2] &| -lFW FWList] [-k Type [-k Type]] [-doc] [-c | -f] [-d] [-x] [-t TargetPath [-external]] [-xml XmlFile] [-v] [-debug] [-h | -help | -?]

 

mkCreateRuntimeView accepts the following options:

Option Description
-c Copy files in the run time view if the source is newer than the existing file in the run time view.
-d Delete unknown files in the run time view. If you put personal files in the run time view, they will be deleted.
-doc Create the run time view for documentation only (*.doc modules).
-debug Debug mode.
-external Do not copy internal and non deliverable data. Only with -t.
-f Copy files in the run time view whatever the source timestamp. This is the default with Windows.
-FW Create the run time view for the frameworks listed.
-h|-help|-? Display the command help.
-k Set the type of frameworks to consider and create the run time view for the frameworks of this type only. Type values are: code, tst, and edu. Default value is all.
-l Create symbolic links rather than copying files. Unix only (default).
-lFW FWlist Create the run time view of the frameworks the names of which are listed in the file located at the path FWlist.
-s Strip the executables. Unix only, and with the -t option. It makes sense with other values than the default $MkmkOS_VAR value.
-t TargetPath Path of the folder where the run time view must be copied. Default is $MkmkOS_VAR. If you use another value as the default value, modules are processed. It means that the modules in $MkmkOS_VAR/code/bin will be copied into the run time view.
-v Verbose mode.
-W WSPath Path of the workspace to process. Default is the current folder.
-x Append to destination. Only with -t.
-xml XmlFile Specify the XML file.

Examples

mkCreateRuntimeView

creates symbolic links from the CNext directory in the current workspace into the $Mkms_OS_VAR of the current workspace.

mkCreateRuntimeView -t /tmp/myCNext -c

creates the run time view of the current workspace into the /tmp/myCNext directory.

mkCreateRuntimeView -W /u/users/gus/ADELE/MYWS

creates the run time view of the /u/users/gus/ADELE/MYWS
