mkRemoveDo
Quick Reference

Abstract

The mkRemoveDo command deletes the framework derived objects from a workspace.

  • Usage
  • Options
  • Examples
  • References

Synopsis

mkRemoveDo [-W WSPath] [FW1 [FW2 ...] | -lFW FWlist | -a ] [-preq | -nopreq] [-fw | -nofw] [-o OS1 [-o OS2 ...]] [-simul] [-jobs #] [-v] [-debug] [-h | -help | -?]

Usage

This command is used to delete all derived objects into read/write frameworks of a development workspace.

A framework is read/write only if a file named IdentityCard.h exists into the IdentityCard folder. Note that mkCopyPreq command will create read only versions for all prerequisites frameworks. Then mkRemoveDo will not remove any files from the frameworks created by the mkCopyPreq command.

An operating system can be indicated to limit the scope of the deletion.

mkRemoveDo deletes the derived objects which are coming from

  • the mkCopyPreq command
  • or from any framework whatever.

Options

mkRemoveDo accepts the following options:

Option Description
-a Consider all the frameworks of the workspace.
-debug Debug mode.
-fw Delete the derived objects of all the frameworks except those that were copied using mkCopyFw only.
-h|-help|-? Display the command help.
-jobs # Multiprocessing mode. # is the maximum number of parallel tasks mkRemoveDo forks. Useful to limit the number of tasks when the machine is heavily used. Default is 5.
-lFW FWlist Delete the derived objects of the frameworks the names of which are listed in the file located at the path FWlist.
-o Set an operating system to consider and delete the derived objects for this operating system only. Default is all available operating systems.
-nofw Do not delete frameworks that were copied using mkCopyFw.
-nopreq Delete the derived objects of all the frameworks except those that were copied using mkCopyPreq.
-preq Delete the derived objects of the frameworks that were copied using mkCopyPreq only.
-simul Simulate the command execution.
-v Verbose mode.
-W WSPath Path of the workspace to process. Default is the current folder.
FW1 [FW2]... Delete the derived objects of FW1, FW2, etc.

Examples

Deletes all the derived objects of all the frameworks of the workspace located in the current folder:

mkRemoveDo

Deletes the intel-a related derived objects of the frameworks that were copied using mkCopyPreq in the workspace located in e:\MyWorkspace:

mkRemoveDo -W e:\MyWorkspace -preq -o intel_a
