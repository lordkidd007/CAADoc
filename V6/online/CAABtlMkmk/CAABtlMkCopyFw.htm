mkCopyFw
Quick Reference

Abstract

The mkCopyFw command copies frameworks from a workspace to another one.

  • Usage
  • Options
  • Examples
  • References

Synopsis

mkCopyFw [-W WSPath] -p Path1;|:Path2;|:... [-no|-o OS1 [-o OS2 ...]] [-update] [-remove] [-nm] [-d] [-f] [-jobs #] [-v][-debug] [FW1 [FW2 ...] | -lFW FWlist] [-simul] [-h | -help | -?]

Usage

mkCopyFw copies the build-time and run-time files of a framework from a workspace into another workspace.

Please note that mkCopyFw replaces the existing frameworks only for a case where these frameworks were previously created by the mkCopyFw command.

mkCopyFw can be used to copy frameworks from an Unix system into Windows system and vice versa. But this functionality can be involved only from a Windows platform.

If one of the frameworks is already existing in the target workspace, an error message is displaying and the copy stops. Use the -remove option which removes existing frameworks before copying them again.

Options

mkCopyFw accepts the following options:

Option Description
-d Copy the CNext data folder.
-debug Debug mode.
-f Copy FunctionTests folders.
-h|-help|-? Display the command help.
-jobs # Multiprocessing mode. # is the maximum number of parallel tasks mkCopyFw forks. Useful to limit the number of tasks when the machine is heavily used. Default is 5.
-lFW FWlist Copy the frameworks the names of which are listed in the file located at the path FWlist.
-nm Do not copy modules (src and Objects directories).
-no Do not copy derived objects. Cannot be used with -o.
-o Set an operating system to consider and copy the derived objects for this operating system. Default is all available operating systems. Cannot be used with -no.
-p Path1[;|:Path2...] Concatenate workspaces where to search for the frameworks to copy. Path1 and Path2 represent workspace root folder paths, and must be separated using a ":" (colon) with Unix and a ";" (semi-colon) with Windows. Windows workspace paths can be mapped drive paths, such a e:\WorkspaceToAdd, or UNC paths, such as \\dsone\rnd\R211rel\BSF.
-remove Clean up in the target workspace the frameworks to be copied, if they already exist in there.
-simul Simulate the command execution.
-update Force the copy if the frameworks already exist in the target workspace. Note that mkCopyFw does not take care of deleted objects.
-v Verbose mode.
-W WSPath Path of the workspace to process. Default is the current folder.
FW1 [FW2]... Copy FW1, FW2, etc.

Examples

Copy the FW1 framework from the r:\R212rel\BSF workspace into the current folder. Data related to all available operating systems are copied.

mkCopyFw -p r:\R211rel\BSF FW1

Copy FW1 and FW2 intel-a related data from the r:\R212rel\BSF workspace into the e:\MyWorkspace folder. If the intel_a related data already exist, it is first removed before being copied.

mkCopyFw -W e:\MyWorkspace -p r:\R211rel\BSF -o intel_a -remove FW1 FW2
