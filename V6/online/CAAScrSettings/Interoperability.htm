Interoperability Setting Repository
Quick Reference

Interoperability

Role: Setting to handle the reading of V4 data

The Interoperability repository has the following characteristics:

  • Primary Tab Page: /General/Compatibility/V4 Data Reading
  • Repository File: Interoperability
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Setting controller:
    • Framework_CPP: CATIAV4Interfaces
    • Framework_IDL: CATIAV4Interfaces
    • Interface_CPP: CATIV4IInteropSettingAtt
    • Interface_IDL: CATIAInteropSettingAtt
    • LateType: CATV4IInteropSettingCtrl

Interoperability setting repository attributes:

  • Code_page
    Role: The "Reading Code page" declares the language to identify the data read if this data is not labeled. It is stored in the CATIA data to be written when saving Version 5 CATPart documents as CATIA Version 4 models.
    • Type: integer
    • Default value: 8
    • Values:
      0 French
      1 Chinese
      2 German
      3 Italian
      4 Japanese
      5 Korean
      6 Swedish
      7 UK_English
      8 US_English
  • Conversion_Table
    Role: The "Characters Equivalence Table" allows to convert characters contained in CATIA V4 documents. It is mainly used to generate V5 compliant names from V4 names by replacing special characters (", *, /, etc...) by standard characters.
    • Type: string
    • Default value: 0
  • DisplayMode
    Role: Enable to decide whether to display or not the elements which were sensitive to the shading mode in CATIA Version 4.
    • Type: integer
    • Default value: 524288
    • Values:
      262144 Display.
      524288 Not display.
  • DisplayV4Text3D
    Role: Enable the reading of the 3D text associated to the V4 elements.
    • Type: integer
    • Default value: 1
    • Values:
      1 Display.
      0 Not Display.
  • Dlname
    Role: Retrieves the DLNAME referenced by the Model.
    • Type: string
    • Default value:
  • Draw
    Role: Disable the reading of DRAW data in a V4 Model.
    • Type: string
    • Default value: Draw
  • PROJECT_File_Path
    Role: Contain the location of the external project file referenced by the V4 model.
    • Type: string
    • Default value:
