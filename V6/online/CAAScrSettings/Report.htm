Report Setting Repository
Quick Reference

Report

Role: Defines some options for the Report generation

Detailed role:Enable the user to deal with these categories of options: Configuration of the Check Report, Input XSL, Report content and Output directory

The Report repository has the following characteristics:

  • Primary Tab Page: /General/Parameters and Measure/Report Generation
  • Repository File: Report
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

Report setting repository attributes:

  • AllCheck
    Role: Defines if the generated report will contain information about the failed checks only
    Detailed role: Enables the user to define if the generated report will contain information about the failed checks only: true means all checks, false means failed checks only
    • Type: boolean
    • Default value: true
  • CheckAdvSelect
    Role: Defines if the generated report will contain information about all the Knowledge Advisor checks
    Detailed role: Enables the user to define if the generated report will contain information about all the Knowledge Advisor checks: true means with all the Knowledge Advisor checks, false means without
    • Type: boolean
    • Default value: true
  • CheckExpSelect
    Role: Defines if the generated report will contain information about all the Knowledge Expert checks
    Detailed role: Enables the user to define if the generated report will contain information about all the Knowledge Expert checks: true means with all the Knowledge Expert checks, false means without
    • Type: boolean
    • Default value: true
  • HTMLIntoCATIA
    Role: Defines if the report will be opened in a CATIA session
    Detailed role: This option is available for Windows only. It enables the user to define if the report will be opened in a CATIA session
    • Type: boolean
    • Default value: true
  • InputXSL
    Role: Selects the XSL style sheet that will be applied to the generated XML report
    Detailed role: Enables the user to select the XSL style sheet that will be applied to the generated XML report (note that this option is available only if the XML configuration setting is set)
    • Type: URN
    • Default value: UNIX: ${CATKnowledgePath}[1]/knowledgeResources/StyleSheet.xsl Windows: ${CATKnowledgePath}[1]\knowledgeResources\StyleSheet.xsl
  • ObjAttr
    Role: Defines if the generated report will contain a list of the objects attributes
    Detailed role: Enables the user to define if the generated report will contain a list of the objects attributes
    • Type: boolean
    • Default value: true
  • OutputDirectory
    Role: Selects the output directory that will contain the generated report
    Detailed role: Enables the user to select the output directory that will contain the generated report
    • Type: URN
    • Default value: UNIX: ${CATTemp} Windows: ${CATTemp}
  • ParametersValue
    Role: Defines if the generated report will contain the parameters values
    Detailed role: Enables the user to define if the generated report will contain the parameters values: true means with the parameters values, false means without
    • Type: boolean
    • Default value: true
  • PassedExp
    Role: Defines if the generated report will contain information about the objects that passed the Expert checks
    Detailed role: Enables the user to define if the generated report will contain information about the objects that passed the Expert checks as well as information about the parameters of these objects (diameter, depth, pitch,...)
    • Type: boolean
    • Default value: true
  • ReportHtml
    Role: Generates a HTML or XML report
    Detailed role: Generates a HTML or XML report
    • Type: integer
    • Default value: 1
    • Values:
      0 Generates a XML report
      1 Generates a HTML report
