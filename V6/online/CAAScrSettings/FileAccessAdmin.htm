FileAccessAdmin Setting Repository
Quick Reference

FileAccessAdmin

Role: Setting to handle the parameters of the File Access Admin Options user interface for the Open/Save/choose commands

Detailed role:This setting allows to customize the behavior of the File Access Open/Save/Choos commands through the user interface. It could be either File or DLName.

The FileAccessAdmin repository has the following characteristics:

  • Primary Tab Page: /General/TEST/File Access Administration
  • Repository File: FileAccessAdmin
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

FileAccessAdmin setting repository attributes:

  • AccessMode
    Role: Control the File Access mode.
    Detailed role: This option allows to Control the File Access type : File or DLName.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default File Access mode : File
    • Values:
      1 File Access mode
      2 DLName Access mode
  • ConfigureDLName
    Role: Control the accessibility of the DLName's configuration.
    Detailed role: This option allows to Control the accessibility of the DLName's configuration : Available or not.
    • Type: integer
    • Default value: 0
    • Default Value Role: By default DLName's configuration accessibility : Available
    • Values:
      0 DLName's configuration accessibility
