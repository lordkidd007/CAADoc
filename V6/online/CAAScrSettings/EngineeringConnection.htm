EngineeringConnection Setting Repository
Quick Reference

EngineeringConnection

Role: Setting to manage Assembly Design options

Detailed role:Engineering Cnx setting contains all options necessary to Engineering Cnx related commands

The EngineeringConnection repository has the following characteristics:

  • Primary Tab Page: /Mechanical Design/Assembly Design/Engineering connection
  • Repository File: EngineeringConnection
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Root tab page /Mechanical Design/Assembly Design/General reused as /Mechanical Design/Assembly Design/Constraints

EngineeringConnection setting repository attributes:

  • BrokenCstsFbdiMode
    Role: File based design import of broken Constraints mode
    Detailed role: According to this attribute's value, the file based design import of broken Constraints can be managed in three differents ways
    • Type: integer
    • Default value: 0
    • Default Value Role: By default Broken Constraints are imported and deactivated
    • Values:
      0 Broken Constraints are imported and deactivated
      1 Broken Constraints are imported
      2 Broken Constraints are not imported
  • ConstraintCreationMode
    Role: Constraint creation mode
    Detailed role: According to this attribute's value, constraints can be created on any geometry or only on published geometry
    • Type: integer
    • Default value: 0
    • Default Value Role: By default constraint can be created on any geometry
    • Values:
      0 The constraint can be created on any kind of geometry
      1 The constraint can only be created on geometry published on the direct child level
      2 The constraint can only be created on geometry published on any assembly level
  • EnggConnectionAnimateSolve
    Role: Animates the update solve of consraints
    Detailed role: According to this attribute's value, the animation of solve of constriant is done or not.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default animation is to be done on solve of constraints
    • Values:
      0 No animation is done on solve of constraints
      1 Animation of products is done during solve of constraints
  • EnggConnectionChildNodeVisualizationStatus
    Role: Controls the visualization display of Engineering Connection in 3D view
    Detailed role: According to this attribute's value, the visualization of Engineering Connection is displayed in the 3D view.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default child node visualization is shown
    • Values:
      0 No child node visualization is shown
      1 Child node visualization is shown on appropriate selections
  • EnggConnectionOrientation
    Role: Sets the orientation of engineering connection to all instances can move
    Detailed role: According to this attribute's value, the default orientation of engineering connection is set to all instances can move.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default Engineering Connection orientation is not defined
    • Values:
      0 Orientation is not defined by default
      1 Engineering Connection orientation is set to all instances can move
  • EnggConnectionSolve
    Role: Controls weather instance will be positioned on creation of constraints or not
    Detailed role: According to this attribute's value, when the constraint is created the instances involves will be position to satisfy the constraint or not
    • Type: integer
    • Default value: 1
    • Default Value Role: By default Instances will be positioned on constraint creation
    • Values:
      0 Instances will not be positioned on constraint creation
      1 Instances will be positioned on constraint creation
  • EnggConnectionType
    Role: Controls the type of Engineering Connection by default
    Detailed role: According to this attribute's value, the type of Engineering Connection is set to the most equivalent.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default Engineering Connection type is set to the equivalent type when ok is pressed
    • Values:
      0 The type of engineering conneciton is not set to the equivalent on pressing ok in the panel
      1 Engineering Connection type is set to the equivalent type when ok is pressed
  • EnggConnectionVisualizationStatus
    Role: Controls the visualization display of Engineering Connection in 3D view
    Detailed role: According to this attribute's value, the visualization of Engineering Connection is displayed in the 3D view.
    • Type: integer
    • Default value: 0
    • Default Value Role: By default no Engineering Connection visualization is shown
    • Values:
      0 No Engineering Connection visualization is shown
      1 Engineering Connection visualization is shown on appropriate selections
  • InterferenceWarningMode
    Role: Interferance Warning when leaving Engineering Connection panel
    Detailed role: According to this attribute's value, a warning can be shown when exiting the Engineering Connection panel if no interference specification have been made or if current interference status does not match current specifications.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default warning are shown
    • Values:
      0 No warning is shown
      1 Warning are shown
  • MRXOrderedList
    Role: Constraints ordered list for Engineering connection creation
    Detailed role: This attribute is the ordered list, used by commands such as Engineering connection creation, and specifies the constraint creation priority. Note that this array of 12 values must not contain any duplicates
    • Type: string
    • Default value: Tangency
    • Default Value Role: Default order is Contact, FixTogetherTransfo, FixTogether, Coincidence, Offset, Angle, Parallelism, Perpendicularity, Length, Roll, Slide, Tangency
  • QuickConstraintMode
    Role: Quick constraint mode
    Detailed role: This attribute is used by commands such as Quick Constraint, which create the most accurate constraint from the context. The priority order may be determined by a predefined list or by the constraints solver
    • Type: integer
    • Default value: 0
    • Default Value Role: By default constraint is created according to the specified order
    • Values:
      0 Use the specified order
      1 Create verified constraint first
  • QuickConstraintOrderedList
    Role: Constraints ordered list for Quick constraints creation
    Detailed role: This attribute is the ordered list, used by commands such as Quick Constraint, and specifies the constraint creation priority. Note that this array of 6 values must not contain any duplicates
    • Type: string
    • Default value: PerpendType
    • Default Value Role: Default order is Contact, Coincidence, Offset, Angle, Parallelism, Perpendicularity
