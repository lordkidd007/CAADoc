AssemblyDesign Setting Repository
Quick Reference

AssemblyDesign

Role: Setting to manage Assembly Design options

Detailed role:Assembly Design setting contains all options necessary to Assembly Design commands

The AssemblyDesign repository has the following characteristics:

  • Primary Tab Page: /Mechanical Design/Assembly Design/General
  • Repository File: AssemblyDesign
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Root tab page /Mechanical Design/Assembly Design/General reused as /Mechanical Design/Assembly Design/Constraints

AssemblyDesign setting repository attributes:

  • AsmPatternRepairMode
    Role: Attribute to manage AsmPatternRepair mode (automatic or manual)
    Detailed role: Automatically repairs assembly pattern, or displays update error whenever a input pattern modification is performed
    • Type: integer
    • Default value: 1
    • Default Value Role: By default assembly pattern is automatically repaired during PLM Update, without configuration management.
    • Values:
      0 Update error pops up during PLM Update forcing manual assembly pattern repair with or without configuration management.
      1 Automatically repairs assembly pattern during PLM Update, without configuration management.
  • BrokenCstsFbdiMode
    Role: File based design import of broken Constraints mode
    Detailed role: According to this attribute's value, the file based design import of broken Constraints can be managed in three differents ways
    • Type: integer
    • Default value: 0
    • Default Value Role: By default Broken Constraints are imported and deactivated
    • Values:
      0 Broken Constraints are imported and deactivated
      1 Broken Constraints are imported
      2 Broken Constraints are not imported
  • ConstraintCreationMode
    Role: Constraint creation mode
    Detailed role: According to this attribute's value, constraints can be created on any geometry or only on published geometry
    • Type: integer
    • Default value: 0
    • Default Value Role: By default constraint can be created on any geometry
    • Values:
      0 The constraint can be created on any kind of geometry
      1 The constraint can only be created on geometry published on the direct child level
      2 The constraint can only be created on geometry published on any assembly level
  • EastCompassDuringUpdate
    Role: Attribute to manage East Compass activation during Update
    Detailed role: The East Compass is activated during Update according to this attribute's value
    • Type: integer
    • Default value: 0
    • Default Value Role: By default East Compass is deactivated during Update
    • Values:
      0 The East Compass is deactivated during Update
      1 The East Compass is activated during Update (potential bad performances)
  • EnggConnectionAnimateSolve
    Role: Animates the update solve of consraints
    Detailed role: According to this attribute's value, the animation of solve of constriant is done or not.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default animation is to be done on solve of constraints
    • Values:
      0 No animation is done on solve of constraints
      1 Animation of products is done during solve of constraints
  • EnggConnectionOrientation
    Role: Sets the orientation of engineering connection to all instances can move
    Detailed role: According to this attribute's value, the default orientation of engineering connection is set to all instances can move.
    • Type: integer
    • Default value: 0
    • Default Value Role: By default Engineering Connection orientation is not defined
    • Values:
      0 Orientation is not defined by default
      1 Engineering Connection orientation is set to all instances can move
  • EnggConnectionType
    Role: Controls the type of Engineering Connection by default
    Detailed role: According to this attribute's value, the type of Engineering Connection is set to the most equivalent.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default Engineering Connection type is set to the equivalent type when ok is pressed
    • Values:
      0 The type of engineering conneciton is not set to the equivalent on pressing ok in the panel
      1 Engineering Connection type is set to the equivalent type when ok is pressed
  • EnggConnectionVisualizationStatus
    Role: Controls the visualization display of Engineering Connection in 3D view
    Detailed role: According to this attribute's value, the visualization of Engineering Connection is displayed in the 3D view.
    • Type: integer
    • Default value: 0
    • Default Value Role: By default no Engineering Connection visualization is shown
    • Values:
      0 No Engineering Connection visualization is shown
      1 Engineering Connection visualization is shown on appropriate selections
  • InterferenceWarningMode
    Role: Interferance Warning when leaving Engineering Connection panel
    Detailed role: According to this attribute's value, a warning can be shown when exiting the Engineering Connection panel if no interference specification have been made or if current interference status does not match current specifications.
    • Type: integer
    • Default value: 1
    • Default Value Role: By default warning are shown
    • Values:
      0 No warning is shown
      1 Warning are shown
  • IsUpToDateAutoMode
    Role: Attribute to manage IsUpToDate mode (automatic or manual)
    Detailed role: Automatically computes real update status, or highlight update icon whenever a modification is performed
    • Type: integer
    • Default value: 0
    • Default Value Role: By default update icon gets highlighted whenever a modification is performed
    • Values:
      0 Update icon gets highlighted whenever a modification is performed
      1 Automatically computes real update status
  • MoveWithFixTExtendMode
    Role: Attribute managing the extension of a move to the components involved in a FixTogether
    Detailed role: When a component involved in a FixTogether is moved, same move may be applied to all other components involved in the same FixTogether, according to this attribute's value
    • Type: integer
    • Default value: 0
    • Default Value Role: By default Move is never extended
    • Values:
      0 Never extend move to all components involved in a FixTogether
      1 Ask question to extend move to all component involved in a FixTogether
      2 always extend move to all component involved in a FixTogether
  • MRXOrderedList
    Role: Constraints ordered list for Engineering connection creation
    Detailed role: This attribute is the ordered list, used by commands such as Engineering connection creation, and specifies the constraint creation priority. Note that this array of 11 values must not contain any duplicates
    • Type: string
    • Default value: Slide
    • Default Value Role: Default order is Contact, FixTogetherTransfo, FixTogether, Coincidence, Offset, Angle, Parallelism, Perpendicularity, Length, Roll, Slide
  • NewExperienceInAssemblyApp
    Role: Attribute to manage Assembly Design user experience
    Detailed role: The new user experience is available in Assembly Design according to this attribute's value
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the new user experience is deactivated in Assembly Design
    • Values:
      0 The new user experience is deactivated in Assembly Design
      1 The new user experience is activated in Assembly Design
  • QuickConstraintMode
    Role: Quick constraint mode
    Detailed role: This attribute is used by commands such as Quick Constraint, which create the most accurate constraint from the context. The priority order may be determined by a predefined list or by the constraints solver
    • Type: integer
    • Default value: 0
    • Default Value Role: By default constraint is created according to the specified order
    • Values:
      0 Use the specified order
      1 Create verified constraint first
  • QuickConstraintOrderedList
    Role: Constraints ordered list for Quick constraints creation
    Detailed role: This attribute is the ordered list, used by commands such as Quick Constraint, and specifies the constraint creation priority. Note that this array of 6 values must not contain any duplicates
    • Type: string
    • Default value: PerpendType
    • Default Value Role: Default order is Contact, Coincidence, Offset, Angle, Parallelism, Perpendicularity
  • UpdateAskedAtSaveMode
    Role: Attribute to manage the update panel at save (display when saved data is out of date or not)
    Detailed role: Depending on this attribute, a PLM Update panel will appear at Save if the data is out of date to ask for update, or not
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the panel never appears at Save
    • Values:
      0 The panel never appears at Save
      1 The panel appears at Save if the data is out of date
  • UpdateBehaviorOnErrorsMode
    Role: Attribute to manage PLM Update behaviour on errors mode (stop or continue)
    Detailed role: Depending to this attribute, the PLM Update will stop on the first error encountered or will update all that is possible to update (all that is not impacted by encountered errors)
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the PLM Update is stoped on first error
    • Values:
      0 The PLM Update is stoped on first error
      1 The PLM Update updates all that is possible to update
  • UpdateBehaviorOnWarningsMode
    Role: Attribute to manage PLM Update behaviour on warnings mode (display on the report or not)
    Detailed role: Depending on this attribute, the PLM Update report will display the encountered warnings or not
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the PLM Update report displays the warnings
    • Values:
      0 The PLM Update report displays the warnings
      1 The PLM Update report hides the warnings
