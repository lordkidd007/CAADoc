Disconnection Setting Repository
Quick Reference

Disconnection

Role: Automatic disconnection setting

Detailed role:Setting used to handle the parameters of automatic disconnection. If the automatic disconnection is enabled, the V5 session will be kill after a user-defined duration of inactivity.

The Disconnection repository has the following characteristics:

  • Primary Tab Page: /General/General
  • Repository File: CATAutoLogoff
  • Version: 1
  • Full Compatibility: 0
  • Upward Compatibility: 0

Disconnection setting repository attributes:

  • Activation
    Role:
    • Alias: AutoLogoffActive
    • Type: boolean
    • Default value: false
  • InactivityDuration
    Role: timeout in seconds before the automatic disconnection when no activity has been detected, if the mechanism is enabled
    • Alias: MaximumInactivity
    • Type: integer
    • Default value: 300
