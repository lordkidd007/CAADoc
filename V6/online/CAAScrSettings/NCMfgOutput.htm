NCMfgOutput Setting Repository
Quick Reference

NCMfgOutput

Role: Lists the settings of the outputs generated in the Machining applications

Detailed role:The output machining settings are useful to manages options on manufacturing programs outputs

The NCMfgOutput repository has the following characteristics:

  • Primary Tab Page: /Manufacturing/Output
  • Repository File: NCMfgOutput
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

NCMfgOutput setting repository attributes:

  • CEPath
    Role: Defines the path where the Controller Emulators are located
    Detailed role: Defines the path where the Controller Emulators are located
    • Type: URN
    • Default value:
  • CEPro
    Role: Choose the Controller Provider company
    Detailed role: Choose the Controller Provider company
    • Type: string
    • Default value:
    • Default Value Role: By default no company preference is done
  • ExportPath
    Role: Defines the export path for NC Files container
    Detailed role: Defines the export path for NC Files container
    • Type: URN
    • Default value: UNIX: ${CATTemp} Windows: ${CATTemp}
  • ExternalCEPath
    Role: Defines the file path of the external Controller Emulators
    Detailed role: Defines the file path of the external Controller Emulators
    • Type: URN
    • Default value: UNIX: ${CATTemp} Windows: ${CATTemp}
  • MinCollisionCheck
    Role: Defines the minimum length for collision detection
    Detailed role: Defines the minimum length for collision detection
    • Type: double
    • Default value: 0.0
    • Default Value Role: By default, the minimum length collision detection is set to 0.
  • NCCodeDirectory
    Role: Defines the path where the NC code files are generated
    Detailed role: Defines the path where the NC code files are generated
    • Type: URN
    • Default value: UNIX: ${CATTemp} Windows: ${CATTemp}
  • NCErrorLimit
    Role: Defines the limit on number of Errors for NCCode generation to stop
    Detailed role: Defines the limit on number of Errors for NCCode generation to stop
    • Type: integer
    • Default value: 10
    • Min value: 0
    • Max value: 10000
  • NCErrorLimitCheck
    Role: Defines if the limit on number of Errors for NCCode generation to stop is to be considered
    Detailed role: Defines if the limit on number of Errors for NCCode generation to stop is to be considered
    • Type: boolean
    • Default value: false
  • NCFileExtension
    Role: Defines the extension name of generated NC code files
    • Type: string
    • Default value: CATNCCode
  • NCWarningLimit
    Role: Defines the limit on number of Warnings for NCCode generation to stop
    Detailed role: Defines the limit on number of Warnings for NCCode generation to stop
    • Type: integer
    • Default value: 10
    • Min value: 0
    • Max value: 10000
  • NCWarningLimitCheck
    Role: Defines if the limit on number of Warnings for NCCode generation to stop is to be considered
    Detailed role: Defines if the limit on number of Warnings for NCCode generation to stop is to be considered
    • Type: boolean
    • Default value: false
  • PostPro
    Role: Choose the Post Processor company
    Detailed role: Choose the Post Processor company
    • Type: string
    • Default value:
    • Default Value Role: By default no company preference is done
  • PPPath
    Role: Defines the path where the Post Processors are located
    Detailed role: Defines the path where the Post Processors are located
    • Type: URN
    • Default value:
  • StoreContactPoints
    Role: Store contact points in toolpath during its computation
    Detailed role: Store contact points in toolpath during its computation
    • Type: boolean
    • Default value: true
  • ToolOutputPoint
    Role: Chooses the reference point to compute toolpath with
    Detailed role: Chooses the reference point to compute toolpath with
    • Type: integer
    • Default value: 1
    • Default Value Role: By default the tool output point is the tool tip
    • Values:
      1 Tool tip
      2 Tool center
      3 Tool center for ball end tools
  • ToolPathEditorAccess
    Role: Allows Tool Path edition
    Detailed role: Allows Tool Path edition
    • Type: boolean
    • Default value: true
  • TPAutoPack
    Role: Pack tool path automatically after computation
    Detailed role: Pack tool path automatically after computation
    • Type: boolean
    • Default value: false
