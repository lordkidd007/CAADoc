Knowledge Setting Repository
Quick Reference

Knowledge

Role: Defines some Knowledge Objects behaviors

Detailed role:Enable the user to deal with these categories of options: Parameter Tree View, Parameter names, Relations update in part context and Design Tables

The Knowledge repository has the following characteristics:

  • Primary Tab Page: /General/Parameters and Measure/Knowledge
  • Repository File: Knowledge
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

Knowledge setting repository attributes:

  • DesignTableDefaultCopyMode
    Role: Defines the defaut mode for copying data in model
    Detailed role: Enables the user to define the defaut mode: to copy data contained in the external source file into the model or not: true means the data will be copied
    • Type: boolean
    • Default value: false
  • DesignTableRepRef
    Role: Defines the defaut mode for rep ref usage in Design Table creation
    Detailed role: Enables the user to define rep ref usage in Design Table creation: true means that the rep ref are retrieved in PLM Chooser during design table creation, false means that plm documents are used
    • Type: boolean
    • Default value: false
  • DesignTableSynchroMode
    Role: Defines the type of synchronisation for Design Tables
    Detailed role: Enables the user to define the type of synchronisation for Design Tables: Automatic at load, Interactive at load or Manual
    • Type: integer
    • Default value: 0
    • Values:
      0 Automatic Synchronisation At Load
      1 Interactive Synchronisation At Load
      2 Manual Synchronisation
  • EncadrementPathnames
    Role: Defines if parameter names should be surrounded by the symbol `
    Detailed role: This option should be set to true if you work with non-Latin characters. If this option is set to false, parameter names should have to be renamed in Latin characters when used in formulas
    • Type: boolean
    • Default value: false
  • ParametersViewWithFormula
    Role: Displays the formulas constraining the parameter in the specification tree
    Detailed role: Displays the formulas constraining the parameter in the specification tree
    • Type: boolean
    • Default value: false
  • ParametersViewWithValue
    Role: Displays the parameter values in the specification tree
    Detailed role: Displays the parameter values in the specification tree
    • Type: boolean
    • Default value: false
  • RelationIntegratedToUpdate
    Role: Enables the user to associate the evaluation of asynchronous relations with the global update
    Detailed role: Enables the user to associate the evaluation of asynchronous relations with the global update
    • Type: boolean
    • Default value: false
  • RelationSynchronousAtCreation
    Role: Enables the user to create synchronous relations
    Detailed role: Enables the user to create synchronous relations: when creating the relation, it defines if the relation will be synchronous (i.e. the evaluation will be launched as soon as one of its parameters is modified) or asynchronous (i.e. the evaluation will be launched when the Part is updated)
    • Type: boolean
    • Default value: true
