AutoLogon Setting Repository
Quick Reference

AutoLogon

Role: Auto Logon activation definition

Detailed role:Controls the Auto Logon mode and which are the Data Sources for which Auto Logon must be done. For the moment, Auto Logon can be done for only one Data Source.

The AutoLogon repository has the following characteristics:

  • Primary Tab Page: /General/Connection
  • Repository File: AutoLogon
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

AutoLogon setting repository attributes:

  • AutoLogonActivated
    Role: Defines whether or not the Auto Logon mode is activated.
    • Type: boolean
    • Default value: false
  • DataSourcesList
    Role: Defines the list of Data Sources for which Auto Logon must be done. For the moment, Auto Logon can be done for only one Data Source.
