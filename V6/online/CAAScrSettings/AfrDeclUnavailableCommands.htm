AfrDeclUnavailableCommands Setting Repository
Quick Reference

AfrDeclUnavailableCommands

Role: Setting to handle the availability of commands

Detailed role:This setting allows to configure the availability of commands belonging to declarative workshop, workbench or add-in.

The AfrDeclUnavailableCommands repository has the following characteristics:

  • Primary Tab Page: /General
  • Repository File: AfrDeclUnavailableCommands
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

AfrDeclUnavailableCommands setting repository attributes:

  • *AddinWorkbench
    Role: Add-in of a workbench containing unavailable commands
    • Type: string
    • Default value:
  • *AddinWorkshop
    Role: Add-in of a workshop containing unavailable commands
    • Type: string
    • Default value:
  • *AdnWbLength
    Role: Number of unavailable commands for the associated add-in of workbench
    • Type: integer
    • Default value: 0
  • *AdnWsLength
    Role: Number of unavailable commands for the associated add-in of workshop
    • Type: integer
    • Default value: 0
  • *WbLength
    Role: Number of unavailable commands for the associated workbench
    • Type: integer
    • Default value: 0
  • *Workbench
    Role: Workbench containing unavailable commands
    • Type: string
    • Default value:
  • *Workshop
    Role: Workshop containing unavailable commands
    • Type: string
    • Default value:
  • *WsLength
    Role: Number of unavailable commands for the associated workshop
    • Type: integer
    • Default value: 0
  • Lock
    Role: Indicates whether this tools option page is locked by an admin.
    • Type: integer
    • Default value: 0
