DNBSimulationSettings Setting Repository
Quick Reference

DNBSimulationSettings

Role: Settings to handle default simulation parameters

Detailed role:Default settings that will be used when performing a simulation. Please note that some applications might have their own ones.

The DNBSimulationSettings repository has the following characteristics:

  • Primary Tab Page: /Infrastructure/Delmia Infrastructure/Simulation
  • Repository File: PLMSimulationSettings
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Root tab page /x reused as /x

DNBSimulationSettings setting repository attributes:

  • SimulationEndTime
    Role: Default user-specified start time for the simulation beginning.
    Detailed role: This time will constraint the simulation start time. The simulation will stop whenever the simulation reaches the end time specified (or earlier if the simulated entity requires it). Please note that if the simulation end time is set to 0, there is no constraint: the simulation will go on until the simulated object ends the simulation itself.
    • Type: double
    • Default value: 0
  • SimulationGraphicsUpdatesYesOrNo
    Role: Activation mode of the graphics update.
    Detailed role: Graphics update can be controlled either by simulation steps or by duration. Although simulation is running, there is no graphics update between the speficied steps or duration. When the graphics update is turned OFF, there is no graphical update, except viewpoint manipulations (zoom, panning, view mode, etc..).
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Graphics update are deactivated
      1 Graphics update are activated
  • SimulationMasterClockStepSize
    Role: Default step size of each simulation.
    Detailed role: The master clock controls the step size of the simulation to provide a specific interval during refresh. Each simulated may have specific step size. However, the master clock allows to control the simulation by constraining the step size. This step size represents the refresh rate of the simulation and not the computation rate (step size for specific solvers).
    • Type: double
    • Default value: 0.1
  • SimulationMasterClockYesOrNo
    Role: Activation mode of the constrained step size.
    Detailed role: The master clock controls the step size of the simulation to provide a specific interval during refresh. Each simulated may have specific step size. However, the master clock allows to control the simulation by constraining the step size. This step size represents the refresh rate of the simulation and not the computation rate (step size for specific solvers). If this option is turned OFF, the step size during a simulation may not be constant.
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Constrained step is OFF
      1 Constrained step is ON
  • SimulationRealTimeYesOrNo
    Role: Activation mode of the real time simulation.
    Detailed role: Real time simulation will synchronize the simulation time with the actual computer time. Please note that the real time simulation will not aceelerate the simulation: it will only delay it to mimic the computer time.
    • Type: unsigned integer
    • Default value: 0
    • Values:
      0 Real time simulation is OFF
      1 Real time simulation is ON
  • SimulationSpecTreeVisibility
    Role: Controls specification tree visibility during simulation.
    Detailed role: For presentation purpose, users can hide the tree during simulation..
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Specification tree is hidden
      1 Specification tree is shown
  • SimulationStartTime
    Role: Default user-specified end time for the simulation end.
    Detailed role: This time will constraint the simulation start time. The simulation will start from the specified time. As a consequence, when the simulation ends, it will restart from the specified point.
    • Type: double
    • Default value: 0
  • SimulationSynchronousRefresh
    Role: Activation mode of the synchroneous refresh mode.
    Detailed role: Allows to activate the synchroneous refresh.
    • Type: unsigned integer
    • Default value: 1
    • Values:
      0 Synchroneous refresh is activated
      1 Synchroneous refresh is deactivated
  • SimulationUpdateRate
    Role: Number of steps between each viewer refresh.
    Detailed role: Graphics update can be controlled either by simulation steps or by duration. Although simulation is running, there is no graphics update between the speficied steps or duration.
    • Type: unsigned integer
    • Default value: 1
  • SimulationUpdateTime
    Role: Interval time between each viewer refresh.
    Detailed role: Graphics update can be controlled either by simulation steps or by duration. Although simulation is running, there is no graphics update between the speficied steps or duration.
    • Type: float
    • Default value: 0.1
