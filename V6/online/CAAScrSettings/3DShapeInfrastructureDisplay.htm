3DShapeInfrastructureDisplay Setting Repository
Quick Reference

3DShapeInfrastructureDisplay

Role: Setting to manage some 3D Shape Infrastructure options

The 3DShapeInfrastructureDisplay repository has the following characteristics:

  • Primary Tab Page: /Infrastructure/3D Shape Infrastructure/Display
  • Repository File: part
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

3DShapeInfrastructureDisplay setting repository attributes:

  • BodiesUnderOperationsInTree
    Role: Displays bodies under assembly operations in tree
    Detailed role: Determines if a Body node is displayed when it is being aggregated under a boolean operation (Add, Assemble, Remove, Intersect, Union Trim).
    • Alias: VisuTool
    • Type: boolean
    • Default value: true
  • ConstraintsInGeometry
    Role: Enables the permanent display of feature parameters and constraints
    Detailed role: Enables constraints to be visualized in the 3D view.
    • Alias: ParmDisplayMode
    • Type: boolean
    • Default value: false
  • ConstraintsNodeInTree
    Role: Displays the constraints in tree
    Detailed role: Displays the tree nodes called "Constraints" that contain the constraints.
    • Alias: VisuConstraint
    • Type: boolean
    • Default value: false
  • DisplayGeometryAfterCurrent
    Role: Displays geometry located after the current feature
    Detailed role: Enables to visualize in the 3D features after the in work object in ordered geometric set and "solid and surface set".
    • Alias: FutureGeometry
    • Type: boolean
    • Default value: false
  • ExpandSketchBasedFeaturesNodeAtCreation
    Role: Expands sketch-based feature nodes at creation to display sketch nodes in the tree
    Detailed role: Determines if tree nodes for sketch-based features are expanded when such elements are created. This will enable to view their sketch node.
    • Alias: CollapseSketch
    • Type: boolean
    • Default value: false
  • ExternalReferencesNodeInTree
    Role: Displays external references in tree
    Detailed role: Displays the tree node called "External References" that contains all external references.
    • Alias: VisuExternal
    • Type: boolean
    • Default value: true
  • NamingMode
    Role: Check the renaming
    Detailed role: Determines how an element can be named through Edit/Properties or any operation creating a feature (Copy-Paste, etc.). When this option is changed, it only affects elements whose name is modified afterwards.
    • Alias: CATMmuSetRenamingMode
    • Type: short integer
    • Default value: 0
    • Values:
      0 No name check
      Detailed role: Naming is rule-free.
      1 Under the same tree node
      Detailed role: No identical names are allowed under the same tree node.
      2 In the main object
      Detailed role: No identical names are allowed in the main object of the current workshop. For instance, no identical names can be given to objects located between a 3D shape and its sketches.
  • OnlyCurrentOperatedSolidSetInGeometry
    Role: Displays only the current operated solid in the geometry area
    Detailed role: Enables to visualize in the 3D only the current operated body feature (operated means being aggregated in a boolean operation), as well as all other bodies and sets directly inserted under the 3D shape feature.
    • Alias: VisuLocalTool
    • Type: boolean
    • Default value: false
  • OnlyCurrentSolidSetInGeometry
    Role: Displays only geometry located in the current solid set
    Detailed role: Enables to visualize in the 3D only features of the solid set that contains the In Work object.
    • Alias: OnlyCurrentBody
    • Type: boolean
    • Default value: false
  • ParametersNodeInTree
    Role: Displays in tree the parameters created using Formula command
    Detailed role: Displays the tree nodes called "Parameters" that contain the parameters created using Formula command.
    • Alias: VisuParameter
    • Type: boolean
    • Default value: false
  • RelationsNodeInTree
    Role: Displays in tree the relations created using Formula command
    Detailed role: Displays the tree nodes called "Relations" that contain the relations created using Formula command.
    • Alias: VisuRelation
    • Type: boolean
    • Default value: false
  • UpdateDependencies
    Role: Update dependencies when updating all
    Detailed role: Updating all in the 3D shape also updates dependencies (shapes containing the imported objects).
    • Tab Page: /Infrastructure/3D Shape Infrastructure/General
    • Type: boolean
    • Default value: false
  • UpdateElementsRefreshed
    Role: Activate local visualization
    Detailed role: Redraws individually updated elements during the update process.
    • Alias: VisuUpdate
    • Tab Page: /Infrastructure/3D Shape Infrastructure/General
    • Type: boolean
    • Default value: false
  • UpdateExternalReferences
    Role: "Synchronize all external references when updating
    Detailed role: Synchronizes all external references with the elements from which they are created to perform an update.
    • Tab Page: /Infrastructure/3D Shape Infrastructure/General
    • Type: boolean
    • Default value: false
  • UpdateMode
    Role: Automatic or manual update of the 3D shape
    Detailed role: Determines how the update of a 3D shape is conducted.
    • Alias: AutoUpdate
    • Tab Page: /Infrastructure/3D Shape Infrastructure/General
    • Type: integer
    • Default value: 1
    • Values:
      1 Automatic
      Detailed role: Automatically updates the 3D shape when it is not updated.
      0 Manual
      Detailed role: Automatically updates only the feature that is modified. The 3D shape has to be updated manually.
  • UpdateStoppedOnError
    Role: Stop update on first error
    Detailed role: Stops the update process on the first detected error.
    • Alias: ContinueUpdate
    • Tab Page: /Infrastructure/3D Shape Infrastructure/General
    • Type: boolean
    • Default value: false
