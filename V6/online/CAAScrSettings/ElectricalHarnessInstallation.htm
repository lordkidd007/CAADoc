ElectricalHarnessInstallation Setting Repository
Quick Reference

ElectricalHarnessInstallation

Role: Electrical Harness Installation setting

Detailed role:Electrical Harness Installation setting

The ElectricalHarnessInstallation repository has the following characteristics:

  • Primary Tab Page: /Equipments/Electrical Harness Installation/Harness Management
  • Repository File: ElectricalHarnessInstallation
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1

ElectricalHarnessInstallation setting repository attributes:

  • AccuracyState
    Role: Setting to manage the balance between performance and accuracy
    Detailed role: Setting to manage the balance between performance and accuracy for the FLEX algorithm
    • Type: integer
    • Default value: 1
    • Min value: 1
    • Max value: 10
  • AllowSystem
    Role: CheckButton to allow System to use a Min Bend Radius for Curve Computation
    Detailed role: Setting to allow System to use a Min Bend Radius for Curve Computation
    • Type: integer
    • Default value: 0
    • Default Value Role: By default The Check Button is UNCHECKED
    • Values:
      0 If Unchecked the User set Bend Radius will be respected
      1 If Checked the System may override the User set Bend Radius
  • AvoidWarningMessagesRepetition
    Role: Check Button to avoid repetition of warning message when building curve
    Detailed role: Avoids repetition of warning message.i.e. when curve is broken OR when curve is built with dichotomy
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the warning message is not avoided
    • Values:
      0 Don't avoid repetition of warning message
      1 Avoid repetition of warning message
  • BNSCollapsed
    Role: Check Button to warn user when BundleSegment is collapsed
    Detailed role: Warns the user when BundleSegment is collapsed
    • Type: integer
    • Default value: 1
    • Default Value Role: By default user gets warning when BundleSegment is collapsed
    • Values:
      0 Dont Warn user when BundleSegment is collapsed
      1 Warn user when BundleSegment is collapsed
  • BNSColorChange
    Role: Check Button to change color of BundleSegment as well
    Detailed role: Changes the color of BundleSegment as well
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the color of BundleSegment doesnot change
    • Values:
      0 Will not Change the color of BundleSegment as well
      1 Will Changes the color of BundleSegment as well
  • BNSColourToSet
    Role: Color to Set.
    • Type: unsigned integer
    • Default value: 0
    • Default Value Role: Red in RGB order
    • Min value: 0
    • Max value: 255
  • BranchCreationFromResources
    Role: Check Button to enable creation of branches using resource data
    Detailed role: Check Button to enable creation of branches using resource data
    • Type: integer
    • Default value: 0
    • Default Value Role: By default branches will be manually created without the use of resource data
    • Values:
      0 Creates branch manually without the use of resource data
      1 Creates branch using resource data only
  • ChoixAlgo
    Role: Setting to manage the algorithm used for branch routing
    Detailed role: Setting to manage the algorithm used for branch routing
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the formal algorithm is used for the branch routing
    • Values:
      0 The formal algorithm is used for the branch routing
      1 The FLEX algorithm is used for the branch routing
  • CollapseGeoHarnessNode
    Role: Collapse Geometrical Harness Node on OK/Cancel of Branch definition panel
    Detailed role: Collapse Geometrical Harness Node in the Spec tree on OK/Cancel of Branch definition panel
    • Type: integer
    • Default value: 0
    • Default Value Role: By default Geometrical Harness Node should not collapse on OK/Cancel of Branch definition panel
    • Values:
      0 Do not Collapse Geometrical Harness Node on OK/Cancel of Branch definition panel
      1 Collapse Geometrical Harness Node on OK/Cancel of Branch definition panel
  • CurveNotElectrical
    Role: Check Button to warn user when Curve is not Electrical
    Detailed role: Warns the user when the Curve is not Electrical
    • Type: integer
    • Default value: 1
    • Default Value Role: By default user gets warning when Curve is not Electrical
    • Values:
      0 Dont Warn user when Curve is not Electrical
      1 Warn user when Curve is not Electrical
  • DefaultLineType
    Role: Check Button to allow the user to set the default line type
    Detailed role: If checked it aloows the user to set the default line type Instantiating the Protection
    • Type: integer
    • Default value: 0
    • Default Value Role: By default user is not allowed to set the default line type
    • Values:
      0 Do not allow the user set the default line type
      1 Allow the user to set the default line type
  • DisableFoldWarningRepetition
    Role: Check Button to enable or disable repetitive warning messages related to fold management
    Detailed role: Check Button to enable or disable repetitive warning messages related to fold management
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the repetitive warning messages related to fold management will appear
    • Values:
      1 Repetitive warning messages related to fold management will appear
      0 Repetitive warning messages related to fold management will not appear
  • FoldAngleHighValue
    Role: Setting to define the highest value for fold angle
    Detailed role: Setting to define the absolute highest value for fold angle
    • Type: double
    • Default value: 3.142857
    • Default Value Role: By default the lowest positive value of fold angle is 180 degree
  • FoldAngleInterval
    Role: Check Button to define the fold angle interval
    Detailed role: Check Button to define the fold angle interval
    • Type: integer
    • Default value: 0
    • Default Value Role: By default fold angle interval is not defined
    • Values:
      1 Fold angle interval defined
      0 Fold angle interval not defined
  • FoldAngleLowValue
    Role: Setting to define the lowest value for fold angle
    Detailed role: Setting to define the lowest value for fold angle
    • Type: double
    • Default value: 0.0
    • Default Value Role: By default the lowest absolute value of fold angle is 0 degree
  • GeometricalLinkState
    Role: Creates geometrical link or not between branch extremity and object to connect to
    Detailed role: Creates geometrical link or not between branch extremity and object to connect to
    • Type: integer
    • Default value: 1
    • Default Value Role: By default a branchable extremity is always geometrically linked to the other segment extremity or bundle connection point
    • Values:
      0 A branchable extremity is not geometrically linked to the other segment extremity or bundle connection point
      1 A branchable extremity is geometrically linked to the other segment extremity or bundle connection point
  • KeepNotUsedState
    Role: Setting to manage the unused branch points
    Detailed role: Setting to manage the unused branch points
    • Type: integer
    • Default value: 1
    • Default Value Role: By default an unused branch point is not deleted
    • Values:
      1 An unused branch point is not deleted
      0 An unused branch point is prompted for deletion
  • L2PGenerateBranchesToCavities
    Role: Check Button to allow generation of branches to cavities
    Detailed role: Check Button to allow generation of branches to cavities
    • Type: integer
    • Default value: 1
    • Default Value Role: By default branches to cavities will be generated
    • Values:
      0 Generate branches to compatible connection point
      1 Generate branches to cavities
  • L2PMapLogCableToPhyWire
    Role: Check Button to create a Physical Wire from a Logical Cable
    Detailed role: Check Button to create a Physical Wire from a Logical Cable in Logical To Physical Generation
    • Type: integer
    • Default value: 0
    • Default Value Role: By default To create a Physical Wire Group from a Logical Cable in Logical To Physical Generation
    • Values:
      0 To create a Physical Wire Group from a Logical Cable in Logical To Physical Generation
      1 To create a Physical Wire from a Logical Cable in Logical To Physical Generation
  • LocalSlackVisu
    Role: Check Button to display labels for Local Slack
    Detailed role: Check Button to display labels for Local Slack on Branch portions
    • Type: integer
    • Default value: 0
    • Default Value Role: By default Labels should not be displayed
    • Values:
      0 Do not display the labels for Local Slack
      1 Display the labels for Local Slack
  • MandatoryGBNState
    Role: Mandatory geometrical harness for multibranchable creation
    Detailed role: A multibranchable must be created under a geometrical harness or not
    • Type: integer
    • Default value: 1
    • Default Value Role: By default a geometrical bundle is mandatory
    • Values:
      0 Geometrical bundle is not mandatory
      1 Geometrical bundle is mandatory
  • NumberOfDocAssistanceState
    Role: Number of multibranchables under a geometrical harness
    Detailed role: The number of multibranchables under a geometrical harness is restricted to one or not
    • Type: integer
    • Default value: 0
    • Default Value Role: By default the number of multibranchables under a geometrical harness is not restricted
    • Values:
      0 Number of multibranchables under geometrical harness not limited
      1 Number of multibranchables under geometrical harness limited to one
  • ProfileTypeState
    Role: Setting to manage the profile type of segments
    Detailed role: Setting to manage the profile type of segments
    • Type: integer
    • Default value: 1
    • Default Value Role: By default any profile types can bet set on a segment
    • Values:
      1 Any profile types can bet set on a segment
      2 Only circular profile type can be set on a segment
  • ProfileValueState
    Role: Setting to manage the update of profiles after wire routing
    Detailed role: Setting to manage the update of profiles after a wire routing is run
    • Type: integer
    • Default value: 1
    • Default Value Role: By default the greatest value of the profile is kept constant when the other is changed according to the routed wires diameter
    • Values:
      1 The greatest value of the profile is kept constant when the other is changed according to the routed wires diameter
      2 The smallest value of the profile is kept constant when the other is changed according to the routed wires diameter
      3 The ratio between the greatest and the smallest profile values is kept constant according to the routed wires diameter
      4 The profiles are not modified after the wire routing
  • RatioCLMode
    Role: Setting to manage the accuracy of the final branch length in constrained length mode
    Detailed role: Setting to manage the accuracy of the final branch length in constrained length mode
    • Type: double
    • Default value: 1
    • Default Value Role: By default the tolerance is set to one percent
  • ReframeOnSupport
    Role: Reframe on support selected for Arrange Segments command
    Detailed role: Reframe on the selected support automatically for Arrange Segments command
    • Type: integer
    • Default value: 1
    • Default Value Role: By default Reframe on the support when the Arrange Segments command is launched
    • Values:
      0 Do not reframe on the support automatically for Arrange Segments command
      1 Reframe on the support automatically for Arrange Segments command
  • Scratchmode
    Role: Check Button to allow the user to use the Scratch Mode
    Detailed role: If checked it aloows the user to use the Sctarch mode while Instantiating the Protection
    • Type: integer
    • Default value: 1
    • Default Value Role: By default user is allowed to use the Scratch mode
    • Values:
      0 Do not allow the user to use the Scratch mode
      1 Allow the user to use the Scratch mode
  • SelectedLineType
    Role: Line Type to Set
    Detailed role: The user is allowed to set the default line type
    • Type: integer
    • Default value: 1
  • UseBCPProfileValuesState
    Role: Setting to Enable Flat Cable Routing Capability during Routing a Branch
    Detailed role: Setting to enable Flat Cable Routing from a Segment Connection Point with Section Properties defined on it
    • Type: integer
    • Default value: 0
    • Default Value Role: By default Section Properties of Segment Connection Point cannot be used for Routing
    • Values:
      0 Section Properties of Segment Connection Point cannot be used for Routing
      1 Section Properties of Segment Connection Point can be used for Flat Cable Routing
  • UserBendRadNotRespected
    Role: Check Button to warn user when User Bend Radius is not Respected
    Detailed role: Warns the user when User Bend Radius is not Respected
    • Type: integer
    • Default value: 1
    • Default Value Role: By default user gets warning when User Bend Radius is not Respected
    • Values:
      0 Dont Warn user when User Bend Radius is not Respected
      1 Warn user when User Bend Radius is not Respected
  • WarnOnBranchCollapse
    Role: Check Button to warn user when Branch collapses
    Detailed role: Warns the user when the Branch collapses i.e. BendRadius is less than Segment Radius
    • Type: integer
    • Default value: 0
    • Default Value Role: By default user does not get warning when Branch collapses
    • Values:
      0 Dont Warn user when Branch is collapsed
      1 Warn user when Branch is collapsed
