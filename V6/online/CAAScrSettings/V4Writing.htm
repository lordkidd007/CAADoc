V4Writing Setting Repository
Quick Reference

V4Writing

Role: Setting to handle the V5->V4 options

The V4Writing repository has the following characteristics:

  • Primary Tab Page: /General/Compatibility/Saving As V4 Data
  • Repository File: V4Writing
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Setting controller:
    • Framework_CPP: CATIAV4Interfaces
    • Framework_IDL: CATIAV4Interfaces
    • Interface_CPP: CATIV4IV4WritingSettingAtt
    • Interface_IDL: CATIAV4WritingSettingAtt
    • LateType: CATV4IV4WritingSettingCtrl

V4Writing setting repository attributes:

  • CleanTolButton
    Role: The cleaning tolerance mode setting parameter manages the cleaning of small elements in V4 model.
    • Type: integer
    • Default value: 0
    • Values:
      0 Use a default tolerance for small V4 elements cleaning.
      1 Use a customized tolerance for small V4 elements cleaning.
  • CleanTolEditor
    Role: The cleaning tolerance value setting parameter manages the cleaning of small elements in V4 model.
    • Type: double
    • Default value: 0.0
  • Code_page_Dest
    Role: The Writing Code Page setting parameter manages the format related with the language used by the customer.
    • Type: integer
    • Default value: 11
    • Min value: 0
    • Max value: 16
  • ModeCreateDisplay
    Role: The Curve Display Mode setting parameter manages the mode of creation of V4 curves (between faces).
    • Type: string
    • Default value: NeverCreate
    • Values:
      AlwaysCreate Always create curves associated to V4 faces boundaries
      KeepConics Create conic curves associated to V4 faces boundaries
      NeverCreate Never create curves associated to V4 faces boundaries
  • ModeErrorDisplay
    Role: The Error Feature Display setting parameter manages the creation of an error feature when the save is not complete.
    • Type: string
    • Default value: Refuse
    • Values:
      Accept Systematic creation of error features
      Ask Ask for error features creation
      Refuse Never create error features
  • Model_Dimension
    Role: The Model Dimension Value setting parameter manages the value of the model dimension usually used in V4.
    • Type: double
    • Default value: 10
  • Model_Factor
    Role: The Multiplication Factor setting parameter manages the conversion of model dimension in millimeters.
    Detailed role: This parameter is used with the Model Dimension Unit parameter in order to stock the value in millimeter in the CATSetting. The Multiplication factor is the ratio between millimeters and the unit selected. For example if the unit selected is Meter, then the Multiplication Factor is 1000. Note that no option is available for this value : it is automatically computed
    • Type: double
    • Default value: 1
  • Model_File_Name
    Role: The Capital Letters setting parameter manages the mode of creation of the V4 model name (in capital letters or not)
    • Type: string
    • Default value: Normal
    • Values:
      Normal Use the characters given for the V4 saved model name
      CapitalLetter Convert all letters of the V4 saved model name into capital letters
  • Model_Unit
    Role: The Model Dimension Unit setting parameter manages the unit used for save as model. As the reference unit used in CATSettings is millimeters, a factor to convert units chosen is necessary. This factor is the Multiplication Factor parameter.
    • Type: integer
    • Default value: 0
    • Min value: 0
    • Max value: 11
  • StartUp_Model_File_Path
    Role: The Initial Model File Path setting parameter manages the location of a V4 model which will be used during save as model.
