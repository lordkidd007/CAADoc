GeneralFind Setting Repository
Quick Reference

GeneralFind

Role: Setting to manage Find command and Power Input options

The GeneralFind repository has the following characteristics:

  • Primary Tab Page: /General/Find
  • Repository File: GeneralFind
  • Version: 1
  • Full Compatibility: 1
  • Upward Compatibility: 1
  • Setting controller:
    • Framework_CPP: CATIAApplicationFrame
    • Framework_IDL: CATIAApplicationFrame
    • Interface_CPP: CATICafSearchSettingAtt
    • Interface_IDL: CATIASearchSettingAtt
    • LateType: CATCafSearchSettingCtrl

GeneralFind setting repository attributes:

  • ContextPriority
    Role: Flag to use the Default search scope when a favorite query is typed in Power Input
    Detailed role: Check this option to supersede the genuine context associated with a favorite query by the default context scope
    when this favorite query is typed in Power Input.
    • Type: boolean
    • Default value: false
  • ContextScope
    Role: Default search scope implicitely used in Power Input
    Detailed role: Context scope to be used by default for search queries entered in Power Input
    if none has been explicitly typed.
    For example, choose InWorkbench if you search mainly in the active element,
    this will avoid typing ",in" after each criterion.
    • Type: integer
    • Default value: 1
    • Values:
      0 Everywhere
      1 InWorkbench
      2 FromWorkbench
      3 FromSelection
      4 VisibleOnScreen
  • DeepSearch
    Role: Flag to activate or deactivate the Deep search option in Find queries
    Detailed role: Check this option to activate the Deep search option in Find queries.
    Then, documents in visualization mode will be transiently loaded in order to find elements.
    • Type: boolean
    • Default value: false
  • MaxDisplay
    Role: Maximum number of elements that can be displayed in the Search results page
    Detailed role: Displaying too many lines can stick more or less the results list so it is recommended to limit this number.
    Set this number to 0 if you don't want any limitation and always want all the elements to be displayed.
    • Type: unsigned integer
    • Default value: 100
    • Min value: 0
    • Max value: 5000
    • Step: 10
  • MaxPSO
    Role: Maximum number of elements that can be pre-highlighted
    Detailed role: Pre-highlighting too many elements can stick the session so it is strongly recommended to limit this number.
    Set this number to 0 if you don't want any limitation and always want all the elements to be pre-highlighted.
    • Type: unsigned integer
    • Default value: 1000
    • Min value: 0
    • Max value: 5000
    • Step: 10
  • Prefix
    Role: Default prefix implicitely used in Power Input
    Detailed role: Prefix to be used by default at the beginning of the string entered in Power Input
    if none has been explicitly typed.
    For example, choose c: if you use Power Input mainly to start commands,
    this will avoid typing "c:" before each command name.
    • Type: string
    • Default value: none
    • Values:
      none none
      s set
      Set_CAP SET (set case sensitive)
      col color
      vis visibility
      l layer
      d dashed
      w weight
      symb symbol
      last last entered
      c command
      f favorite
      t type
      n name
      Name_CAP NAME (name case sensitive)
      graph name in graph
      NameInGraph_CAP NAME IN GRAPH (name in graph case sensitive)
