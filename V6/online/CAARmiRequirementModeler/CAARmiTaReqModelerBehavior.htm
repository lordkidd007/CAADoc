Requirement Modeler Behaviors

Quick Reference

This article describes for each PLM Class of the Requirement Modeler how you can invoke the Query ops, CRUD, and lifecycle operations. Navigation and Edition can be done using Generic APIs [1] as well as Modeler specific APIs [2]

On PLM Object in Database

Behaviors Supported How Is It Supported

Expand Yes CATAdpOpener
Open Yes CATAdpOpener
Save Yes CATAdpSaver
Lock/Unlock Yes CATAdpLockServices
Versioning Yes CATAdpVersioningServices
Change Maturity Yes CATAdpMaturityServices
