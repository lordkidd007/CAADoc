Knowledge Advisor Changes
Technical Article

Abstract

This article explains how to transform your V5 Automation macros to take advantage of the Open V6 PLM 2.0 infrastructure. It focuses on Knowledge Advisor.

Two V5 Knowledge Advisor Automation objects do not exist any longer with V6:

  • The FeatureGenerator object

    This object is exposed in V5, but without any implementation. So it cannot be used in V5, should not be found in V5 macros, and is removed in V6.

  • The Loop object

    As a specialization of the KnowledgeActivateObject object, it does not provide any properties or methods in V5. If you used it in V5, you can replace it with this object.

The setting objects are replaced with the generic one and XML files. Refer to KnowledgeInterfaces changed objects for details, and to Automating Settings to know how to use the generic setting object.
