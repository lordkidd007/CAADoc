What You Will Learn With This Use Case

This use case illustrates the section entitled "Creating a New Mechanical Feature" of the Mechanical Modeler chapter. Its main intent is to help you to make your first steps in creating a new geometrical feature,  a geometrical feature being a mechanical feature with a geometrical result [1].

You will learn how to:

  • Create a new geometrical feature.

    How to create a new StartUp, how to store it in a feature's catalog and how to instantiate it. For this part, you can refer to the "Creating a New StartUp Deriving from a Mechanical StartUp" technical article [2] for details.

  • Integrate it in standard DS Behavior.

    A feature is completely described by a set of behaviors, in other words interfaces. Some are inherited from the mechanical feature StartUp, but some need to be implemented. Among them, there are the mandatory interfaces (CATIFmFeatureBehaviorCustomization for example) and the Dassault Systèmes behavior interfaces to be fully integrated. The technical article entitled "Integrating a New Mechanical Feature" gives you an overview of these interfaces [3].

The Combined Curve use case forms a set of articles, each describing the creation or the integration. All of these articles illustrate how to integrate some components coming from various DS frameworks, such as the Feature Modeler framework, the Geometric Modeler frameworks or the Dialog and DialogEngine frameworks. Their intent is to focus on the use of the Mechanical Applications platform frameworks: CATMecModUseItf, CATMecModLiveUseItf, CATMecModUIUseItf, CATMecModExtendItf, and CATMecModUIExtendItf. A prerequisite knowledge of other frameworks may be required to fully understand this use case, some links with other CAA use cases will help you navigate among them.

Before getting to the use case, it is important understand  what is a Combined Curve. This is the goal of the next section. You can skip it if you are already familiar with this concept.

What Is a Combined Curve

A Combined Curve is a curve resulting of the intersection of two surfaces, each of them being extruded from a curve along a direction.

Consequently, a Combined Curve has two input curves also named profiles and two input directions.

Fig.1:Combined Curve Inputs

The two extruded surfaces are only temporary hidden support objects.

The Combined Curve Use Case

The Combined Curve is a use case of the CAAMechanicalModelerExtend.edu framework that illustrates the Mechanical Modeler extended capabilities.

What Does the Combined Curve Use Case Do

The final intent of this use case is to create a wireframe feature of the Generative Shape Design app. It can be divided into several steps:

  1. Creating the Combined Curve StartUp catalog - Creating Combined Curve's Catalog

    Here is an image of the "CAAMmrCombinedCurveCatalog" feature's catalog containing the CombinedCurve StartUp. You can note the input specifications: two curves and two directions, four features.

    Fig.2: The CombinedCurve StartUp

    The CombinedCurve is a mechanical feature and more precisely a geometrical feature, so the StartUp hierarchy is the following:

    Fig.3: The StartUps Hierarchy
  2. Creating an interface to instantiate the new Combined Curve StartUp - Creating Combined Curve's Factory Interface

    To create instances of the CombinedCurve StartUp, you should create and implement a factory interface on the specification container [8]. This interface creates a new instance but do not set the newly instance in a geometrical features set [7].

    Fig.4: The CAAIMmrCombCrvFactory Interface

    CATPrtCont is the container containing all the specifications of the Part feature. This factory interface uses the "CAAMmrCombinedCurveCatalog" catalog file created in the first step.

  3. Creating an interface to modify or retrieve the input specifications of the Combined Curve instances.- Creating Combined Curve's Interface of Type

    In the CATIFmFeaturebehaviorCustomization implementation, or in the Combined Curve edition command, it is mandatory to have an access to the input specifications of the new feature. It is the role of the CAAIMmrCombinedCurve interface.

    Fig.5: The CAAIMmrCombinedCurve Interface

    With this interface you can retrieve or modify the two curves and the two directions.

  4. Integrating the Combined Curve

    The technical article [3] explains that for a feature deriving from the GeometricalElement3D StartUp - Fig.3 - the interfaces to take into account to integrate it in DS behaviors can be classified in three cases:

    • The mandatory interfaces
      • CATIFmFeatureBehaviorCustomization - Building Combined Curves

        This interface enables you to build the result of the feature. Reading this technical article [4] is strongly recommended before getting to the use case.

      • CATIReplace- Modifying Combined Curve Specifications

        This interface enables you to define the available elements for a replacement, and makes the replacement. The way to implement it is to use the adaptor class CATMmrReplaceAdapter.

      • CATIMf3DBehavior - Defining the Combined Curve's Type

        This interface enables you specify that the CombinedCurve StartUp is a surfacic feature. Without this implementation, you will have wrong behaviors for instances aggregated in a solid and surfacic features set.

      • CATInputDescription - Integrating Combined Curves inside Ordered Sets

        This interface enables you to aggregate the instances of the CombinedCurve StartUp into ordered sets: in other words: Ordered Geometrical Set features and Body features.

    • The Dassault Systèmes Behavior interfaces
      • CATIMechanicalProperties - Enabling the Build Deactivation

        This interface enables you to manage the build's activation status of the combined curve. The CATIFmFeatureBehaviorCustomization implementation takes the activation status of the combined curve into account before generating the result. Interactively you can propose to the end user to choose the state, if you insert a switch into the contextual menu; It will be explained in the Adding a Contextual Sub-Menu article. See Fig.7

      • CATIEdit - Editing Combined Curves

        This interface of the ApplicationFrame framework enables editing a Combined Curve instance. It launches the CAAMmrCombCrvPanelStCmd state dialog command. This command can be used in the creation mode and in the edition mode. For this last mode, refer to the "Creating Interactively Instances" step.

        The CAAMmrCombCrvPanelStCmd state dialog command uses a dialog box to help the end user enter/modify the input.

        Fig.6: The Combined Curve Definition Dialog Box

      • CATIContextualSubMenu - Adding a Contextual Sub-Menu

        This interface of the ApplicationFrame framework enables you to add commands after the Definition command. This command is in the sub-menu of the contextual menu associated with the Combined Curve.

        Fig.7: The Contextual Sub-Menu of the Combined Curve Object

        This use case explains how to implement this interface and how to retrieve Dassault Systèmes (or other) commands.

    • For specific case interfaces
      • CATIReplaceUI

        See the Modifying Combined Curve Specifications article.

         

      • CATIIcon

        The MechanicalFeature feature - Fig.1- already implements the CATIIcon interface. For a simple case as the Combined Curve, you only need to provide an icon whose name is: I_StartUpName.bmp where StartUpName is the name of the StartUp. In the CNext/Resources/graphic/normal directory of the CAAMechanicalModelerExtend.edu framework, you will thus find the I_CombinedCurve.bmp file.

        Fig.9: The Combined Curve Icon

        In short, the Combined Curve, deriving from a mechanical StartUp, benefits from mechanical behaviors, but it should or could implement other interfaces to be fully integrated.

      • Creating Interactively Instances

        To create new instances of the Combined Curve, you will create an add-in of the Generative Shape Design app into which you will add the Combined Curve Creation command. Creating an Add-in in the Generative Shape Design App

Overview of the Combined Curve Use Case Articles

The documentation of the Combined Curve use case contains the following articles:

  • Creating Combined Curve's Catalog. This article describes the creation of the catalog containing the new feature StartUp definition.
  • Creating Combined Curve's Factory Interface. This article describes the creation and the implementation of the new feature factory interface.
  • Creating Combined Curve's Interface of Type. This article describes the creation and the implementation of the interface which enables you to retrieve or modify the input specifications of the new feature.
  • Building Combined Curves. This article describes the implementation of the CATIFmFeatureBehaviorCustomization interface on the new feature.
  • Enabling the Build Deactivation. This article describes the implementation of the CATIMechanicalProperties interface on the new feature.
  • Integrating Combined Curves inside Ordered Sets. This describes the implementation of the CATInputDescription interface on the new feature.
  • Defining the Combined Curve's Type. This describes the implementation of the CATIMf3DBehavior interface on the new feature.
  • Modifying Combined Curve Specifications. This article describes the implementation of the replace mechanism on the new feature.
  • Editing Combined Curves. This article describes editing the new feature through a dialog box and implementing the CATIEdit interface.
  • Adding a Contextual Sub-Menu. This article describes the CATIContextualSubMenu interface implementation
  • Creating an Add-in in the Generative Shape Design App. This article describes creating an Add-in of the Generative Shape Design app to insert the new feature creation command.

How to Launch the Combined Curve Use Case

To launch the Combined Curve use case, you will need to set up the build time environment, then compile the following modules of the CAAMechanicalModelerExtend.edu framework along with their prerequisites:

  • CAAMmrCombinedCurve.m
  • CAAMmrCombinedCurveAddIn.m
  • CAAMmrCombinedCurveReplace.m
  • CAAMmrCombinedCurveUI.m
  • CAAMmrCombinedCurveSample.m

Next, execute the use case [6]. Several steps must be followed in order to completely execute the use case:

  • Create the Combined Curve catalog as described in the Creating Combined Curve's Catalog article.
  • By batch, launch the following command:

    mkrun -c CAAMmrCombinedCurveSample Repository Server UserID Password SecurityCtx

    Where:

    Repository
    Server The server as Host:port_number/RootURI
    User The user name
    Password The user password to authenticate the user
    SecurityCtx A string representing the security context (Role.Organization.Project). Choose Role as Designer (as the use case modifies the PLM data)

    The result of this executable is provided. A "CAAMmrCombinedCurve.3dxml" file is stored in the InputData directory of the CAAMechanicalModelerExtend.edu framework, at the path specified below.

    InstallRootFolder\CAADoc\CAAMechanicalModelerExtend.edu\InputData\CAAMmrCombinedCurve.3dxml\

    where InstallRootFolder [6] is the folder where the API CD-ROM is installed.

    This 3D XML file contains the resulting PLM Representation Reference with these characteristics:

    • PLM_ExternalID: CAAMmrCombinedCurve
    • majorrevision: ---

    Please refer to the document [6] for how to import and reimport such a 3D XML file in your PLM repository.

  • Launch CATIA. When the application is ready, follow scenarios described below:

    The Creation scenario:

    • Launch Generative Shape Design app from the 3D quadrant of the compass.
    • Activate Wireframe section of the action bar.
    • Create four points with the Point command.
    • Create two lines with the Line command.
    • Create two splines with the Spline command.
    • Activate CAA MechMod Extend UC section of the action bar.
    • Create a Combined Curve with the Combined Curve command .

    The Edition scenario:

    • Double click the Combined Curve instance.

    The Contextual-SubMenu scenario:

    • Right click the Combined Curve and select Parent/Children.
    • Right click the Combined Curve and select Activate.
    • Right click the Combined Curve and select Deactivate.

    Once the Combined Curve is created, try the Replace scenario:

    • Open CAACombinedCurveReplace (*).
    • Select Line.1 (second direction).
    • Choose the Replace item in the contextual menu
      • Select Line.5.
      • Click OK in the Replace dialog box.
    • Select Line.3 (second curve).
    • Choose the Replace item in the contextual menu:
      • Select Line.6.
      • Click OK in the Replace dialog box.
    • Select Line.5 (new second direction).
    • Choose the Replace item in the contextual menu:
      • Select an edge of Pad.1.
      • Click OK in the Replace dialog box.
    • Select Line.6 (new second curve).
    • Choose the Replace item in the contextual menu:
      • Select Line.3.
      • Click on the green arrow to invert Line.3.
      • Click OK in the Replace dialog box.

      (*)This represents a PLM Representation Reference which is provided in the CAAMmrCombinedCurveReplace.3dxml file, at the path specified below.

      InstallRootFolder\CAADoc\CAAMechanicalModelerExtend.edu\InputData\CAAMmrCombinedCurveReplace.3dxml\

      where InstallRootFolder [6] is the folder where the API CD-ROM is installed.

      This 3D XML file contains a PLM Representation Reference with these characteristics:

      • PLM_ExternalID: CAAMmrCombinedCurveReplace
      • majorrevision: ---

      Please refer to the document [6] for how to import and reimport such a 3D XML file in your PLM repository.

    Where to Find the Combined Curve Code

    The Combined Curve use case is made of a several modules of the CAAMechanicalModelerExtend.edu framework:

    InstallRootFolder\CAADoc\CAAMechanicalModelerExtend.edu\

    where InstallRootFolder [6] is the folder where the API CD-ROM is installed.

    These modules are:

    • CAAMmrCombinedSample.m - This module contains one main program, CAAMmrCombinedSample.cpp. It is an executable batch program that was used to create a Part feature containing a Combined Curve. After its creation, the new Combined curve is updated.
    • CAAMmrCombinedCurve.m - This module contains the implementations of the CATIFmFeatureBehaviorCustomization (manages the build operation called during the update of the Combined Curve), the definition and the implementation of the CAAIMmrCombinedCurveFactory (manages the Combined Curve StartUp instantiation), the CAAIMmrCombinedCurve (manages the access to the input specifications of the Combined Curve), the CATInputDescription, the CATIMechanicalProperties, and the CATIMf3DBehavior interface.
      • CAAEMmrCombinedCurveBehavior.cpp, the header file is in the LocalInterfaces directory
      • CAAEMmrCombCrvFactory.cpp, the header file is in the LocalInterfaces directory
      • CAAEMmrCombinedCurve.cpp, the header file is in the LocalInterfaces directory
      • CAAIMmrCombCrvFactory.cpp, the header file is in the PublicInterfaces directory of the CAAMechanicalModelerExtend.edu framework:
      • CAAIMmrCombinedCurve.cpp, the header file is in the PublicInterfaces directory of the CAAMechanicalModelerExtend.edu framework
      • CAAEMmrCombCrvInputDescription.cpp, the header file is in the LocalInterfaces directory
      • CAAEMmrCombinedCurveMechProp.cpp, the header file is in the LocalInterfaces directory
      • CAAEMmrCombCrvMf3DBehavior.cpp, the header file is in the LocalInterfaces directory
      • CAAEMmrCombCrvCkeFeature.cpp, the header file is in the LocalInterfaces directory
    • CAAMmrCombinedCurveAddIn.m - This module contains the implementation of the CATIShapeDesignWorkshopAddin interface. In other words, it defines an Add-in to the Generative Shape Design app.
      • CAAEMmrCombinedCurveAdn.cpp, the header file is in the LocalInterfaces directory
    • CAAMmrCombinedCurveReplace.m - This module contains the implementation of the CATIReplace and CATIReplaceUI interfaces. It enables integration of the Combined Curve in the replace mechanism.
      • CAAEMmrCombinedCurveReplace.cpp, the header file is in the LocalInterfaces directory
      • CAAEMmrCombinedCurveReplaceUI.cpp, the header file is in the LocalInterfaces directory
    • CAAMmrCombinedCurveUI.m - This module contains the implementation of the CATIContextualSubMenu and CATIEdit interfaces. The implementation of this last interface launches a state command with a Dialog box to create Combined Curve instances.
      • CAAEMmrCombinedCurveEdit, the header file is in the LocalInterfaces directory
      • CAAMmrCombCrvPanelStCmd, the header file is in the LocalInterfaces directory
      • CAAEMmrCombinedCurveDlg, the header file is in the LocalInterfaces directory
      • CAAEMmrCombinedCurveContSubMenu, the header file is in the LocalInterfaces directory
