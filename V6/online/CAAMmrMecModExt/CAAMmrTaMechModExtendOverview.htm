Mechanical Modeler Extend Overview
Technical Article

Abstract

This article provides an overview of the Mechanical Modeler Extend functionalities. A prerequisite knowledge of the Mechanical Modeler [1], the Feature Modeler [2] and the Feature Modeler Extension [3] is recommended to fully understand the Mechanical Modeler Extend articles.

Objectives

The objective of Mechanical Modeler Extend is to provide an infrastructure and tools that enable you to create and integrate in CATIA your own new mechanical features, in case DS mechanical features or user features do not fit your needs.

The following articles will help you to follow all the necessary steps:

  • "Creating a New StartUp Deriving from a Mechanical StartUp" [4].

    This article describes from which Mechanical StartUp you should derive, and how to create the StartUp.

  • "Integrating the New Mechanical Feature in Standard Behaviors" [5].

    By deriving from a Mechanical StartUp, your feature automatically benefits from Mechanical behaviors that you do not have to redefine. But there is a set of interfaces which are to take into account for a full integration of the feature in the DS Applications. This article presents the main DS Application behaviors and gives the interfaces implied in these behaviors. They are classified by importance: the mandatory interfaces, the interfaces to be standard or the interfaces to implement only for specific cases.

  • "Integrating the new Mechanical Feature in the Update Mechanism" [6].

    This article explains the CATIFmFeatureBehaviorCustomization interface implementation.

  • "Verifying the Validity of a Geometrical Feature" [7].

    For a new geometrical feature, the last step is the validation step. After the StartUp creation and the CATIFmFeatureBehaviorCustomization implementation, you should verify the feature model and check the result. The mechanical infrastructure provides a tool to do this verification. This article describes how to use it and gives you some recommendations to correct errors if any.

Refer to the "Creating a new Geometrical Feature: The Combined Curve" article for a concrete use case [8].

Mechanical Modeler Architecture

Frameworks

The new packaging model in V6 distinguishes frameworks with APIs that enable to do standard tasks from frameworks with API's that enable to do advanced tasks. For Mechanical Modeler, the standard APIs enable to manage DS Mechanical Features, whereas the extend APIs enable to manage new applicative Features, derived from DS Mechanical Features.

  • CATMecModUseItf / CATMecModLiveUseItf

    These two frameworks provide interfaces and classes to manage DS Mechanical Modeler Features.

  • CATMecModExtendItf

    This framework provides interfaces and classes to manage your own Features, derived from Mechanical Modeler Features.

  • CATRepTemplateUseItf

    This framework contains the interfaces to manage and create user features and power copies. They are "black box" features which are built on other mechanical features. The technical article entitled "An Overview of the Power Copies and User Features" [9] gives you details about these powerful features.

  • CATMecModUIUseItf / CATMecModUIExtendItf

    These frameworks contain tools and interfaces to manage the user interface of the mechanical applications for respectively Use and Extend scopes.

Scalability

The Mechanical Modeler offers several usage levels which are described here by increasing complexity:

  1. The first level enables you to consult the contents of a 3D Shape in order to do analysis or data transfers.
  2. The second level enables you to create your application by using supplied features, the Part Design and the Generative Shape Design features.
  3. The third level enables you to create new mechanical features from existing ones. It is possible thanks to the Power Copy or the User feature mechanism.
  4. The fourth and last level enables you to create your own mechanical features. This advanced step requires to have the extend scope frameworks.
