Introduction

The manufacturing system model describes the operational realization of the manufacturing process.

  • Who (driven by capacity): numbers of systems, paths between systems.
  • When (driven by cycle time): analyzed durations, constraints (time, precedence, data) between operations.
  • How: kind of operation, work instructions.

Manufacturing System Topology

Aggregation

A manufacturing system:

  • Is the entity producing the manufactured product.
  • Contains operational specifications (operations) for the physical world.
  • Has a number of inputs/outputs.
  • May define a scope of implementation of a process.
Fig. 2: Aggregation of Manufacturing Systems

A manufacturing operation:

  • Implements manufacturing process
  • Has a duration
  • May be described with data in PLM representation
  • May be connected to others operations thru Time Constraints
  • Is implemented by one physical program of physical resource
Fig. 3: Aggregation of Manufacturing Operations

Constraints

Different kind of constraints can be set between manufacturing systems:

Material path:

  • It is a the connection between father/child and brother systems.
  • It describes the possible paths for the material path between inputs/outputs of systems.
  • It is taken into account by the time solver.
Fig. 4: Material Path between Two Manufacturing Systems

Time constraints:

  • It is a connection between operations (of same or different systems).
  • It defines a constraints network solved with a time solver.
  • It defines the planning of the work to do along the time.
  • There are 4 kinds of constraints : Finish to start, start to finish, Start to start, Start to finish. All with optional delay.

Scopes

A scope connects a manufacturing process reference with a manufacturing system reference that is by definition a description of the operations necessary and sufficient to realize the transformation. It is aggregated by the manufacturing system reference and points to the manufacturing process reference. There can be {0..n} manufacturing system references pointing to one manufacturing process reference. These systems represent different possible solutions to realize the transformation.

Implement Links

Implement links could connect an implemented instance of manufacturing process to N manufacturing operation instances. Each implement link is aggregated by the manufacturing system defining the scope of implementation.

Manufacturing System and Operation Types

A manufacturing system has one of the following types:

  • General
  • Transformation
  • Transfer
  • Storage.

A manufacturing operation has one of the following types:

  • General
  • Punctual
  • Curve
  • Loading
  • Unloading
  • Remove material
  • Transfer.
