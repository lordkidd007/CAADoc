What You Will Learn With This Use Case

By implementing the CATIImmSearchAddin, a toolbar with its commands can be made active in the PLM Search workbench.

The CAAIpaSearchAddin Use Case

CAAIpaSearchAddin is a use case of the CAAImmVPMInterfaces.edu framework that illustrates the CATImmVPMInterfaces framework capabilities.

What Does CAAIpaSearchAddin Do

The goal of CAAIpaSearchAddin is to demonstrate how to implement the CATIImmSearchAddin interface. It consists of the following parts:

  • Code to create the toolbar and the layout of the commands.
  • Code to be executed when the command is selected.
  • Resource definition for the visual aspects of the interface.

Our toolbar appears on top of the CATIA Messenger frame and contains a single command represented by the icon.

How to Launch CAAIpaSearchAddin

To launch CAAIpaSearchAddin:

  1. Set the current folder to InstallRootFolder\CAADoc\CAAImmVPMInterfaces.edu
  2. Set up the build time environment and build the CAAIpaSearchAddin.m module (see reference [1])
  3. Edit the CNext\code\dictionary\CAAImmVPMInterfaces.edu.dico dictionary by uncommenting the #CAA# from the following lines: 
    # CAAIpaSearchAddin use case
#CAA# CAAIpaSearchAddinLT  CATIImmSearchAddin  libCAAIpaSearchAddin
#CAA# CAAIpaSearchAddinLT  CATIWorkbenchAddin  libCAAIpaSearchAddin
  4. Install the dictionary and other resource files by executing mkCreateRuntimeView
  5. Start 3DEXPERIENCE by executing mkrun -c 3DEXPERIENCE
  6. Connect to ENOVIA (or any other connector)
  7. Search for an existing Product from the PLM Toolbar, for example typing "prd: *"
  8. Display the toolbar named "CAA PLM Search addin toolbar" if necessary (menu View / Toolbars / CAA PLM Search addin toolbar)

  9. Click on the command and a dialog box will appear:

Step-by-Step

There are two parts in CAAIpaSearchAddin:

  1. Toolbar Creation and Layout
  2. User Interface

We will now detail each of those sections:

Toolbar Creation and Layout

The implementation of CATIImmNavAddin is provided by our CAAIpaSearchAddin class. Its purpose is to create the toolbar and to do the layout the commands on it. Only two methods need to be defined: CreateToolbars and CreateCommands.

...
class CAAIpaSearchAddin: public CATBaseUnknown 
{

CATDeclareClass;

public:

        // Standard constructors and destructors for an implementation class
        // -----------------------------------------------------------------
        CAAIpaSearchAddin();
        virtual ~CAAIpaSearchAddin();

        /**
         * @see ApplicationFrame.CATIWorkbenchAddin#CreateToolbars
         */
        CATCmdContainer * CreateToolbars();

        /**
         * @see ApplicationFrame.CATIWorkbenchAddin#CreateCommands
         */
        void CreateCommands();

private:
        // The copy constructor and the equal operator must not be implemented
        // -------------------------------------------------------------------
        CAAIpaSearchAddin(CAAIpaSearchAddin &);
        CAAIpaSearchAddin& operator=(CAAIpaSearchAddin&);

};
...

Create Toolbars

The purpose of the CreateToolbars method is to create all the command headers of our toolbar. In this case a toolbar named CAAIpaSearchAddinToolbar (pointer toolbar name pToolbarContainer) with a command named CAAIpaSearchAddinCmdHdr is created.

...
CATCmdContainer * CAAIpaSearchAddin::CreateToolbars ()
{
        CAAIpaSearchAddinTRACE(">> CAAIpaSearchAddin::CreateToolbars()");

        // Declaration of toolbar for Addin
        NewAccess(CATCmdContainer, pToolbarContainer,
                  CAAIpaSearchAddinToolbar);
        NewAccess(CATCmdStarter, pCommandStarter,
                  CAAIpaSearchAddinStarter);

        // Arranging the command within the toolbar
        SetAccessCommand(pCommandStarter, "CAAIpaSearchAddinCmdHdr");
        SetAccessChild(pToolbarContainer, pCommandStarter);

        int visibleFlag = 1;
        AddToolbarView (pToolbarContainer, visibleFlag, Top);

        CAAIpaSearchAddinTRACE("<< CAAIpaSearchAddin::CreateToolbars()");
        return pToolbarContainer;
}

...

Create Commands

The purpose of the CreateCommands method is to create all the command headers of our toolbar. Our command header class CAAIpaSearchAddinHeader, is defined. Here a command named CAAIpaSearchAddinCmdHdr is created. It is located in the CAAIpaSearchAddin shared library and its class is CAAIpaSearchAddinCmd.

User Interface

There are two visible objects: the toolbar and its command.

The title of the toolbar named CAAIpaSearchAddinToolbar is defined by the CAAIpaSearchAddinLT.CATNls which can be found in InstallRootDirectory/CAAImmVPMInterfaces.edu/CNext/resources/msgcatalog directory.

CAAIpaSearchAddinToolbar.Title = "CAA PLM Search addin toolbar";

The resources of the CAAIpaSearchAddinCmd1 are defined by three files:

  1. CAAIpaSearchAddinHeader.CATNls (located in the InstallRootDirectory/CAAImmVPMInterfaces.edu/CNext/resources/msgcatalog directory): 
    CAAIpaSearchAddinHeader.CAAIpaSearchAddinCmd1.Title = "Search addin command";
CAAIpaSearchAddinHeader.CAAIpaSearchAddinCmd1.Help = "Search addin command long help text";
CAAIpaSearchAddinHeader.CAAIpaSearchAddinCmd1.ShortHelp = "Search addin command info";

    These texts correspond to the screen tip, the documentation line and the command prompt input

  2. CAAIpaSearchAddinHeader.CATRsc (located in the InstallRootDirectory/CAAImmVPMInterfaces.edu/CNext/resources/msgcatalog directory) defines the bitmap name for the command icon: 
    CAAIpaSearchAddinHeader.CAAIpaSearchAddinCmd1.Icon.Normal="CAAIpaSearchAddin_I";
  3. CAAIpaSearchAddin_I.gif (located in the InstallRootDirectory/CAAImmVPMInterfaces.edu/CNext/resources/graphic/icons/normal directory) contains the bitmap the command icon:

In Short

This use case has demonstrated how to create a toolbar with a command:

  • Implement the CATIImmSearchAddin interface to create a toolbar and layout the commands.
  • Derive from CATCommand to implement the command action.
  • Define resource files for the user interface.
