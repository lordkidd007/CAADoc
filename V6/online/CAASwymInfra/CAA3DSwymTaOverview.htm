3DSwym Role and Openness
Technical Article

Abstract

3DSwym enable cross-discipline collaboration of users anywhere in the enterprise. This article is an overview of its capabilities, and what sort of openness it proposes.

3DSwym Social Innovation

3DSwym service is the backend of Social and Collaborative Apps. Those Apps enables users a lot of things. Let's quickly sum them up.

  • Consume and React
    • Explore & Discover Social Content and Communities
    • Endorse, Comment
    • Share
  • Create
    • Communities
    • Social Content like Post, iQuestions, iDeas, Wiki
    • Social Media like 3D, Photos, Videos, Audio
  • Listen
    • Monitor Community Activity
    • Measure Social Content impact
    • Manage Community settings
  • Social
    • Rich Profil
    • Notification
    • Suggestion
    • Messaging

Platform Isolation

3DSwym is based on the multi-tenancy [1] principle. Through the 3DEXPERIENCE platform, a tenant is also named a platform instance [2].

On the public cloud solution, there are as many platform instances as customer having pay for one instance. Otherwise, for the private Cloud and onPremise solutions, there are only one platform instance.

A user must be first granted to be able to use a platform instance. It is the role of the 3DEXPERIENCE platform administrator.

So, there are two API [2] to get the Platform instance list for the connected user, and for each of them the appropriate 3DSwym service url.

Secured Access

Like expressed by this whitepaper on our www.3ds.com web site [3], the security is on the heart of the 3DEXPERIENCE platform. When using 3DSwym service you have to be familiar with three points of security:

  • Authentication

    3DSwym, like all the services of the 3DEXPERIENCE platform, requires an authentication. The protocol is based on the CAS protocol, with a centralized management through the 3DPassport [4]. So, before using 3SDSwym you must perform a login [5].

  • CSRF token

    To reinforce the security, 3DSwym requires a token to be exchanged when consuming its web services. It is provided by an initial WEB service call (/api/index/tk) [5]

  • Cross-domain

    All the browsers recommended by the 3DEXPERIENCE platform prevents cross-domain access. So, interactively, your code should rely on DS Api to consume them [6].

If the first point is the only one visible by the end-user, the three are concepts the developper must know.

REST Web Services

3DSwym service exposes resources through the REST protocol. We inform the reader the current API are subject to change in the next release.

The 3DSwym service is related to resources (in bold below). We can split them in three categories

  1. Data
    • community: List members of a community, list commnunities of the connected user, info about a specific community.
    • post:
      • List posts of a community, create ones, modify them, delete them
      • Manage post's comments: list, create, update and delete.
    • media:
      • List medias of a community, create ones, modify them, delete them
      • Manage media's comments: list, create, update and delete.
    • question:
      • List iQuestion of a community, create ones, modify them, delete them
      • Manage iQuestion's answer: list, create, update, delete, rate and valid.
      • Manage iQuestion's comments: list, create, update and delete.
  2. People
    • user: resource dedicated for the people having an account inside 3DSwym, and dealing with the previous data.
  3. Others
    • passport: To perform login, logout implemented on CAS Protocol - -
    • index: to get the CRSF token for secury purpose
    • Exalead: To perform 3DSwym data queries

There are examples showing how to consume them through a batch like this one [7], or through UI inside a Widget App [8].

6WTags

The 3DSwym data are implicated in the 6WTag [9] process. Just below some examples of predicates which can be associated with 3DSwYm subjects:

  • ds6w:what/ds6w:language
  • ds6w:what/ds6w:social/ds6w:comments
  • ds6w:what/ds6w:social/ds6w:endorsements
  • ds6w:what/ds6w:social/ds6w:question/ds6w:answers
  • ds6w:when/ds6w:modified
  • ds6w:when/ds6w:created
  • ds6w:who/ds6w:responsible
  • ds6w:where/ds6w:location/ds6w:country

See the dedicated use case [10] .

In Short

3DSwym is a service to share multi-discipline information and from anywhere in the enterprise. There are web services based on REST protocole to deal with this service of the 3DEXPERIENCE platform. Their use is protected by different security protocols like CAS for authentication.
