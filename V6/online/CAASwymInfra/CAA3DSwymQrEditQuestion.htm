Edit Question

Role

Edits an existing iQuestion.

Request

 Method  Path 
POST api/question/update

Parameters

None.

POST Body

The value is a JSON composed as follows:

 Name   Require   Value   Description 
params yes The value is a JSON composed as follows:
 Name   Require   Value   Description 
category_id no A positive number. Identifier of the category the current object belong to. The category is one from the list associated to the community where the blog post has been published.
community_id yes A positive number. Identifier of the community where the iQuestion will be edited.
published yes A boolean:
  • 0: draft.
  • 1: published.
Should always be 0 has long as the iQuestion has not been edited with additional information. 		Status of the iQuestion.
qna_id yes A positive number Identifier of the iQuestion to edit.
question yes A string. Format is HTML-like (see Format HTML-like). iQuestion message.
title yes A string. Title of the iQuestion.
Container for all post parameters.
request_id yes 0 For internal use.

Response

The response is a JSON composed as follows:

 Key   Description   Value   Applicability 
monitoring no
nb_result For internal use. A number. equals to 1 on success yes
request_id no
result Status of the edition. A boolean:
  • true: edited.
  • false: not edited.
yes
success Status of the request on server. A boolean:
  • true: success.
  • false: failure.
yes
