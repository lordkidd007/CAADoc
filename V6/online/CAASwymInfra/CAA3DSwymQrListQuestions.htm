List Questions

Role

Returns a list of iQuestions belonging to a community.

Request

 Method  Path 
GET api/question/list

Parameters

The GET parameters (see GET Method) are:

 Name   Require   Value   Description 
community_id yes A positive number. Identifier of the community to list iQuestion from.
limit no A number greater than one. The maximum number of elements returned in the list.
page no A number greater than one. Optional parameter to be used with "limit".For paginated result (see Paginated Result).
published no A boolean:
  • 0: list only unpublished (draft).
  • 1: list only published.
 Published status of iQuestion to list.

Response

The response is a standard 3DSwYm JSON (see JSON as Output) where the key result is an array. Each element of the array is a JSON composed as follows:

 Key   Description   Value   Applicability 
NbDocuments Number of documents attached to the iQuestion. A positive number. yes
author Information on the author of the blog post.

The value is a JSON composed as follows:

 Key   Description   Value   Applicability 
accept_share_info no
city City part of the author address. A string. An empty string if information is not shared. yes
company Company of the author of the post. A string. An empty string if information is not shared. yes
country Country part of the author address. A string. yes
ds_user no
email E-mail of the author of the post. A e-mail string. Ex: john.doe@ds.com. An empty string if information is not shared. yes
first_name Author of the post first name. A string. An empty string if information is not shared. yes
hash_key Author of the post crypted identifier. A hash key. yes
id no
job_title Job title of the author of the post. Example: CEO. A string. An empty string if information is not shared. yes
last_name Author of the post last name. A string. yes
login no
nickname no

username Username of the author of the post. A String. yes
zip Zip part of the author address. An empty string if information is not shared. yes
yes

community_title Community title into which the iQuestion has been published. A string. yes
count_answer Number of answers to the iQuestion. A positive number. yes
count_comments Number of comments on the answers to the iQuestion. A positive number. yes
hash_key iQuestion author crypted identifier. A hash key. yes
