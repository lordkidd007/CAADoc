Add Answer Comment

Role

Adds a comment to an existing answer of an iQuestion. Returns the added comment.

Request

 Method  Path 
POST api/answer/addcomment

Parameters

None.

POST Body

The value is a JSON composed as follows:

 Name   Require   Value   Description 
params yes The value is a JSON composed as follows:
 Name   Require   Value   Description 
comment yes A string. Format is HTML-like (see Format HTML-like). Text of the comment.
qna_answer_id yes A positive number. identifier of the answer into which the comment will be added.
Container for all POST parameters.
request_id yes 0 For internal use.

Response

The response is a standard 3DSwYm JSON (see JSON as Output) where the key result is itself a JSON composed as follows:

 Key   Description   Value   Applicability 
_acl Access Control List (ACL) of the comment for the current user.

The value is a JSON composed as follows:

 Key   Description   Value   Applicability 
delete Access Control List (ACL) for comment deletion

The value is a JSON composed as follows:

 Key   Description   Value   Applicability 
access Accessibility for the current user. A boolean:
  • true: current user can access.
  • false: current user cannot access.
 yes
groupIdentifier no
yes

The value is a JSON composed as follows:
