nresults no A positive number. Maximum number of results in the response.
order_by no A string. Possible values are:
  • asc: return results in ascending order (preferred display value).
  • desc: return results in descending order.
 To be used with order_field.
order_field no A string. Possible values are:
  • update
  • relevance (default)
  • last_connection
 Order the result. See order_by to choose ascending or descending order.
query yes A string. The token #all can be used to return all results. It is possible to filter on a specific attribute, for example media_type, and combine filters separated by a simple space. By default the combination is a logical AND; to do a logical OR, you must use the operator OR. A term can be exceluded by using the – (minus) operator. (see Exalead official documentation [EDIT MANUALLY https://www.exalead.com/search/web/search-syntax/] for more information). Ex: media_type:3d_model Ex: #all AND (type:media) Ex: androbot OR (media_type:3d_model) Ex: AND community_id: AND (type:media OR type=posts) Ex: industry -automotive Search keyword on common text attributes (title, description...).

start no A positive number. For paginated result, the index of the first item returned starting 0. For instance, to have the ten results from 11 to 20, set start=12 and nresults=10

  • true: refine is zapped.
  • false: refine is not zapped.
