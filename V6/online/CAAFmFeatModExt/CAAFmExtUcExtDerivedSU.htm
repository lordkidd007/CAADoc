Creating an Externally Derived StartUp
Use Case

Abstract

This article discusses the CAAOsmExtDerivedSU use case. This use case explains how to create a new StartUp deriving from an existing one found in another catalog.

What You Will Learn With This Use Case

This use case is intended to help you understand how to create a new StartUp based on an already existing one found in an external catalog.

Before getting to the use case itself, you should have a good understanding of what a feature object is and how it is created [1].

The CAAOsmExtDerivedSU Use Case

CAAOsmExtDerivedSU is a use case of the CAAFeatureModelerExt.edu framework that illustrates FeatureModelerExt framework capabilities.

What Does CAAOsmExtDerivedSU Do

The goal of CAAOsmExtDerivedSU is to illustrate creating new StartUps based on StartUps defined in another catalog as the current one. To exemplify this, it uses the "CAAOsmNovel" StartUp defined in the catalog created by the CAAOsmCatalogSU use case [2]. Here is an image of the contents of this catalog:

Fig.1 CAAOsmCatalogSU.CATfct Catalog Contents

A new StartUp called "CAAOsmHistNovel" and deriving from "CAAOsmNovel" is created within the new catalog, CAAOsmExtDerivedSU.CATfct. To this new StartUp, a few new attributes are added. This StartUp will, therefore, have all of the attributes of "CAAOsmNovel" in addition to its own particular ones.

The client identification of the newly created CAAOsmExtDerivedSU catalog is "CAAOsmClientId2". Note that the client identification of the CAAOsmCatalogSU is useless.

How to Launch CAAOsmExtDerivedSU

To launch CAAOsmExtDerivedSU, you will need to set up the run time environment and then execute the CATfctEditorAssisant tool as indicated.

$ mkrun -c sh
$ cd your_workspace_root
$ # Create empty catalog and empty osm
$ CATfctEditorAssistant -create-new-catalog -catalog-name CAAOsmExtDerivedSU.CATfct -with-client-id CAAOsmClientId2 -into-directory outptudirectoryforcreatedcatalog  
$ # Upgrade the empty catalog with the input osm
$ CATfctEditorAssistant -update-catalog CAAOsmExtDerivedSU -with-client-id CAAOsmClientId2 -with-osm $PWD -into-directory outptudirectoryforupdatedcatalog  
$ exit # the mkrun shell

Notes:

CATfctEditorAssistant looks for the catalog and outputs it in two different locations:

  • On input, the catalog argument is stripped of its directory and looked up in the run-time view. At your workspace level, it is the workspace_root/OS_directory/resources/graphic directory.
  • On output, the catalog is written at the location specified by the catalog argument

The place to store catalogs in your workspace is workspace_root/CNext/resources/graphic. Any catalog found in this directory will be automatically copied to the run-time view workspace_rootCAADoc/OS_directory/resources/graphic by the mkrtv command.

Where to Find the CAAOsmExtDerivedSU OSM

The CAAOsmExtDerivedSU.osm file is located in the InputData directory of the CAAFeatureModelerExt.edu framework:

InstallRootFolder\CAADoc\CAAFeatureModelerExt.edu\InputData

where InstallRootFolder [3] is the root folder of your API installation.

Step-by-Step

Just for recall, the definition of the CAAOsmNovel within CAAOsmCatalogSU:

...
   feature CAAOsmNovel CAAOsmBook #startup #inheritability(caa){
     string Author
     component NovelChapter #list
   }
...

A feature is derived from another one just by specifying it on the OSM feature statement:

...
   feature derived base {
...

However, if the base feature is located in another catalog, the feature@catalog notation must be used. Since CAAOsmNovel is in the CAAOsmCatalogSU.CATfct catalog, it must be specified using this notation: CAAOsmNovel@`CAAOsmCatalogSU.CATfct`.

Other that this external catalog reference, the rest of the OSM is plain regular.

document `CAAOsmExtDerivedSU.CATfct` {
  container RootCont #root {
    feature CAAOsmHistNovel CAAOsmNovel@`CAAOsmCatalogSU.CATfct` #startup {
      int TimePeriod
        string Subject
    }
  }
}

Without the #inheritability(caa) for CAAOsmNovel, a such derivation, external to the current catalog, could not be possible.

In Short

This use case has demonstrated how to create a StartUp that derives from another StartUp located in an external catalog.
