Feature Modeler Recommendations

Abstract

This article tries to give you some tips, or at least recommendations, when starting a new application based on the Feature Modeler. This article will be regularly updated .

Recommendations

Respect Model View Control Paradigm 

Respect the MVC (Model, View, Controller) paradigm means

  • The commands (controller) must not contain themselves Feature modeler APIs. The commands must call API encapsulating the model operations. These API, delivered by the responsible of the application, could be used by either by a command or by a batch code.
  • The Model APIs must not contain view code. For example, an model PAI must not contain a call to refresh the viewers or the specifications tree.

Explicit Model Operation

In your code implementing a model's operation think simple, and avoid to base your code on implicit code. Implicit code are those occurring in callbacks, in customization, implementation overwriting. In other words, avoid to have you model operations split in several parts by different piece of code. 

The more your code is explicit and simple the more it is easy to debug.
