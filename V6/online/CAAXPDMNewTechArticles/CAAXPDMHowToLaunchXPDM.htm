Launching MyX-PDM Use Case

MyX-PDM functionalities are integrated in MyX-CAD application in the form of addins. To use this addins we have to launch MyX-CAD application and load all this addins. Launching MyX-CAD application is explained in How to Launch MyX-CAD[1].

Once MyX-CAD is started successfully, we can launch MyX-PDM using MyX-PDM launcher. Following are the steps involved in starting both applications :

  • Start the Eclipse with workspace for MyX-PDM and X-CAD Application. Import the project workspace.
  • Build the workspace along with CAAVIMyXPDM.edu and CAAVIMyXPDMIntegration.edu frameworks. If it build successfully we will get the jar files located at $workspace$\win_b64\docs\javaserver
  • Open Project Explorer. Here we get the list of all modules and frameworks.
  • Select MyXPDMLauncher module , as shown in Fig.1
Fig.1:Project Explorer

Right click on module name, select Run as from menu and choose Java Application. This is shown in Fig.2

Fig.2:Run Application

In case of successful execution of above command following window Fig.3 appears :

Fig.3:MyX-PDM Launcher

"Install path " : Select the path of directory which contains runtime view.For 64-bit system it is,  $workspace$\win_b64

When we click on "Launch MyX-PDM", information about ongoing processes will be displayed in MyX-PDM information area. Fig.4 shows the sucessful execution of My-XPDM launcher. In this case,login to MyX-PDM operation is performed at MyX-CAD side.

Refer to How to Use MyX-PDM ?[2] for details about this application.

Fig.4: MyX-PDM Application

In case of using X-PDM addin functionalities (e.g. Send to 3DExperience Platform , Receive from 3DExperience Platform), we have to start few applications which are deployed on application server.

Application which are deployed on Application server(e.g. Tomee Plus) can be launched by starting the application server. i.e. execute startup.bat present at $ApplicationServer$\bin. If it successfully executes then it shows message as: INFO : Server startup in --- ms.(Miliseconds mentioned in ms ).

Please refer to Description[3] before launching the usecases.
