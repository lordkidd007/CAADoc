Basic Objective

Basic objective of this functionality is that user will be able to exchange data between any PDM and 3DEXPERIENCE Platform. Overview of this integration is explained in [1] technical article.Here,we have illustrated this functionality by using CAA sample data. This sample data has been created by using MyX-CAD application. MyX-CAD application creates component and assembly format data.

In this usecase,we are illustrating exchange of both type of data created by CAA Sample (MyX-CAD) application. This data has been stored in local hard drive with .CAAcmp and .CAAasm format.

 Basic steps for this usecase are :

  • Before Connection
  • Import Scenario
  • Export Scenario

Before connection

Here we will be describing the exact flow of the information/command through various components/applications and also the operations performed by the respective components/applications upon receiving the information/command.

To begin with user need to install 3DEXPERIENCE Platform and Gateway setup. To know more about components need to be installed refer to installation section of technical article[2].

Make sure that all basic checks are done as mentioned in the [3] article.

Import Scenario

Create or Open existing Component or assembly in MyX-CAD application which need to be import. If we create new data then it is necessary to save it locally.

Basic steps involved during import of data between MyX-PDM and 3DEXPERIENCE Platform are explained in Import Scenario section of technical article [2].

Here we are explaining about processing details at X-PDM adapter.These steps are listed below :

  • X-PDM adapter reads component or assembly file data.
  • Create XPDMXML file for this data.
  • Using event bus services create event.This event has bulk or metadata attachment details associated with it.
  • Meta or bulk data associated with the event will be copied to gateway's shadow directory. It uploads the data to shared file server between gateways.
  • X-PDM adapter sends this event to X-PDM gateway.
  • X-PDM gateway send this event to 3DEXPERIENCE gateway and gateway will process it further depending on it's processing information.
  • Adapter notifies to user about event processing feedback received from gateway.

Export Scenario

Any V6 command,Batch or API invokes the "Send to XPDM" command from V6 application.

Basic steps involved during export of data between MyX-PDM and 3DEXPERIENCE Platform are explained in Export Scenario section of technical article [2].

Here we are explaining about processing details at X-PDM adapter.These steps are listed below :

  • 3DEXPERIENCE gateways sends the event with it's processing information and data associated with it to X-PDM gateway.
  • X-PDM adapter fetches and processes the events from X-PDM gateway which are waiting for processing.
  • Adapter downloads the data from shared file server and copy it to directory which is mentioned as per users request.
  • Adapter reads XPDMXML file data and notify to user about this.
  • Adapter sends processing feedback to X-PDM gateway.
  • X-PDM gateway sends event proceesing result feedback to 3DEXPERIENCE gateway.

InShort

This article describes basic steps involved in exchanging data between MyXPDM and 3DEXPERIENCE Platform. Role of X-PDM adapter has been explained in details for both import and export scenario.
