Integration of X-PDM with the 3DEXPERIENCE Platform - An Overview
Technical Article

Abstract

PDM stands for Product Data Management software. Integration of PDM with 3DEXPERIENCE Platform allows user to transfer the data from any PDM to the 3DEXPERIENCE Platform. This article provides an overview of such integration.

WHY Integrate X-PDM with 3DEXPERIENCE Platform ?

PLM and PDM systems are highly flexible and complex systems that companies configure to meet their existing business process needs. Companies need an interface between these systems that is fast, efficient and flexible enough to meet the needs of both systems.

Adapter for X-PDM brings additional exchange scenarios to meet these challenges.

Adapter for X-PDM aims at supporting the following processes:

  • When some of the design is performed in a non-DS system and must be integrated in an Engineering structure.
  • When the design authored in 3DEXPERIENCE Platform must be propagated to a non-DS system that manages product release or downstream processes.

HOW does X-PDM allow data co-operation with the 3DEXPERIENCE Platform ?

To integrate X-PDM with the 3DEXPERIENCE Platform we are using event exchange services via Adapter. Adapters provide the link between PDM servers and Gateways. It retrieves event data,fetch it and processes it as mentioned in event types and documentation.It also sends the event to their controller. This processing step makes the necessary changes to the PDM objects so that it can be understand by 3DEXPERIENCE Platform.

For details about adapter refer to Bus Adapter [1].

Web services are used to exchange the events between V6 gateway and X-PDM gateway. X-PDM event exchange is based on XPG events services.

X-PDM integration consists of component depicted below [Fig.1]:

Store Adapter plays a very important role in achieving this integration. Adapters provides the link between the PDM server and Gateway.

These are responsible for processing the data associated with events which  are received at gateway. These processes can make changes to PDM objects. Adapter can send events to their local controllers.

In this way,adapter provides the indirect link between two PDM systems.Role of adapter is to make outgoing event compatible with other PDM and read incoming events.

Customer have to create their own adapter for their application.

Capabilities Scope

Basic objective of this integration is to provide capabilities for exchanging the data between two PDM systems :

  • Import - Send the data from X-PDM to 3DEXPERIENCE Platform.
  • Export -  Receive and Process the data in X-PDM, sent thourgh 3DEXPERIENCE Platform.

These two scenarios have been documented through usecase. Usecase illustrates the steps involved in realizing these operations.

Along with these scenarios,some other operations for retrieving the gateway configuration information are documented through different usecases.

Web services operations dedicated for gateway monitoring operations are also documented through dedicated usecases.

In Short

The article deals with basics of X-PDM and how it is integrated with 3DEXPERIENCE Platform.Any application can provide the user interface to perform different operations with X-PDM.Bus Adapter completes the action required and notifies customer about status of processing.
