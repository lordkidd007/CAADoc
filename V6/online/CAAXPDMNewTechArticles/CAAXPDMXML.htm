Basic Objective

The basic objective of this article is that,user will be able to understand about XPDMXML. This article gives detailed information of what is XPDMXML and mapping which is used while creating XPDMXML.

Introduction

XML for XPDM exchange is called as XPDMXML. While exchanging the information between 3DEXPERIENCE Platform and any PDM, XPDMXML plays a very important role. XPDMXML is the XML file which contains information about PLM Metadata.PLM Metadata has metadata definition and non-CAD information.

XPDMXML contains information about the instantiation of any V6 Domain like Product structure,EBOM etc. These domain objects are combined in structures. Properties of objects are represented by attributes of node or different tags. XML element Associated Files represent the representation or any Native App File.

Creation of XPDMXML is handled by Bus adapter. This is explained in Sample Bus Adapter for MyX-PDM[1].

General XPDMXML

Every XPDMXML has some common elements. This elements are listed below :

  1. InfoHeader
  2. XPDMXML_Structure
  3. File
  4. ListOfObjectBase

It is not mandatory to have this elements in XPDMXML but if exist then it should exist in same sequence as shown in  Fig.2

Fig.2 General XPDMXML

Significance of different components of Fig.2 is given below :

  1. XPDMXML : XPDMXML file must have main container named as XPDMXML.This helps to identify and differentiate it from other XML files.
  2. InfoHeader : This is header present in XPDMXML. It contains information about XPDMXML in name and value pairs.
  3. XPDMXML_Structure : This element is used for representing the collection of objects representing structure. Name of this node will be replaced by name which is mentioned in domain specific schema.This name is dependent on V6 domain which we are instanciating for writting XPDXML. In every domain,class representing structure is derived from StructureBase abstract class. Refer to XPDMXML Core and Domain[2] for details.
    e.g. Product Domain -> ProductStructure , EBOM Domain -> EBOMStructure, Requirement Domain -> RequirementStructure etc.
    For example, writting XPDMXML by instanciation with Product domain refer to XPDMXML for Product Domain [3].
  4. File : This describes physical file associated with XPDXML instance file. It contains file information about Name, Location, Format as XML element.
  5. ListofObjectBaseType : This describes the unique identifier of the object that can be reused to reference this object across different XPDMXML documents. It consist of two components :
    External: identifier as it is known by the source and the target systems.
    XID: optional identifier of the object as it is known by the V6 source.
