Abstract

XPDMXML for CAA Component

MyX-CAD and MyX-PDM supports two data types which are Component and Assembly.

Component is treated as basic data type in MyX-CAD application. Component can be saved in file with .CAAcmp extension. Component consist of points and lines. Points has X,Y and Z co-ordinates.CAAcmp file is a simple XML file,it saves the information about points,lines and volume of geometry in node structure.

Creation of Component data is explained in How to Use My-XCAD Data Type [3].

In Fig.1 sample CAA component file is dispalyed. This component consist of 3 points and 3 lines which forms triangle geometry. Here,volume is also defined for triangle in terms of height.

Geometry is the root tag which has attribute name. Here value of name represents the name given to component while saving it in .CAAcmp file.

PointRef tag contains information about identifier of point and X,Y,Z co-ordinates of it. LineRef  has information about identifier for line and identifier of points which are forming this line.

VolumeRef tag has information related to volume mentioned in terms of height, identifier of volume and lines which are creating a closed geometry.

Table.1. XPDMXML Attributes
Attribute Comment
UniqueID Defines the unique identifier of the object that can be reused to reference this object across different XPDMXML documents. Unique ID consist of two attributes
1. External: attribute contains the stable identifier value which is known to both source and target system.
2. XID : Optional identifier of the object as it is known by 3DEXPERIENCE Platform.
Owned Defines parent root i.e. product reference. Value is ID of product reference which holds this Rep3DAggr.
FileRef Defines associated files. Value is ID of File node which represents the associated file with this Rep3DAggr.
Location Defined only for File node.This tag represents the name of associated file.
Format Defined only for File node.This tag represents format of associated file. We use either CGR or CGM depending on the attribute GeometryType.
GeometryType attribute defines the type of geometry used in associated file. We can use either "tessellated" or "exact".
