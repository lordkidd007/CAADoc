Sample Bus Adapter for MyX-PDM
Use Case

Abstract

This article describes about sample bus adapter which is used in exchanging the data between two PDMs i.e. MyX-PDM and 3DEXPERIENCE Platform. Here it is important to understand that Sample Bus adapter for MyX-PDM is specific to CAA usecases. Depending upon data types,user can create his own adapters in his own application.

Basic Objective

This article gives detailed information about sample bus adapter which is dedicated for executing CAA usecases. From this illustration, user will understand about role of an adapter in exchanging the data and how to create his own adapter in any application.

Introduction

While exchanging data between two PDMs ,we are using two adapters:
1. V6 Gateway Adapters : This adapters need to be present at 3DEXPERIENCE side. This will be managed by 3DEXPERIENCE Platform itself.
2. MyX-PDM Sample bus adapter : This bus adapter is present at MyX-PDM side. This adapter need to be managed by user of MyX-PDM.

Here it is important to understand that user can create their own adapter to exchange the event between MyX-PDM and 3DEXPERIENCE Platform.

Sample bus adapter for MyX-PDM provides the link between MyX-PDM gateway and MyX-PDM server(PDM server). We have designed an application which is integrated with different functionalities related to PDM. Refer to MyX-PDM Application[1] for details about this application.

Sample Bus Adapter for MyX-PDM

MyX-CAD application supports two types of data creation. We can create Component of .CAACmp and Assembly of .CAAAsm formats in MyX-CAD application. Creation of these data is explained in How to Use MyX-CAD data Type [2].

We can convert MyX-CAD application data into xml file format i.e. XPDMXML using bus adapter. Sample Bus adapter contains the implementation of converting CAA Sample data into XPDMXML File. Details about this implementation is given in XPDMXML for Basic Data Type[3] and XPDMXML for Assembly Data Type[4].

Sample Bus adapter contains the implementation of events bus services. Synchronous Communication between MyX-PDM gateway and 3DEXPERIENCE gateway is achieved with event bus services.

This implementation is given in Send Sample to 3DEXPERIENCE Platform[5] and Processing of Remote Events[6] usecases.

Sample bus adapter also contains the implementation of monitoring web services which are used for monitoring the gateway operation. This implementation is given in Gateway Monitoring services[7].

How Events are processed with MyX-PDM Adapter?

MyX-CAD application provides the user interface for MyX-PDM functionalities. User can trigger the commands for exchanging the data between MyX-PDM and 3DEXPERIENCE Platform. How to use this application and triger operations is explained in MyX-PDM application[1].

MyX-PDM Adapter supports two types of processing depending on the type of event :

Processing of Local Events :

[Fig.2] shows how local events are processed at MyX-PDM Adapter. This processing involves sequential processes when user invokes "Send to 3DEXPERIENCE Platform" command.Following is the explanation about each step shown in Fig.2:

Fig.2 Processing of Event at MyX-PDM
  1. When user invokes command "Send Sample to 3DEXPERIENCE Platform", control of MyX-CAD application will be transfered to MyX-PDM adapter.This is explained by (1) operation.
  2. Adapter process and generates the data needed for event for i.e. XPDMXML is generated along with the meta data. An object filled with unique event path will be generated.This is explained by (2) process.

Depending on the associated file type there are two ways of sending Meta Data file.

  1. If the event requires bulk data files or if the XPDMXML file size is too big then it must be firstly stored into the event path i.e. copied to shadow directory and secondly referenced inside input parameter.This is explained by (3) operation.
  2. If the events only require Meta data and if this file size is small then it can be directly sent to the Gateway without any file transfer by adding the content of this file as a string input parameter.This is explained by (4) operation.
  3. The last step is to effectively call Send with all the determined parameters (e.g.target site id and event name etc). Event will proceed towards the 3DEXPERIENCE Gateway for the processing. This is represented with (5) operation. When this event will be sent to 3DEXPERIENCE gateway it's processing will be handled by V6 adapters.
    4. Depending on the processing at V6 adapter, X-PDM gateway receives the feedback and notifies to user about this.

Processing of Remote Events :

[Fig.3] shows how remote events are processed at MyX-PDM Adapter. This processing involves sequential processes happening in execution of "Processing of Remote Events" usecase . Following is the explanation about each step shown in Fig.3:

Fig.3 Processing of Remote Event at MyX-PDM
  1. Any V6 command or Batch  invokes "Send to X-PDM" command. After completing local processing,3DEXPERIENCE gateway sends an event to X-PDM gateway for remote processing.This is explained by (1). Events that required processing at XPDM will be in waiting state.
  2. Sample bus adapter will make a call to X-PDM gateway and checks for events that require processing at X-PDM.This is explained by (2) operation.
  3. Sample bus adapter polls event from X-PDM gateway.It also retrieves the necessary information about processing from event. Supported event name gives the details about meta and bulk data that are associated with this type of event. This is explained by (3) operation.
  4. If meta or bulkdata is associated with the event then sample bus adapter fetch them from the event path (4) and copy it into local shadow directory.
  5. Sample bus adapter processes this event and data depending on the business logic mentioned by customer(5). Bus Adapter notifies to user about this processing status(6).
  6. Feedback of this processing i.e. Processing result or return code will be sent to X-PDM Gateway(7).
  7. X-PDM gateway transfer this processing feedback to 3DEXPERIENCE gateway(8).

In Short

This article gives detailed information about the sample bus adapter for MyX-PDM. User will understand the role of adapter involved in data exchange between two PDMs. This article also describes about some important steps in creating a new bus adapter while creating a new application.
