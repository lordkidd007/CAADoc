XPDMXML for Assembly Data type
Technical Article

Abstract

This article describes about CAA specific XPDMXML for Assembly data type used in MyX-CAD application.We are using XPDMXML for exchaning the PLM information between MyX-CAD application and 3DEXPERIENCE Platform. Here it is important to understand that, XPDMXML for assembly  is only for CAA Assembly (.CAAasm) data type. Depending upon data types,user can create his own XPDMXML.

 MyX-PDM and MyX-CAD are sample application,user can replace it with his own application or use existing.

Basic Objective

The basic objective of this article is that,user will be able to understand about XPDMXML used in CAA application. This article gives information about XPDMXML with CAA Assembly data.

Introduction

In MyX-CAD application, when we are performing operation Send to 3DExperience Platform. XPDMXML file will be created for opened assembly structure.Refer to How to use MyX-DPM ?[1] for how different operations are performed with MyX-PDM. Creation of XPDMXML is role of Bus adapter. This is explained in Sample Bus Adapter for MyX-PDM[2].

XPDMXML for CAA Assembly Data Type

MyX-CAD and MyX-PDM supports two data types which are Component and Assembly. Assembly consist of collection of different components or different assembly.

Creation of Assembly data is explained in How to Use My-XCAD Data Type [3].

Here we have taken a simple structure of assembly file [Fig.1]. Structure contains the One component(MyComp_1.CAAcmp) and one assembly(Assembly_2.CAAasm) file with one component(MyComp_2.CAAcmp) under the main assembly(Assembly_1.CAAasm) file/node. We have mapped this assembly file to XPDMXML file by using the Sample Bus Adapter.

In MyX-PDM, mapping of CAA assembly to Product model shown below[Fig.3].

Fig.3 Assembly in Product Structure

As mentioned in above figure, RootElement is product reference. This holds different product instances. Here CAAcomponent is treated as Product which aggregates Rep3DAggr. Rep3DAggr holds the informaton about reference files. We are using the same mapping as we used in comonent XPDMXML. CAA component XPDMXML is explained in XPDMXML for Basic Data Type[4].

Fig.4 displays the XPDMXML file generated for this sample assembly structure.

Fig.4 Assembly in Product Structure

Here a new element ProductInst will be added which represent the component. This product instance has some additional tags.

Owned tag represents the value of product reference which holds this product instance. As we know root element with ID_0 holds all product instances. so in all product instances value of Owned tag is ID_0.

Instancing tag represents the value of product reference which is instantiated. Here product reference with ID_1 is instantiated in product instance with ID_3. So value of this tag is ID_1.

RelativeTransformation3D tag represents relative positioning between two product instances.This information is given in terms of Rotation3D and Translation3D matrix.
