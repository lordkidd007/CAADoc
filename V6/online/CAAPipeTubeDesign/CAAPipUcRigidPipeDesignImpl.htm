What You Will Learn With This Use Case

This use case is intended to help you make your first steps in programming with CATIA Piping and Tubing Interfaces.
Its main intent is to provide you information about it’s use, access and navigate rigid pipe.

The CAAPipRigidPipeDesignImpl Use Case

CAAPipRigidPipeDesignImpl.m is a use case of the CAAPipeTubeDesign.edu framework that illustrates the CATIA Piping and Tubing Interface framework capabilities.

What Does CAAPipRigidPipeDesignImpl Do

The goal of CAAPipRigidPipeDesignImpl use case is to show how to use and navigate rigid pipe.

How to Launch CAAPipRigidPipeDesignImpl

To launch CAAPipRigidPipeDesignImpl, you will need to set up the build time environment, then compile CAAPipRigidPipeDesignImpl along with its prerequisites, set up the run time environment, and then execute the sample.

To launch the use case, execute the following command:

mkrun -c "CAAPipRigidPipeDesignImpl Repository Server User Password SecutityCtx Version PLMType PLM_ExternalID1 PLM_ExternalID2"

where:

Repository
Server The server as protocol://Host:port_number/RootURI
User The user name.
Password The user password to authenticate the user.
SecurityCtx A string representing the security context (Role.Organization.Project). Choose Role as Designer.
Version The V_Version attribute value of the reference to be opened.
PLMType A PLM Type for component to be opened.
PLM_ExternalID1 The value of the PLM_ExternalID of a PLM Reference or PLM Representation Reference be browsed for Physical Structure.
PLM_ExternalID2 The value of the PLM_ExternalID of a PLM Reference or PLM Representation Reference be browsed for Logical Structure.

* The input model is retrieved by importing the Rigid_Route.3dxml file from the CAAPipeTubeDesign.edu framework.

InstallRootFolder\CAADoc\CAAPipeTubeDesign.edu\InputData\Rigid_Route.3dxml

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

This 3D XML file contains the root Product Reference with the following attributes:

  • PLM_ExternalID1 : Physical_Root
  • PLM_ExternalID2 : Logical_Root
  • PLMType : VPMReference
  • Version : A

 The image above shows the product structure used for this use-case.

Attention: The Name field in Properties dialog for the product is now refered as attribute PLM_ExternalID.
                  Whenever user imports the Rigid_Route.3dxml the PLM_ExternalID value for PLM_ExternalID1 and PLM_ExternalID2 will be different as per user CATIA session
                  as it is genearted by system automatically.
                  Please verify the PLM_ExtrenalID value carefully with Name field before executing the use case.

Where to Find the CAAPipRigidPipeDesignImpl Code

The CAAPipRigidPipeDesignImpl sample is made of a single class named CAAPipRigidPipeDesignImpl located in the CAAPipRigidPipeDesignImpl.m module of the CAAPipeTubeDesign.edu framework: InstallRootFolder\CAAPipeTubeDesign.edu\CAAPipRigidPipeDesignImpl.m\

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

This sample deals with the following classes or methods:

CAAAdpCreateSession Class to create a batch session and connects to a repository.
CAAOpenPLMComponent Class to retrieve the Product Reference or Representation Reference.
CATAdpOpener Class to open the Logical Root Product.
CATIPrdOccurrenceMngt Class to create the root occurrence for the Root Product factory.
