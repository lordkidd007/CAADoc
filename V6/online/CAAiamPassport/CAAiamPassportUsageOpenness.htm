3DPassport Role and Openness
Technical Article

Abstract

This article exposes the role of the 3DPassport server among the 3DEXPERIENCE platform, and the capabilities it offers for your own server, or information system.

Role

The 3DPassport server has a key role while providing the unique authentication and identification managenement interface for the 3DEXPERIENCE platform. let's detail it means.

Authentication

The authentication is the process to validate the identity of someone to grant him/her to access resources. The 3DPassport provides this capability through the CAS protocol. The Central Authentication Service (CAS) is a single sign-on protocol for the web. Its purpose is to permit a user to access multiple applications while providing their credentials (such as userid and password) only once. It also allows web applications to authenticate users without gaining access to a user's security credentials, such as a password. The name CAS also refers to a software package that implements this protocol.

The end users once authenticated to the 3DEXPERIENCE platform can access all its servers, and their data.

There is also the SLO, single logout out. Both mecanisms are explained in the referenced article [1]

Information System Integration

The 3DPassport is the application managing the repositories containing the Enterprise information systems enabling users to be connected to the 3DEXPERIENCE platform.

The platform administator can connect the enterprise system information when it is a LDAP or a database. Please refer to the installation documentation (Managing Repositories article) for the details of this capabality.

Federation

The federation defines mechanisms for allowing disparate security realms to broker information on identities, identity attributes and authentication.

Through the bridge, the end-user can access the web applications of its own enterprise as well those of the 3DEXPERIENCE platform with a unique authentication while both systems have their own authentication protocol.

Identity Management and Administration

The 3DPassport provides to the end-user capabilities to create different user accounts like described in the article "Creating End User Passport Credentials Manually in the Local MySQL Database", of the product documentation.

Openness

This sections describes the two kind of openness requiring the REST web services exposed by the 3DPassport.

SSO & SLO for 3DEXPERIENCE platform Servers

Each time you need a connection to the 3DEXPERIENCE platform to reach one of its service, you will need an authentication. The API is the login.

https://<3DPassport Server Authority>/login

According the parameters, and the HTTP/HTTPS method, you can:

  • Get a login ticket, a security token to realize an authentication,
  • Realize an authentication by providing end-user credential,
  • Login for a given service,

There is also the logout service.

https://<3DPassport Server Authority>/logout

Please refer to the 3DSwym use case [2] to see how these API are used.

CAS Protocol For Your Own Servers

You have also the capability to implement the CAS client on your own server. You will have to use the following apis:

https://<3DPassport Server Authority>/proxy
https://<3DPassport Server Authority>/proxyValidate

In Short

The 3DPassport is the service for managing the SSO and SLO for the 3DEXPERIENCE platform through the CAS protocol. Using the login API you can authenticate your application, and access all the resources of the 3DEXPERIENCE platform services.
