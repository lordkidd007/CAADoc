CAS Proxy

Role

Provides proxy tickets to services that have acquired proxy-granting tickets and will be proxying authentication to back-end services.

Request

Method Path
GET /proxy

Parameters

Name Required Value Description
pgt YES the proxy-granting ticket acquired by the service during service ticket or proxy ticket validation
targetService YES The service identifier of the back-end service. It can be an URL or not, since all back-end services are not always web services. However, the service identifier specified here must match the "service" parameter specified to /proxyValidate upon validation of the proxy ticket.

Response

Returns an XML-formatted CAS serviceResponse document as described in the XML schema in the section Appendix A of the CAS Protocol article
