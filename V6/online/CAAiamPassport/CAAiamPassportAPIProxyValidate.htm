CAS Proxy Validate

Role

Performs a validation of the service ticket as well the proxy ticket.

Request

Method Path
GET /proxyValidate

Parameters

Name Required Value Description
service YES An URL service. The format is URL-encoded. The identifier of the service for which the ticket was issued
ticket YES The service ticket issued by /login
pgtUrl NO An URL service. The format is URL-encoded. The URL of the proxy callback
renew NO true If this parameter is set, ticket validation will only succeed if the service ticket was issued from the presentation of the user's primary credentials. It will fail if the ticket was issued from a single sign-on session

Response

Returns an XML-formatted CAS serviceResponse document as described in the XML schema in the section Appendix A of the CAS Protocol article
