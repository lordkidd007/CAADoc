CAS Log out

Role

Destroys a CAS Session.

It means deleting the CATSTGC cookie associated with the request, and deleting each service ticket associated with the url services associated with this CATSTGC. Moreover, each url service is called to perform a log out which can be effective or not according the provider url service.

Request

Method Path
GET /logout

Parameters

Name Required Value Description
service NO An URL service. The format is URL-encoded. The browser might be automatically redirected to the service URL after the log out is performed by the CAS server. If this service URL is not specified, a notification message can be displayed (it depends on the service provider implementation)

Response

The response code can be:

  • 302, a redirection

    The log out is successul. The url of redirection is the input url service

  • 200,

    The logout is successul.

  • Error
