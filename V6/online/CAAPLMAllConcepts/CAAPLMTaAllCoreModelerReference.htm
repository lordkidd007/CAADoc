Introduction

We have defined the PLM Class [2] as a structure having:

  • PLM Attributes,
  • PLM Relations,
  • PLM Behaviors.  

The first two information are stored in the PLM Dictionary, the last being defined by code.

The PLM Core Modeler is formed with six PLM Classes. This set of core PLM classes will be the base of all PLM modelers [3]: the PLM attributes, the PLM relations, the PLM behaviors defined at this core level will be available for all PLM modelers. It means that we should describe them.

What does the Article Contains?

In the first chapter you will find for each PLM Core classes a complete description:

  • A definition,
  • The PLM symbol (those generally used to represent the PLM object on diagram),
  • The link to the CAA Encyclopedia page displaying the public PLM Attributes,
  • The list of relations,
  • The list of behaviors.

And then in the last chapter, we explain what are the relationships between the six PLM Classes.

PLM Behaviors

The  PLM Behavior can be divided in four categories:

  • Intrinsic   : aggregation, instantiation possibilities
  • Qeops       : Query , Expand, Open and Save,
  • Life Cycle : maturity, versioning [6], lock
  • Crud         : Create, Update, Delete

The goal of this article is to give the contract between a PLM Core class and these PLM behaviors. For all PLM Core Class we use the same table to sum up the behaviors.

Behavior Categorie Behavior Name Available Note
  • Behavior Categories: one among the four described just above
  • Behavior name: the exact name of the behavior
  • Available:  The value can be Yes , No, modeler dependant (MD). 
    • No :  the behavior has no sense for the PLM Core Class. Whatever the PLM Modeler it will be never possible.
    • Yes: the behavior has a sense  for the PLM Core Class, and whatever the PLM Modeler it  will be available.
    • MD : the behavior has a sense for the PLM Core Class, but the final decision is done by the PLM Modeler.

PLM Instance

Definition

The PLM Instance is an instance of a PLM Reference. The name of the PLM Core Instance class is PLMCoreInstance.

PLM symbol

PLM Instance as represented by the above illustration.

PLM Attributes:

click here : PLMCoreInstance

PLM Relations

  • IsAggregatedBy : It gives the PLM Reference aggregating the PLM Instance.
  • IsInstanceOf  : It gives the PLM Reference  which is the reference of the instance.

The two relations are defined by a private PLM attribute which contains only one element (a PLM Instance can be aggregated by and is an instance of only one PLM reference)

PLM Port

Definition

The PLM Port provides a stable abstraction of private data to the outside, isolating both parties (the outside and the private data) from each other.

The name of the PLM Core Port class is PLMPort.

  • IsReferencing : it gives the PLM object which is referenced. It can be a PLM Port, a PLM Instance, or a PLM Representation Instance

In Short

The PLM Core Modeler is based on six PLM Classes:

  • PLMCoreReference, a PLM Reference
  • PLMCoreInstance, a PLM Instance
  • PLMCoreRepInstance, a PLM Representation Instance
  • PLMCoreRepReference, a PLM Representation Reference.
  • PLMPort, a PLM port
  • PLMConnection,  a PLM Connection
