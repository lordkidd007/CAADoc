Introduction

In DS PLM products, data is stored in PLM Repository. Users can work concurrently inside a PLM Repository to create, browse and modify data. But it is also frequently needed to share data across PLM Repositories in order to enable concurrent engineering on a wider scale, as described in the following chart:

For example, PLM Repositories need to communicate with each other in the following cases:

  •     between geographical sites of the same company.
  •     between different companies: an OEM and its suppliers, a manufacturer and its clients.
  •     between a catalog issuer and all companies using the catalog parts.
  •     between PLM Workgroups.

Data Exchange is the capability to transfer data between PLM Repositories, or between a PLM Repository and an external system.

Dassault Systems proposes complementary solutions to handle this requirement:

  •     PLM Briefcase
  •     PLM Replication
  •     3DXML
  •     Neutral Formats

This article gives an overview of those solutions and their respective features.

Please note that there are no CAA APIs exposed for data exchange. Data Exchange is available only through interactive commands.

Data Exchange Solutions Overview

PLM Data Exchange is the process to extract/transport/import PLM data from one PLM Repository to another. All solutions described below are based on the generation of exchangeable files of various formats. They all follow the following macro-steps.

  • Export step: Extract a selection of data from a repository, and generate exchange files. Various processing might happen here (filtering, conversion, simplification...).
  • Import step: Integrate the content of an exchange file in the receiving repository. Various processing might happen here (conversion, links reroute, update...).
  • Transport step: Transport the exchange files from the physical location where they were generated to the physical location(s) were they need to be imported.

The transport step is not described in this article, as Dassault Systems does not intend to provide any specific middleware solution.

PLM Replication

PLM Replication is a DS solution for distributed PLM authoring over distant geographical sites. It enables multiple PLM repositories to share the same PLM components and to be regularly synchronized with each other. It supports co-design scenarios where each site contributes to a common design. Note that this solution could be used inside a single company that wants to dispatch the design work on multiple geographical sites, or between multiple companies that agreed to share the same PLM system and same work methodology.

PLM Briefcase

PLM Briefcase is a Dassault Systems' file format for lossless transport of PLM data from one PLM repository to another. Using PLM Briefcase, users can easily copy PLM components from one PLM Repository to another. It supports simple carry-over scenarios, where one user creates a new design by duplicating an existing one.

3DXML

3DXML is a universal, lighweight XML-based format proposed by Dassault Systems. It is meant to become the preferred format for mass market diffusion of 3D data, and integration with other companies' systems. 3DXML can be previewed with standalone DS tools (3DXML Viewer an Internet Explorer Plug-In) and it can be embedded in Microsoft Office documents. Refer to Dassault-Systems' presentation of 3DXML on www.3DS.com. 3DXML enables Data Exchange as it can be imported in a PLM Repository for clash analysis, digital mock-up or as a working context for supplier.

 Neutral Formats

Neutral formats are industry standards for Engineering Data Exchange. Some of those standards are supported by PLM products, for examples: Step AP203, Step AP214, IGES. Thus, it is possible to export a CATIA session to one of those formats, or import a neutral format file into a CATIA session. Neutral formats enable data exchange data between PLM repositories and external systems.

Below we give explanations concerning the features of the different solutions above. 

 Format

This section describes the file format produced by each solution. One major difference between those formats is that some are open standards, while others are proprietary to Dassault Systems.

  • PLM Replication generates multiple files, including a proprietary XML-formatted file describing the PLM components and a set of raw binary files extracted from the vault. Those files are not visible to the end-user, since they are meant to be generated and transported through automatic batch processes.
  • PLM Briefcase uses a proprietary format with "PLMBriefcase" extension. This is a binary transport format whose content can not be browsed or modified. It includes a header section describing the content of the briefcase.
  • 3DXML is an open XML format, associated with cgr geometry files. All files are embedded in a single zip archive with ".3DXML" extension. The XML-formatted data is documented an public. It can be generated, browsed or modified by any-one. 3DXML also supports private XML sections that can be accessed only with a dedicated license.
  • Neutral Formats generated by DS products comply to industry standards. Refer to the official documentation of Step AP203, Step AP214 and IGES.

Compatibility

This section describes the systems supported by each technology.

  • PLM Products: Compatible with "DS PLM Products" means that the technology can be used only between PLM systems provided by Dassault Systems. All Phase 2 PLM products are supported, but this technology can not be used with legacy Phase1 Products or systems from other companies.
  • All compliant systems: Compatible with "All compliant systems" means that the technology uses an open standard that can be supported by arbitrary products, provided by Dassault System or another company. Included compliant systems are all Phase 2 PLM products and Phase 1 legacy DS products.

Also note that for PLM Briefcase and PLM Replication, the two communicating PLM systems must share the same customization and the same release level.

Scalability

This section compares the data-exchange technologies based on their capability to handles high volume of data. We distinguished 3 levels of scalability:

  • CAD session: This means that the technology is limited to the amount of data that can hold a CAD session. This is the case for the  Neutral Formats that can only be exported and imported through a CAD session.
  • Full Model: This means that the technology can handle volumes of data that exceed the CAD session memory limit. Those technologies are adapted for exchanging medium-sized models (below 10.000 PLM components). Hence PLM Briefcase and 3DXML have low memory requirements because they transfer data directly between the repository and the exchange file. They both use data compression to generate smaller files. They are limited by the maximum size of the transport file (for example 2GB for PLMBriefcase).
  • Full Repository: This means that the technology is designed for handling up to the full content of a repository. PLM Replication can efficiently synchronize the full content of a repository by using a "delta" exchange mode ( only modified parts of PLM entities are transported ).

Sharing Strategy

This section compares the way PLM components are handled when imported in a repository. We distinguish two main sharing modes:

  • Duplicate: This means that the technology duplicates PLM components at import. For PLM Briefcase, Neutral Formats and 3DXML, the import operation creates new PLM components. Those new components have their own PLM Identificator, which is independent from the original exported components' PLM Identificator. Consequently, the imported data and the original data can be modified independently. Data can be imported multiple times in the same  PLM Repository, which will generate as many independent data sets.
  • Replicate: This means that the technology replicates PLM components at import. Using PLM Replication, the import operation does not create new PLM components, but creates a view of the same component. All replicates of the same original component share the same PLM Identificator. To avoid conflicts, an "Site Ownership" mechanism guarantees that only one of the PLM Repositories is given the right to modify a given PLM component. The Site Ownership associated to a PLM entity can be transferred from one PLM Repository to another.

Exchanged Information

This section describes the type of data that each technology is able to transport.

  • Full PLM description: This means that the technology is able to transport the full description of each PLM components of the model. This description includes PLM attributes, PLM extensions , Semantic Relations and Stream-Descriptors. In particular, PLM Briefcase and PLM Replication enable the lossless transport of a PLM component, with complete specification and relational information.
  • Product Structure and Geometry: This means that the technology is only capable of exporting the Product (or Process) Structure and the 3D geometry information. Consequently,  the specification information can not be exchanged using Neutral Formats and 3DXML. Some Neutral Formats provide a partial support of relation information.

Process

This section compares how each technology can be used by the end-users:

  • Admin Batch: Export and Import operations are operated through batch under the control of an administrator. For PLM Replication, those batches can be configured by declaring distant repositories and defining persistent objects (called Replication Packages) that describe the set of PLM components to be exported.  
  • Export/Import from session: This means that Export/Import are interactive commands available to any user in the CAD session. Export requires loading data into the CAD session before exporting its content. Import loads data into a CAD session, that must be saved for to persist the imported data in the PLM Repository.
  • Export/Import from repository: This means that Export/Import are interactive commands available to any user in VPMNav. The user defines the data set to be exported through queries and 3DLive navigation within the PLM Repository.  For example, no CAD session is needed for exporting/importing 3DXML and PLM Briefcase files since data is transferred directly between the repository and the data exchange file.

Downstream Usage

This section describes the scenarios that are targeted by each technology.

  • Collaborative Authoring: Means that the technology supports concurrent engineering on a shared model across multiple PLM Repositories (also called virtual co-location because users work as if they were connected to a single PLM repository). All PLM authoring applications are supported.
  • Carry-Over: Means that the technology enables the lossless cloning of data over multiple repositories. This makes it possible to punctually integrate another company's design, or start a new project from an existing design. It does not fit cases where responsabilities are intricated or where long term collaborative work requires frequent updates.
  • Browse/DMU/Preview: Means that the technology can be used to enable 3D navigation, Clash analysis and Digital Mock-up over a distant site. Exported data can also be used as a read-only working context for a supplier.
