PLM Modeler and Generic APIs
Technical Article

Abstract

This article first lists the APIs that are exposed through modeler independent frameworks: CATPLMIntegrationAccess, KnowledgeInterfaces and CATPLMComponentInterfaces. Nevertheless it does not mean that those API are automatically supported by all PLM Modeler. The last chapter explains how you can reach the information per PLM Modeler.

Generic APIs

Inside CATPLMIntegrationAccess, KnowledgeInterfaces and CATPLMComponentInterfaces you have a set of APIs which could be available whatever the PLM Modeler. We have split these API in two parts:

  • On PLM Object in Database
  • On PLM Object in Session

On PLM Object in Database

This array shows the APIs and the associated PLM Behavior [1].

API PLM Behavior Fw
CATAdpQueryServices Query CATPLMIntegrationAccess
CATAdpOpener::CompleteAndOpen(PLM Reference) Open & Expand CATPLMIntegrationUse
CATAdpOpener::CompleteAndOpen(PLM Representation Reference) Open CATPLMIntegrationUse
CATAdpLockServices Lock/Unlock CATPLMIntegrationAccess
CATAdpVersionningServices Versioning CATPLMIntegrationAccess
CATAdpMaturityServices Change Maturity CATPLMIntegrationAccess
CATAdpQueryServices::GetIdentificationSet Identifier Set Values CATPLMIntegrationAccess

All these API, except the query, have the same input: the PLM object represented by a CATIAdpPLMIdentificator interface pointer. The query API is the means to retrieve a CATIAdpPLMIdentificator interface pointer.

On PLM Object in Session

Once the PLM object is opened in session (authoring Session [2]) , you could invoke these following interfaces:

API Behavior Fw
CATAdpSaver Save  CATPLMIntegrationUse
CATIPLMNavReference, CATIPLMNavRepReference, CATIPLMNavRepInstance,

CATIPLMNavInstance, CATIPLMNavOccurrence

 Navigation CATPLMComponentInterfaces
CATICkeObject Public PLM Attributes (R/W) KnowledgeInterfaces
CATIPLMNavEntity Public PLM Attributes (R) CATPLMComponentInterfaces
CATIPLMComponent PLM Class type and PLM Object Identifier CATPLMIdentificationAccess
CATIPLMIdentifierSet Identifier Set as a string CATPLMComponentInterfaces

The Save can be invoked for a specific PLM object, or for all the session.

PLM Modeler and Generic APIs

Except the Query which is possible whatever the PLM Modeler, for the other APIs, the support must be explicitly given by a CAA article. 

In the PLM Modelers section of the CAA Encyclopedia, you will find articles "XXX Behaviors" (where XXX is the name of the modeler) describing the behavior of the PLM Modeler for those APIs.  

If a such article does not exist, by default, the APIs listed above (except the Query), are not supported for any PLM Modeler.
