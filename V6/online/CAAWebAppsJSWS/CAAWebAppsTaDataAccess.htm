About Service Data Access
Technical Article

Abstract

This article is about accessing resources on servers from a Widget App running in the 3DDashboard. For any server to be reached by a widget, there is at least the cross-domain protection to solve. But if the server is one of the 3DEXPERIENCE Platform, in addition, an authentication to the 3DPassport must be performed. So, we provide API to easily solve these two protections.

3DEXPERIENCE Platform Services

If your widget needs data from a server of the 3DEXPERIENCE platform, your widget must be trusted [1]. So if your widget is launched from Run Your Apps, it will not be possible.

The API to succeed such a request is provided by the DS/WAFData/WAFData module. It is the authenticatedRequest function exposed by this AMD [2] module.

Please refer to the 3DSwym use case [3] for an example.

Others

Unlike the previous case, there is no constraint about the widget itself. It can be trusted or not.

The API to succeed such a request is also provided by the DS/WAFData/WAFData module. It is the proxifiedRequest function exposed by this AMD [2] module.

Please refer to the referenced use case [4] for an example.
