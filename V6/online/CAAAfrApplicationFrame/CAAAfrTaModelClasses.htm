Using Application Frame model classes
Technical Article

Abstract

This article explains how to handle non exposed Application Frame model classes (CATCmd classes )

  • Apps and Add-ins

Apps and Add-ins

CATCmd-prefixed classes are the main classes to deal with apps and add-ins. They do not expose anything, except their types handled using pointers. All these classes have been removed from CAA V6, but can still be referenced.

CATCmd  Classes Description
CATCmdWorkbench The whole set of commands available at a given time for a given UI-activable object.
CATCmdContainer A starter container. Apps are containers, and  as well as sections, and groups of commands in a section.. 
CATCmdAccess The base class of all the other classes.
CATCmdStarter The leaf object in a container to which a command header is attached.

Example

In following file, CATCmdWorkbench is used as type of return for CreateWorkbench method. One line is simply added at the begining to declare the class : 

#include "CATBaseUnknown.h"  // Needed to derive from CATBaseUnknown

class CATCmdWorkbench;       // Needed by CreateWorkbench

class CAAAfrGeoCreationWkb : public CATBaseUnknown
{
  CATDeclareClass;
  public:
     CAAAfrGeoCreationWkb();
     virtual ~CAAAfrGeoCreationWkb();

     void CreateCommands();
     CATCmdWorkbench * CreateWorkbench();
};

In Short

CATCmd-prefixed classes even if removed, can still be referenced in V6.
