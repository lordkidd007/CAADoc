The CAAGeometry Use Case

CAAGeometry is a use case distributed on several frameworks: CAAApplicationFrame.edu, CAACATIAApplicationFrm.edu, CAADialogEngine.edu, CAAVisualization.edu, CAAObjectModelerBase.edu and CAASystem.edu. They illustrate AfrFoundation, AfrProperties, AfrSelection, AfrStandard, CATIAApplicationFrame, Dialog, DialogEngine, Visualization, VisualizationBase and System framework capabilities.

What Does the CAAGeometry Use Case Do

The CAAGeometry use case is mainly a set of commands which group together the main interfaces and behaviors of the frame/command domains. These commands are used to edit the CAAGeometry model. This data model has two apps: The CAA : Create Geom app and the CAA : Analyse Geom app.

The CAA Apps in Compass

The CAAGeometry apps consists in commands which are mostly located in two dedicated sections of the action bar: the CAA Geometry Creation Section and the CAA Geometry Analysis Section. Some commands are located in common sections, such as the Tools section and the View section, and some general commands are even located in the fixed area of the action bar. One command is located in the App Options dialog box.

The CAA Geometry Creation Section

There are five groups of commands in this section:

  • The select command
  • Basic geometry creation commands : point, line, plane, etc...
  • Surfacic creation commands
  • Solids creation commands
  • Boolean operation commands
The CAA Geometry Analysis Section

There is just one group of mathematical analysis commands in this section.

The View Section

In the standard View section, we have added three commands that change the view point.

The Tools Section

In the standard Tools seection, we have added two sets of commands: one group contains commands that enable to show the current model in a new customized tab, and another group contains clipping commands.

The Fixed Area

Three generic commands were added in this section, that are independant of the current data model.

The App Options box
 
One command was added in App Options box, that enables to set a color option thanks to a combo.

The use case shows the implementation of some interfaces on the elements:

  • The CATIEdit implementation
    On the Plane element
  • The CATIContextualMenu implementation
    On the Ellipse element
  • The CATIContextualSubMenu implementation
    On the Plane element

There are other behaviors:

  • The Edit Properties
    On the Circle element
  • The Preferences pages
      For the "CAA : Create Geom" app
    For the "CAA : Analyse Geom" app
  • Customized command header

    There are two reasons to customize a command header:

    1. Set command availability depending on a context.
    2. Customize the default graphic representation. Just below, two examples are given:

      The left picture shows a combo of colors [32], and on right two editors [34]. These are specialized command headers.

  • Tools Palette

    You can provide the end user with some options. These options are command header instances integrated in the Tools Palette. It is a specific toolbar created by the frame application. An app can add options [33], or a command [35].

    On this above picture, you can see a command header instance represented by two editors [34], and inserted in the Tools Palette by the "CAA Create Geom" app [33]. The other four icons are command header instances created in the Cuboid command [35].

The Geometrical Creation app contains commands to create components of the CAAGeometry document. These components are the following:

  • CAASysPoint, for point elements
  • CAASysLine, for line elements
  • CAASysPlane, for plane elements
  • CAASysPolyline, for polyline, triangle and rectangle elements
  • CAASysCircle, for circle elements
  • CAASysEllipse, for ellipse elements
  • CAASysCuboid, for cuboid elements
  • CAASysCylinder, for cylinder elements

The root component, not created by a command, is CAASysGeomRootObj. This component implements some interfaces, the main one being CATIUIActivate to provide the app of the CAAGeometry model.

Launch 3DEXPERIENCE. When the application is ready, follow the scenarios described below:

  • The Apps Transitions scenario
    • In the 3D quadrant of the compass,  select CAA : Create Geom app.
    • In the 3D quadrant of the compass, select  CAA : Analyse Geom app.
    • In the 3D quadrant of the compass, select  CAA : Analyse Geom app.

    Two CAAGeometry tabs are open.

  • The Creation Objects scenario: (The complete scenarios are described in the article given in the brackets)

    With the CAAGeometry  basic commands:

    • Create points with the Point Command [19]
    • Create lines with the Line Command [28]
    • Create planes with the Plane Command [15]
    • Create circles with the Circle Command [14]
    • Create ellipses with the Ellipse Command
    • Create rectangles with the Rectangle Command ( In the Insert/Multi-Lines menu ) [13]
    • Create triangles with the Triangle Command ( In the Insert/Multi-Lines menu )
    • Create polylines with the Polyline Command ( In the Insert/Multi-Lines menu ) [20]

    With the CAA : Create Geom app commands:

    • In the 3D quadrant of the compass,  select CAA : Create Geom app
    • Create Cylinders with the Cylinder Commands
    • Create Cuboids with the Cuboid Command

    With the CAA : Analyse Geom app commands:

    • In the 3D quadrant of the compass, select  CAA : Analyse Geom app
    • In the action bar, select the Type of Element Command [22]
    • In the action bar, select the Logical Command [22]
    • In the action bar, select the Numerical Command
  • The Edition scenario [11]
    • Double click on a plane
    • Double click on a point
  • The Contextual-SubMenu / Contextual Menu [10] scenario
    • Right click a plane, select Plane.x Object and select Element Count...
    • Right click an ellipse and select Circle
  • The New Tab scenario: [7]
    • In the action bar, activate Tools section and launch Histogram Chart Tab command.
    • Click OK
  • The Edit Properties scenario [12]
    • Right click the circle, and select Properties in the contextual menu
    • Click More
    • Click OK
  • The Preferences scenario [25] then [9]
    • In the top bar, select Me Services and click Preferences
    • Click Infrastructure in the left part of the dialog box
    • In the list of apps displayed, click CAA:Create Geom
    • Change some options
    • Click OK
    • In the 3D quadrant of the compass, select CAA : Analyse Geom
    • Select Circle discretization command
    • In the top bar, select Me Services and click Preferences
    • Click Infrastructure in the left part of the dialog box
    • In the list of apps displayed, click CAA: Create Geom
    • Change some options
    • Click OK
  • The Interruptible Task scenario [24]
    • In the "CAA Geometry Analysis Section", launch the Progress Task command to see the Interruptible Task dialog box:
    • Click Compute
      • The Computing Dialog box appears without the Cancel button. Wait until the end of the process
    • Check Interruptible Task in the Interruptible Task dialog box
    • Click Compute
      • The Computing Dialog box appears with the Cancel button.
      • Click Cancel before the end of the process.
    • Click Compute
      • The Computing Dialog box appears with the Cancel button. Wait until the end of the process
    • Click Close in the Interruptible Task dialog box.
  • The Bounding Element scenario [23]
    • In the "CAA Geometry Analysis Section", launch the Bounding Element command to see the Model Bounding Sphere dialog box:
    • Click Compute
    • Click Close in the Model Bounding Sphere dialog box
  • The Temporary Objects/ISO scenario [29]
    • Create some Point elements
    • In the Tools section, launch the Clipping By Box command
    • Select the ISO Selection text ( first temporary object) located at the origin of the model (0,0,0)
    • Select a Point, the origin of the clipping box. A trihedral is located at the selected point (second temporary object)
    • Move the mouse, click left to define the clipping box size and end the command. (The temporary bounding box is the third temporary object.) All the points outside this bounding box are removed from the model.

Where to Find the CAAGeometry Code

The CAAGeometry use case is made of a several modules distributed in several frameworks:

  • CAAApplicationFrame.edu
  • CAACATIAApplicationFrm.edu
  • CAASystem.edu
  • CAADialogEngine.edu
  • CAAVisualization.edu
  • CAAObjectModelerBase.edu

These frameworks are are in InstallRootFolder\CAADoc which is the folder where the API CD-ROM is installed.

CAAApplicationFrame.edu

Module Name Module Contents
CAAAfrGeoAnalysisWbench.m This module contains the "CAA : Analyse Geom" app definition. It is one of two apps of the CAAGeometry data model.
CAAAfrGeoCreationWbench.m This module contains the "CAA : Create Geom" app definition. It is one of two apps of the CAAGeometry data model. [1]
CAAAfrGeoCreationWkbAddin.m This module contains one Add-in of the "CAA : Create Geom" app. [2]
CAAAfrGeoWksAddin.m This module contains one Add-in of the CAAGeometry data model.
CAAAfrGeoWksAddin2.m This module contains the second Add-in of the CAAGeometry data model. This Add-in contains the "Histogram Chart Tab" command [7]
CAAAfrGeoDocument.m

CAAAfrGeoModel.m

These two modules contain the necessary interface implementations to visualize the CAAGeometry model.
CAAAfrGeoSubMenu.m This module contains the CATIContextualSubMenu implementation for a component of the CAAGeometry model.
CAAAfrGeoWindows.m This module contains the definition of the "Histogram Chart Tab" [7] tab and of the default tab of the CAASample model [8].
CAAAfrGeoEdition.m This module contains the CATIEdit implementation for some components of the CAAGeometry model. [11]
CAAAfrGeoCommands.m This module contains some commands used by the CAAGeometry app. [23]
CAAAfrProgressTask.m This module contains a command which implements the CATIProgressTask interface. [24]
CAAAfrGeneralWksAddin.m This module contains a global Add-in  [30]. It means that all the commands launched from a command header defined in this add-in are available whatever the data model.
CAAAfrSpecializedCommandHeader.m This module contains the definition of the two specialized command headers: a combo header [32] and editor header [34]
CAAAfrPaletteOptions.m This module contains the implementation of the CATIAfrPaletteOptions interface on the geometry creation app.

CAACATIAApplicationFrm.edu

Module Name Module Contents
CAACafContextualMenu.m This module contains the CATIContextualMenu implementation for a component of the CAAGeometry model. [10]
CAACafEltToolsOptions.m This module creates the Elements property tab that contains three frames: Identifier, Max Points/Curve, and Implicit Points. This property page is associated with the "CAA: Create Geom" app. [9]
CAACafViewToolsOptions.m This module creates the View property tab that contains the 3DRepresentation frame. This property tab is also associated with the "CAA : Create Geom" app.
CAACafUseToolsOptions.m This module contains a command which uses the "Elements" property tab options.
CAACafEditTextureProp.m This module contains the CATIEditProperties implementation for some components of the CAAGeometry model. It is the "Texture" properties. [12]
CAACafEditColorProp.m This module contains the CATIEditProperties implementation for some components of the CAAGeometry model. It is the "Color" properties.
CAACafRootProperties.m This module contains the CATIRootProperties implementation.
CAACafGeoDocument.m This module contains the CATIDocumentEdit implementation for the CAAGeometry model
CAACafViewerFeedback.m This module contains the code to see the usage of a specific notification sent by a viewer. This notification give you information about the mouse position, list of elements under the mouse ....

CAASystem.edu

Module Name Module Contents
CAASysGeoModelComp.m This module contains the implementation of all the components contained in the CAAGeometry model. It contains the CATICreateInstance implementations too.
CAASysGeoModelInt.m This module contains the definition of some interfaces implemented by the CAAGeometry model components. It is the interface of type: the interface which characterizes the component.
CAASysGeoModelImpl.m This module contains the data extensions for the interfaces defined in the previous module.

CAADialogEngine.edu

Module Name Module Contents
CAADegGeoCommands.m This module contains all the state commands of the CAAGeometry model. [13, .., 22]
CAADegMultiDocCmd.m This module contains the command which derives from CATMultiDocumentCommand and which is associated with a CATOtherDocumentAgent.

CAAVisualization.edu

Module Name Module Contents
CAAVisGeoModel.m This module contains the CATIVis3DGeoVisu implementation for the CAAGeometry model components. [26]
CAAVisModelForRep.m This module contains the creation of the CAAVisTextModel component. This component is a used as temporary object to represent the "ISO Selection" Text in the "Clipping By Box" command. [29]
CAAVisWireBoxComp.m This module contains the creation of the CAAVisWireBox component. This component is a used as temporary object to represent the bounding box in the "Clipping By Box" command. [29]

CAAObjectModelerBase.edu

Module Name Module Contents
CAAOmbGeoNavigate.m This module contains the CATINavigateObject implementation for all the CAAGeometry model components and the CATINavigModify implementation for Point and Root componants. [27]
