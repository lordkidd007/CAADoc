What You Will Learn With This Use Case

This use case is intended to show you the structure and the contents of a documentation file tree [1] for an add-on on the 3DEXPERIENCE.

The CAADoc Use Case

CAADoc is a use case of the CAAApplicationFrame.edu framework that illustrates contextual help on commands created in the CAAGeometry sample use case.

What Does CAADoc Do

CAAGeometry is a sample model that contains a set of commands in two apps: CAA:Create Geom and CAA:Analyse Geom. For some commands there is a contextual help. To do that, we have created a documentation file tree which respects a structure to enable the visualization of HTML pages when the end user clicks F1 on a command.

Launch 3DEXPERIENCE. When the application is ready:

  • Launch CAA:Create Geom app from 3D quadrant of the compass.
  • On Me Services menu, click Preferences
    • In the General section, select the Help tab page.
    • Uncheck the Technical documentation Online button.
    • In the Technical documentation Locationeditor, enter InstallRootFolder\CAADoc\CAAApplicationFrame.edu\CAADoc where InstallRootFolder [2] is the folder where the API CD-ROM is installed.
    • Click OK
  • From the CAA Geometry Creation section of the action bar, launch the Point command .
  • Click F1

    The Point command is a command defined for CAAGeometry apps. At the right of the page, the Related Topics section contains links towards the other commands defined for CAAGeometry apps.

    The window contains three parts:

    • At the bottom right, the page associated with the command,
    • At the top, the banner with the print icon,
    • At the bottom left, the table of contents.
  • Launch the Cuboid command
  • Click F1

    The page in the lower righthand frame is replaced by the following HTML page about the Cuboid command :

    Fig.2: The Cuboid Command Page

    The Cuboid command is a command available in the CAA: Create Geom app. At the right of the page, the Related Topics section contains links toward the other commands of the app.

  • Click the Geometry Analysis Workbench link.

    The page in the lower righthand frame is replaced by the following HTML page about the Geometry Analysis workbench:

How to Launch CAADoc

To launch CAADoc , you will need to set up the build time environment, then compile all the modules of the following frameworks:

  • CAAApplicationFrame.edu
  • CAACATIAApplicationFrame.edu
  • CAADialogEngine.edu
  • CAASystem.edu
  • CAAVisualization.edu

Next set up the run time environment, and then execute the use case [2].

Then, in the window where you run the mkrun command, do not type the module name on the command line, but type 3DEXPERIENCE instead. When the application is ready make the scenario given in the above section.

Where to Find the CAADoc Files

The CAADoc use case is made of several files located in the CAADoc module of the CAAApplicationFrame.edu framework:

InstallRootFolder\CAADoc\CAAApplicationFrame.edu\CAADoc\

where InstallRootFolder [2] is the folder where the API CD-ROM is installed.

Step-by-Step

There are two logical steps in CAADoc:

  1. Creating the Documentation File Tree Structure and Contents
  2. Updating the Table Of Contents

Creating the Documentation File Tree Structure

The CAADoc folder contains two sub-folders: the English and the resources folders.

Contents of the English Folder

The English folder contains HTML, javascript and css files used to build the frame infrastructure, a Custom_TOC.xml file describing the global table of contents as well as a set of sub-folders:

The AnaWb, CreWb and GeoWs are folders associated with the CAA:Analyse Geom app, the CAA:Create Geom app and the CAAGeometry model, respectively.

  • Detail of the AnaWb folder contents:

    The AnaWb folder contains:

    • structure.xml: a XML file defining the table of contents for Geometry Analysis Workbench documentation.
    • AnaWbpr01.htm: the home page associated with the CAA:Analyse Geom app.
    • AnaWbboundingelement.htm: the page for the Bounding Element command.
  • Detail of the CreWb folder contents:

    The CreWb folder contains:

    • structure.xml: a XML file defining the table of contents for Geometry Creation Workbench documentation.
    • CreWbpr01.htm: the page associated with the CAA:Create Geom app.
    • CreWbcuboid, CreWbcylinder1 and CreWbcylinder2 HTML pages for the Cuboid, Cylinder 1 and Cylinder 2 commands respectively. - [Fig.2 picture]
  • Detail of the GeoWs folder contents:

    The GeoWs folder contains:

    • structure.xml: a XML file defining the table of contents for Geometry Workshop documentation.
    • GeoWspr01.htm: the page associated with the Geometry model.
    • GeoWsChartWindow, GeoWsdump, GeoWsexplore, GeoWsplane and GeoWspoint HTML pages for the Chart window, Element count, Explore, Plane and Point commands respectively. - [Fig.1 picture]

The FrameImages folder contains all the images used by the frame infrastructure.

The FrontmatterMap folder contains a light version of the DSDocHome.htm home page of the DS User Assistance.

Contents of the resources folder

This folder contains only the msgcatalog sub-folder.

This msgcatalog folder contains the CommonId2url.CATNls mapping file [1]. 

CAAAfrGeometryWksHeader.CAAAfrPlaneHdr = "GeoWs/GeoWsplane.htm";
CAAAfrGeometryWksHeader.CAAAfrPointHdr = "GeoWs/GeoWspoint.htm";
CAAAfrGeometryWksHeader.CAAAfrExploreHdr = "GeoWs/GeoWsexplore.htm";
CAAAfrDumpCommandHeader.CAAAfrDumpHdr = "GeoWs/GeoWsdump.htm";
CAAAfrGeoChartWindowAdnHeader.CAAAfrHistogramChartWindowHdr = "GeoWs/GeoWsChartWindow.htm";
CAAAfrGeoAnalysisWkbHeader.CAAAfrBoundingEltHdr = "AnaWb/AnaWbboundingelement.htm";
CAAAfrGeoCreationWkbHeader.CAAAfrCuboidHdr = "CreWb/CreWbcuboid.htm";
CAAAfrGeoCreationWkbHeader.CAAAfrCylinder1Hdr = "CreWb/CreWbcylinder1.htm";
CAAAfrGeoCreationWkbHeader.CAAAfrCylinder2Hdr = "CreWb/CreWbcylinder2.htm";

The left part of each statement in the previous file is the URL link while the right part is the relative path to the associated topic file.

In the CAAAfrGeometryWksHeader.CATRsc the resources file associated with the CAAAfrGeometryWksHeader class you find:

CAAAfrGeometryWksHeader.CAAAfrPointHdr.LongHelpId = "CAAAfrGeometryWksHeader.CAAAfrPointHdr";
CAAAfrGeometryWksHeader.CAAAfrPlaneHdr.LongHelpId = "CAAAfrGeometryWksHeader.CAAAfrPlaneHdr";
CAAAfrGeometryWksHeader.CAAAfrExploreHdr.LongHelpId = "CAAAfrGeometryWksHeader.CAAAfrExploreHdr";

The left part of each statement in the previous file is the contextual help resource key of the command while the right part is the URL link of the associated topic file.

The Pages Format

The standard HTML pages are formated by the DSDocXML.css style sheet located in the English folder.

In order to use the DSDoc style sheet, the following line has to be added to the HTML source code page:

<link rel="stylesheet" type="text/css" href="../DSDocXML.css">

Information in this file is placed within an overall table of class table1 to match the DSDoc style.

Updating the tables of contents

The Custom_TOC.xml file provided in this use case describes the global table of contents of the provided documentation:

<?xml version='1.0' encoding='UTF-8' ?>
<Root xml:lang = "en">
 <ITEM name = "DS User Assistance" icon = "FrameImages/DSDocIcon_home.gif" href = "FrontmatterMap/DSDocHome.htm"/>
 <DITEM name = "Geometry Workshop" icon = "FrameImages/DSDocIcon_help.gif" href = "GeoWs/GeoWspr01.htm" childtoc = "GeoWs/structure.xml"/>
 <DITEM name = "Geometry Analysis workbench" icon = "FrameImages/DSDocIcon_help.gif" href = "AnaWb/AnaWbpr01.htm" childtoc = "AnaWb/structure.xml"/>
 <DITEM name = "Geometry Creation Workbench" icon = "FrameImages/DSDocIcon_help.gif" href = "CreWb/CreWbpr01.htm" childtoc = "CreWb/structure.xml"/>
</Root>

It's a XML file containing a <Root> element containing two kind of elements:

  • DITEM defines a node of the table of contents with the following attributes:

    • name: is the name displayed in the table of contents
    • childtoc: indicates the relative path of the file describing the content of this node
    • href: indicates the html page linked to the node
    • icon: is the icon displayed in the table of contents.

  • ITEM defines a leaf of the table of contents with the following attributes:

    • name: is the name displayed in the table of contents
    • href: indicates the html page linked to the node
    • icon: is the icon displayed in the table of contents.

Each structure.xml individual table of contents has the same structure, but should contain ITEM elements only.
