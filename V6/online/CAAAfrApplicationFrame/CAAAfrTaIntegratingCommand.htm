How to Make Your Commands Available
Technical Article

Abstract

Once you have completed your command development, or even during its development stage, your may want to test them in the application frame, and then make them available to your customers. To do this, you need to decide where to integrate them, and then proceed to the integration. This article gives you tips and explains how to carry out the integration depending on your location decision. Integrating a script as a command is also described. For further information and examples, you can refer to the use cases "Creating an App"[1] and "Creating an Add_in"[2].

Integrating Your Commands

To make your new commands available to your customers, you need to integrate them into one of the following objects:

  • An app: your commands will be available as long as this app is the active one. You can either create a new app, or modify an existing one. In this case, you need to have the rights to modify the app source files.
  • An add-in to either a model or an app, allowing you to add your commands without modifying the model or the app. Creating an add-in makes you fully independent from the model or the app provider, and protects your commands from future evolutions of the model or the app. You can also modify an existing add-in, providing you have the rights to modify its source files. The add-in and the commands it contains are available with the model or the app.

Into an App

You can either create a new app or modify a existing one.

  • To create a new app for you commands, refer to the use case "Creating an App" [1]
  • To integrate your commands into an existing app, you need to do the following:
    1. Decide where you want to integrate your commands.
    2. Create one or several command header classes, or reuse existing ones you have already created for your app.
    3. Modify the CreateCommands method of the app class to add your command header instantiations.
    4. Modify the CreateWorkbench method to create the command starters and place them in application frame: for example in a section of the action bar.
    5. Rebuild the app shared libraries and DLLs using mkmk.
    6. Modify the command header resource files to add the resources you commands will use to display and provide help, such as the titles, the icons, the various help messages, the accelerators and mnemonics.

Into an Add-in

Because you are not their provider, or because you don't want to modify them, you may prefer to remain independent from the app into which you intend to integrate your commands. In this case, the add-in is the solution. You can either create a new add-in, or modify a existing one if you have the rights.

Also, an add-in is the only way to add your commands either to a model, making them available for all the apps associated with the model, either to the general app, making them available for all apps.

  • To create an add-in for your commands, refer to the use case "Creating an Add-in" [2]
  • To integrate your commands into an existing add-in, you need to do the following:
    1. Decide where you want to integrate your commands.
    2. Create one or several command header classes, or reuse existing ones
    3. Modify the CreateCommands method of the add-in class to add your command header instantiations
    4. Modify the CreateToolbars method to create the command starters and put them where you have choosen
    5. Rebuild the add-in shared libraries and DLLs
    6. Modify the command header resource files to add the resources your commands will use to display and provide help, such as the icons, the various help messages, the accelerators and mnemonics.

Integrating a Script as a Command

You can easily customize the action bar by adding your favorite macros. This reduces the macro access time for the macros you often run.

  This task explains how to add a macro as a command in the action bar.
  1. In the Tools section of the action bar, the Macros command enables you to manage your macros.
2. From the top bar, right click and select Display->Customize to launch the Customize panel. In the Sections tab, select the section in which you want to add your macro. Click on Add Commands.
3. In the list of commands, select the macro you want to add in the action bar.

The macro is now ready to be launched from the action bar.

In Short

To make your command available in the application frame, you need to integrate it into an app or an add-in. You can also add scripts as commands in the action bar.
