3DS Workspace Explorer

The Workspace Explorer shows you the actual tree structure of your workspaces, made up of frameworks, modules, and data folders. It helps you navigate this tree structure and includes menus contextual to the workspace and to its components.

  • Opening the Workspace Explorer
  • Comparing with the Solution Explorer
  • Synchronizing the Solution Explorer
Before you begin:

Open the appropriate workspace.

Opening the Workspace Explorer

  1. On the VIEW menu, point to 3DS Views, and click 3DS Workspace Explorer.

    The 3DS - Workspace Explorer opens.

The icons depict the following objects:

Icon Description
Workspace
Framework
Module
Data

Contextual menus provide commands for each of them. See Interface Description.

A toolbar located at the bottom left part of the view includes two commands:

  1. New Workspace. See Creating a Workspace.
  2. Open Workspace. See Opening a Workspace.

These commands are also available in the File menu, in the New and Open submenus respectively, as 3D Workspace commands.

Comparing with the Solution Explorer

Both the 3DS Workspace Explorer and the Solution Explorer display the actual workspace tree structure. The 3DS Workspace Explorer displays only the frameworks, modules, and data folder. To dive deeper, use the Explore command available in each contextual menu and which opens a Windows Explorer. The Solution Explorer recursively opens all the folders below a framework, a module, or a data folder, down to the files.

To explore a module from the 3DS Workspace Explorer, right click, for example CAADlgOnIdle.m, and click Explore. A Windows Explorer opens displaying the module contents.

To explore a module from the Solution Explorer, click the triangle signs in front of CAADlgOnIdle.m, and of its subfolders. This displays the folder and file tree from within the Solution Explorer.

Synchronizing the Solution Explorer

When you have added, removed, or modified frameworks, modules, or source files using the 3DS Workspace Explorer, you can synchronize the Solution Explorer thanks to Add to Solution, Add All to Solution, or Refresh Solution Project commands in the framework and module contextual menus.

Synchronization commands are:

Command Applies to Description
Open Solution Workspace Opens the Solution Explorer.
Add to Solution Framework, Module Creates a Visual Studio project for this framework or module and adds it to the current Solution Explorer.
Add All to Solution Framework Recursively creates a Visual Studio project for the framework and its modules, and add them to the current Solution Explorer.
Refresh Solution Framework, Module Refreshes the current Solution Explorer with the changes carried out to the framework or module from the Workspace Explorer. Note that a change to a module or a module creation must be refreshed from the module node and not from the framework node.
