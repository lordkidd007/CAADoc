Adding a Comment

The Add Comments command enables you to add a comment above a method in the currently opened header file.
Before you begin:
  • Open a workspace with at least a framework and a module.
  • Display the Solution Explorer.
  • Open a header file from the Solution Explorer.
  • To use keyboard shortcuts, such as CTRL+Shift+D, these shortcuts must have been enabled. See Accessing and Customizing Settings.

  1. In the currently edited header file, locate the cursor in the method for which you want to add a comment, or select this method.

    Warning: Pay attention not to select beyond the method closing semi-colon.

  2. On the EDIT menu, point to 3DS Edit, and click Add Comments, or press CTRL+Shift+D.

    A comment is added above this method. The method is collapsed. Click the + sign below the comment to expand it.

    The comment is a template to document the method and looks as follows:

    /**
 *
 * <br><b>Role</b>:
 * @param iFirstParam
 *
 * @param iSecondParam
     *
 * @return
 *
 */
HRESULT Method(CATBaseUnknown* iFirstParam, CATBaseUnknown* iSecondParam);

    It includes:

    • A first empty line to describe in one short sentence what the method does.
    • A Role section to precisely detail what the method does and how.
    • As many @param sections as parameters in the method signature.
    • A @return section.

    Below is such a documented method using the example of the QueryInterface method. 

    /**
 * Retrieves a pointer to a given interface.
 * <br><b>Role</b>: Using an interface pointer to a component and given the IID of another interface,
 * QueryInterface searches whether this component implements this other interface. If the search is 
 * successful, QueryInterface returns a pointer to that interface against the component. 
 * @param iIID
 *   The interface identifier for which a pointer is requested.
 * @param oPPV
 *   he address where the returned pointer to the interface is located.
 * @return
 *   An HRESULT value. 
 *   <br><b>Legal values</b>:
 *   <dl>
 *     <dt>S_OK</dt>
 *     <dd>The query succeeds.</dd>
 *     <dt>E_NOINTERFACE </dt>
 *     <dd>The interface does not exist.</dd>
 *     <dt>E_FAIL </dt>
 *     <dd>The object is not valid.</dd>
 *     <dt>E_OUTOFMEMORY </dt>
 *     <dd>One memory allocation fails</bb>
 *     <dt>E_UNEXPECTED </dt>
 *     <dd>The query fails for any other reason</dd>
 *   </dl>
 */
virtual HRESULT __stdcall QueryInterface(const IID &iIID, void **oPPV);
