Generating XML Test Cases

This task shows you how to generate an XML test case.
Before you begin:
  • Open a workspace with at least a framework and a module.
  • Display the Solution Explorer.

  1. In the Solution Explorer, right click a test framework.

  2. In the contextual menu, point to Add, and click New Item.

    The Add New Item dialog box opens.

  3. In Installed, click the  arrow in front of 3DS Templates to expand its contents, if not yet expanded.

  4. Click Tests.

    The test case templates are displayed in the central pane.

  5. In the central pane, click XML test case.

  6. In the Name box, type the name you want to assign to the Test Case, such as CAAxxxTest.

    Warning: This name is used to create the test file name. Bear in mind that:

    • All the characters are allowed except those listed below:

      * ? \ / < > " ' | :

    • With Windows, full path names must not exceed 260 characters.

  7. Click Add.

    An XML file is created with the name you give, such as CAAxxxTest.xml. This file is opened in the Code Editor window.

  8. Edit this file.

    See the Native Apps Unit Tester documentation to get information about filling a test XML file.

  9. On the File menu, click Save CAAxxxTest.xml.

The file is saved in the framework, in the folder FunctionTests/TestCases.
