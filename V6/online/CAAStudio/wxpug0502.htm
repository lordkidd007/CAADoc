Compiling

Your application is now ready to be built.
Before you begin:
  • Open a workspace with at least a framework and a module.
  • Update the framework IdentityCard file.
  • Check that your prerequisites are up-to-date, of update them.

Before being able to run the written application, you must build your programs with the appropriate compiler and linker. This is not a simple task especially when you must take into account various locations of source files, multiple operating systems, several languages, multiple compilers with different and numerous options, and so on. Those who have experienced makefiles management know the problem.

With mkmk, Native Apps Interactive Development Environment proposes a single tool to compile and link applications wherever you are working and whatever the programming language you use.

The first time you compile your program, use the mkmk builder which not only compiles your program but also generates information that is used afterwards to accelerate compilations or to compile just one file.

Command Description Run mkmk Command
Build with mkmk Opens a dialog box to set mkmk options and then runs mkmk mkmk with options set in the dialog box.
Build All with mkmk Runs mkmk upon the whole selected module, framework, or workspace, keeping all the previously set option. mkmk -a ...
Link All Runs mklink upon the whole selected framework or workspace to link-edit it. mklnk or mklnk -a
Link Module Runs mklink upon the selected module to link-edit it. mklnk
Build ID Cards Runs mkCi upon the selected framework or workspace to build the identity cards. mkCi or mkCi -a
Clean Build Runs mkRemoveDo upon the selected framework or workspace to remove all the derived objects. mkRemoveDo or mkRemoveDo -a

To set build options:

  1. In the 3DS - Workspace Explorer, right click the workspace name, a framework, or a module, possibly point to Build, and click Build with mkmk.

    The 3DS - Build options dialog box opens.

  2. Check Force mkmk update (-u) to force mkmk to rebuild the build time data, even if it already exists.

  3. Check Activate debug mode (-g) to generate debugging data.

  4. Check Activate warning detection (-w) to issue warning messages.

  5. Check Activate edit/continue option (32 bits) to activate the edit and continue debug mode while compiling.

    Warning: This option must not be used when compiling 64-bit code.

  6. In More options, possibly type additional mkmk options.

  7. Click Build.

    mkmk is run upon the workspace, framework, or module with the declared options.

The Output windows opens, if it was not already opened, and displays the mkmk log.

If errors occurred, of if warnings or messages are output, they are displayed in the Error List window. Click Error List to display it.

To correct the errors, double click the error row. The file in error opens.
