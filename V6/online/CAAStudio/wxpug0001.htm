Installing Native Apps Interactive Development Environment

This task explains you how to install the Native Apps Interactive Development Environment customization from the VSIX installed with the Native Apps Development Toolset.

Native Apps Interactive Development Environment enables you to manage your development workspaces and to access from within Visual Studio the Native Apps Development Toolset such as the Native Apps Builder (mkmk), the Native Apps Unit Tester (mkodt), or the Native Apps Source Checker (mkscc).

This task also explains you how to uninstall Native Apps Interactive Development Environment.

  • Uninstalling Native Apps Interactive Development Environment
Before you begin: Note that:
  • You should have first installed the Native Apps Development Toolset CD ROM and set up the Native Apps Development Toolset. See Installing Native Apps Development Toolset.
  • You should also have set up the Native Apps Development Toolset licenses. See Setting up the Native Apps Development Toolset Licenses.

Installing Native Apps Interactive Development Environment

Native Apps Interactive Development Environment is installed from the 3DEXPOpenNativeAppsExtensions.vsix installation program installed with Native Apps Development Toolset. This VSIX is located in intel_a\code\bin folder just below the folder where you have installed Native Apps Development Toolset (C:\Program Files (x86)\Dassault Systemes\B417 by default). This VSIX is valid for Microsoft Visual Studio 2012 only.

To install Native Apps Interactive Development Environment:

  1. Using an explorer, locate the 3DSOpenNativeAppsExt2012.vsix installation program in your Native Apps Development Toolset installation.

    Default location: C:\Program Files (x86)\Dassault Systemes\B417\intel_a\code\bin\3DSOpenNativeAppsExt2012.vsix

  2. Double click 3DSOpenNativeAppsExt2012.vsix.

    The VSIX Installer dialog box opens. The Visual Studio version detected is displayed and checked.

  3. Click Install.

    The dialog displays a progress bar that then turns to Installation Complete.

  4. Click Close.

    Native Apps Interactive Development Environment is now installed.

  5. Start Visual Studio.

  6. On the TOOLS menu, click Options.

    The Options dialog box opens.

  7. Scroll down and click General tab under the 3DS option.

  8. In Tools Initialization Command Path, set the path to the tck_init command.

    This command is located in the intel_a\code\command folder in the folder where you have installed Native Apps Interactive Development Environment. The default path of this command is C:\Program Files (x86)\Dassault Systemes\B417\intel_a\code\command\tck_init.bat

  9. In Company, type the name of your company.

  10. Click OK.

  11. Exit Visual Studio.

    You can now install the documentation. See Installing Native Apps Interactive Development Environment Documentation.

Warning: If you have uninstalled a previous version of this vsix, check that the tools initialization command path (tck_init) is consistent with the current installation. Since the setting files are not deleted when uninstalling to let your workspaces available with the new version, this path might be wrong.

To check it, and possibly change it, see Accessing and Customizing Settings.

Uninstalling Native Apps Interactive Development Environment

To uninstall the Native Apps Interactive Development Environment:

  1. Launch Visual Studio.

  2. On the TOOLS menu, click Extensions and Updates.

    The Extensions and Updates dialog box opens.

  3. In the central pane, click 3DEXPERIENCE Open Extensions for Native Apps.

  4. Click Uninstall.

    A prompt opens to ask you whether you want to uninstall 3DEXPERIENCE Open Extensions for Native Apps.

  5. Click Yes.

    The 3DEXPERIENCE Open Extensions for Native Apps are removed from your computer.

Important:
  • The Native Apps Interactive Development Environment customization is no longer available from Visual Studio, but the vsix from which you have installed Native Apps Interactive Development Environment is not removed from your computer, and you can re-install it later on using the Visual Studio Extension Installer, as explained above.
  • The other tools, such as the Native Apps Builder (mkmk), the Native Apps Unit Tester (mkodt), etc., are not embedded in this package, and are thus not de-installed.
