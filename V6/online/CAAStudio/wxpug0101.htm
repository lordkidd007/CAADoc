Creating a Workspace

This task shows you how to create a workspace.

Before you begin: Note that:
  • You can create a workspace from the FILE menu or from the 3DS - Workspace Explorer.
  • You can disable run time environment computing.
  • By default, the Visual Studio solution associated with the workspace is created in the ToolsData\VisualStudio2012 folder below the workspace root folder. You can choose to manage it in another folder.
Warning: If you work with several V5 and V6 release levels, always check that the tools initialization command path is set to the tools installation path of the release level used by the workspace to create. For example, when you want to create a V5R23 workspace, set the tools initialization command path to the one in your V5R23 CAA RADE installation path. When afterwards you want to create a R2015x workspace, reset the tools initialization command path to the one in your R2015x Native Apps Development Toolset installation path. See Accessing and Customizing Settings.

  1. Open the 3DS - New Workspace dialog box using one of these ways:

    • On the FILE menu, point to New, and click 3DS Workspace.
    • In the 3DS - Workspace Explorer toolbar, click .

    If a workspace is current and displayed in the 3DS - Workspace Explorer, a prompt asks you whether you want to save its associated solution.

    If the 3DS Workspace Explorer was not opened, it opens, and the 3DS - New Workspace dialog box opens.

  2. In Location, type a folder path to create a new folder, such as E:\MyWorkspace.
    You can also click Browse to select an existing folder and fill the Location box, and then complete the selected path by typing the folder name into which you want to store the workspace frameworks.

    Warning: The folder you set for the workspace must not exist. Otherwise, a prompt is issued to warn you that a workspace cannot be created in an existing, even empty, folder. 

  3. In Tools level, select the tools level you want to use for this workspace.

  4. To disable run time computing, click in front of Open Options and check Disable run time environment computing.

    Disabling run time environment computing improves workspace opening performance if you do not need the run time environment during your working session, that is, for example if you do not need to run tests or debug your code.

  5. To create the workspace associated solution in another folder than the default one, in Associated Visual Studio solution location, either click Browse to select, or type the path of an existing solution.

  6. Click OK.

    The workspace is created and assigned the name of the folder. If the typed folder did not exist, it is created. The 3DS Workspace Explorer is refreshed with the newly created workspace. An empty solution is created, and if the Solution Explorer is displayed, it is refreshed with the created solution.
