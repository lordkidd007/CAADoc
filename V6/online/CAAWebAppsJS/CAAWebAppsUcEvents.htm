What You Will Learn With This Use Case

In this article you will learn about the exposed events dispatched by the 3DDashboard platform.

The CAAWebAppsEvents Use Case

CAAWebAppsEvents is a use case of the CAAWebAppsJS.edu framework.

What Does CAAWebAppsEvents Do

CAAWebAppsEvents is a widget application listening to events sent by the 3DDashboard platform. The list of events is presented in the referenced article [1].

The callback method associated with each one of them simply triggers an alert dialog box containing the name of the event, and its data if any.

Fig.1When the widget is launched

Just above, the display when the widget is launched. You can see that the onLoad Event is dispatched.

We invite you to play with the widget, and check different scenarios:

  • Reload the page, the onLoad Event is dispatched
  • Resize the page, or a frame inside the dashboard, the onResize Event is dispatched
  • Maximise the widget, two events : onResize and then onViewChange (with maximized as info)
  • Minimize the widget (come back to its normal size), two events are dispatched : onResize and then onViewChange (with windowed as info)
  • 'Edit Preferences' (Preferences command in the menu), the event onEdit is triggered, and then when the preferences are edited, there are 2 events: onRefresh, and then endEdit.
  • 'Delete' the widget (Delete command in the menu), the event onDestroy is triggered

How to Launch CAAWebAppsEvents

To launch CAAWebAppsEvents, you will need to set up the build time environment, then compile CAAWebAppsEvents along with its prerequisites, set up the run time environment, and then execute the use case [2]

Where to Find the CAAWebAppsEvents Code

The CAAWebAppsEvents use case is made of a unique file located in the CAAWebAppsJS.edu framework:

  • InstallRootFolder\CAADoc\CAAWebAppsJS.edu\CAAWebAppsEvents.mweb\src\CAAWebAppsEvents.html, the widget html web page

where InstallRootFolder [2] is the folder where the API CD-ROM is installed.

Step-by-Step

There are 2 mains steps:

  • Declaring the listeners
  • Writing callback method

Declaring the listeners

We have declared the listeners, and their respective callback functions as follows: 

<script>
     ...
     var myWidget = {
        events: {
           onXXX: function() {
               alert('onXXX callback');						
           }, 
           onYYY: function () {
               alert('onXXX callback');
           },
           ...
        }
     };
     widget.addEvent('onXXX',myWidget.events.onXXX);
     widget.addEvent('onYYY',myWidget.events.onYYY);
   ...

That we could have also written:

<script>
    ...
            
    var myWidget = {
				
         onXXX: function() {
            alert('onXXX callback');						
         },
         onYYY: function () {
            alert('onXXX callback');
         },
         ...
    };
    widget.addEvent('onXXX',myWidget.onXXX);
    widget.addEvent('onYYY',myWidget.onYYY);
    ...

Writing callback method

There are two kinds of callbacks: the one with or without arguments as input.

<script>
    ...
    var myWidget = {
       ...		
       onResize: function () {
          alert('onResize callback');
       },
       onKeyboardAction: function (key) {
          alert('onKeyboardAction:'  + ' / ' + key);				
       },
       ...
    ...

In Short

This use case shows how to be informed when the 3DDashboard sends events after end-user interactions.
