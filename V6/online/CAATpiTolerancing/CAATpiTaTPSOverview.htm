Some Technological Product Specification Concepts

The CATTPSUseItf and CATTPSBaseUseItf frameworks provides interfaces designed to describe the TPS domain. All these TPS are dedicated to qualify surfaces they are applied too. In practice, it allows you to handle the whole life cycle of technological specification objects. Its specificity consists in a semantic capture of those specifications. This enables clients to share and reuse this semantic. It also inhibits interpretation misunderstanding between the different applications accessing such a data base. This framework has been first dedicated to dimensioning and tolerancing but is to be extended to handle roughness, hardness, and furthermore user specifications.

TPS Objects

CATTPSSet is a tolerancing set. It groups TPS features and manages standards for all TPS included. A set can contain other sets. It implements:

  • CATITPSComponent, basic interface for all tolerancing objects.
  • CATITPSSet, allows user to retrieve standards and TPS included in that set.

The Tolerancing Set or Annotation Set may be retrieved from the 3D Shape Representation Reference by using GetSet method of the CATITPSServicesContainers interface. Two methods are defined to retrieve the Annotation Set from the 3D Shape Representation Reference or to retrieve the Tolerancing container from the 3D Shape Representation Reference.[1]

There is no need to create an Annotation Set, an annotation Set is created as soon as an annotation is created. Meanwhile, it is possible to create an Annotation Set by using the method RetrieveOrCreateCurrentTPSSet defined in CATITPSServicesContainers. A pointer on this interface may be retrieved by using CATTPSInstantiateUseComponent class.

TPS Features are Technological Product Specifications. They implement:

  • CATITPSComponent.
  • CATITPS, designed to manage TPS links with surfaces
  • CATITPSSemanticValidity, allows user to retrieve type, supertype and behavioral interfaces.
  • SuperType Interface, describes the general type of feature among Datum, Dimension, Form, Position, Orientation, RunOut, NonSemantic.
  • Type Interface, describes the specific type of the feature.
  • Behavioral Interfaces, provide access to features behaviors and attributes.

TTRS: Technologically and Topologically Related Surfaces. Component provided by the CATTTRSUseItf framework used to link TPS with surfaces.

CATITPSList is an interface to manage a list of CATITPSComponent. This object list is transient.

Copy Paste, Life Cycle and Update

Main of the CAA 2DLayout for 3D Design features implements CATIUseEntity interface. This interface allows to use DataCommonProtocolServices and DataCommonProtocolCCPServices services. [2]

Service Methods
DataCommonProtocolServices CanDelete, Delete, IsUpToDate, Update
DataCommonProtocolCCPServices CopyAndPasteTo, GetAvailableFormats, CopyAndPasteSpecialTo

Notes:

  • CATIUseEntity interface is implemented by the CAA features contained in the Tolerancing Modeler.
  • TPS Annotation Set Result feature and TPS Capture feature can only be deleted.
  • TPS Annotation Set, TPS View, TPS Annotation, TPS Noa, TPSWeld, TPS CGxxx features can be deleted and updated by using API defined in CATIUseEntity interface.
