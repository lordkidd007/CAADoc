What You Will Learn With This Use Case

Most importantly, the user learns to create an Implement link between Logical Instance and Functional Connection. In addition we will learn to create object in context (OIC) for Logical Instance and Functional Connection.

The CAALilCreateConToInstImplLink Use Case

CAALilCreateConToInstImplLink is a use case of the CAALogImplLinkUseItf.edu framework that illustrates CATLogicalImplementLinkUseItf framework capabilities.

What Does CAALilCreateConToInstImplLink Do

The Use case basically talks about creation of an Implement link between Logical Instance and Functional Connection.

  • Initially we load the Logical Reference. The loading mode is "Authoring" mode to enable to load in session all the Logical structure. The possible aggregated Implement relations will be loaded too.
  • Next we load the Functional Reference under the same condition.
  • Then important step is to retrieve path of first instances up to Source and Target.
  • Further we create an object in context (OIC) for Logical Instance (to represent the Source) as well as for Functional Connection (to represent the Target)
  • Before implement link creation, we retrieve the count of the Source objects from Target object and count Target objects from Source object.
  • Next we create Implement link between Source and Target OIC.
  • Lastly, we confirm the success of the creation of an implement link by confirming if the count of implement relation Target from a Source finally is +1 compared to the count prior to the implement link creation. On similar lines, we also confirm if the count of implement relation sources from our Target under consideration, is +1 compared to the count prior to the implement link creation.

How to Launch CAALilCreateConToInstImplLink

To launch CAALilCreateConToInstImplLink, you will need to set up the build time environment, then compile CAALilCreateConToInstImplLink along with its prerequisites, set up the run time environment, and then execute the use case [1]. 

To launch the use case execute the command:

mkrun -c "CAALilCreateConToInstImplLink Repository Server User Password SecurityCtx LogicalPLMType FunctionalPLMType 

                                -LogRef Attribute_name1 Attribute_value1 [Attribute_name1 Attribute_value1 ...]

                                -FuncRef Attribute_name2 Attribute_value2 [Attribute_name2 Attribute_value2 ...]

                                -PLMIDLogInst PLM_ExternalID_value1 [PLM_ExternalID_value1 ... ]

                                -PLMIDFuncInst PLM_ExternalID_value2 [PLM_ExternalID_value2 ... ]"

where:

Repository
Server The server as Host:Port_number/URI
User The user name
Password The user password to authenticate the user
SecurityCtx A string representing the security context (Role.Organization.Project) Choose Role as Designer( as the use case we are creating new Implement link (the PLM object))
LogicalPLMType Logical PLMType name
FunctionalPLMType Functional PLMType name
Attribute_name1 The identification attribute name for Logical ref (IDSet Attribute)
Attribute_value1 The identification attribute value for Logical ref (IDSet Attribute)
Attribute_name2 The identification attribute name for Functional ref (IDSet Attribute)
Attribute_value2 The identification attribute value for Functional Ref (IDSet Attribute)
PLM_ExternalID_value1 The PLMExtenalID value list for Path of first inst for Logical Instance
PLM_ExternalID_value2 The PLMExtenalID value list for Path of first inst for Functional Instance

The input model for this use case is retrieved by importing the CAALilCreateConToInstImplLink.3dxml file which located at:

InstallRootDirectory\CAADoc\CAALogImplLinkUseItf.edu\InputData

CAALilCreateConToInstImplLink.3dxml this 3D XML file contains the Logical Root Reference and Functional Root Reference with the following attributes:

  • PLM_ExternalID: CAALilCreateConToInstImplLink_Log_Root
  • Version: ---
  • PLM_ExternalID: CAALilCreateConToInstImplLink_Func_Root
  • Version: ---

And the Instance names are as below

  • PLM_PLM_ExternalID_value1: CAALilCreateConToInstImplLink_Func_Child1.1
  • PLM_PLM_ExternalID_value2: CAALilCreateConToInstImplLink_Log_Child1.1

Where to Find the CAALilCreateConToInstImplLink Code

The CAALilCreateConToInstImplLink use case is defined at the location:

InstallRootDirectory\CAADoc\CAALogImplLinkUseItf.edu\CAALilCreateConToInstImplLink.m\

where InstallRootDirectory [1] is the directory where the CAA CD-ROM is installed.

Step-by-Step

  • Prolog
  • Load the Logical Reference And Functional Reference in the session
  • Create - if necessary - the Ref/Ref Implement Link
  • Create the two Object in context (one for the source, and one for the target)
  • Retrieve the Implement link manager
  • Check Count of source from target and Target from source.
  • Create Instance to Connection Implement link
  • Check Count of source from target and Target from source.
  • Epilog

Prolog 

The CAALilCreateConToInstImplLink use case begins by creating a session and connecting to the Input Repository with an appropriate role (Designer, in this case). The Use Case  "Creating a Basic PLM Batch" [2] deals with Opening and Closing a PLM Session.

Load the Logical Reference And Functional Reference in the session

We begin with by retrieving the Identificator of input Logical Reference and Functional Reference. Further we load the Logical Reference and Functional Reference in the session. For details of loading Reference in session refer [3] use case.

Create - if necessary - the Ref/Ref Implement Link

Before creating a inst/inst link, an implement link between the two references must exist. 

...
hr = CAALilCheckRefRefImplementLink(piPLMCompOnLogRoot, piPLMCompOnFunctionalRoot);
if (FAILED(hr)) return 1;
...

This method is explained in the referenced use case [6].

Create the two Object in context (one for the source, and one for the target)

Now we will create Object in Context for Logical instance. To create OIC we need to retrieve Path of first Instances.

Creates the Object in context for Logical Instance

...
   CATListPtrCATIPLMComponent ListComponentLogicalPathOfInst; 
   CATIPLMNavReference *piNavRefOnEndLogRef = NULL;
   hr = RetrievePathOfInstancesAndLastInstRef(piPLMCompOnLogRoot ,
					ListLogicalPLMIDAttrValueOfFirstInst,
					ListComponentLogicalPathOfInst,
					piNavRefOnEndLogRef,
					level);
    ...
    CATOmbObjectInContext * piOmbObjCtxOnLogicalInst = NULL;
    hr = CATOmbObjectInContext::CreateObjectInContext(ListComponentLogicalPathOfInst, NULL, NULL, piOmbObjCtxOnLogicalInst);
...

A call to RetrievePathOfInstancesAndLastInstRef create path of first instances of Source Logical Port object and the Reference of the last element(piNavRefOnEndLogRef) [4]. Here last element retrieval is useless since we just want path of instances (ListComponentLogicalPathOfInst) for creation of OIC of Instance, only for reusability and convenience of code we use RetrievePathOfInstancesAndLastInstRef method.  (RetrievePathOfInstancesAndLastInstRef same method is also used in next step, where last element is useful).   

The CreateObjectInContext of CATOmbObjectInContext call in this case takes a Path of first Instance list namely ListComponentLogicalPathOfInst and second and third arguments are NULL, then API returns the Object in context (piOmbObjCtxOnLogicalInst) for Inst as last argument.

Creates the Object in context for Functional Connection

Next we will create Object in Context for Functional Flow. In very similar manner as we have created OIC of Logical port.

...
    CATListPtrCATIPLMComponent ListComponentFuncPathOfInst;
    CATIPLMNavReference *piNavRefOnEndFunctionalRef = NULL;
    level =1;
    hr = RetrievePathOfInstancesAndLastInstRef(piPLMCompOnFunctionalRoot,
		                         ListPLMIDAttrValueOfFunctionalPathOfInst,
					 ListComponentFuncPathOfInst,
					 piNavRefOnEndFunctionalRef,
					 level);
    ...
    CATIFctConnections  *  piFctConnectionsOnRoot  =  NULL; 
    hr= piNavRefOnEndFunctionalRef->QueryInterface(IID_CATIFctConnections,(void**)&piFctConnectionsOnRoot);
    ...
    CATListValCATIFctConnection_var listFunctionalConnections ;
    hr = piFctConnectionsOnRoot->List(listFunctionalConnections);
    ...
    CATIFctConnection_var spFctConnectionOnFirstConnection = listFunctionalConnections[1];
    ...
    CATOmbObjectInContext * piOmbObjCtxOnFunctionalConnection = NULL;
    hr = CATOmbObjectInContext::CreateObjectInContext(ListComponentFuncPathOfInst, NULL, spFctConnectionOnFirstConnection, piOmbObjCtxOnFunctionalConnection);
...

A call to RetrievePathOfInstancesAndLastInstRef create path of first instances of Source Logical Port object and the Reference of the last element(piNavRefOnEndLogRef) [4].   The last element aggregates Functional Connection, so retrieval of last element object is important using RetrievePathOfInstancesAndLastInstRef method. 

The CreateObjectInContext of CATOmbObjectInContext call in this case takes a Path of first Instance list namely ListComponentLogicalPathOfInst and second argument is NULL, and third argument is Functional Connection Object(spFctConnectionOnFirstConnection), then API returns the Object in context (piOmbObjCtxOnFunctionalConnection) for Connection as last argument.

Retrieve the Implement link manager

Now here we proceed to retrieve Implement link manager. This Implement manager will provide services to manage the Implement link.  

...
   CATILogicalImplementLinkManager* piImplementLinkManager = NULL;
   hr = CATLogicalImplementLinkManagerProvider::GetImplementLinkManager(piImplementLinkManager);
...

A call to GetImplementLinkManager of CATLogicalImplementLinkProvider call returns the Logical Implement manager piImplementLinkManager.

Retrieve Count of Source from Target and Target from Source

Now we retrieve count of targets associated with source object in context (Logical Port). Then further we retrieve count of Source from Target object in context (Functional Port). 

...
    int countOfTargetsBeforeNewLinkCreation = 0; 
    hr = RetrieveCountOfTargets(piImplementLinkManager,piOmbObjCtxOnLogicalInst,countOfTargetsBeforeNewLinkCreation);
    ...
    int countOfSourceBeforeNewLinkCreation = 0;
    hr = RetrieveCountOfSource(piImplementLinkManager,piOmbObjCtxOnFunctionalConnection,countOfSourceBeforeNewLinkCreation);
...

A call to RetrieveCountOfTargets returns a Count of target countOfTargetsBeforeNewLinkCreation. The details of this local method is explained in another use case [5].

A call to RetrieveCountOfSource returns a count of source countOfSourceBeforeNewLinkCreation [5].

Create Instance to Connection Implement link

Now we will create the instance to instance Implement link between the created Logical Port OIC as source and Functional Port OIC as target. 

...
   CATListValCATICkeParm_var EmptyList ; 
   CATILogicalImplementLink * pImplementLinkConnection = NULL ; 
   hr =  piImplementLinkManager->CreateImplementRelation(NULL_var, NULL, EmptyList, 
                                                         piOmbObjCtxOnLogicalInst,
		                                       piOmbObjCtxOnFunctionalConnection,
                            		            pImplementLinkConnection );
...

A call to CreateImplementRelation of CATILogicalImplementLinkManager on piImplementLinkManager creates a Implement link between two input Objects (piOmbObjCtxOnLogicalInst, piOmbObjCtxOnFunctionalConnection).

First argument is NULL_var due to that default Type will be taken into consideration. Next argument Environment is NULL in this case current set environment will be taken into consideration. Third argument Attribute list is empty since BL will valuate attribute values. 

Check Count of Source from Target and Target from Source

Now we will again retrieve the count of source as well target as we have retrieved in earlier steps, but here we expect the count is increment one (+1). Since we have created Implement link between target and source, so count must be increment one (+1)  to previous count.   

...
    int countOfTargetsAfterNewLinkCreation = 0;
    hr = RetrieveCountOfTargets(piImplementLinkManager,piOmbObjCtxOnLogicalInst,countOfTargetsAfterNewLinkCreation);
    if ((FAILED(hr))||((countOfTargetsBeforeNewLinkCreation+1) != countOfTargetsAfterNewLinkCreation)) return 1;
    ...	
    int countOfSourceAfterNewLinkCreation = 0;
    hr = RetrieveCountOfSource(piImplementLinkManager,piOmbObjCtxOnFunctionalConnection,countOfSourceAfterNewLinkCreation);
    if ((FAILED(hr))||((countOfSourceBeforeNewLinkCreation+1) != countOfSourceAfterNewLinkCreation )) return 1;
...

A call to RetrieveCountOfTargets returns a count of target countOfTargetsAfterNewLinkCreation [5].

A call to RetrieveCountOfSource returns a count of source countOfSourceAfterNewLinkCreation [5].

Epilog

The CAALilCreateConToInstImplLink use case ends by simply saving and closing the PLM session. Since all use cases necessarily conclude by saving and closing the PLM session, we have a dedicated article on this topic namely "Creating a Basic PLM Batch" [2]. 

In Short

The Use Case illustrates the creation of the Connection to Instance implement relation link.
