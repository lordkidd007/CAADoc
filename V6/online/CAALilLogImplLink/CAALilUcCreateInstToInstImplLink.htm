What You Will Learn With This Use Case

Most importantly, the user learns to create an Implement link between Logical Instance and Functional Instance. In addition we will learn to create object in context (OIC) for Logical Instance and Functional Instance.

The CAALilCreateInstToInstImplLink Use Case

CAALilCreateInstToInstImplLink is a use case of the CAALogImplLinkUseItf.edu framework that illustrates CATLogicalImplementLinkUseItf framework capabilities.

What Does CAALilCreateInstToInstImplLink Do

The Use case basically talks about creation of an Implement link between a Logical Instance and a Functional Instance.

  • Initially we load the Logical Reference. The loading mode is "Authoring" mode to enable to load in session all the Logical structure. The possible aggregated Implement relations will be loaded too.
  • Next we load the Functional Reference under the same condition.
  • Then important step is to retrieve path of first instances up to Source and Target.
  • Further we create an object in context (OIC) for Logical Instance (to represent the Source) as well as for Functional Instance (to represent the Target)
  • Before implement link creation, we retrieve the count of the Source objects from Target object and count Target objects from Source object.
  • Next we create Implement link between Source and Target OIC.
  • Lastly, we confirm the success of the creation of an implement link by confirming if the count of implement relation Target from a Source finally is +1 compared to the count prior to the implement link creation. On similar lines, we also confirm if the count of implement relation sources from our Target under consideration, is +1 compared to the count prior to the implement link creation.

How to Launch CAALilCreateInstToInstImplLink

To launch CAALilCreateInstToInstImplLink, you will need to set up the build time environment, then compile CAALilCreateInstToInstImplLink along with its prerequisites, set up the run time environment, and then execute the use case [1]. 

To launch the use case execute the command:

mkrun -c "CAALilCreateInstToInstImplLink Repository Server User Password SecurityCtx LogicalPLMType FunctionalPLMType 

                                -LogRef Attribute_name1 Attribute_value1 [Attribute_name1 Attribute_value1 ...]

                                -FuncRef Attribute_name2 Attribute_value2 [Attribute_name2 Attribute_value2 ...]

                                -PLMIDLogInst PLM_ExternalID_value1 [PLM_ExternalID_value1 ... ]

                                -PLMIDFuncInst PLM_ExternalID_value2 [PLM_ExternalID_value2 ... ]"

where:

Repository
Server The server as Host:Port_number/RootURI
User The user name
Password The user password to authenticate the user
SecurityCtx A string representing the security context (Role.Organization.Project) choose Role as Designer( as the use case we are creating new Implement link (the PLM object))
LogicalPLMType Logical PLMType name
FunctionalPLMType Functional PLMType name
Attribute_name1 The identification attribute name for Logical ref (IDSet Attribute)
Attribute_value1 The identification attribute value for Logical ref (IDSet Attribute)
Attribute_name2 The identification attribute name for Functional ref (IDSet Attribute)
Attribute_value2 The identification attribute value for Functional Ref (IDSet Attribute)
PLM_ExternalID_value1 The PLMExtenalID value list for Path of first inst for Logical Instance
PLM_ExternalID_value2 The PLMExtenalID value list for Path of first inst for Functional Instance

The input model for this use case is retrieved by importing the CAALilCreateInstToInstImplLink.3dxml file which located at:

InstallRootDirectory\CAADoc\CAALogImplLinkUseItf.edu\InputData

CAALilCreateInstToInstImplLink.3dxml this 3D XML file contains the root Logical Reference and Functional Reference with the following attributes:

  • PLM_ExternalID: CAALilCreateInstToInstImplLink_Log_Root
  • Version: ---
  • PLM_ExternalID: CAALilCreateInstToInstImplLink_Func_Root
  • Version: ---

And the Instance names are as below

  • PLM_PLM_ExternalID_value1: CAALilCreateInstToInstImplLink_Func_Child1.1
  • PLM_PLM_ExternalID_value2: CAALilCreateInstToInstImplLink_Log_Child1.1

Where to Find the CAALilCreateInstToInstImplLink Code

The CAALilCreateInstToInstImplLink use case is defined at the location:

InstallRootDirectory\CAADoc\CAALogImplLinkUseItf.edu\CAALilCreateInstToInstImplLink.m\

where InstallRootDirectory [3] is the directory where the CAA CD-ROM is installed.

Step-by-Step

  • Prolog
  • Load the Functional Reference And Logical Reference in the session in Authoring Mode
  • Create - if necessary - the Ref/Ref Implement Link
  • Create two Object in context (one for the Source, and one for the Target)
  • Retrieve the Implement link manager
  • Retrieve Count of Source from Target and Target from Source.
  • Create Instance to Instance Implement link
  • Check Count of Source from Target and Target from Source.
  • Epilog

Prolog 

The CAALilCreateInstToInstImplLink use case begins by creating a session and connecting to the Input Repository with an appropriate role (designer, in this case). The Use Case  "Creating a Basic PLM Batch" [2] deals with Opening and Closing a PLM Session.

Load the Functional Reference And Logical Reference in the session in Authoring Mode

We begin with by retrieving the Identificator of input Logical Reference and Functional Reference. Further we load the Logical Reference and Functional Reference in the session. For details of loading Reference in session refer [3] use case.

Create - if necessary - the Ref/Ref Implement Link

Before creating a inst/inst link, an implement link between the two references must exist. 

...
hr = CAALilCheckRefRefImplementLink(piPLMCompOnLogicalRoot, piPLMCompOnFunctionalRoot);
if (FAILED(hr)) return 1;
...

The local method CAALilCheckRefRefImplementLink creates an implement link between the two input references as described in the referenced use case [5]. But the creation can fail if an implement link already exists. In this case we use the CATError  returned by CreateImplementRelation to analyze the error.

If the creation is a success, the Ref/Ref implement link relation does not exist, otherwise if the error is "ImplementLinkERR_03", the link already exists. Have a look to the class documentation for the other error numbers.

Create two Object in context (one for the Source, and one for the Target)

Now we will create object in context (OIC) for Logical instance. To create OIC we need to retrieve Path of first Instances. 

...
    CATListPtrCATIPLMComponent ListComponentLogicalPathOfInst; 
    int level =1;
    hr = RetrievePathOfInstances(piPLMCompOnLogicalRoot,
	                        ListLogicalPLMIDAttrValueOfFirstInst,
 			      ListComponentLogicalPathOfInst,
			      level);
    ...
    CATOmbObjectInContext * piOmbObjCtxOnLogicalInst = NULL;
    hr = CATOmbObjectInContext::CreateObjectInContext(ListComponentLogicalPathOfInst, NULL, NULL, piOmbObjCtxOnLogicalInst);
...

A call to RetrievePathOfInstances create path of first instances of Source Logical instance object [#]. 

The CreateObjectInContext of CATOmbObjectInContext call in this case takes a Path of first Instance list namely ListComponentLogicalPathOfInst and other two arguments are NULL, then API returns the Object in context (piPLMCompOnLogicalRoot) as last argument.

Next we will create object in context for Functional Instance. In very similar manner as we have created OIC of Logical Instance.

...
    CATListPtrCATIPLMComponent ListComponentFuncPathOfInst;
    level =1;
    hr = RetrievePathOfInstances(piPLMCompOnFunctionalRoot,
  	                       ListPLMIDAttrValueOfFunctionalPathOfInst,
			     ListComponentFuncPathOfInst,
			     level);
    ...
    CATOmbObjectInContext * piOmbObjCtxOnFunctionalInst = NULL;
    hr = CATOmbObjectInContext::CreateObjectInContext(ListComponentFuncPathOfInst, NULL, NULL, piOmbObjCtxOnFunctionalInst);
...

A call to RetrievePathOfInstances create path of first instances of Target Functional instance object [#]. 

The CreateObjectInContext of CATOmbObjectInContext call in this case takes a Path of first Instance list namely ListComponentFuncPathOfInst and other two arguments are NULL, then API returns the object in context (piOmbObjCtxOnFunctionalInst) as last argument.

Retrieve the Implement link manager

Now here we proceed to retrieve Implement link manager. This Implement link manager will provide services to manage the Implement link.  

...
   CATILogicalImplementLinkManager* piImplementLinkManager = NULL;
   hr = CATLogicalImplementLinkManagerProvider::GetImplementLinkManager(piImplementLinkManager);
...

A call to GetImplementLinkManager of CATLogicalImplementLinkProvider call returns the Logical Implement manager piImplementLinkManager.

Retrieve Count of Source from Target and Target from Source

Now we retrieve count of Targets associated with Source object in context piOmbObjCtxOnLogicalInst (Logical Instance). Then further we retrieve count of Source from Target object in context piOmbObjCtxOnFunctionalInst (Functional Instance). 

...
    int countOfTargetsBeforeNewLinkCreation = 0; 
    hr = RetrieveCountOfTargets(piImplementLinkManager,piOmbObjCtxOnLogicalInst,countOfTargetsBeforeNewLinkCreation);
    ...
    int countOfSourceBeforeNewLinkCreation = 0;
    hr = RetrieveCountOfSource(piImplementLinkManager,piOmbObjCtxOnFunctionalInst,countOfSourceBeforeNewLinkCreation);
...

A call to RetrieveCountOfTargets returns a count of Target countOfTargetsBeforeNewLinkCreation. The details of this local method is explained in another use case [4].

A call to RetrieveCountOfSource returns a count of Source countOfSourceBeforeNewLinkCreation [4].

Create Instance to Instance Implement link

Now we will create the Instance to Instance Implement link between the created Logical Instance OIC as Source and Functional Instance OIC as Target. 

...
    CATListValCATICkeParm_var EmptyList ; 
    CATILogicalImplementLink * pImplementLinkConnection = NULL ; 
    hr =  piImplementLinkManager->CreateImplementRelation(NULL_var, NULL, EmptyList, piOmbObjCtxOnLogicalInst,
		                                        piOmbObjCtxOnFunctionalInst, pImplementLinkConnection );
...

A call to CreateImplementRelation of CATILogicalImplementLinkManager on piImplementLinkManager creates a Implement link between two input Objects (piOmbObjCtxOnLogicalInst, piOmbObjCtxOnFunctionalInst).

First argument is NULL_var due to that default Type will be taken into consideration. Next argument Environment is NULL in this case current set environment will be taken into consideration. Third argument Attribute list is empty since BL will valuate attribute values. 

Check Count of Source from Target and Target from Source

Now we will again retrieve count of Source as well Target as we have retrieved in earlier steps, but here we expect the count is increment one (+1). Since we have created Implement link between Target and Source, so count must be increment one (+1)  to previous count.  

...
    int countOfTargetsAfterNewLinkCreation = 0;
    hr = RetrieveCountOfTargets(piImplementLinkManager,piOmbObjCtxOnLogicalInst,countOfTargetsAfterNewLinkCreation);
    if ((FAILED(hr))||((countOfTargetsBeforeNewLinkCreation+1) != countOfTargetsAfterNewLinkCreation)) return 1;
    ...
    int countOfSourceAfterNewLinkCreation = 0;
    hr = RetrieveCountOfSource(piImplementLinkManager,piOmbObjCtxOnFunctionalInst,countOfSourceAfterNewLinkCreation);
    if ((FAILED(hr))||((countOfSourceBeforeNewLinkCreation+1) != countOfSourceAfterNewLinkCreation )) return 1;
...

A call to RetrieveCountOfTargets returns a count of Target countOfTargetsAfterNewLinkCreation [4].

A call to RetrieveCountOfSource returns a count of Source countOfSourceAfterNewLinkCreation [4].

Epilog

The CAALilCreateInstToInstImplLink use case ends by simply saving and closing the PLM session. Since all use cases necessarily conclude by saving and closing the PLM session, we have a dedicated article on this topic namely "Creating a Basic PLM Batch" [2]. 

RetrievePathOfInstances

The RetrievePathOfInstances method browse through the input model structure recursively, and creates list of path of first instances as per the input PLM_ExternalID list.   

HRESULT RetrievePathOfInstances(CATIPLMNavReference *piPLMNavRefOnRef,
			    CATListOfCATUnicodeString ListPLMIDAttrValueOfPathOfInst,
			    CATListPtrCATIPLMComponent &ComponentList,
			    int Level)
{

	CATListPtrCATIPLMNavEntity childrenList;
	CATPLMCoreType coreType = PLMCoreInstance;
	rc = piPLMNavRefOnRef->ListChildren(childrenList,1,&coreType);

A call to ListChildren of CATListPtrCATIPLMNavEntity retrieves the List of children Objects.

If the children object count is more than one we proceed to browse further.  

        int j=1 ;
        while ( (j<=childrenList.Size()) && (SUCCEEDED(rc)) )
        {

	   CATIPLMNavEntity * pNavEntityOnchild = childrenList[j];
		
   	   CATListOfCATUnicodeString listattr , listvalattr;
	   listattr.Append("PLM_ExternalID");
	   pNavEntityOnchild->GetPublicAttributes(listattr, listvalattr);
		
  	   if (listvalattr[1] == ListPLMIDAttrValueOfPathOfInst[Level] )
	   {
		CATIPLMComponent * piPLMCompOnCurrentInst = NULL ;
		rc = pNavEntityOnchild->QueryInterface(IID_CATIPLMComponent, (void**) &piPLMCompOnCurrentInst);
		
   		ComponentList.Append(piPLMCompOnCurrentInst);
		
		CATIPLMNavInstance * pNavInstanceOnCurrentChild = NULL ;
		rc = pNavEntityOnchild->QueryInterface(IID_CATIPLMNavInstance, (void**) &pNavInstanceOnCurrentChild);
			
 		CATIPLMNavReference* pReferenceOnChild = NULL ;
		rc = pNavInstanceOnCurrentChild->GetReferenceInstanceOf(pReferenceOnChild);
				
		Level++;
		rc = RetrievePathOfInstances(pReferenceOnChild,ListPLMIDAttrValueOfPathOfInst,ComponentList,Level) ;
	}
}	
...

Here we browse through the input mode structure with generic APIs, so that this method will be used for the both models Logical as well Functional.

we browse through recursively for matching each PLM_ExternalID from each level. Further after matching PLM_ExternalID with instance name we append the instance in the Path of first element list.

And finally this method will fill the list of path of first instance element list.
