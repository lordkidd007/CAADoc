interface CATIDOMNamedNodeMap

Interface used to represent collections of nodes that can be accessed by name.
Note that does not inherit from ; CATIDOMNamedNodeMaps are not maintained in any particular order. Objects contained in an object implementing may also be accessed by an ordinal index, but this is simply to allow convenient enumeration of the contents of a , and does not imply that the DOM specifies an order to these Nodes.

objects in the DOM are live.

o SetNamedItem
public virtual SetNamedItem( const iNodeToStore,
oOldNode)
Adds a node using its nodeName attribute. If a node with that name is already present in this map, it is replaced by the new one.
As the nodeName attribute is used to derive the name which the node must be stored under, multiple nodes of certain types (those that have a "special" string value) cannot be stored as the names would clash. This is seen as preferable to allowing nodes to be aliased.
